## 零、开篇：大模型工程师的 Python 和普通 Python 有什么不同

用 Python 写个脚本和用 Python 写大模型工程，差距就像会骑自行车和能驾驶载重卡车。

大模型工程对 Python 的要求集中在四块：

1. **面向对象**：PyTorch 的 `nn.Module` 体系、Transformers 的 `PreTrainedModel` 继承链，全是 OOP
2. **装饰器**：`@torch.no_grad()`、`@property`、`@staticmethod`、FastAPI 的路由、LangChain 的 `@chain`——代码里到处是装饰器
3. **异步编程**：LLM 推理服务、并发批量 API 调用、Agent 并行工具调用全靠 `async/await`
4. **类型提示**：Pydantic 数据校验、现代 Python 工程的类型安全、IDE 补全都依赖类型系统

本章不重复你已经知道的 Python 基础，只聚焦这四块"大模型级别"的用法，每块都给出在真实大模型工程里的实际案例。

## 一、面向对象：继承体系与魔术方法

### 1.1 初印象：用费曼视角理解

**面向对象就是把"数据"和"能对数据做的操作"绑在一起，叫做"类"**。

类是模具，对象是用模具生产出来的东西。你定义一次"汽车"类（有颜色、有速度、能加速、能刹车），就能造出成千上万辆各有特色的"汽车"实例。

**继承**就是"复用模具"：你不用重新设计跑车，只需说"跑车是汽车的一种，多了一个涡轮增压"。

在大模型工程里，你会天天用到的继承结构：

```
nn.Module（PyTorch 基类）
   └── PreTrainedModel（HF Transformers 基类）
           └── LlamaModel（具体模型）
```

每一个自定义网络都是 `nn.Module` 的子类。

### 1.2 类的基本结构

```python
class LinearLayer:
    """单层线性变换，展示 Python 类的核心概念"""

    # 类变量（所有实例共享）
    layer_count = 0

    def __init__(self, in_features: int, out_features: int):
        """构造函数：初始化实例变量"""
        self.in_features = in_features
        self.out_features = out_features
        # 用 Xavier 初始化权重
        import numpy as np
        scale = np.sqrt(2.0 / (in_features + out_features))
        self.weight = np.random.randn(out_features, in_features) * scale
        self.bias = np.zeros(out_features)
        LinearLayer.layer_count += 1

    def forward(self, x):
        """前向传播"""
        return x @ self.weight.T + self.bias

    def __call__(self, x):
        """让实例像函数一样调用：layer(x)"""
        return self.forward(x)

    def __repr__(self):
        """控制 print(layer) 的输出"""
        return f"LinearLayer(in={self.in_features}, out={self.out_features})"

    def num_parameters(self):
        return self.weight.size + self.bias.size

    @classmethod
    def total_layers(cls):
        """类方法：操作类变量"""
        return cls.layer_count

    @staticmethod
    def activation_options():
        """静态方法：和类/实例都无关，只是逻辑上归属于此类"""
        return ["relu", "gelu", "silu"]


layer = LinearLayer(512, 256)
print(layer)                          # LinearLayer(in=512, out=256)
print(LinearLayer.total_layers())     # 1
print(LinearLayer.activation_options())
```

### 1.3 继承与多态：nn.Module 的本质

```python
import numpy as np

class Module:
    """简化版 nn.Module，说明继承体系"""

    def __init__(self):
        self._parameters = {}
        self._modules = {}

    def register_parameter(self, name, param):
        self._parameters[name] = param

    def register_module(self, name, module):
        self._modules[name] = module

    def parameters(self):
        """递归收集所有参数——这是 optimizer.parameters() 能工作的原因"""
        for p in self._parameters.values():
            yield p
        for m in self._modules.values():
            yield from m.parameters()

    def forward(self, x):
        raise NotImplementedError("子类必须实现 forward()")

    def __call__(self, *args, **kwargs):
        return self.forward(*args, **kwargs)


class Linear(Module):
    def __init__(self, in_f, out_f):
        super().__init__()
        w = np.random.randn(out_f, in_f) * 0.01
        b = np.zeros(out_f)
        self.register_parameter("weight", w)
        self.register_parameter("bias", b)

    def forward(self, x):
        return x @ self._parameters["weight"].T + self._parameters["bias"]


class MLP(Module):
    def __init__(self, dims):
        super().__init__()
        for i, (d_in, d_out) in enumerate(zip(dims[:-1], dims[1:])):
            self.register_module(f"layer_{i}", Linear(d_in, d_out))

    def forward(self, x):
        for m in self._modules.values():
            x = np.maximum(m(x), 0)  # ReLU
        return x


mlp = MLP([128, 64, 32, 10])
x = np.random.randn(4, 128)
out = mlp(x)
print(out.shape)  # (4, 10)
param_count = sum(p.size for p in mlp.parameters())
print(f"总参数量：{param_count}")
```

这段代码基本还原了真实 `nn.Module` 的骨骼结构，读懂它你就理解了 PyTorch 参数管理的原理。

### 1.4 常用魔术方法（Dunder Methods）

大模型工程里高频出现的魔术方法：

| 方法 | 触发场景 | 大模型里的用例 |
|------|---------|---------------|
| `__init__` | 构造 | 初始化权重、配置 |
| `__call__` | `obj(x)` | `nn.Module` 前向 |
| `__repr__` | `repr(obj)` | 打印模型结构 |
| `__len__` | `len(obj)` | Dataset 长度 |
| `__getitem__` | `obj[i]` | Dataset 按索引取样本 |
| `__iter__` | `for x in obj` | DataLoader 迭代 |
| `__enter__` / `__exit__` | `with` 语句 | `torch.no_grad()` 上下文管理 |
| `__contains__` | `x in obj` | 词表查询 |

```python
class TextDataset:
    """最小可运行的 Dataset，展示 __len__ 和 __getitem__"""
    def __init__(self, texts, tokenizer, max_len=512):
        self.texts = texts
        self.tokenizer = tokenizer
        self.max_len = max_len

    def __len__(self):
        return len(self.texts)

    def __getitem__(self, idx):
        enc = self.tokenizer(
            self.texts[idx],
            max_length=self.max_len,
            truncation=True,
            padding="max_length",
            return_tensors="pt",
        )
        return enc["input_ids"].squeeze(), enc["attention_mask"].squeeze()
```

PyTorch 的 `DataLoader` 只需要你实现这两个方法，其他的（批次、shuffle、并行加载）它全包了。

## 二、装饰器：深度学习代码里最重要的语法糖

### 2.1 初印象：用费曼视角理解

**装饰器就是给函数"套一层壳"，在不改变函数本体的情况下，给它加点料**。

想象你在餐厅点了一份牛排。正常的厨师（函数）给你烤好端上来。现在套上一个"计时装饰器"，就变成了：先按下秒表、再烤牛排、再停秒表——菜还是原来的菜，只是外面多了个计时的动作。

在大模型代码里：
- `@torch.no_grad()` — 套上这层壳，函数里的所有操作都不计算梯度（省显存）
- `@property` — 让一个方法像属性一样访问（`.num_parameters` 而非 `.num_parameters()`）
- `@staticmethod` / `@classmethod` — 控制方法的"归属感"
- `@app.route("/generate")` — FastAPI 路由注册
- `@retry(max_attempts=3)` — 自动重试失败的 API 调用

### 2.2 从零理解：装饰器的本质

装饰器不过是"接受函数、返回函数"的函数：

```python
# 最简单的装饰器：打印函数调用时间
import time
import functools

def timer(func):
    @functools.wraps(func)   # 保留原函数的 __name__, __doc__ 等元信息
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} 耗时 {elapsed:.4f}s")
        return result
    return wrapper

@timer
def train_one_step(batch):
    time.sleep(0.05)  # 模拟训练
    return {"loss": 1.23}

# @timer 等价于：train_one_step = timer(train_one_step)
result = train_one_step(None)
```

### 2.3 带参数的装饰器

当装饰器本身需要配置参数时，再多包一层：

```python
def retry(max_attempts=3, exceptions=(Exception,)):
    """带参数的装饰器：自动重试失败的操作，常用于 LLM API 调用"""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_attempts:
                        raise
                    wait = 2 ** attempt   # 指数退避
                    print(f"第 {attempt} 次失败: {e}，{wait}s 后重试...")
                    time.sleep(wait)
        return wrapper
    return decorator

@retry(max_attempts=3, exceptions=(ConnectionError, TimeoutError))
def call_llm_api(prompt: str) -> str:
    # 实际调用 OpenAI / Claude / 本地模型
    return "模型响应"
```

### 2.4 类装饰器和 @property

```python
class ModelConfig:
    def __init__(self, hidden_size: int, num_layers: int, vocab_size: int):
        self._hidden_size = hidden_size
        self._num_layers = num_layers
        self._vocab_size = vocab_size

    @property
    def hidden_size(self):
        return self._hidden_size

    @hidden_size.setter
    def hidden_size(self, value):
        if value % 64 != 0:
            raise ValueError("hidden_size 必须是 64 的倍数")
        self._hidden_size = value

    @property
    def num_parameters_estimate(self):
        """估算参数量（只读属性，不需要 setter）"""
        # 简化估算：每层约 12 * hidden^2
        return 12 * self._hidden_size ** 2 * self._num_layers

    @classmethod
    def from_pretrained_name(cls, name: str):
        """工厂方法：根据名称创建预设配置"""
        presets = {
            "llama-7b":  cls(4096, 32, 32000),
            "llama-13b": cls(5120, 40, 32000),
            "llama-70b": cls(8192, 80, 32000),
        }
        if name not in presets:
            raise ValueError(f"未知模型：{name}，可选：{list(presets)}")
        return presets[name]


cfg = ModelConfig.from_pretrained_name("llama-7b")
print(cfg.num_parameters_estimate)   # 约 6.4B
```

### 2.5 大模型工程中的实战装饰器

```python
import functools
import logging

logger = logging.getLogger(__name__)

def log_io(func):
    """记录函数输入输出，用于调试 Agent 工具调用"""
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        logger.debug(f"调用 {func.__name__}，参数: args={args}, kwargs={kwargs}")
        result = func(*args, **kwargs)
        logger.debug(f"{func.__name__} 返回: {result!r}")
        return result
    return wrapper

def cache_result(maxsize=128):
    """带 LRU 缓存的装饰器，常用于嵌入计算或 API 结果缓存"""
    def decorator(func):
        @functools.lru_cache(maxsize=maxsize)
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            return func(*args, **kwargs)
        return wrapper
    return decorator

@log_io
@cache_result(maxsize=1024)
def get_embedding(text: str) -> tuple:
    """获取文本嵌入，结果被缓存（tuple 可哈希，list 不行）"""
    # 实际调用嵌入模型
    import hashlib
    return (hashlib.md5(text.encode()).hexdigest(),)
```

## 三、生成器与迭代器：大数据集的内存法宝

### 3.1 初印象：用费曼视角理解

**生成器就是一个"按需出货的仓库"**。

普通函数一次把所有结果算完装进一个列表返回给你——相当于整个仓库的货一次全运到你家。如果货有一千万件，你家根本放不下。

生成器则是"你要一件给你一件"，只有你 `next()` 的时候它才计算并吐出下一个。内存里同时只有那一件货。

**这正是大模型数据处理的标配**：训练集动辄几百 GB，不可能全部 load 进内存，必须用生成器/迭代器按 batch 喂。

### 3.2 生成器的写法

```python
def token_stream(text: str, tokenizer, chunk_size: int = 512):
    """
    把大文档切成 chunk 并 tokenize，用生成器逐块产出。
    无论文档多大，内存里同时只有一个 chunk。
    """
    tokens = tokenizer.encode(text)
    for i in range(0, len(tokens), chunk_size):
        chunk = tokens[i: i + chunk_size]
        yield chunk   # 关键字 yield：暂停、返回、下次从这里继续

def batch_generator(dataset, batch_size: int):
    """把数据集按 batch_size 分组，逐 batch 产出"""
    batch = []
    for sample in dataset:
        batch.append(sample)
        if len(batch) == batch_size:
            yield batch
            batch = []
    if batch:   # 最后一个不满的 batch 也要产出
        yield batch

# 使用
texts = ["很长的文档..."] * 10000

for batch in batch_generator(texts, batch_size=32):
    # 每次内存里只有 32 个样本
    pass
```

```python
# 生成器表达式（和列表推导式一样的语法，但用圆括号，不立即求值）
```

## 四、异步编程：并发 LLM 调用的关键

### 4.1 初印象：用费曼视角理解

**异步就是"等待时不发呆"**。

想象你同时在三个餐厅点餐。同步的做法是：去第一家点餐，等餐上来吃完，再去第二家……异步的做法是：三家同时点好单，然后哪家先好就先去取哪家。

**大模型工程里大量 I/O 等待**：
- 调用 OpenAI API：等网络返回（几秒）
- 读取磁盘上的训练数据
- RAG 检索：同时查多个向量库
- Agent 并行调用多个工具

这些场景都是"CPU 闲着、程序在等"。异步让你在等待时去做别的事，大幅提升吞吐量。

### 4.2 核心概念

Python 异步的三个关键词：

| 关键词 | 含义 |
|--------|------|
| `async def` | 定义一个协程函数（coroutine function） |
| `await` | 挂起当前协程，把控制权交还给事件循环，等待结果 |
| `asyncio.run()` | 启动事件循环，运行顶层协程 |

```python
import asyncio
import time

# 同步版本：串行
def fetch_sync(url: str) -> str:
    time.sleep(1)   # 模拟网络延迟
    return f"response from {url}"

def main_sync():
    start = time.perf_counter()
    for url in ["url_a", "url_b", "url_c"]:
        fetch_sync(url)
    print(f"同步耗时: {time.perf_counter()-start:.1f}s")  # ~3.0s

# 异步版本：并发
async def fetch_async(url: str) -> str:
    await asyncio.sleep(1)   # 模拟网络延迟，await 期间让出控制权
    return f"response from {url}"

async def main_async():
    start = time.perf_counter()
    # gather 让三个请求同时发出
    results = await asyncio.gather(
        fetch_async("url_a"),
        fetch_async("url_b"),
        fetch_async("url_c"),
    )
    print(f"异步耗时: {time.perf_counter()-start:.1f}s")  # ~1.0s
    return results

asyncio.run(main_async())
```

### 4.3 实战：并发调用多个 LLM API

```python
import asyncio
import aiohttp
from typing import List

async def call_claude(session: aiohttp.ClientSession, prompt: str, api_key: str) -> str:
    """异步调用 Anthropic API"""
    url = "https://api.anthropic.com/v1/messages"
    headers = {
        "x-api-key": api_key,
        "anthropic-version": "2023-06-01",
        "content-type": "application/json",
    }
    payload = {
        "model": "claude-sonnet-4-5",
        "max_tokens": 1024,
        "messages": [{"role": "user", "content": prompt}],
    }
    async with session.post(url, headers=headers, json=payload) as resp:
        data = await resp.json()
        return data["content"][0]["text"]

async def batch_llm_call(prompts: List[str], api_key: str) -> List[str]:
    """并发处理多个 prompt，有并发限制防止触发速率限制"""
    semaphore = asyncio.Semaphore(10)   # 最多同时 10 个并发请求

    async def call_with_limit(prompt):
        async with semaphore:
            return await call_claude(session, prompt, api_key)

    async with aiohttp.ClientSession() as session:
        tasks = [call_with_limit(p) for p in prompts]
        return await asyncio.gather(*tasks)

# 用法
```

## 五、类型提示：现代 Python 工程标配

### 5.1 初印象：用费曼视角理解

**类型提示就是给代码加"标签"**，告诉你（和 IDE）这个变量应该是什么类型。

Python 本身是动态语言，类型提示**不影响运行**，但有三大收益：

1. **IDE 补全更准确**：你写 `model.` 时，IDE 知道 model 是什么类型，能列出所有方法
2. **静态类型检查**（mypy / pyright）：在运行前就发现"你把 list 传给了需要 str 的地方"
3. **代码即文档**：看函数签名就知道它接受什么、返回什么

HuggingFace Transformers、LangChain 等库全都大量使用类型提示，读不懂就很难深入这些代码。

### 5.2 基础类型提示

```python
from typing import Optional, Union, List, Dict, Tuple, Callable, Any
from typing import TypeVar, Generic

# 基础
def tokenize(text: str, max_length: int = 512) -> List[int]:
    ...

# Optional = 可以是某类型，也可以是 None
def get_embedding(text: str, model: Optional[str] = None) -> List[float]:
    ...

# Union = 多个可能类型之一（Python 3.10+ 可用 X | Y）
def load_model(path: str | int) -> "nn.Module":
    ...

# Dict
ModelConfig = Dict[str, Union[int, float, str, bool]]

# Callable：接受函数作为参数
def apply_fn(data: List[Any], fn: Callable[[Any], Any]) -> List[Any]:
    return [fn(item) for item in data]
```

### 5.3 Pydantic：类型提示 + 数据验证

Pydantic 是大模型工程里最常用的数据验证库，LangChain、FastAPI、LlamaIndex 全都基于它：

```python
from pydantic import BaseModel, Field, field_validator
from typing import Optional, List
from enum import Enum

class ModelFamily(str, Enum):
    LLAMA = "llama"
    QWEN = "qwen"
    MISTRAL = "mistral"

class LLMConfig(BaseModel):
    model_name: str = Field(..., description="模型名称")
    model_family: ModelFamily = ModelFamily.LLAMA
    max_tokens: int = Field(default=2048, ge=1, le=128000)
    temperature: float = Field(default=0.7, ge=0.0, le=2.0)
    top_p: float = Field(default=0.9, ge=0.0, le=1.0)
    system_prompt: Optional[str] = None
    stop_sequences: List[str] = Field(default_factory=list)

    @field_validator("temperature")
    @classmethod
    def temp_and_top_p_conflict(cls, v, info):
        """temperature=0 时不应同时使用 top_p 采样"""

    model_config = {"frozen": True}   # 不可变（类似 dataclass frozen=True）

# 自动验证，类型错误会抛异常
cfg = LLMConfig(

print(cfg.model_dump_json(indent=2)) # 转 JSON
```

### 5.4 TypeVar 与泛型：理解 Transformers 源码

```python
from typing import TypeVar, Generic, Type

T = TypeVar("T", bound="BaseModel")

class Registry(Generic[T]):
    """泛型注册表：在 HF AutoModel 这类"自动类"里大量使用"""
    _registry: Dict[str, Type[T]] = {}

    @classmethod
    def register(cls, name: str):
        def decorator(model_cls: Type[T]):
            cls._registry[name] = model_cls
            return model_cls
        return decorator

    @classmethod
    def create(cls, name: str, **kwargs) -> T:
        if name not in cls._registry:
            raise KeyError(f"未注册的模型: {name}")
        return cls._registry[name](**kwargs)
```

## 六、常见陷阱

**1. 可变默认参数陷阱**：

```python
# 错误：所有调用共享同一个列表
def append_to(element, to=[]):
    to.append(element)
    return to

print(append_to(1))  # [1]
print(append_to(2))  # [1, 2]  惊不惊喜？

# 正确
def append_to(element, to=None):
    if to is None:
        to = []
    to.append(element)
    return to
```

**2. 装饰器忘记 `@functools.wraps`**：会丢失原函数的 `__name__`、`__doc__`，调试时报错信息混乱，类型检查也会失效。

**3. 异步函数忘记 `await`**：调用 `async def` 函数时不加 `await`，得到的是协程对象而非结果，且不会执行。Python 3.11+ 会发出 `RuntimeWarning`。

**4. 在异步代码里调用阻塞 IO**：

```python
import asyncio
import time

# 错误：在异步函数里用同步 sleep，会阻塞整个事件循环
async def bad():
    time.sleep(1)  # 阻塞！其他协程无法运行

# 正确：用异步版本
async def good():
    await asyncio.sleep(1)  # 只让出当前协程，其他照常跑

# 如果必须调用阻塞操作（如同步库）
async def use_sync_lib():
    loop = asyncio.get_event_loop()
    result = await loop.run_in_executor(None, blocking_function)
```

**5. 生成器只能遍历一次**：

```python
gen = (x**2 for x in range(5))
list(gen)   # [0, 1, 4, 9, 16]
list(gen)   # []  已经耗尽，需要重新创建
```

**6. 类型提示只是提示，不保证运行时正确**：`mypy`/`pyright` 做静态检查，但 Python 解释器运行时不强制。要运行时验证，用 Pydantic 或 `isinstance`。

## 七、面试常见问题

**Q1：装饰器是什么？用一句话解释。**

装饰器是一个接受函数、返回函数的函数，用于在不修改原函数代码的前提下给它增加额外功能。`@timer` 写在函数上面，等价于 `func = timer(func)`。

**Q2：`@classmethod` 和 `@staticmethod` 的区别？**

- `@classmethod`：第一个参数是类本身（`cls`），可以访问和修改类变量，常用于工厂方法（如 `Model.from_pretrained()`）
- `@staticmethod`：没有 `self` 或 `cls`，完全是普通函数，只是逻辑上属于这个类，不能访问类或实例的任何状态

**Q3：`yield` 和 `return` 的区别？**

`return` 结束函数并返回值；`yield` 暂停函数、返回一个值，下次调用 `next()` 时从暂停处继续。有 `yield` 的函数是生成器函数，调用它不立即执行，而是返回一个生成器对象。

**Q4：Python 的 GIL 对多线程有什么影响？大模型工程里用多线程还是多进程？**

GIL（全局解释器锁）使得同一时刻只有一个线程在执行 Python 字节码，所以 CPU 密集型任务用多线程没有并行加速效果。

- **I/O 密集型**（API 调用、文件读写）→ 用多线程或 `asyncio`（GIL 在 I/O 等待期间释放）
- **CPU 密集型**（数据预处理、tokenization）→ 用多进程（`multiprocessing`，每个进程独立 GIL）
- **GPU 计算**（训练、推理）→ Python 代码只是调度，实际计算在 GPU 上，GIL 不影响

**Q5：`async/await` 和多线程的区别？什么时候用哪个？**

- 多线程：操作系统调度，适合阻塞性 I/O（无法修改为 async 的同步库）
- `async/await`：用户态调度（事件循环），开销更小，适合大量短时 I/O（API 调用、数据库查询）

大模型工程里并发调用 LLM API，优先用 `asyncio.gather`；调用没有 async 版本的老库，用 `loop.run_in_executor`。

**Q6：什么是闭包？装饰器里的闭包起什么作用？**

闭包是指一个函数"记住"了定义时所在的外部作用域的变量，即使那个外部函数已经返回了。装饰器里的 `wrapper` 函数就是闭包——它记住了外部的 `func` 变量，这样调用 `wrapper` 时才能调用原来的函数。

**Q7：Pydantic 和 dataclass 的区别？**

- `dataclass`：Python 内置，只做"自动生成 `__init__`、`__repr__` 等"，没有运行时类型验证
- `Pydantic BaseModel`：额外做运行时类型校验、类型强制转换、JSON 序列化/反序列化，还有字段级验证器

大模型 API 的请求/响应、LLM 应用的配置文件，用 Pydantic 更安全。

**Q8：`*args` 和 `**kwargs` 是什么？有什么应用场景？**

- `*args`：把位置参数打包成 tuple，接受任意数量的位置参数
- `**kwargs`：把关键字参数打包成 dict，接受任意数量的命名参数

装饰器的 `wrapper(*args, **kwargs)` 这样写是为了透明地把参数转发给被装饰的原函数，不管原函数有什么参数都能通用。

**Q9：`__enter__` 和 `__exit__` 是干嘛的？**

实现了这两个方法的对象可以用在 `with` 语句里。`with` 块开始时调用 `__enter__`，结束时（不管是正常结束还是抛异常）调用 `__exit__`。`torch.no_grad()` 就是这样实现的：进入时关闭梯度计算，离开时恢复。

**Q10：类型提示在运行时有用吗？**

类型提示本身在运行时不强制验证。但 `from __future__ import annotations` 之后，注解会被延迟求值。Pydantic 在运行时会读取类型提示并做验证。`typing.get_type_hints()` 可以在运行时获取类型信息。
