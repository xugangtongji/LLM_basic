## 零、开篇：大模型是四种范式的融合体

很多人以为大模型只是"超大的监督学习"，其实它是四种学习范式的精妙融合：

- **预训练**（大量无标注文本）→ **自监督学习**
- **SFT 指令微调**（人写的指令-回答对）→ **监督学习**
- **RLHF 对齐**（人类偏好排序）→ **强化学习**
- **词向量空间、CLIP 嵌入**→ **无监督 / 对比学习**

搞懂四种范式的边界和特点，你才能真正理解大模型训练流水线的每一步在做什么，以及为什么要这样做。

## 一、监督学习（Supervised Learning）

### 1.1 初印象：用费曼视角理解

**监督学习就是"带答案做题"**。

老师给你 1000 道数学题，每道题都附有正确答案。你反复对照答案改正思路，考试时就能独立解题。

机器学习里：题目 = 输入 $x$，答案 = 标签 $y$，"对照答案改思路"= 最小化损失函数，"能独立解题"= 泛化到新数据。

**大模型里的监督学习出现在哪里**：
- **SFT（监督微调）**：人工编写指令 + 标准回答，模型从中学习"这类问题该怎么回答"
- **奖励模型（RM）训练**：标注者给多个回答排序，从排序对构造监督信号
- **下游任务微调**：把预训练模型接上分类头，用标注数据有监督训练

### 1.2 形式化定义

给定数据集 $\mathcal{D} = \{(x_1, y_1), \ldots, (x_n, y_n)\}$，目标是学习映射 $f_\theta: \mathcal{X} \to \mathcal{Y}$，最小化：

$$\mathcal{L}(\theta) = \frac{1}{n} \sum_{i=1}^n \ell(f_\theta(x_i), y_i)$$

**不同任务对应的损失函数**：

| 任务类型 | 标签 $y$ | 损失函数 | 大模型应用 |
|---------|---------|---------|-----------|
| 多分类 | 类别整数 | 交叉熵 CrossEntropy | 分类 head、SFT token 预测 |
| 二分类 | 0 / 1 | BCEWithLogitsLoss | 奖励模型二分类变体 |
| 回归 | 实数 | MSE / MAE | 奖励模型分数预测 |
| 序列生成 | token 序列 | NLL（逐位交叉熵之和） | 语言模型 SFT |

### 1.3 SFT 的监督学习本质

SFT 的损失函数是：

$$\mathcal{L}_{\text{SFT}} = -\sum_{t=1}^{T} \mathbb{1}[\text{response token}] \cdot \log P_\theta(w_t \mid w_{<t})$$

其中 $\mathbb{1}[\text{response token}]$ 是掩码——**只对回答部分计算 loss，指令部分不计算**。这是因为我们希望模型学会"给定指令时如何回答"，而不是去预测指令本身。

```python
import torch
import torch.nn.functional as F

def sft_loss(logits, labels, ignore_index=-100):
    """
    logits: (B, L, V)  模型输出
    labels: (B, L)     目标 token，指令部分置为 ignore_index
    """
    # CrossEntropyLoss 内部做 log_softmax，并自动忽略 ignore_index=-100 的位置
    loss = F.cross_entropy(
        logits.view(-1, logits.size(-1)),  # (B*L, V)
        labels.view(-1),                   # (B*L,)
        ignore_index=ignore_index,
        reduction="mean",
    )
    return loss

# 构造示例
B, L, V = 2, 10, 32000
logits = torch.randn(B, L, V)
labels = torch.randint(0, V, (B, L))
# 把前 4 个 token（指令部分）设为 ignore
labels[:, :4] = -100

loss = sft_loss(logits, labels)
print(f"SFT Loss: {loss.item():.4f}")
```

## 二、无监督学习（Unsupervised Learning）

### 2.1 初印象：用费曼视角理解

**无监督学习就是"没有答案自己找规律"**。

把 100 万篇文章扔给模型，不告诉它任何标签，让它自己发现"哪些词意思相近"、"这段文字属于什么主题"、"这些句子有什么共同点"。这就是无监督学习。

它的核心假设是：**数据本身的分布蕴含了足够丰富的结构**，不需要人来打标签就能学到有用的表示。

### 2.2 三类主要方法

**聚类（Clustering）**：把相似的数据点归到一组。

- K-means、层次聚类、DBSCAN
- 大模型中的应用：对训练数据去重和多样性采样、对模型的隐状态聚类分析

**降维（Dimensionality Reduction）**：把高维数据压缩到低维而保留主要结构。

- PCA、t-SNE、UMAP
- 大模型中的应用：可视化词向量空间、分析模型隐层表示

**密度估计（Density Estimation）**：学习数据的概率分布。

- GMM（高斯混合模型）、核密度估计
- 大模型中的应用：VAE 的隐变量先验、异常检测

## 三、自监督学习（Self-Supervised Learning）：大模型的根基

### 3.1 初印象：用费曼视角理解

**自监督学习是"用数据本身制造监督信号"**。

没有人给标签，但数据本身可以给：

- 把句子中的一个词盖住，让模型猜这个词（BERT 的 MLM）
- 把句子的后半部分盖住，让模型预测下一个词（GPT 的 CLM）
- 把图像的一块区域盖住，让模型重建（MAE）
- 给同一张图做两种增强，让模型认出它们是同一张（SimCLR 对比学习）

**自监督和无监督的区别**：无监督是"完全没有目标"地学结构；自监督是"用数据制造了一个伪标签"，本质上还是在做有监督的预测任务，只是标签来自数据自身。

这就是为什么<自监督学习可以充分利用互联网上海量的无标注文本>，却又能学到如此强大的表示。

### 3.2 三大自监督范式

**范式一：自回归语言模型（Autoregressive LM / CLM）**

**代表**：GPT-1/2/3/4、LLaMA、Qwen、DeepSeek

训练目标：预测下一个 token

$$\mathcal{L}_{\text{CLM}} = -\sum_{t=1}^{T} \log P_\theta(w_t \mid w_1, \dots, w_{t-1})$$

- 使用 Causal Mask，每个位置只能看到左侧的 token
- 天然适合**文本生成**任务
- 预训练后具备 In-context Learning 能力（Zero/Few-shot）

**范式二：掩码语言模型（Masked LM / MLM）**

**代表**：BERT、RoBERTa、DeBERTa

训练目标：预测被掩盖的 token

$$\mathcal{L}_{\text{MLM}} = -\sum_{t \in \mathcal{M}} \log P_\theta(w_t \mid w_{\setminus \mathcal{M}})$$

其中 $\mathcal{M}$ 是被随机掩盖的位置集合，模型可以同时看到左右两侧的上下文（双向注意力）。

- 适合**理解类**任务（分类、NER、问答）
- 不适合直接做生成（因为推理时没有遮掩可以预测）

**BERT 的 Masking 策略**：随机选 15% 的 token，其中 80% 替换成 `[MASK]`，10% 替换成随机词，10% 保持不变——这样模型无法"偷懒"只关注 `[MASK]` 位置，必须学习每个位置的真实表示。

**范式三：对比学习（Contrastive Learning）**

**代表**：CLIP（图文）、SimCLR（视觉）、SimCSE（文本）

训练目标：把正样本对拉近，负样本对推远

$$\mathcal{L}_{\text{InfoNCE}} = -\log \frac{\exp(\text{sim}(z_i, z_i^+) / \tau)}{\sum_{j=1}^{N} \exp(\text{sim}(z_i, z_j) / \tau)}$$

其中 $z_i^+$ 是 $z_i$ 的正样本对，$\tau$ 是温度系数，$N$ 是 batch 内样本数。

CLIP 的做法：同一图文对是正样本，batch 内其他所有图文组合是负样本：

```python
import torch
import torch.nn.functional as F

def clip_loss(image_features, text_features, temperature=0.07):
    """
    image_features: (N, D)，已归一化
    text_features:  (N, D)，已归一化
    N: batch size，每张图对应一段文字
    """
    # 相似度矩阵 (N, N)：第 i 行第 j 列是图 i 和文 j 的相似度
    logits = (image_features @ text_features.T) / temperature

    # 对角线是正样本对，其余是负样本
    # 图→文 loss：每行的正确类别是对角线 i
    # 文→图 loss：每列的正确类别是对角线 i
    labels = torch.arange(len(image_features), device=image_features.device)
    loss_i = F.cross_entropy(logits, labels)           # 图视角
    loss_t = F.cross_entropy(logits.T, labels)         # 文视角
    return (loss_i + loss_t) / 2

# 模拟
N, D = 32, 512
img_feat = F.normalize(torch.randn(N, D), dim=-1)
txt_feat = F.normalize(torch.randn(N, D), dim=-1)
loss = clip_loss(img_feat, txt_feat)
print(f"CLIP Loss: {loss.item():.4f}")
```

### 3.3 自监督学习如何产生 Zero-shot 能力

这是大模型最神奇的特性之一。模型在预训练时从未见过"写一首七言绝句"这样的指令，但经过预训练后，它往往能做到。为什么？

**原因一：任务格式已经出现在训练数据里**
互联网上的文本包含了无数示例：FAQ、教程、问答帖子。模型见过"问题-答案"的格式，就学会了这个模式。

**原因二：语言是任务的通用接口**
语言本身能描述任何任务。自回归语言模型的训练目标"预测下一个词"，恰好让它学会了"给定上下文完成文本"——而任何任务都可以被表述成"给定上下文完成文本"。

**原因三：涌现能力（Emergent Abilities）**
规模足够大之后（据观察约 10B+ 参数），模型突然获得了小模型不具备的能力，比如多步推理、类比迁移。这被称为涌现，其机制目前仍有争议，但是不可否认的实验现象。

## 四、强化学习（Reinforcement Learning）

### 4.1 初印象：用费曼视角理解

**强化学习就是"试错学习"**——没有人告诉你每一步对不对，只在你做完一系列操作之后，给你一个总体评分（奖励），你自己摸索出哪种策略能得到更高的分。

类比：你在玩一款你完全不懂规则的游戏。没有说明书，没有老师，只有得分。通过大量尝试，你逐渐发现"这种操作序列能得高分"，于是坚持这种风格——这就是 RL。

**大模型里的强化学习**出现在对齐阶段：

- **RLHF（PPO）**：人类对模型回答的偏好充当奖励信号，PPO 算法让模型策略朝"人类更喜欢的方向"优化
- **GRPO / DeepSeek R1**：用规则化奖励（答案是否正确）训练推理模型
- **RLAIF**：用另一个 AI 模型替代人类打分

### 4.2 核心概念

**四要素**：

| 要素 | RL 术语 | 大模型里的对应 |
|------|---------|--------------|
| 智能体 | Agent | LLM（策略模型 $\pi_\theta$） |
| 环境 | Environment | 用户输入 + 奖励模型 |
| 状态 | State | 当前对话上下文 |
| 动作 | Action | 生成下一个 token |
| 奖励 | Reward | 奖励模型给完整回答的分数 |

**策略梯度**（Policy Gradient）：

$$\nabla_\theta \mathcal{J}(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[R(\tau) \nabla_\theta \log \pi_\theta(\tau)]$$

通俗：如果一个回答得分高（$R > 0$），就增大生成这个回答的概率；得分低就降低。

**PPO 的 KL 约束**（防止模型跑偏）：

$$\mathcal{L}_{\text{PPO}} = \mathbb{E}[R(\tau)] - \beta \cdot D_{KL}(\pi_\theta \| \pi_{\text{ref}})$$

```python
# PPO 奖励塑形（Reward Shaping）的核心逻辑
# 在 trl 库中的简化版实现

def compute_reward_with_kl_penalty(
    reward_scores,    # 奖励模型打分 (B,)
    log_probs_new,    # 新策略的 log 概率 (B, L)
    log_probs_ref,    # 参考策略的 log 概率 (B, L)
    kl_coef: float = 0.1,
):
    """
    实际奖励 = 奖励模型分数 - KL 惩罚
    只在最后一个 token 处加奖励（episode 末尾）
    """
    # 逐 token 的 KL 散度（近似）
    kl_per_token = log_probs_new - log_probs_ref    # (B, L)
    kl_penalty = kl_coef * kl_per_token.sum(dim=-1) # (B,)

    # 最终奖励：外部奖励 - KL 惩罚
    total_reward = reward_scores - kl_penalty
    return total_reward
```

### 4.3 四种范式在大模型训练流水线中的位置

```
第一阶段：预训练（万亿 token 无标注文本）
    范式：自监督学习（自回归 CLM）
    目标：让模型学会语言的基本规律、世界知识

第二阶段：SFT 指令微调（数万条指令-回答对）
    范式：监督学习
    目标：让模型学会按照指令格式回答

第三阶段：奖励模型训练（人工偏好排序数据）
    范式：监督学习（二分类或回归）
    目标：学一个能评估回答质量的打分模型

第四阶段：RLHF 对齐（PPO / DPO / GRPO）
    范式：强化学习 / 偏好优化
    目标：让模型的回答风格更符合人类偏好

可选阶段：对比学习预训练（CLIP、SimCSE 等）
    范式：自监督对比学习
    目标：获得高质量的多模态嵌入空间
```

## 五、半监督学习与元学习

### 5.1 半监督学习

**初印象**：大量无标签数据 + 少量有标签数据，两者结合。

在大模型时代，半监督学习更多以**伪标签**（pseudo-labeling）形式出现：先用有标签数据训练一个小模型，再用它给无标签数据打标签，把置信度高的伪标签加入训练集扩大规模。

**在 LLM 里的体现**：
- Self-training（自训练）：用强模型的输出训练弱模型（蒸馏的一种）
- 合成数据（Synthetic Data）：用 GPT-4 给没有标注的领域数据生成标注，再 SFT 小模型

### 5.2 元学习（Meta-learning）/ 少样本学习

**初印象**：**"学会学习"** — 不是学会一个具体任务，而是学会"如何快速适应新任务"。

**大模型的 In-Context Learning（ICL）本质上是元学习的体现**：

```
Few-shot prompt 结构：
[任务描述]
[示例 1: 输入 → 输出]
[示例 2: 输入 → 输出]
[示例 3: 输入 → 输出]
[新问题: 输入 → ???]
```

模型不需要更新任何参数，只是看了几个例子就能解决新问题。这表明预训练赋予了模型一种"元学习能力"——从上下文中快速推断出任务的规律。

## 六、四种范式的核心对比

| 维度 | 监督学习 | 无监督学习 | 自监督学习 | 强化学习 |
|------|---------|-----------|-----------|---------|
| 标签来源 | 人工标注 | 无标签 | 数据自生成 | 环境奖励信号 |
| 数据效率 | 低（贵） | 高（大量数据） | 高（大量数据） | 低（需大量交互） |
| 优化目标 | 最小化预测误差 | 无显式目标/重建 | 预测被遮盖部分 | 最大化累积奖励 |
| 典型算法 | 交叉熵 + SGD | K-means / PCA | CLM / MLM / InfoNCE | PPO / DPO / GRPO |
| 大模型中的阶段 | SFT、RM 训练 | 数据分析 | 预训练 | RLHF 对齐 |
| 主要优点 | 直接、精准 | 不需要标注 | 可利用海量数据 | 可优化非可微目标 |
| 主要缺点 | 标注贵、泛化弱 | 难以评估质量 | 预训练目标和下游任务不对齐 | 奖励稀疏、不稳定 |

## 七、面试常见问题

**Q1：自监督学习和无监督学习的区别是什么？**

无监督学习完全没有目标，模型自己找数据的内在结构（聚类、降维）。自监督学习则用数据的一部分预测另一部分，制造了一个有明确损失函数的预测任务——从这个角度看，它本质上是有监督的，只是标签来自数据本身而非人工标注。GPT 预测下一个词是自监督，不是无监督。

**Q2：BERT 和 GPT 的预训练目标有什么区别？各自适合什么任务？**

BERT 用 MLM（掩码语言模型）：随机盖住 15% 的词，模型双向看上下文来预测。适合理解类任务（分类、NER、问答），因为双向注意力能充分理解上下文语义。

GPT 用 CLM（因果语言模型）：从左到右逐词预测，只能看前面的上下文。适合生成类任务（对话、写作、代码生成），因为训练目标和生成过程完全一致。

代价各有取舍：BERT 能利用双向信息，但不能直接生成；GPT 天然能生成，但预训练时没有充分利用右侧上下文。

**Q3：RLHF 中的强化学习解决了什么问题？为什么不直接用 SFT？**

SFT 的问题：模型学习的是模仿"给出的示例"，但示例的质量有限，而且 SFT 的目标（Token 级别交叉熵）不能直接优化"回答是否有帮助"这个整体质量指标。

RLHF 用人类偏好作为奖励信号，把不可微的"人类满意度"转化成可优化的奖励，让模型可以超越示例数据，生成"比任何示例都好"的回答。同时 KL 约束防止模型走偏（reward hacking）。

**Q4：什么是 In-Context Learning？为什么大模型能做到？**

ICL（上下文学习）是指模型在推理时只看几个示例（不更新参数），就能适应新任务的能力。

可能原因有几种解释：预训练数据中包含了大量"输入-输出"格式的文本；模型参数中隐式地存储了"元学习"能力，能从 prompt 的模式推断任务类型；也有人认为 ICL 等价于对隐式贝叶斯推断的近似。目前学界对此尚无定论，但大规模预训练是其必要条件已达成共识。

**Q5：自监督的对比学习（CLIP）和自回归语言模型（GPT）有什么本质区别？**

方向不同：
- CLM（GPT）：生成式，学会"给定前缀生成后续"，结果是一个能生成文本的模型
- 对比学习（CLIP）：判别式，学会"哪些数据点应该靠近、哪些应该远离"，结果是一个高质量的嵌入空间

用途不同：CLM 主要用于生成；对比学习主要用于检索、Zero-shot 分类、跨模态对齐。两者不互斥，现代多模态模型（GPT-4o、Gemini）往往结合两者。

**Q6：为什么强化学习在 LLM 中特别难训练？**

三大挑战：奖励稀疏（一个完整的回答才有一个分数，而回答可能有几百个 token，很难知道哪个 token 导致了好分数或差分数）；奖励黑客（模型可能学会用语言技巧糊弄奖励模型而非真正改善质量）；训练不稳定（PPO 的超参数敏感，梯度方差大）。DPO 等方法就是为了规避显式的 RL 而设计的。

**Q7：伪标签（Pseudo Labeling）和自监督学习有什么关系？**

两者都不依赖人工标注，但机制不同。自监督学习用数据自身的结构制造标签（遮掩-预测）；伪标签是先用一个已经训练好的模型对无标注数据做推断，把推断结果当作标签用于后续训练。伪标签更接近半监督学习，而且会引入噪声（模型自身的预测误差）。在 LLM 里，用 GPT-4 生成的数据训练小模型，本质上就是伪标签 + 知识蒸馏的组合。

**Q8：元学习（Meta-Learning）的"学会学习"具体是什么意思？**

传统机器学习学的是"如何解决特定任务"（对某个 $f: x \to y$ 拟合）。元学习学的是"如何快速学习任意新任务"，即优化的目标是"从 K 个样本中适应新任务的速度和质量"。大模型的 ICL 被认为是一种隐式的元学习——预训练中模型见过各种各样的任务格式，获得了"看几个例子就能推断规律"的能力。
