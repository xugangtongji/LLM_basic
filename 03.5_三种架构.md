## 零、开篇：同样是 Transformer，为什么 BERT 和 GPT 如此不同

2018 年，OpenAI 发布了 GPT，Google 发布了 BERT。两者都基于 Transformer，但用途完全不同：

- GPT 是"写作高手"，给你一个开头，它能继续写下去
- BERT 是"阅读理解高手"，给它一段文章，它能理解语义、回答问题

同样的 Transformer 积木，为什么能搭出这么不同的两样东西？

答案藏在一个关键设计里：**注意力的方向**。

## 一、注意力方向：决定一切的设计选择

### 1.1 初印象：用费曼视角理解

想象两种不同的读书方式：

**读法一（双向阅读）**：你拿到一本书，可以跳来跳去随意翻阅——看第一章时可以参考第十章，分析某个句子时可以联系上下文任意段落。

**读法二（单向阅读）**：你读一本悬疑小说，被要求一页一页往下看，绝对不能翻后面的剧情。你读到某个词时，只能参考之前读过的内容。

- 读法一适合"理解"任务——你有全部信息，慢慢分析
- 读法二适合"预测/生成"任务——你模拟真实生成场景，根据已有信息猜下一步

**Encoder 用的是读法一（双向），Decoder 用的是读法二（单向）。**

### 1.2 具体到注意力矩阵

以句子"今天 天气 很 好"（4个词）为例：

**双向注意力（Encoder）——每个词能看所有词**：

```
注意力矩阵（每行和为1，×=可以看）：

          今天   天气   很    好
今天  [  ×      ×      ×     ×  ]   ← 处理"今天"时，能看所有4个词
天气  [  ×      ×      ×     ×  ]   ← 处理"天气"时，能看所有4个词
很    [  ×      ×      ×     ×  ]   ← 处理"很"时，能看所有4个词
好    [  ×      ×      ×     ×  ]   ← 处理"好"时，能看所有4个词
```

**单向注意力（Decoder）——每个词只能看自己和之前的词**：

```
注意力矩阵（×=可以看，○=被掩码屏蔽）：

          今天   天气   很    好
今天  [  ×      ○      ○     ○  ]   ← 处理"今天"时，只能看自己
天气  [  ×      ×      ○     ○  ]   ← 处理"天气"时，能看"今天"和"天气"
很    [  ×      ×      ×     ○  ]   ← 处理"很"时，能看前3个词
好    [  ×      ×      ×     ×  ]   ← 处理"好"时，能看所有词
```

这一个设计差异，就导致了 Encoder 和 Decoder 完全不同的能力和用途。

## 二、Encoder-only：深度理解型

### 2.1 初印象：用费曼视角理解

**Encoder 是一个"深度阅读机器"——它把输入文字变成一组包含丰富上下文信息的向量，但不生成任何新文字。**

因为每个词都能看到所有其他词（双向注意力），Encoder 对每个词的理解是"全局的"——"苹果"在"我买了一个苹果"和"苹果公司发布了新手机"中，Encoder 能产生截然不同的向量表示，精确捕捉每种语境下的语义。

### 2.2 Encoder 的工作流程

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class EncoderBlock(nn.Module):
    """
    Encoder 块：双向 Self-Attention（无因果掩码）+ FFN。
    和 Decoder 块的唯一区别：没有因果掩码，每个词能看到所有词。
    """
    def __init__(self, d_model: int, n_heads: int, d_ff: int, dropout: float = 0.1):
        super().__init__()
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.attention = nn.MultiheadAttention(
            d_model, n_heads, dropout=dropout, batch_first=True
        )
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model),
            nn.Dropout(dropout),
        )

    def forward(self, x: torch.Tensor, padding_mask=None):
        # 注意：这里没有因果掩码！
        # 每个词可以看到序列中所有其他词（双向）
        x_norm = self.norm1(x)
        attn_out, _ = self.attention(
            x_norm, x_norm, x_norm,
            key_padding_mask=padding_mask   # 只屏蔽 padding，不屏蔽未来
        )
        x = x + attn_out
        x = x + self.ffn(self.norm2(x))
        return x


class BERTStyleModel(nn.Module):
    """
    BERT 风格的 Encoder-only 模型。
    用途：把输入文字转换成语义向量，用于分类、问答、命名实体识别等。
    """
    def __init__(self, vocab_size, d_model, n_heads, n_layers, d_ff, num_classes=None):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, d_model)
        self.pos_embedding = nn.Embedding(512, d_model)
        self.blocks = nn.ModuleList([
            EncoderBlock(d_model, n_heads, d_ff) for _ in range(n_layers)
        ])
        self.final_norm = nn.LayerNorm(d_model)
        # 可选：在 [CLS] token 上接分类头
        if num_classes:
            self.classifier = nn.Linear(d_model, num_classes)
        else:
            self.classifier = None

    def forward(self, token_ids):
        B, L = token_ids.shape
        positions = torch.arange(L, device=token_ids.device)
        x = self.embedding(token_ids) + self.pos_embedding(positions)

        for block in self.blocks:
            x = block(x)
        x = self.final_norm(x)

        if self.classifier:
            # 取第一个 token ([CLS]) 的输出做分类
            cls_output = x[:, 0, :]       # (B, d_model)
            return self.classifier(cls_output)   # (B, num_classes)
        return x   # (B, L, d_model)：每个 token 的上下文表示


# 验证
bert = BERTStyleModel(vocab_size=1000, d_model=64, n_heads=4,
                       n_layers=4, d_ff=256, num_classes=3)
token_ids = torch.randint(0, 1000, (2, 20))
logits = bert(token_ids)
print(f"分类输出: {logits.shape}")   # (2, 3)  ← 2个句子，3分类
```

### 2.3 Encoder 的典型应用

| 任务 | 输入 | 输出 | 代表模型 |
|------|------|------|---------|
| 文本分类 | 一句话 | 类别标签 | BERT |
| 情感分析 | 评论文本 | 正/负/中性 | RoBERTa |
| 命名实体识别 | 句子 | 每个词的实体标签 | BERT |
| 语义相似度 | 两句话 | 相似度分数 | Sentence-BERT |
| **RAG 检索的 Embedding** | 文档片段 | 向量表示 | BGE、E5、all-MiniLM |

**最后一行特别重要**：现在 RAG 系统里用来把文档转换成向量的"嵌入模型"，基本上都是 Encoder-only 模型。你输入一段文字，它输出一个稠密向量，用于向量检索。

## 三、Decoder-only：生成型

### 3.1 初印象：用费曼视角理解

**Decoder 是一个"续写机器"——给定已有文字，预测下一个词应该是什么。**

因为使用了因果掩码（单向注意力），模型在预测第 $t$ 个词时，只能看到前 $t-1$ 个词。这让模型的训练方式和推理方式完全一致：训练时预测"下一个词是什么"，推理时也是一步步预测下一个词。

### 3.2 自回归生成的过程

```
问题："法国的首都是"

步骤1: 输入["法国的首都是"]       → 模型预测下一个词 → "巴"   (概率最高)
步骤2: 输入["法国的首都是", "巴"] → 模型预测下一个词 → "黎"
步骤3: 输入["法国的首都是", "巴", "黎"] → 预测 → "。"（或 EOS）
步骤4: 遇到结束符，停止生成

最终输出："法国的首都是巴黎。"
```

这种一步一步生成的方式叫做**自回归（Autoregressive）生成**——每次的输出作为下一次的输入。

### 3.3 Decoder 的典型应用

| 任务 | 代表模型 |
|------|---------|
| 对话聊天 | ChatGPT、Claude、Qwen-Chat |
| 文章写作 | GPT-4、Gemini |
| 代码生成 | GitHub Copilot、DeepSeek-Coder |
| 指令执行 | LLaMA-3-Instruct、Mistral-Instruct |
| 数学推理 | DeepSeek-R1、o1 |

**几乎所有你现在用的 AI 助手，底层都是 Decoder-only 模型。**

## 四、Encoder-Decoder：转换型

### 4.1 初印象：用费曼视角理解

**Encoder-Decoder 是一个"翻译机器"——先完整理解输入（Encoder），再根据理解逐步生成输出（Decoder）。**

这种架构适合"输入和输出是不同语言/格式"的任务：

- 中文 → 英文（机器翻译）
- 长文章 → 短摘要（文本摘要）
- 语音 → 文字（语音识别）
- 表格 → 描述（结构化数据到自然语言）

### 4.2 Cross-Attention：两个部分如何沟通

Encoder-Decoder 架构里有一个特殊的机制叫 **Cross-Attention（交叉注意力）**：

- Encoder 处理完输入，输出一组向量（代表对输入的理解）
- Decoder 每生成一个词时，通过 Cross-Attention 去查询 Encoder 的输出
- Cross-Attention 里，Q 来自 Decoder，K/V 来自 Encoder——Decoder 问"我现在需要什么信息"，Encoder 回答"我理解的内容里哪些和你的需求相关"

```python
class CrossAttention(nn.Module):
    """
    Cross-Attention：Q 来自 Decoder，K/V 来自 Encoder。
    用于 Encoder-Decoder 架构中让 Decoder 查询 Encoder 的信息。
    """
    def __init__(self, d_model: int, n_heads: int):
        super().__init__()
        self.attention = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.norm = nn.LayerNorm(d_model)

    def forward(self, decoder_x, encoder_output):
        """
        decoder_x:      (B, target_len, d_model)  Decoder 当前状态
        encoder_output: (B, source_len, d_model)  Encoder 的完整输出
        """
        x_norm = self.norm(decoder_x)
        # Q 来自 Decoder，K/V 来自 Encoder
        attn_out, attn_weights = self.attention(
            query = x_norm,           # Decoder 问："我需要什么？"
            key   = encoder_output,   # Encoder 答："我有这些内容"
            value = encoder_output,   # Encoder 提供实际信息
        )
        return decoder_x + attn_out, attn_weights


class EncoderDecoderLayer(nn.Module):
    """
    Encoder-Decoder 架构的 Decoder 层：
    含三个子层：
    1. Masked Self-Attention（Decoder 内部，单向）
    2. Cross-Attention（查询 Encoder 输出）
    3. FFN
    """
    def __init__(self, d_model: int, n_heads: int, d_ff: int):
        super().__init__()
        # 1. Decoder 自身的因果注意力
        self.self_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.norm1 = nn.LayerNorm(d_model)
        # 2. 交叉注意力
        self.cross_attn = CrossAttention(d_model, n_heads)
        # 3. FFN
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
        )
        self.norm3 = nn.LayerNorm(d_model)

    def forward(self, x, encoder_output, causal_mask=None):
        # 步骤1：Decoder 内部的因果 Self-Attention
        x_norm = self.norm1(x)
        self_out, _ = self.self_attn(x_norm, x_norm, x_norm, attn_mask=causal_mask)
        x = x + self_out

        # 步骤2：Cross-Attention（查询 Encoder）
        x, cross_weights = self.cross_attn(x, encoder_output)

        # 步骤3：FFN
        x = x + self.ffn(self.norm3(x))
        return x, cross_weights


# 验证 Encoder-Decoder 的数据流
d_model, n_heads, d_ff = 64, 4, 256
enc_layer = EncoderBlock(d_model, n_heads, d_ff)
dec_layer = EncoderDecoderLayer(d_model, n_heads, d_ff)

source = torch.randn(2, 10, d_model)   # 源语言：2句话，每句10个token
target = torch.randn(2, 7,  d_model)   # 目标语言：2句话，每句7个token

encoder_out = enc_layer(source)              # Encoder 处理源语言
L = target.size(1)
causal_mask = torch.triu(torch.ones(L, L), diagonal=1).bool()
decoder_out, weights = dec_layer(target, encoder_out, causal_mask)

print(f"Encoder 输出: {encoder_out.shape}")   # (2, 10, 64)
print(f"Decoder 输出: {decoder_out.shape}")   # (2, 7, 64)
print(f"Cross-Attention 权重: {weights.shape}")  # (2, 7, 10)  ← 目标每词关注源句哪些词
```

## 五、三种架构横向对比

### 5.1 一张表看清楚

| 维度 | Encoder-only | Decoder-only | Encoder-Decoder |
|------|-------------|--------------|-----------------|
| **注意力方向** | 双向（全部可见）| 单向（只看过去）| Enc 双向 + Dec 单向 |
| **擅长任务** | 理解、分类 | 生成、对话 | 翻译、摘要 |
| **预训练目标** | 预测被掩盖的词（MLM）| 预测下一个词（CLM）| 重建被破坏的序列 |
| **代表模型** | BERT、RoBERTa | GPT、LLaMA、Claude | T5、BART、Whisper |
| **当前主流地位** | 嵌入模型、分类器 | **几乎所有主流 LLM** | 特定场景专用 |

### 5.2 为什么 Decoder-only 成为大模型的主流？

这是一个有意思的问题——为什么当今最强的模型（GPT-4、Claude、LLaMA、Qwen）全都是 Decoder-only？

**原因一：预训练目标统一**

Decoder-only 的训练目标（预测下一个词）和推理方式（逐词生成）完全一致——训练时怎么用，推理时就怎么用，没有不匹配的问题。

Encoder 的 MLM 训练（随机掩盖词再预测）和实际使用场景有一些差距，需要额外的微调来适配。

**原因二：涌现能力（Emergent Ability）**

研究发现，当 Decoder-only 模型参数量超过一定规模（约 10B+），会突然出现之前没有的能力：多步推理、代码生成、数学解题……这种"规模带来质变"的特性，在 Encoder-only 模型上不明显。

**原因三：In-Context Learning（上下文学习）**

只需要在 prompt 里给几个例子，Decoder-only 模型就能学会新任务，无需重新训练——这叫 Few-Shot Learning。这种能力让 Decoder-only 模型极其灵活，一个模型可以完成数百种任务。

**原因四：工程更简单**

Decoder-only 只有一个组件（Decoder），没有 Encoder 和 Cross-Attention，实现和调试更简单，更容易规模化。

## 六、选择架构的实用指南

```
你的任务是什么？

├── 需要"理解"文本，生成一个标签/向量/分数
│   ├── 文本分类（正面/负面/中性）
│   ├── 情感分析
│   ├── 把文档转成向量用于搜索（RAG 检索）
│   ├── 文本相似度比较
│   └── → 选 Encoder-only（BERT、BGE、E5 等）
│
├── 需要"生成"文本，从输入产生新的文字
│   ├── 聊天对话
│   ├── 文章写作、代码生成
│   ├── 问题回答（开放式）
│   ├── 指令执行
│   └── → 选 Decoder-only（GPT、LLaMA、Qwen 等）
│
└── 需要把一种形式"转换"成另一种形式
    ├── 语言翻译（中文→英文）
    ├── 长文摘要（文章→几句话）
    ├── 语音转文字
    └── → 选 Encoder-Decoder（T5、BART、Whisper 等）

注意：现代 Decoder-only 大模型（如 GPT-4、Claude）
也能做翻译和摘要！
但专门的 Encoder-Decoder 在这些任务上往往更高效，
因为它能用更少的参数达到相近的效果。
```

## 八、一个具体例子：同一句话，三种模型的处理方式

输入：**"这部电影的特效非常震撼，但剧情有些薄弱。"**

**Encoder-only（如 BERT）处理方式**：

```
输入 → 双向注意力处理整句话 → 每个词都充分理解了上下文
→ 取 [CLS] 的向量接分类头
→ 输出："正面"(30%) / "中性"(55%) / "负面"(15%)  → 预测：中性
```

**Decoder-only（如 GPT）处理方式**：

```
输入 → 单向逐词预测
→ "这"→"部"→"电"→"影"→"的"→...
→ 在给定这句话后，预测下一个词：
→ "总体" 或 "综上" 或 "建议" ... （生成续写）
或者配合 prompt：
→ "请给以下评论评分（1-5分）：这部电影..."
→ 生成："4"（分）
```

**Encoder-Decoder（如 T5）处理方式**：

```
Encoder 完整理解输入句
↓（Cross-Attention）
Decoder 根据 prompt 生成输出

Prompt: "情感分析：这部电影的特效..."
→ 生成："中性"

Prompt: "总结：这部电影的特效..."
→ 生成："特效出色但剧情欠佳"
```

## 九、面试常见问题

**Q1：BERT 和 GPT 的根本区别是什么？**

最根本的区别是注意力方向。BERT 使用双向注意力（Encoder），每个词可以同时看到前后所有词，非常适合理解类任务（分类、NER、问答）。GPT 使用单向因果注意力（Decoder），每个词只能看到自己和之前的词，适合生成类任务（对话、写作）。预训练目标也不同：BERT 随机掩盖词并预测（MLM），GPT 预测下一个词（CLM）。

**Q2：为什么现代大模型（GPT-4、Claude、LLaMA）几乎都是 Decoder-only？**

三个主要原因：预训练目标（预测下一词）和推理方式（逐词生成）完全一致，没有训练-推理差距；规模扩展时表现出更强的涌现能力（推理、代码等）；具备 In-Context Learning 能力，一个模型可以零/少样本完成数百种任务，灵活性极强。

**Q3：Cross-Attention 是什么？和 Self-Attention 有什么区别？**

Self-Attention 中 Q、K、V 来自同一个序列（词与自身序列交互）。Cross-Attention 中 Q 来自 Decoder，K/V 来自 Encoder 的输出（两个序列之间交互）。Cross-Attention 让 Decoder 在生成每个词时，能动态查询 Encoder 对源序列的理解，是 Encoder-Decoder 架构中两个部分沟通的桥梁。

**Q4：什么任务该选 Encoder，什么任务该选 Decoder？**

需要"理解"文本、输出标签/向量的任务选 Encoder（文本分类、情感分析、RAG 的嵌入模型）。需要"生成"文本、逐词输出的任务选 Decoder（对话、写作、代码生成）。需要把一种形式"转换"成另一种形式的任务可以选 Encoder-Decoder（翻译、摘要、语音识别），但现代 Decoder-only 大模型也能完成这些任务，只是参数利用效率可能略低。

**Q5：Encoder-only 模型（BERT）为什么不能直接用于文本生成？**

BERT 的双向注意力让每个词都能看到"未来"的词，这在生成任务中是不合法的——你不能在还没生成第 5 个词时就让第 3 个词看到第 5 个词。如果强行用 BERT 生成，就需要把它改成单向（加上因果掩码），但那样就和 BERT 的预训练方式不匹配了。所以 BERT 擅长理解，GPT 擅长生成，两者有本质的架构差异。
