## 一、正则化快速回顾：大模型视角的补充

### 1.1 Dropout 在大模型中的命运

**已经讲过**：Dropout 在前向时随机关闭 $p$ 比例的神经元，等效集成效应，防止特征共适应。

**大模型特有视角**：

大模型预训练中，Dropout 的使用正在减少。数据量足够大时，数据本身就是最好的正则化——模型没有机会过拟合单个样本，Dropout 的收益急剧降低，却带来额外计算和不确定性。

```python
# 主流 LLM 的 Dropout 配置
dropout_configs = {
    "GPT-2（2019）":    {"attn_dropout": 0.1, "ffn_dropout": 0.1},
    "GPT-3（2020）":    {"attn_dropout": 0.1, "ffn_dropout": 0.1},
    "LLaMA-1（2023）":  {"attn_dropout": 0.0, "ffn_dropout": 0.0},   # 取消
    "LLaMA-2（2023）":  {"attn_dropout": 0.0, "ffn_dropout": 0.0},
    "LLaMA-3（2024）":  {"attn_dropout": 0.0, "ffn_dropout": 0.0},
    "Mistral（2023）":  {"attn_dropout": 0.0, "ffn_dropout": 0.0},
    "Qwen-2（2024）":   {"attn_dropout": 0.0, "ffn_dropout": 0.0},
}
# 结论：2022 年后几乎所有大规模 LLM 预训练都关掉了 Dropout
# SFT / RLHF 微调（数据量小）有时会开 0.05-0.1 的小 dropout
```

**什么时候还要用 Dropout**：
- **SFT 微调**：数据量小（几万条），有过拟合风险时，在 attention 或 FFN 处开 0.05-0.1
- **小模型**（< 1B）在小数据集上微调时
- **RLHF 奖励模型**：训练数据往往只有几千到几万对，Dropout 是必要正则化

### 1.2 Label Smoothing 在 LLM 中的使用

**已经讲过**（1.1.4 信息论）：Label Smoothing 把 one-hot 目标分布软化，防止模型过度自信。

**大模型特有视角**：大多数 LLM 预训练不用 Label Smoothing（或用极小值如 0.0-0.1），因为：
- 预训练是自回归语言建模，有时候"下一个词"确实存在多个等价的正确答案，强行 smoothing 会引入噪声
- 超大数据量本身的多样性已经提供了足够的软化效果

SFT 阶段有时会用 `label_smoothing=0.1` 来防止模型对 SFT 数据过拟合。

### 1.3 数据增强的 LLM 形式

传统数据增强（图像翻转、裁剪、颜色抖动）在 LLM 里没有直接对应，但有三种类似机制：

**回译（Back-Translation）**：把句子翻译成另一种语言再翻译回来，生成语义等价但词汇不同的变体。

**指令扩充（Instruction Augmentation）**：用强模型把同一任务改写成多种指令形式，扩充 SFT 数据多样性（Evol-Instruct、WizardLM 的核心思路）。

**混合精度 Dropout（MCD，推理时不关 Dropout）**：在推理时保持 Dropout 开启，对同一输入采样多次，计算置信度区间——这是贝叶斯不确定性估计的近似，在安全敏感场景下有应用价值。

### 1.4 梯度惩罚与权重范数约束

大模型有时用以下技巧防止训练不稳定（区别于正则化，更接近稳定性手段）：

```python
import torch
import torch.nn as nn

# 技巧 1：QK LayerNorm（Attention 的 Q 和 K 各加 LayerNorm，防止 attention logit 爆炸）
class StableAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        self.q_norm = nn.LayerNorm(d_model // n_heads)  # 对 head 维度 norm
        self.k_norm = nn.LayerNorm(d_model // n_heads)

    # 在 Q K 做矩阵乘之前先 norm，防止 QK^T 数值过大

# 技巧 2：z-loss（对 logit 的稳定化正则）
# 用于 MoE 模型中防止路由 logit 无限增大
def z_loss(router_logits, coeff=1e-3):
    """让路由 logit 的 log_sum_exp 不过大"""
    return coeff * torch.logsumexp(router_logits, dim=-1).pow(2).mean()
```

## 二、残差连接的设计哲学

### 2.1 初印象：用费曼视角理解

**残差连接是"给信号留一条永远畅通的高速公路"**。

普通深层网络：信号每过一层都要经过变换，层数深了，信号就变得面目全非，梯度也随之消失。

残差连接：在变换之外，强制保留一条从输入直达输出的路径：$y = x + F(x)$。信号可以选择"不经过变换"直接通过。这条捷径同时服务于两个目的——梯度可以无损反传（2.2.1 讲过），信息也可以无损前传。

### 2.2 梯度视角（已讲）与表达力视角（新补充）

**梯度视角**（2.2.1 已详细讲）：反向传播时，$\partial h^{(l+1)} / \partial h^{(l)} = I + \partial F / \partial h^{(l)}$，恒等矩阵 $I$ 保证梯度不消失。

**表达力视角**（补充）：

假设不加残差，100 层网络要学习一个"接近恒等变换"的函数（比如某些层对输入的改动很微小）。每一层的权重必须精确配置到输出 ≈ 输入——这在高维空间里极难优化。

加了残差之后，每层 $F(x)$ 只需要学习"与恒等变换的残差（差值）"——如果这一层最优的改动是"什么都不变"，那 $F(x) = 0$ 就行，而零权重是很容易用梯度下降找到的。这就是"残差学习比从头学习恒等更容易"的直觉。

### 2.3 残差的三种形态

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# 形态一：标准残差（ResNet 风格，加法前不 Norm）
class ResidualBlock(nn.Module):
    def __init__(self, d):
        super().__init__()
        self.fn = nn.Sequential(nn.Linear(d, d), nn.ReLU(), nn.Linear(d, d))
        self.norm = nn.BatchNorm1d(d)

    def forward(self, x):
        return self.norm(x + self.fn(x))   # Post-Norm 加法

# 形态二：Pre-Norm 残差（现代 LLM 标配）
class PreNormResidual(nn.Module):
    def __init__(self, d, fn):
        super().__init__()
        self.norm = nn.LayerNorm(d)
        self.fn = fn

    def forward(self, x):
        return x + self.fn(self.norm(x))   # 先 Norm，残差路径完全干净

# 形态三：Gated Residual（门控残差，扩展表达力）
class GatedResidual(nn.Module):
    """用可学习门控决定"走捷径"还是"走变换路径"的比例"""
    def __init__(self, d, fn):
        super().__init__()
        self.fn = fn
        self.gate = nn.Parameter(torch.zeros(d))  # 初始化为 0 = 全走捷径

    def forward(self, x):
        alpha = torch.sigmoid(self.gate)
        return alpha * self.fn(x) + (1 - alpha) * x
```

### 2.4 残差的宏观效果：网络集成的近似

有一个颇具洞察的观点：含有 $L$ 个残差块的网络在前向传播时可以展开为 $2^L$ 条不同深度的路径之和（每个残差块可以"走捷径"或"走变换路径"）。这 $2^L$ 条路径相当于一个庞大的隐式集成。

实验验证：随机"关掉"（short-circuit）一部分残差块，ResNet 和 Transformer 的性能下降远比普通网络平滑，这正是集成效应的体现。这也解释了为什么 Transformer 某些层可以被直接剪掉（layer pruning）而性能损失较小。

## 三、注意力机制雏形：从 RNN 到 Bahdanau Attention

### 3.1 初印象：用费曼视角理解

**注意力机制的诞生，是为了解决 RNN 的"信息瓶颈"问题**。

经典 RNN 编码器把整句话压缩成一个固定大小的向量（Context Vector），解码器从这个向量"解压"生成翻译。问题是：一个向量装不下一整句话的信息，句子越长，这个瓶颈越严重。

Bahdanau 在 2014 年的洞见：**与其让解码器每步都读同一个 context vector，不如让它每步"自己决定看源句的哪些部分"**。每一步生成词时，模型动态地在所有编码器隐状态上分配权重（注意力权重），加权平均得到当前步的 context——这就是注意力机制的起点。

### 3.2 Bahdanau Attention 的数学形式

给定编码器隐状态序列 $\{h_1, \ldots, h_n\} \in \mathbb{R}^d$，解码器当前隐状态 $s_{t-1} \in \mathbb{R}^d$：

**第一步：计算对齐分数（Alignment Score）**

$$e_{ti} = \text{score}(s_{t-1}, h_i) = v^T \tanh(W_s s_{t-1} + W_h h_i)$$

这是一个小型神经网络，学习"解码器状态 $s_{t-1}$ 与编码器状态 $h_i$ 的相关性"。

**第二步：Softmax 归一化得到注意力权重**

$$\alpha_{ti} = \frac{\exp(e_{ti})}{\sum_{j=1}^n \exp(e_{tj})}$$

**第三步：加权求和得到 Context Vector**

$$c_t = \sum_{i=1}^n \alpha_{ti} h_i$$

**第四步：Context + 解码器状态生成输出**

$$s_t = f(s_{t-1}, y_{t-1}, c_t)$$

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class BahdanauAttention(nn.Module):
    """Bahdanau（加法）注意力机制"""

    def __init__(self, encoder_dim: int, decoder_dim: int, attn_dim: int):
        super().__init__()
        self.W_s = nn.Linear(decoder_dim, attn_dim, bias=False)  # 解码器状态变换
        self.W_h = nn.Linear(encoder_dim, attn_dim, bias=False)  # 编码器状态变换
        self.v   = nn.Linear(attn_dim, 1, bias=False)            # 对齐得分

    def forward(self, decoder_state, encoder_outputs):
        """
        decoder_state:    (B, decoder_dim)
        encoder_outputs:  (B, src_len, encoder_dim)
        返回：context (B, encoder_dim)，weights (B, src_len)
        """
        # 广播解码器状态到每个源位置
        s = self.W_s(decoder_state).unsqueeze(1)   # (B, 1, attn_dim)
        h = self.W_h(encoder_outputs)              # (B, src_len, attn_dim)

        # 计算对齐得分
        energy = self.v(torch.tanh(s + h)).squeeze(-1)  # (B, src_len)

        # Softmax 得到注意力权重
        weights = F.softmax(energy, dim=-1)   # (B, src_len)

        # 加权求和得到 context
        context = (weights.unsqueeze(-1) * encoder_outputs).sum(dim=1)  # (B, encoder_dim)

        return context, weights


# 演示
B, src_len, enc_dim, dec_dim, attn_dim = 4, 20, 256, 256, 128
attn = BahdanauAttention(enc_dim, dec_dim, attn_dim)

encoder_outputs = torch.randn(B, src_len, enc_dim)
decoder_state   = torch.randn(B, dec_dim)

context, weights = attn(decoder_state, encoder_outputs)
print(f"Context: {context.shape}")   # (4, 256)
print(f"Weights: {weights.shape}")   # (4, 20)，每个值在 [0,1] 且每行和为 1
print(f"注意力权重之和: {weights.sum(dim=-1).mean().item():.4f}")  # ≈ 1.0
```

### 3.3 从 Bahdanau 到 Self-Attention：一步之遥

理解了 Bahdanau Attention，Self-Attention 就是自然的推广：

**Bahdanau Attention**：Query 来自解码器状态，Key/Value 来自编码器输出，是**跨序列**的注意力。

**Self-Attention**：Query、Key、Value 全部来自**同一序列**，每个位置可以关注序列中的任意其他位置。

具体对应关系：

| Bahdanau | Self-Attention | 作用 |
|----------|---------------|------|
| $s_{t-1}$（解码器状态） | $Q = XW_Q$ | 查询"我现在关注什么" |
| $h_i$（编码器隐状态，用于得分） | $K = XW_K$ | 被查询的"关键词" |
| $h_i$（编码器隐状态，用于加权） | $V = XW_V$ | 实际提取的"内容" |
| 加法得分 $v^T\tanh(W_s s + W_h h)$ | 点积得分 $QK^T / \sqrt{d_k}$ | 更高效，GPU 友好 |

```python
import torch
import torch.nn.functional as F

def self_attention(X, W_Q, W_K, W_V, causal_mask=True):
    """
    最简 Self-Attention，展示与 Bahdanau 的直接对应关系
    X: (B, L, D)
    """
    B, L, D = X.shape
    d_k = W_Q.shape[-1]   # key 维度

    # 和 Bahdanau 类比：Q ≈ 解码器状态，K/V ≈ 编码器隐状态
    Q = X @ W_Q   # (B, L, d_k)   -- "我想关注什么"
    K = X @ W_K   # (B, L, d_k)   -- "我的关键词是什么"
    V = X @ W_V   # (B, L, D)     -- "我携带的内容是什么"

    # 点积得分（比 Bahdanau 的加法更高效）
    scores = Q @ K.transpose(-2, -1) / (d_k ** 0.5)   # (B, L, L)

    # 因果掩码（只能看过去，不能看未来）
    if causal_mask:
        mask = torch.tril(torch.ones(L, L, device=X.device)).bool()
        scores = scores.masked_fill(~mask, float("-inf"))

    # Softmax → 注意力权重（对应 Bahdanau 的 alpha）
    weights = F.softmax(scores, dim=-1)   # (B, L, L)

    # 加权求和（对应 Bahdanau 的 context 计算）
    output = weights @ V   # (B, L, D)
    return output, weights

# 测试
B, L, D, d_k = 2, 10, 64, 32
X = torch.randn(B, L, D)
W_Q = torch.randn(D, d_k)
W_K = torch.randn(D, d_k)
W_V = torch.randn(D, D)
out, w = self_attention(X, W_Q, W_K, W_V)
print(f"Self-Attention 输出: {out.shape}")  # (2, 10, 64)
print(f"注意力权重: {w.shape}")             # (2, 10, 10)
```

### 3.4 两种注意力得分函数的比较

| 得分函数 | 公式 | 特点 |
|---------|------|------|
| 加法（Bahdanau） | $v^T \tanh(W_s s + W_h h)$ | 参数多（$W_s, W_h, v$），表达力强，计算慢 |
| 点积（Luong / Transformer） | $q^T k / \sqrt{d_k}$ | 无额外参数，GPU 友好，可用矩阵乘法批量计算 |
| 缩放点积 | $q^T k / \sqrt{d_k}$ | 点积 + 稳定性缩放，Transformer 的标准 |

**为什么 Transformer 选择点积而非加法**：点积可以表达成矩阵乘法（$QK^T$），批量计算极其高效，在 GPU 上可以利用 Tensor Core；而加法注意力需要逐对计算，难以高效并行。

## 五、面试常见问题

**Q1：大规模预训练为什么关掉了 Dropout？**

两个原因：数据量足够大时，模型没有机会过拟合单个样本，Dropout 的防过拟合效果可以忽略；同时 Dropout 的随机性会使训练结果不完全确定，增加调试难度，且对超大模型的计算成本有细微影响。2022 年之后几乎所有主流 LLM（LLaMA、Qwen、Mistral）在预训练阶段均 dropout=0.0。

**Q2：残差连接解决了哪两个维度的问题？**

梯度维度：反向传播时梯度路径包含恒等矩阵 $I$，无论网络多深梯度都有保底，不会消失（2.2.1 详细推导）。表达力维度：每层只需要学习"与恒等变换的残差"，如果某层最优操作是"什么都不变"，$F(x) = 0$ 就是答案，零权重极易优化，让深层网络能有效地选择"跳过某些层"。

**Q3：Bahdanau Attention 的三个步骤是什么？**

第一步：对每个编码器位置 $h_i$ 和当前解码器状态 $s_t$，用一个小神经网络计算对齐分数 $e_{ti} = v^T \tanh(W_s s_t + W_h h_i)$。第二步：对所有位置的得分做 Softmax，得到归一化的注意力权重 $\alpha_{ti}$。第三步：以权重 $\alpha_{ti}$ 对所有编码器隐状态加权求和，得到当前步的 context vector $c_t = \sum_i \alpha_{ti} h_i$。

**Q4：Self-Attention 和 Bahdanau Attention 的核心区别是什么？**

来源不同：Bahdanau 的 Q（查询）来自解码器，K/V（被查询的）来自编码器，是跨序列注意力。Self-Attention 的 Q、K、V 全部来自同一序列，每个位置可以关注序列内任意其他位置，是序列内的自注意力。得分计算不同：Bahdanau 用加法得分（小神经网络），Transformer 用缩放点积（矩阵乘法，GPU 友好且无额外参数）。

**Q5：为什么 Transformer 用点积注意力而不用 Bahdanau 的加法注意力？**

效率：点积 $QK^T$ 可以表达为一次矩阵乘法，对整个序列并行计算，在 GPU 上极其高效（可利用 Tensor Core）。加法注意力需要逐对计算，难以高效批量化。参数量：点积注意力的得分计算无需额外参数，加法注意力需要 $W_s, W_h, v$ 三组参数。缩放 $1/\sqrt{d_k}$ 解决了点积在高维时数值偏大的问题（会让 Softmax 饱和）。

**Q6：QK LayerNorm 是什么？为什么有些 LLM 要加它？**

QK LayerNorm 在 Multi-Head Attention 中，对每个头的 Q 和 K 向量分别做 LayerNorm，再计算点积得分。目的是防止 $QK^T$ 的数值过大导致 Softmax 极度饱和（注意力权重接近 one-hot，梯度几乎消失）。这在超大模型（70B+）或长序列训练中尤为重要，是一种额外的训练稳定性手段，Gemma-2 等模型有采用。

**Q7：残差连接的"网络集成"视角是什么意思？**

一个含 $L$ 个残差块的网络，可以展开为 $2^L$ 条不同深度的路径（每个残差块在每个样本上选择"走捷径"或"走变换路径"），这 $2^L$ 条路径的输出相加，相当于隐式地集成了指数级数量的子网络。这解释了为什么残差网络对部分层被随机关闭（layer dropping）的鲁棒性很强，也是 Transformer 可以做 Layer-wise 剪枝的理论基础。
