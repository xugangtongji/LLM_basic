## 一、BERT 家族：理解型模型的奠基者

### 1.1 BERT（2018，Google）

**一句话定位**：第一个大规模预训练的双向 Transformer，刷新了 11 项 NLP 任务的记录，开启了"预训练-微调"的范式。

**架构特点**：

| 组件 | 设计 |
|------|------|
| 架构类型 | Encoder-only（双向注意力）|
| 预训练目标 | MLM（掩码语言模型）+ NSP（下句预测）|
| 位置编码 | 可学习绝对位置编码 |
| 归一化 | Post-LayerNorm |
| 注意力 | 标准 MHA（无 GQA）|
| 激活函数 | GELU |
| 典型配置 | BERT-base: 12层, d=768, 12头, 110M 参数 |

**BERT 的核心贡献**：

第一次证明了"在大规模无标注文本上预训练，再在下游任务微调"的范式可以打败所有特定任务的专门模型。这个"预训练+微调"的范式，至今仍是整个 LLM 工程的基础逻辑。

**BERT 的局限**：
- 双向注意力无法生成（无法自回归）
- NSP 任务被后来证明没什么用
- 推理时行为和训练时（有 MASK token）不完全一致

```python
# BERT 的预训练任务：MLM
# 随机遮盖 15% 的 token，其中：
# 80% 替换成 [MASK]
# 10% 替换成随机词
# 10% 保持不变
# 目的：让模型不能"偷懒"只看 [MASK] 位置

import torch
import torch.nn as nn

def bert_mlm_mask(input_ids: torch.Tensor, vocab_size: int,
                   mask_token_id: int, mask_prob: float = 0.15):
    """BERT 的 MLM 掩码策略"""
    labels = input_ids.clone()
    # 随机选 15% 的位置
    prob_matrix = torch.full(input_ids.shape, mask_prob)
    masked_indices = torch.bernoulli(prob_matrix).bool()

    # 这些位置是需要预测的（其余设为 -100，CrossEntropy 忽略）
    labels[~masked_indices] = -100

    # 80%：替换为 [MASK]
    replace_with_mask = torch.bernoulli(torch.full(input_ids.shape, 0.8)).bool()
    input_ids[masked_indices & replace_with_mask] = mask_token_id

    # 10%：替换为随机词
    replace_with_random = torch.bernoulli(torch.full(input_ids.shape, 0.5)).bool()
    random_tokens = torch.randint(vocab_size, input_ids.shape)
    condition = masked_indices & ~replace_with_mask & replace_with_random
    input_ids[condition] = random_tokens[condition]

    # 10%：保持不变（隐式处理）
    return input_ids, labels

# 示例
vocab_size = 30522  # BERT 原始词表大小
input_ids = torch.randint(0, vocab_size, (2, 20))
masked_ids, labels = bert_mlm_mask(input_ids, vocab_size, mask_token_id=103)
print(f"掩码前: {input_ids[0, :8].tolist()}")
print(f"掩码后: {masked_ids[0, :8].tolist()}")
print(f"标签:   {labels[0, :8].tolist()}")  # -100 表示不计算 loss
```

### 1.2 BERT 的重要继承者

**RoBERTa（2019，Facebook）**：去掉 NSP，增大批次、增加训练数据，证明 BERT 大幅欠训练。是目前仍然非常实用的理解模型。

**DeBERTa（2020，Microsoft）**：引入解耦注意力（词内容和位置分开编码）和增强掩码解码器，多项 GLUE 任务超过人类水平。

**现代 Embedding 模型（BGE、E5、GTE 等）**：基于 BERT 架构，专门为语义相似度和向量检索优化，是 RAG 系统里最常用的组件。

## 二、GPT 家族：生成型模型的奠基者

### 2.1 GPT-1 到 GPT-3 的演进

**GPT-1（2018，OpenAI）**：第一个大规模生成式预训练 Transformer（1.17 亿参数）。证明了 Decoder-only + CLM 训练的可行性。

**GPT-2（2019，OpenAI）**：参数量扩大到 15 亿。第一个让人震惊的语言生成模型，OpenAI 最初拒绝完整开放权重（"太危险"）。从这个版本开始，Pre-LayerNorm 成为标准。

**GPT-3（2020，OpenAI）**：参数量扩大到 1750 亿。证明"大力出奇迹"——规模足够大时，模型无需微调也能完成各种任务（Zero-shot/Few-shot）。同时带出了 Scaling Law 和 Chinchilla 等一系列关于规模的研究。

### 2.2 GPT-2 的架构（现代 LLM 的模板）

GPT-2 架构至今仍是很多研究的基准，也是 3.4 章完整实现的原型：

```python
# GPT-2 的关键配置（与原始 Transformer 的主要区别）
gpt2_configs = {
    "GPT-2 Small":  {"n_layers": 12, "d_model": 768,  "n_heads": 12, "d_ff": 3072},
    "GPT-2 Medium": {"n_layers": 24, "d_model": 1024, "n_heads": 16, "d_ff": 4096},
    "GPT-2 Large":  {"n_layers": 36, "d_model": 1280, "n_heads": 20, "d_ff": 5120},
    "GPT-2 XL":     {"n_layers": 48, "d_model": 1600, "n_heads": 25, "d_ff": 6400},
}

# GPT-2 vs 原始 Transformer 的架构变化：
gpt2_vs_original = {
    "归一化位置": "Post-Norm → Pre-Norm（GPT-2 的重大贡献）",
    "位置编码": "固定正弦 → 可学习绝对编码",
    "激活函数": "ReLU → GELU",
    "上下文长度": "512 token → 1024 token",
    "参数": "有 bias → 有 bias（LLaMA 才去掉）",

for k, v in gpt2_vs_original.items():
    print(f"  {k}: {v}")
```

### 2.3 InstructGPT 与 ChatGPT：RLHF 的工程化

GPT-3 虽然强大，但不"听话"——它会续写你给的任何内容，不管是否有害，也不理解你的真实意图。

InstructGPT（2022）引入了 RLHF（人类反馈强化学习）三阶段训练：

```
阶段1：SFT（监督微调）
  人工写高质量的指令→回答对
  让模型学会"按指令回答"的格式

阶段2：奖励模型训练（RM）
  让人类对多个回答排序
  训练一个模型来模拟"人类偏好打分"

阶段3：PPO 强化学习
  用奖励模型的分数作为奖励信号
  用 PPO 算法优化语言模型的生成策略
  KL 约束防止模型跑偏
```

ChatGPT 基本就是 InstructGPT 的产品化版本，这套 RLHF 流程奠定了所有现代对话模型（Claude、Gemini、Qwen-Chat 等）的训练基础。

## 三、LLaMA 家族：开源 LLM 的标杆

### 3.1 LLaMA-1（2023，Meta）

**历史意义**：第一个高质量、完全开源的大型语言模型，打破了"大模型必须闭源"的局面，直接催生了开源 LLM 生态的爆发。

**架构关键改进**（相比 GPT-3）：

```python
# LLaMA-1 的四个关键架构改进
llama1_innovations = {
    "Pre-RMSNorm（而非 LayerNorm）": """
        RMSNorm 比 LayerNorm 快约 7-64%（去掉了均值计算）
        同时取消了 Norm 的 bias 参数
        现代 LLM 的标配""",

    "SwiGLU FFN（而非 GELU）": """
        FFN 从两个线性层扩展为 SwiGLU 的三个线性层
        hidden_dim = (8/3) × d_model（约 2/3 × 4d = 8d/3）
        效果更好，参数量与标准 FFN 相当""",

    "RoPE（而非可学习绝对位置编码）": """
        旋转位置编码，相对位置感知更好
        天然支持长序列外推
        现代 LLM 的位置编码标配""",

    "去掉所有 Bias": """
        线性层全部 bias=False
        稍微减少参数量，效果相当
        简化了模型结构""",
}

for key, desc in llama1_innovations.items():
    print(f"\n{key}:{desc}")
```

### 3.2 LLaMA-2（2023，Meta）

**核心改进**：引入 GQA（分组查询注意力），大幅减少 KV Cache 显存；上下文长度从 2K 扩展到 4K；发布了对话版本（LLaMA-2-Chat，用 RLHF 对齐）。

### 3.3 LLaMA-3（2024，Meta）

**核心改进**：词表从 32K 扩大到 128K（更好的多语言支持，尤其是代码）；训练数据从 2T 增加到 15T token；GQA 配置优化；发布了 8B、70B、405B 三个规模。

LLaMA 3 的架构与 GPT-2 的关键差异：位置编码从绝对演进到 RoPE，MHA 基本被 GQA 取代，SwiGLU 取代了 GELU。但核心的 Decoder-only + Pre-Norm + 残差连接的骨架保持不变。

```python
# LLaMA-3 配置速查
llama3_configs = {
    "LLaMA-3-8B":  {
        "n_layers": 32, "d_model": 4096, "n_heads": 32,

        "d_ff": 14336,     # SwiGLU 的 hidden_dim
        "vocab_size": 128256,
        "max_seq_len": 8192,
        "rope_theta": 500000,   # 扩大 base，支持更长上下文
    },
    "LLaMA-3-70B": {

        "n_kv_heads": 8,   # GQA：8 个 KV 头（比 8B 节省更多）
```

## 四、T5 与 BART：Encoder-Decoder 家族

### 4.1 T5（2019，Google）

**一句话定位**："万能文本转文本"模型，把所有 NLP 任务都统一成文本→文本的格式。

T5 的核心洞见：无论是翻译、摘要、分类还是问答，都可以表达为：

```
翻译: "translate English to German: The house is wonderful."
      → "Das Haus ist wunderbar."

情感分析: "sentiment: This movie is great!"
         → "positive"

问答: "question: What is the capital of France? context: France is a country..."
    → "Paris"
```

这种统一格式让 T5 无需为每个任务单独设计输出层，一个模型可以处理所有任务。

**架构特点**：Encoder-Decoder；相对位置偏置（非绝对位置编码）；没有使用 Pre-Norm（用了 Post-Norm，但 norm 在 FFN 和 Attention 外面）。

### 4.2 BART（2019，Facebook）

**定位**：专注于文本生成任务（摘要、对话）的 Encoder-Decoder 模型。

BART 的预训练方式独特：对输入做各种"破坏"（删词、随机打乱句子、掩盖词段），再训练模型将被破坏的文本还原为原始文本——这种"去噪自编码"的方式让 BART 在摘要等生成任务上特别强。

### 4.3 Whisper（2022，OpenAI）

**定位**：语音识别的 Encoder-Decoder 模型，把 Encoder-Decoder 架构成功迁移到语音领域。

Encoder 处理音频 spectrogram（频谱图），Decoder 通过 Cross-Attention 查询 Encoder 输出，自回归生成文字。Whisper 在多语言语音识别上表现卓越，至今是最广泛使用的开源 ASR 模型。

## 五、DeepSeek 家族：效率与性能的双重突破

### 5.1 DeepSeek-V2（2024）：MLA 的首次亮相

DeepSeek-V2 最重要的贡献是 MLA（Multi-head Latent Attention）——把 KV Cache 压缩进一个低维潜在向量，比 GQA 更激进地节省显存。

**MLA vs GQA 的 KV Cache 对比**（以 DeepSeek-V2 为例）：
- MHA（假设基准）：KV 大小 = 128 维 × 层数 × 序列长度
- GQA（8 KV 头）：减少约 75%
- MLA：减少约 93%（压缩到极低维的潜在表示）

### 5.2 DeepSeek-V3（2024）：开源世界的里程碑

DeepSeek-V3 是截至 2024 年底最强的开源模型之一，与 GPT-4o 等顶级闭源模型性能接近，但训练成本出人意料地低（约 600 万美元）。

**关键技术**：

```python
# DeepSeek-V3 的核心架构创新

deepseek_v3_features = {
    "MoE（混合专家）架构": """
        总参数 671B，但每次前向传播只激活约 37B（5.5%）
        256 个专家，每次激活 8 个路由专家 + 1 个共享专家
        使得计算成本仅相当于一个 37B 的稠密模型""",

    "MLA（多头潜在注意力）": """
        继承自 DeepSeek-V2
        KV Cache 压缩到极低维，推理显存大幅减少""",

    "无辅助损失的负载均衡": """
        MoE 模型容易出现"专家坍缩"——大多数 token 都路由到同几个专家
        DeepSeek-V3 用了无辅助损失的负载均衡策略
        比需要额外 loss 的方法更简洁有效""",

    "FP8 混合精度训练": """
        首次在如此大规模（671B）模型上成功使用 FP8 训练
        训练速度提升，成本降低""",
}
```

### 5.3 DeepSeek-R1（2025）：推理模型的新范式

DeepSeek-R1 专注于数学和代码等需要"慢思考"的任务，核心创新是 GRPO 训练策略：

```
传统 RLHF（PPO）：
  需要奖励模型 → 需要大量人工标注偏好数据 → 工程复杂

GRPO（Group Relative Policy Optimization）：
  对同一个问题生成多个答案（Group）
  用答案的正确性（是否真的解对了数学题）作为奖励
  组内相对比较，不需要单独的奖励模型
  效果：自然涌现出"思维链"推理行为
```

R1 证明了：规则化的奖励信号（对/错）+ 大规模 GRPO，可以训练出强大的推理能力，无需人工标注大量"思维过程"数据。

## 九、从架构角度理解模型能力

### 9.1 为什么 Decoder-only 比 Encoder-Decoder 更适合扩展？

Encoder-Decoder 有一个隐蔽的问题：Encoder 和 Decoder 是两套独立参数，而且 Cross-Attention 也需要参数。同样的总参数量，Encoder-Decoder 的"纯生成能力"实际上只有 Decoder-only 的一半。

Decoder-only 把全部参数都用于同一个生成任务，扩展性更强。

### 9.2 词表大小为什么越来越大？

早期 BERT（30K）→ GPT-2（50K）→ LLaMA-1（32K）→ LLaMA-3（128K）→ Qwen-2.5（152K）

词表越大：
- 需要的 token 数更少来表示同样的文字（提高计算效率）
- 对非英语语言友好（中文、代码等不再需要切成很多小 token）
- Embedding 层参数增大（但 Weight Tying 后不增加推理计算量）

### 9.3 为什么现代 LLM 都用 RoPE + GQA + SwiGLU + Pre-RMSNorm？

这四个选择都有明确的工程动机：

```
RoPE：相对位置感知 + KV Cache 友好 + 长序列外推能力
GQA：在几乎不损失质量的情况下，KV Cache 节省 75%-87.5%
SwiGLU：比 GELU FFN 在同等参数量下有更好的表达能力
Pre-RMSNorm：Pre-Norm 训练更稳定，RMSNorm 比 LayerNorm 快 7-64%
```

这四个选择的组合，是当前工程界对"效果-效率-稳定性"三角权衡的最优解。

## 十、面试常见问题

**Q1：BERT 和 GPT 的预训练方式有什么根本区别？**

BERT 用 MLM（Masked Language Model）：随机遮盖 15% 的 token，让模型用双向上下文预测被遮盖的词；同时用 NSP 预测两个句子是否相邻（后来被证明作用不大）。GPT 用 CLM（Causal Language Model）：从左到右逐词预测下一个词，只能看到左侧上下文。前者适合理解，后者适合生成，这决定了两者的架构选择（双向 Encoder vs 单向 Decoder）。

**Q2：LLaMA 相比 GPT-3 做了哪些重要的架构改进？**

四个关键改进：Pre-RMSNorm（训练更稳定，速度更快）；SwiGLU FFN（在同等参数量下效果更好）；RoPE（相对位置感知，支持更好的长度外推）；去掉所有 Bias（减少参数，效果相当）。加上后续的 GQA（LLaMA-2/3 引入），这五个改进构成了现代开源 LLM 的事实标准架构。

**Q3：DeepSeek-V3 为什么训练成本低但效果好？**

两个核心原因：MoE 架构（671B 总参数，但每次前向只激活约 37B，计算成本约等于一个 37B 稠密模型），使得训练更多参数但每步计算成本可控；FP8 混合精度训练（在如此大规模首次成功应用），训练速度更快、成本更低。此外还有 MLA（减少 KV Cache 推理成本）和无辅助损失负载均衡（使 MoE 训练更稳定）。

**Q4：T5 和 BERT 都是在理解任务上表现好，有什么核心区别？**

架构不同：BERT 是 Encoder-only，T5 是 Encoder-Decoder。T5 的核心理念是把所有任务统一成"文本转文本"格式，用同一个模型处理翻译、摘要、分类等所有任务，不需要为每个任务设计不同的输出头。BERT 则需要在 [CLS] 向量上接不同的任务头。T5 更灵活（生成任意文本），BERT 更高效（只做分类/抽取）。

**Q5：Qwen-2.5 的词表为什么有 15 万个 token？这有什么代价？**

大词表（152K）让中文、日文等每个汉字/词有独立 token，不需要拆成多个子词，编码效率更高，理解中文语义也更准确；同时有更多代码 token，对编程任务友好。代价是 Embedding 矩阵更大（152K × 3584 ≈ 5.4 亿参数）。但 Weight Tying 让 LM Head 和 Embedding 共享权重，推理时不增加额外计算量；主要代价是 Embedding 查找的内存占用略大。

**Q6：为什么说现代大模型七年来架构"没有革命性变化"？**

核心骨架（Decoder-only + Pre-Norm + Self-Attention + FFN + 残差连接）自 GPT-2 以来几乎没变。改变的是"零件的质量"：位置编码从绝对→RoPE，归一化从 LayerNorm→RMSNorm，FFN 激活从 GELU→SwiGLU，注意力从 MHA→GQA，但这些都是"同一台机器换了更好的零件"，而非"发明了一台新机器"。真正的结构创新（MoE、MLA、Mamba）相对较少，而且 MoE 已经逐渐成为主流。
