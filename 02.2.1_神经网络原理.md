## 零、开篇：反向传播是深度学习的心脏

1986 年之前，没有人知道如何高效训练多层神经网络。Rumelhart、Hinton、Williams 提出反向传播（Backpropagation）之后，"深度学习"才真正成为可能。

但理解反向传播不止是"知道有这个算法"——你需要理解它的每一步在做什么，才能真正理解：

- 为什么 sigmoid 网络训练 10 层就困难，而 Transformer 能训练 100 层甚至更深
- 为什么残差连接（ResNet、Transformer 的核心）能从根本上解决梯度消失
- 为什么 LayerNorm 必须放在哪里才对
- 为什么初始化策略会让训练成功或失败

## 一、神经网络的基本结构

### 1.1 初印象：用费曼视角理解

**神经网络是一个由"线性变换 + 非线性激活"堆叠起来的函数**。

想象你要学习一个复杂的函数（比如"给定一段文字，判断情感"）。单一线性函数（如逻辑回归）只能画直线，无法表示复杂的弯曲边界。

解决方案：堆叠多层"线性变换 + 弯曲函数（激活函数）"。每一层先做线性投影，再用非线性函数引入弯曲能力，叠足够多层，理论上能逼近任意复杂函数（普适逼近定理）。

**两个关键点**：
1. 没有非线性激活，多层线性变换等价于单层线性——白叠
2. 激活函数选择决定了"弯曲的方式"，直接影响梯度能否顺畅传播

### 1.2 从感知机到 MLP

**感知机（Perceptron，1957）**：最简单的神经网络，单个神经元：

$$\hat{y} = \text{sign}(\mathbf{w}^T \mathbf{x} + b)$$

只能分类线性可分数据（异或问题直接挂掉）。

**多层感知机（MLP）**：多个感知机叠加，中间层用连续可微的激活函数代替 sign：

$$
\mathbf{h}^{(1)} = \sigma(W^{(1)}\mathbf{x} + \mathbf{b}^{(1)})
$$
$$
\mathbf{h}^{(2)} = \sigma(W^{(2)}\mathbf{h}^{(1)} + \mathbf{b}^{(2)})
$$
$$
\hat{y} = W^{(3)}\mathbf{h}^{(2)} + \mathbf{b}^{(3)}
$$

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class MLP(nn.Module):
    """三层 MLP，完整展示神经网络基本结构"""

    def __init__(self, input_dim, hidden_dims, output_dim, activation="relu"):
        super().__init__()
        dims = [input_dim] + hidden_dims + [output_dim]
        self.layers = nn.ModuleList([
            nn.Linear(dims[i], dims[i+1])
            for i in range(len(dims) - 1)
        ])
        self.act = {
            "relu":  F.relu,
            "gelu":  F.gelu,
            "tanh":  torch.tanh,
            "silu":  F.silu,
        }[activation]
        self.n_layers = len(self.layers)

    def forward(self, x):
        for i, layer in enumerate(self.layers):
            x = layer(x)
            # 最后一层不加激活（输出层）
            if i < self.n_layers - 1:
                x = self.act(x)
        return x

# 使用
model = MLP(input_dim=784, hidden_dims=[512, 256], output_dim=10, activation="relu")
x = torch.randn(32, 784)
logits = model(x)
print(f"输出形状: {logits.shape}")   # (32, 10)

# 统计参数量
total = sum(p.numel() for p in model.parameters())
print(f"总参数量: {total:,}")        # 784*512 + 512*256 + 256*10 + 偏置
```

## 二、前向传播：信息的流动

### 2.1 初印象：用费曼视角理解

**前向传播就是"数据从左到右流过网络，每一层加工一次"**。

输入进去，经过第一层变换+激活得到中间表示，再经过第二层……直到输出层得到预测值。这个过程纯粹是数学计算，没有"学习"发生——学习在反向传播里。

PyTorch 的神奇之处在于：**前向传播时它悄悄记录了每一步操作的"本地梯度"**，为后续的反向传播做准备。这就是计算图（动态图）的精髓。

### 2.2 逐步手算一个两层网络

```python
import torch

torch.manual_seed(42)

# 网络参数
W1 = torch.randn(4, 3, requires_grad=True)  # 第一层权重 (hidden, input)
b1 = torch.zeros(4, requires_grad=True)
W2 = torch.randn(2, 4, requires_grad=True)  # 第二层权重 (output, hidden)
b2 = torch.zeros(2, requires_grad=True)

# 输入和目标
x = torch.randn(3)     # 单个样本，3 维输入
y = torch.tensor([1.0, 0.0])  # 目标

# 前向传播（手动，每步打印形状）
z1 = W1 @ x + b1          # 线性变换   (4,)
a1 = torch.relu(z1)        # ReLU 激活  (4,)
z2 = W2 @ a1 + b2          # 线性变换   (2,)
# 输出层不加激活（回归或后接 softmax）
loss = 0.5 * ((z2 - y) ** 2).sum()  # MSE 损失

print(f"z1={z1.detach().numpy().round(3)}")
print(f"a1={a1.detach().numpy().round(3)}")
print(f"z2={z2.detach().numpy().round(3)}")
print(f"loss={loss.item():.4f}")
```

## 三、反向传播：梯度的逆流

### 3.1 初印象：用费曼视角理解

**反向传播就是"从错误出发，追溯每个参数的责任"**。

你做了一道题，答错了。怎么改进？先看最后一步是哪里出错（输出层），再往前追究是哪一步引发了这个错误（倒数第二层），再往前……这正是链式法则的精髓：从输出到输入，逐层用"链式法则"把"损失对参数的偏导"传回去。

### 3.2 手推反向传播

对上面的两层网络，手动计算梯度：

```python
# 反向传播（PyTorch autograd 版本，验证手动计算）
loss.backward()

print("\n--- PyTorch 自动求导结果 ---")
print(f"dL/dW2 = \n{W2.grad.numpy().round(4)}")
print(f"dL/db2 = {b2.grad.numpy().round(4)}")
print(f"dL/dW1 = \n{W1.grad.numpy().round(4)}")
print(f"dL/db1 = {b1.grad.numpy().round(4)}")

# --- 手动推导（验证）---
with torch.no_grad():
    # 输出层梯度
    dL_dz2 = z2 - y                           # MSE 对 z2 的梯度: (2,)
    dL_dW2 = dL_dz2.unsqueeze(1) @ a1.unsqueeze(0)  # (2,1)@(1,4) = (2,4)
    dL_db2 = dL_dz2                            # (2,)

    # 反传到第一层
    dL_da1 = W2.T @ dL_dz2                    # (4,2)@(2,) = (4,)
    dL_dz1 = dL_da1 * (z1 > 0).float()        # ReLU 导数：z1>0 处为 1，否则为 0
    dL_dW1 = dL_dz1.unsqueeze(1) @ x.unsqueeze(0)   # (4,1)@(1,3) = (4,3)
    dL_db1 = dL_dz1                            # (4,)

print("\n--- 手动推导结果 ---")
print(f"dL/dW2 = \n{dL_dW2.numpy().round(4)}")
print(f"dL/dW1 = \n{dL_dW1.numpy().round(4)}")
print("两者完全一致！" if torch.allclose(W2.grad, dL_dW2) and
                         torch.allclose(W1.grad, dL_dW1) else "有误差，检查推导")
```

### 3.3 计算图视角：为什么 autograd 这么高效

每个操作的"本地梯度"只依赖于这个操作本身的输入输出，与全局无关。反向传播只是把这些本地梯度通过链式法则串起来——**全局梯度 = 本地梯度的乘积链**。

这是 autograd 设计的核心：只要每个操作知道自己的 `forward` 和 `backward`（本地梯度如何计算），框架就能自动组合出全局梯度，用户不需要手推任何公式。

## 四、梯度消失：深度训练的历史难题

### 4.1 初印象：用费曼视角理解

**梯度消失就是"传话游戏里信息越传越弱，传到最后什么都没剩下"**。

20 个人站成一排传话：第一个人说了一个很重要的信息，传到最后一个人那里，已经模糊到几乎听不见了。反向传播也是一样——误差信号从输出层向输入层传播时，经过每一层都会乘以一个导数值，如果这个值反复小于 1，乘到最后就接近 0 了。

靠近输入的层（提取基础特征的层）几乎收不到任何梯度信号，所以几乎无法学习。这就是为什么早期深度神经网络（sigmoid / tanh 激活）超过 5-6 层就训练不起来。

### 4.2 数学根源

链式法则展开，损失对第 $l$ 层参数的梯度：

$$\frac{\partial L}{\partial W^{(l)}} = \frac{\partial L}{\partial \mathbf{h}^{(L)}} \cdot \prod_{k=l}^{L-1} \frac{\partial \mathbf{h}^{(k+1)}}{\partial \mathbf{h}^{(k)}}$$

每一项 $\frac{\partial \mathbf{h}^{(k+1)}}{\partial \mathbf{h}^{(k)}} = \text{diag}(\sigma'(\mathbf{z}^{(k)})) \cdot W^{(k)}$ 包含激活函数的导数。

**sigmoid 的问题**：$\sigma'(z) = \sigma(z)(1-\sigma(z)) \in (0, 0.25]$，最大值仅 0.25。

$$\text{10 层后梯度量级} \approx 0.25^{10} \approx 10^{-6}$$

梯度从 1 衰减到百万分之一，早已无法更新参数。

```python
import numpy as np
import matplotlib.pyplot as plt

# 可视化不同激活函数的导数范围对梯度的影响
def sigmoid(x): return 1 / (1 + np.exp(-x))
def sigmoid_grad(x): s = sigmoid(x); return s * (1 - s)
def tanh_grad(x): return 1 - np.tanh(x) ** 2
def relu_grad(x): return (x > 0).astype(float)

x = np.linspace(-4, 4, 300)
fig, axes = plt.subplots(1, 3, figsize=(12, 4))

for ax, (name, fn, grad_fn, color) in zip(axes, [
    ("Sigmoid", sigmoid, sigmoid_grad, "blue"),
    ("Tanh",    np.tanh, tanh_grad,   "orange"),
    ("ReLU",    lambda x: np.maximum(0, x), relu_grad, "green"),
]):
    ax2 = ax.twinx()
    ax.plot(x, fn(x), color=color, label="函数值")
    ax2.plot(x, grad_fn(x), color=color, linestyle="--", alpha=0.7, label="导数")
    ax.set_title(f"{name}\n导数范围: [{grad_fn(x).min():.2f}, {grad_fn(x).max():.2f}]")
    ax.set_xlabel("x"); ax.set_ylabel("函数值"); ax2.set_ylabel("导数")
    ax.legend(loc="upper left"); ax2.legend(loc="upper right")

plt.suptitle("激活函数及其导数（导数大小决定梯度消失/爆炸）", y=1.02)
plt.tight_layout()
plt.savefig("activation_gradients.png", dpi=150, bbox_inches="tight")

# 定量模拟：10层网络，不同激活函数，梯度幅度的变化
n_layers = 20
grad_magnitudes = {"sigmoid": [], "tanh": [], "relu": []}

for _ in range(100):  # 100 次随机实验
    x_val = np.random.randn()
    g_sig = g_tanh = g_relu = 1.0
    for _ in range(n_layers):
        z = np.random.randn() * 0.5
        g_sig  *= sigmoid_grad(z)
        g_tanh *= tanh_grad(z)
        g_relu *= relu_grad(z + abs(z))  # 确保部分激活

    grad_magnitudes["sigmoid"].append(abs(g_sig))
    grad_magnitudes["tanh"].append(abs(g_tanh))
    grad_magnitudes["relu"].append(abs(g_relu))

for name, vals in grad_magnitudes.items():
    print(f"{name:8s} 梯度幅度（{n_layers}层后）: "
          f"均值={np.mean(vals):.2e}  中位数={np.median(vals):.2e}")
```

### 4.3 梯度爆炸

与梯度消失相反：如果每层的导数 × 权重矩阵的谱范数 > 1，梯度指数级增大。

症状：loss 出现 NaN，`nvidia-smi` 显示显存突然跳变，权重变成 inf。

```python
# 检测梯度爆炸的实用代码片段
def check_gradients(model, step, log_every=100):
    """训练循环里插入这段，监控梯度健康"""
    if step % log_every != 0:
        return
    total_norm = 0.0
    for name, p in model.named_parameters():
        if p.grad is not None:
            param_norm = p.grad.data.norm(2).item()
            total_norm += param_norm ** 2
            if param_norm > 10:    # 单参数梯度超过 10 就警告
                print(f"  [{step}] 高梯度警告: {name} grad_norm={param_norm:.2f}")
    total_norm = total_norm ** 0.5
    if total_norm > 100:
        print(f"  [{step}] 梯度爆炸！global_norm={total_norm:.2f}")
    return total_norm
```

## 五、解决方案全景：让梯度顺畅流动

### 5.1 解决方案一：换激活函数（ReLU 及其变体）

**ReLU（Rectified Linear Unit）**：$f(x) = \max(0, x)$

- 正区间导数恒为 1，不会饱和，梯度不消失
- 计算极简，收敛快
- 问题：负区间导数为 0（Dead ReLU）——某些神经元永久"死亡"

```python
# 激活函数家族对比
import torch

x = torch.linspace(-3, 3, 100)

activations = {
    "ReLU":    torch.relu(x),
    "Leaky ReLU": torch.where(x > 0, x, 0.01 * x),
    "GELU":    x * torch.distributions.Normal(0,1).cdf(x),    # 近似
    "SiLU":    x * torch.sigmoid(x),    # Swish
    "ELU":     torch.where(x > 0, x, torch.exp(x) - 1),
}

for name, y in activations.items():
    # 导数（有限差分近似）
    dy = torch.diff(y) / torch.diff(x)
    print(f"{name:12s}  导数范围: [{dy.min():.3f}, {dy.max():.3f}]  "
          f"负值区域最小导数: {dy[x[:-1] < -1].mean():.3f}")
```

**大模型中的激活函数演进**：

| 模型 | 激活函数 | FFN 维度 |
|------|---------|---------|
| GPT-2 | GELU | $4 \times d_{\text{model}}$ |
| LLaMA 1/2 | SiLU（SwiGLU 变体） | $\frac{8}{3} \times d_{\text{model}}$ |
| LLaMA 3 / Qwen2 | SiLU（SwiGLU） | $\frac{8}{3} \times d_{\text{model}}$ |
| DeepSeek-V2 | SiLU（SwiGLU）+ MoE | — |

**GELU 为什么比 ReLU 在 Transformer 里更好**：GELU 在 0 附近是光滑曲线（ReLU 有折角），梯度连续，在 Transformer 的 FFN 层经验上收敛更稳定，效果更好。

### 5.2 解决方案二：合理的权重初始化

```python
import torch
import torch.nn as nn

# 问题：随机初始化权重太大或太小
# 太小 → 激活值指数衰减 → 梯度消失
# 太大 → 激活值指数增大 → 梯度爆炸

n_layers = 50
input_dim = 256

def simulate_forward_pass(init_std):
    """模拟深层网络前向传播，观察激活值方差的变化"""
    x = torch.randn(1, input_dim)
    variances = [x.var().item()]
    for _ in range(n_layers):
        W = torch.randn(input_dim, input_dim) * init_std
        x = torch.relu(W @ x.T).T
        variances.append(x.var().item())
    return variances

# Xavier 初始化（适合 sigmoid / tanh）
def xavier_std(fan_in, fan_out):
    return (2 / (fan_in + fan_out)) ** 0.5

# He / Kaiming 初始化（适合 ReLU 族）
def he_std(fan_in):
    return (2 / fan_in) ** 0.5  # ReLU 使方差减半，所以乘 2 补偿

fan = input_dim
print(f"Xavier std: {xavier_std(fan, fan):.4f}")
print(f"He std:     {he_std(fan):.4f}")

# PyTorch 内置初始化
linear = nn.Linear(256, 256)
nn.init.xavier_uniform_(linear.weight)     # Xavier
nn.init.kaiming_normal_(linear.weight,
                         mode="fan_in",
                         nonlinearity="relu")  # He

# 对比三种初始化策略下深层网络的激活方差
import matplotlib.pyplot as plt
plt.figure(figsize=(10, 4))
for std, label, color in [
    (0.01, "太小 (std=0.01)", "red"),
    (he_std(fan), "He 初始化", "green"),
    (1.0,  "太大 (std=1.0)", "orange"),
]:
    vars_ = simulate_forward_pass(std)
    plt.semilogy(vars_, label=label, color=color, linewidth=2)

plt.xlabel("层数"); plt.ylabel("激活值方差（log scale）")
plt.title(f"不同初始化策略下 {n_layers} 层 ReLU 网络的激活方差变化")
plt.legend(); plt.grid(alpha=0.3)
plt.tight_layout()
plt.savefig("initialization.png", dpi=150)
```

### 5.3 解决方案三：归一化层

归一化层在每层之后对激活值做标准化，让梯度不会因为激活值尺度偏离而失控。

```python
# BatchNorm vs LayerNorm vs RMSNorm 的对比
import torch
import torch.nn as nn

x = torch.randn(4, 8, 512)  # (batch, seq_len, d_model)

# BatchNorm：沿 batch 维度归一化（对 NLP / 序列不友好，不常用）
# bn = nn.BatchNorm1d(512)

# LayerNorm：沿特征维度归一化（Transformer 的标配）
ln = nn.LayerNorm(512)
x_ln = ln(x)
print(f"LayerNorm 后均值: {x_ln.mean(dim=-1).abs().max().item():.6f}")  # ≈ 0
print(f"LayerNorm 后标准差: {x_ln.std(dim=-1).mean().item():.4f}")      # ≈ 1

# RMSNorm：只除以 RMS，不减均值（LLaMA/Qwen 用这个，更简洁，效果相当）
class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-8):
        super().__init__()
        self.scale = nn.Parameter(torch.ones(dim))
        self.eps = eps

    def forward(self, x):
        rms = (x.pow(2).mean(dim=-1, keepdim=True) + self.eps).sqrt()
        return x / rms * self.scale

rms = RMSNorm(512)
x_rms = rms(x)
print(f"RMSNorm 后 RMS: {x_rms.pow(2).mean(dim=-1).sqrt().mean().item():.4f}")  # ≈ 1
```

**Pre-LN vs Post-LN（现代 LLM 的关键选择）**：

- **Post-LN**（原始 Transformer）：Norm 在残差相加之后。梯度不稳定，需要精心 warmup。
- **Pre-LN**（GPT-2 之后主流）：Norm 在残差内部的子层之前。梯度更稳定，训练更顺畅，LLaMA/Qwen/Claude 全用这个。

```python
class TransformerLayerPreLN(nn.Module):
    """Pre-LN：先 Norm，后计算，再残差相加"""
    def __init__(self, d_model, n_heads, d_ff):
        super().__init__()
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.attn  = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ff    = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
        )

    def forward(self, x):
        # 关键：先 norm，再 attn，再加回去（残差路径上没有 norm）
        x = x + self.attn(self.norm1(x), self.norm1(x), self.norm1(x))[0]
        x = x + self.ff(self.norm2(x))
        return x
```

### 5.4 解决方案四：残差连接（最重要）

残差连接让梯度不再只能通过非线性路径流动，而是有一条"高速公路"直接从后层传到前层，梯度不会消失在任意深度的网络中。

**数学分析**：

无残差连接（L 层网络）：

$$\frac{\partial L}{\partial \mathbf{h}^{(l)}} = \prod_{k=l}^{L} \frac{\partial \mathbf{h}^{(k+1)}}{\partial \mathbf{h}^{(k)}}$$

有残差连接 $\mathbf{h}^{(l+1)} = \mathbf{h}^{(l)} + F^{(l)}(\mathbf{h}^{(l)})$：

$$\frac{\partial \mathbf{h}^{(l+1)}}{\partial \mathbf{h}^{(l)}} = I + \frac{\partial F^{(l)}}{\partial \mathbf{h}^{(l)}}$$

反向传播时，梯度包含恒等矩阵 $I$——**无论 $F$ 的导数多小，梯度中总有一个"1"保底**，梯度不会消失。

```python
import torch
import torch.nn as nn
import matplotlib.pyplot as plt

class PlainBlock(nn.Module):
    """无残差连接的普通块"""
    def __init__(self, dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(dim, dim),
            nn.ReLU(),
            nn.Linear(dim, dim),
            nn.ReLU(),
        )
    def forward(self, x): return self.net(x)

class ResidualBlock(nn.Module):
    """有残差连接的块"""
    def __init__(self, dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(dim, dim),
            nn.ReLU(),
            nn.Linear(dim, dim),
        )
    def forward(self, x): return x + self.net(x)   # 核心：加上输入 x

# 对比 50 层网络的梯度流
dim, n_layers = 64, 50

for BlockClass, label in [(PlainBlock, "普通 MLP（无残差）"),
                           (ResidualBlock, "ResNet（有残差）")]:
    blocks = nn.Sequential(*[BlockClass(dim) for _ in range(n_layers)])
    x = torch.randn(1, dim, requires_grad=True)
    loss = blocks(x).sum()
    loss.backward()

    grad_norm = x.grad.norm().item()
    print(f"{label}: 输入层梯度范数 = {grad_norm:.6f}")
    # 普通 MLP 梯度接近 0，ResNet 梯度正常
```

### 5.5 解决方案五：梯度裁剪

对于爆炸梯度，最直接的方案：

```python
# 梯度裁剪：把全局梯度范数裁剪到 max_norm 以内
# 保持方向不变，只缩小幅度

optimizer.zero_grad()
loss.backward()

# 在 step 之前裁剪
grad_norm = torch.nn.utils.clip_grad_norm_(
    model.parameters(),
    max_norm=1.0,        # 阈值，常用 0.5-5.0
    norm_type=2,         # L2 范数
)

optimizer.step()

# 注意：裁剪全局范数（所有参数梯度拼在一起的向量范数）
# 而不是逐参数裁剪（会改变方向）
```

## 六、综合：Transformer 为什么能训练百层

把上面所有知识点串联起来，理解为什么现代 LLM 的架构选择如此设计：

```
Transformer 层的每个组件都有对应的梯度稳定化手段：

Pre-RMSNorm
    ↓  (归一化，防止激活值尺度爆炸)
Multi-Head Self-Attention
    ↓  (Attention 的 QK 内积除以 √d_k，防止 softmax 饱和)
    + 残差连接 (梯度高速公路，100 层后梯度还在)
Pre-RMSNorm
    ↓  (第二次归一化)
FFN (Linear → SiLU → Linear)
    ↓  (SiLU 导数范围友好，不饱和)
    + 残差连接 (再一条梯度高速公路)
```

加上 He 初始化、AdamW 自适应学习率、梯度裁剪，让深达 80-128 层的 Transformer 能稳定训练数万步。

## 七、面试常见问题

**Q1：梯度消失和梯度爆炸分别是什么原因导致的？**

两者都来自反向传播的链式法则——梯度是逐层偏导数的乘积。如果每层的激活函数导数或权重矩阵的谱范数小于 1，乘积随层数指数级缩小，就是梯度消失；如果大于 1，乘积指数级增大，就是梯度爆炸。sigmoid 的最大导数只有 0.25，10 层后梯度就剩 $0.25^{10} \approx 10^{-6}$，典型的梯度消失。

**Q2：ReLU 为什么比 sigmoid 更适合深层网络？**

sigmoid 在正负饱和区导数趋向 0（最大 0.25），导致梯度消失。ReLU 在正区间导数恒为 1，不会饱和，梯度可以完整传播。缺点是负区间导数为 0（Dead ReLU），一旦神经元输出永久为负就再也无法更新。GELU / SiLU 是更平滑的改进，在大模型里是主流。

**Q3：残差连接为什么能解决梯度消失？**

因为残差连接 $h^{(l+1)} = h^{(l)} + F(h^{(l)})$ 让梯度反传时包含恒等矩阵项 $I$：$\partial h^{(l+1)} / \partial h^{(l)} = I + \partial F / \partial h^{(l)}$。无论 $F$ 的梯度多小，这个"1"永远在，梯度不会在任意深度的网络中消失。ResNet 凭此训练了 1000 层网络，Transformer 凭此训练了 100+ 层。

**Q4：Xavier 初始化和 He 初始化分别适合什么情况？**

Xavier 初始化设计假设是线性或类线性的激活函数（sigmoid / tanh），目标是让前向传播各层的方差保持一致：$\text{std} = \sqrt{2/(n_{in}+n_{out})}$。He（Kaiming）初始化针对 ReLU，ReLU 的输出方差是输入的一半（负半轴全部置零），所以需要乘 2 补偿：$\text{std} = \sqrt{2/n_{in}}$。现代 PyTorch 里 `nn.Linear` 默认 Kaiming Uniform 初始化。

**Q5：Pre-LN 和 Post-LN 有什么区别？现代 LLM 用哪个？**

Post-LN 是原始 Transformer 的做法（LayerNorm 在残差加和之后），梯度不稳定，需要精心设计的 warmup 和学习率调度。Pre-LN 把 LayerNorm 移到子层计算之前，残差路径完全不经过归一化，梯度更稳定，不需要特别精心的调参。几乎所有现代大模型（LLaMA、Qwen、GPT-4 系列）都用 Pre-LN。

**Q6：梯度裁剪裁的是什么？为什么不能逐参数裁剪？**

梯度裁剪裁的是"所有参数梯度拼成一个大向量"的 L2 范数（全局范数）。如果范数超过阈值，按比例缩小整个梯度向量，保持各参数梯度的相对方向不变。逐参数裁剪会把每个参数的梯度独立截断，破坏了参数之间的相对比例关系，改变了更新方向，反而可能让训练不稳定。

**Q7：为什么 Attention 的得分要除以 $\sqrt{d_k}$？这和梯度有什么关系？**

如果 $Q, K$ 的元素是标准正态分布，$QK^T$ 的元素方差为 $d_k$（$d_k$ 个独立正态变量乘积之和的方差）。不除的话，$d_k = 64$ 时得分标准差为 8，进入 softmax 后接近 one-hot 分布，softmax 导数接近 0，梯度直接消失。除以 $\sqrt{d_k}$ 把方差拉回 1，softmax 输出不再饱和，梯度可以正常流动。

**Q8：深层网络为什么要用 Batch/Layer Norm？**

随着层数增加，哪怕初始化合理，激活值的均值和方差也会随训练逐渐漂移（Internal Covariate Shift），导致梯度失控。归一化层强制每层输入的分布稳定在均值 0、方差 1 附近，切断了各层之间统计量的耦合，让梯度传播更稳定，也让每层可以使用更大的学习率，加速收敛。
