## 零、开篇：一篇论文改变了整个 AI 领域

先做一个小测试，请用一句话回答：

> "他说他知道那个人不喜欢他"

这句话里最后的"他"，指的是谁？你可能需要读好几遍，在脑子里把几个"他"对应起来。人类处理这种"指代关系"驾轻就熟，但在 2015 年之前，计算机在这件事上非常吃力。**这就是 NLP（自然语言处理）长期面临的核心挑战之一：让机器理解语言中的长距离关系**。

2017 年，Google Brain 的 Vaswani 等人发表《Attention Is All You Need》。在此之前，序列建模的标准工具是 LSTM 和 GRU——串行计算、难以并行、长距离依赖困难。这篇论文的核心主张激进到令人怀疑：**把 RNN 完全扔掉，只用注意力机制**。事实证明这个主张完全正确——Transformer 不仅在机器翻译上打败了 RNN，还在接下来的几年里席卷了 NLP、视觉、语音、多模态的所有主要任务。

理解 Transformer 不只是"知道它的结构"，而是理解**它的每一个设计决策解决了什么问题**。本章先用故事的方式讲清楚这条路是怎么走过来的（前半部分无公式），再建立严谨的架构全局框架（后半部分含代码与估算公式），后续三章逐一深入各个核心组件。

## 一、为什么需要一种新架构：从 RNN 到 Attention

### 1.1 第一代工具：RNN，有记忆但记性不好

**RNN 就像一个读书边做笔记的学生——每读一个词，就更新一下自己的"印象"**。这个"印象"叫做**隐状态（hidden state）**。读完"今天"，印象更新一次；读完"今天天气"，印象再更新一次……一直读到句子末尾，隐状态里理论上包含了整句话的信息。

听起来很完美，问题出在两个地方。

**问题一：记性太差。** 想象你在看一部电影，边看边用一张固定大小的便利贴记录剧情。每出现一个新情节，你就把旧便利贴撕掉重写一张。电影越长，你越不得不把之前的细节压缩掉。看到最后，你可能已经完全不记得第一幕谁是坏人了。

```
输入句子：
"昨天 我 在 图书馆 遇到 了 一个 多年 不见 的 老朋友 ，他 带着 一本 厚厚 的 书 ，
那 本 书 的 封面 是 蓝色 的 ，而 他 本人 看起来 比 以前 老 了 很多 。"

RNN 读到 "老了很多" 时，"昨天"、"图书馆"、"老朋友"
已经在隐状态里被严重稀释，几乎消失了。
```

这就是 RNN 的"**长期记忆问题**"。在数学上，它有一个名字叫**梯度消失**——训练信号从句子末尾传回句子开头时会越来越弱，直到几乎消失。（我们在 2.2.1 讲过这个问题。）

**问题二：只能一个词一个词地读，太慢。** RNN 必须**严格按顺序处理**：读完第 1 个词才能读第 2 个词。这就像一个只有一条流水线的工厂，不管你有多少工人，每次只能干一件活。在 GPU 时代，这是巨大的浪费——GPU 的优势在于**大规模并行计算**，而 RNN 根本无法利用。

### 1.2 第二代工具：LSTM，记性更好，但问题依然存在

LSTM（长短期记忆网络）在 RNN 基础上加入了三个"阀门"（遗忘门、输入门、输出门），让模型能主动决定：哪些旧信息可以丢弃、哪些新信息值得记住、当前该输出什么。这让 LSTM 的"记性"比 RNN 好很多，能处理更长的序列。

但 LSTM **没有解决串行计算慢**的问题——它仍然是一个词一个词地处理。而且即使记性更好，它仍然是通过一条"信息高速公路"把所有信息压缩传递的，句子非常长（几百个词）时，最前面的信息还是很难完整传到最后。

**用一句话总结 RNN/LSTM 的根本局限**：它们是"顺序处理"的机器，只能沿着时间轴从左到右传递信息。

### 1.3 转折点：Attention 机制的出现

2015 年，研究者发现了一个改变一切的思路：

> "与其让模型把所有信息压缩成一个向量，不如让模型在生成每个词时，自己去看一遍整个输入序列，然后决定当前最该关注哪些部分。"

这就是**注意力机制（Attention）**的基本思想。用翻译来理解：翻译"I love machine learning"时，当模型要生成"机器"这个词，它会回头看整句英文，意识到"machine"对生成"机器"最重要（最高注意力），"learning"有点关系（中等），"I"、"love"关系不大（很低）。这就像你看照片时眼睛自然落在最重要的区域，而不是把每个像素平等地看一遍。

注意力机制让模型有了一个新能力：**在处理每个词时，都能直接访问整个序列的任何位置**，不再需要信息通过隐状态一步步传递。

```
没有注意力：
A → B → C → D → E（信息从 A 传到 E 要经过 B、C、D，路途遥远）

有注意力：
A ←→ E（A 和 E 可以直接交流，路径长度永远是 1）
B ←→ E
...所有词对之间都可以直接交流
```

## 二、Transformer：注意力就是全部

### 2.1 一个激进的想法

2017 年，Google 团队做了个激进的决定：**"既然注意力机制这么好用，那就把 RNN 完全扔掉，只用注意力。"** 论文标题就叫《Attention Is All You Need》。

这个想法在当时相当大胆——RNN 已经统治 NLP 多年，很多人觉得注意力只是"RNN 的辅助工具"。但实验结果说明了一切：**纯注意力的 Transformer 在翻译任务上打败了最好的 RNN 模型，而且训练速度快得多**。

### 2.2 Transformer 的三个核心优势

**优势一：并行计算。** RNN 必须一个词一个词地处理（第 $n$ 个词必须等第 $n-1$ 个）；Transformer 可以同时计算所有词对之间的关系，**整个序列一次性全部处理**。打个比方：RNN 是一个人读完一页才能读下一页，Transformer 是几百个人同时读这本书的不同页、读完互相交流。GPU 最擅长同时做很多计算，Transformer 完美利用了这一点。

**优势二：直接建模长距离依赖。** 任意两个词之间的关系，通过一次注意力计算就能直接建立，不需要信息"长途跋涉"穿越中间的词。处理"he said he thought he liked him"这类复杂指代时，比 RNN 容易得多。

**优势三：可以无限扩展。** RNN 的隐状态大小固定，能存储的信息有上限；Transformer 可以通过增加层数、增大维度、增加注意力头数不断扩展。事实证明，**Transformer 的能力随参数量增加几乎线性提升**——从 GPT-1 的 1 亿参数到 GPT-3 的 1750 亿参数，底层都是 Transformer。

### 2.3 Transformer 的代价

没有完美的方案。Transformer 有一个明显的代价：**计算量随序列长度的平方增长**。序列长度变成原来的 2 倍，计算量变成 4 倍；变成 10 倍，计算量变成 100 倍。所以早期 Transformer 处理长文本（比如一本书）代价极高。这个问题后来有很多解决方案（Flash Attention、稀疏注意力等），我们会在 3.8 高效注意力中讲。

### 2.4 为什么"只用注意力"能成功：位置编码

在 2017 年，很多人怀疑：没有 RNN 的记忆传递，模型怎么知道词语的顺序？注意力机制不就是把所有词"平等"地看一遍，失去了顺序信息吗？

**确实如此——注意力机制本身不感知顺序**（把所有词的顺序打乱，注意力计算结果相同，只是对应关系变了）。但 Transformer 有个补充设计叫**位置编码（Positional Encoding）**：在把词送进模型之前，给每个词附加一个"位置标签"。这样模型既能"随机访问"任意位置（注意力机制），又知道每个词的顺序（位置编码）。位置编码是 3.2 和 3.7 的主要内容。

### 2.5 一张图看懂演进历程

```
1986  RNN（循环神经网络）        能处理序列，有记忆
      问题：梯度消失、无法并行、长序列遗忘
         ↓
1997  LSTM（长短期记忆）          门控机制改善记忆能力
      问题：还是无法并行，长序列仍有瓶颈
         ↓
2015  Attention 机制加入 RNN     翻译质量大幅提升

      优点：完全并行、直接长距离依赖、可扩展；问题：计算量 O(L²)

         ↓
```

## 三、原始 Transformer 的整体架构

下面进入严谨部分。RNN 序列模型面临三个致命问题——串行计算、梯度消失、信息瓶颈——Transformer 用三个对应设计一次性解决：**Self-Attention**（每个位置直接和所有其他位置交互，天然并行）、**全局连接**（任意两个位置路径长度为 1）、**无信息瓶颈**（编码器输出完整隐状态序列，解码器通过注意力随机访问任意位置）。

```
输入序列                          目标序列（训练时已知，推理时逐步生成）
    ↓                                    ↓
Token Embedding                   Token Embedding
    +                                    +
Positional Encoding               Positional Encoding
    ↓                                    ↓
┌─────────────────────┐     ┌──────────────────────────────┐
│    Encoder Stack    │     │       Decoder Stack          │
│   ×N 层             │     │       ×N 层                  │
│  ┌───────────────┐  │     │  ┌────────────────────────┐ │
│  │ Multi-Head    │  │     │  │ Masked Multi-Head       │ │
│  │ Self-Attention│  │     │  │ Self-Attention（Causal）│ │
│  └───────────────┘  │     │  └────────────────────────┘ │
│         ↓           │     │              ↓               │
│  ┌───────────────┐  │     │  ┌────────────────────────┐ │
│  │  Add & Norm   │  │──→  │  │ Cross-Attention         │ │
│  └───────────────┘  │     │  │ (Q←Decoder, K/V←Encoder│ │
│         ↓           │     │  └────────────────────────┘ │
│  ┌───────────────┐  │     │              ↓               │
│  │  Feed-Forward │  │     │  ┌────────────────────────┐ │
│  └───────────────┘  │     │  │  Feed-Forward          │ │
│         ↓           │     │  └────────────────────────┘ │
│  ┌───────────────┐  │     └──────────────────────────────┘
│  │  Add & Norm   │  │                  ↓
│  └───────────────┘  │            Linear + Softmax
└─────────────────────┘                  ↓
                                   输出词概率分布
```

## 四、三种架构变体：Encoder-only、Decoder-only、Encoder-Decoder

**原始 Transformer 是双人乐队（编码器 + 解码器），但后来人们发现有时候独唱更好**：

- 任务是"理解"（分类、NER、问答）→ 只要编码器，让每个 token 都看到全局信息
- 任务是"生成"（对话、写作、代码）→ 只要解码器，自左向右逐步生成
- 任务是"转换"（翻译、摘要）→ 两者都要，编码器理解输入，解码器生成输出

### 4.1 Encoder-only：双向理解模型

每个 token 可以看到序列中**所有**其他 token（全双向 Self-Attention）。没有因果掩码，模型天然理解上下文。**典型代表**：BERT、RoBERTa、DeBERTa、ALBERT。**适合任务**：文本分类、命名实体识别、情感分析、句子相似度、RAG 中的 Embedding 模型。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class EncoderBlock(nn.Module):
    """Encoder 块：全双向 Self-Attention + FFN"""
    def __init__(self, d_model: int, n_heads: int, d_ff: int, dropout: float = 0.1):
        super().__init__()
        self.attn  = nn.MultiheadAttention(d_model, n_heads, dropout=dropout, batch_first=True)
        self.ff    = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.drop  = nn.Dropout(dropout)

    def forward(self, x: torch.Tensor, key_padding_mask=None):
        # Pre-Norm
        x = x + self.drop(self.attn(
            self.norm1(x), self.norm1(x), self.norm1(x),
            key_padding_mask=key_padding_mask
        )[0])
        x = x + self.drop(self.ff(self.norm2(x)))
        return x
```

### 4.2 Decoder-only：因果生成模型

每个 token **只能看到自己和它之前**的 token（因果掩码 / Causal Mask）。输出是自回归的：预测下一个 token，把它拼到输入后面，继续预测。**典型代表**：GPT 系列、LLaMA 系列、Qwen、Mistral、DeepSeek、Claude。**适合任务**：文本生成、对话、代码补全、指令跟随——**几乎所有现代 LLM 都是 Decoder-only**。

```python
class DecoderOnlyBlock(nn.Module):
    """Decoder-only 块：Causal Self-Attention + FFN（无 Cross-Attention）"""
    def __init__(self, d_model: int, n_heads: int, d_ff: int):
        super().__init__()
        self.attn  = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ff    = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)

    def forward(self, x: torch.Tensor):
        B, L, D = x.shape
        # 因果掩码：每个位置只能看到 <= 自己位置的 token
        causal_mask = torch.triu(torch.ones(L, L, device=x.device), diagonal=1).bool()

        x_norm = self.norm1(x)
        attn_out, _ = self.attn(x_norm, x_norm, x_norm, attn_mask=causal_mask)
        x = x + attn_out
        x = x + self.ff(self.norm2(x))
        return x
```

### 4.3 Encoder-Decoder：条件生成模型

编码器用全双向 Self-Attention 理解源序列；解码器用 Causal Self-Attention 生成目标序列，同时用 Cross-Attention 查询编码器输出。**典型代表**：T5、BART、mBART、Whisper（语音识别）。**适合任务**：机器翻译、摘要、语音识别、文档问答（需要精确映射输入到输出的任务）。

```python
class CrossAttentionDecoderBlock(nn.Module):
    """Encoder-Decoder 架构的 Decoder 块，含 Cross-Attention"""
    def __init__(self, d_model: int, n_heads: int, d_ff: int):
        super().__init__()
        # 1. Causal Self-Attention
        self.self_attn  = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        # 2. Cross-Attention（Q 来自 Decoder，K/V 来自 Encoder 输出）
        self.cross_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ff    = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.norm3 = nn.LayerNorm(d_model)

    def forward(self, x, encoder_output, causal_mask=None):
        # Causal Self-Attention
        x_n = self.norm1(x)
        x = x + self.self_attn(x_n, x_n, x_n, attn_mask=causal_mask)[0]
        # Cross-Attention：Q 来自 Decoder（x），K/V 来自 Encoder
        x_n = self.norm2(x)
        x = x + self.cross_attn(x_n, encoder_output, encoder_output)[0]
        x = x + self.ff(self.norm3(x))
        return x
```

### 4.4 三种架构对比速查表

| 维度 | Encoder-only | Decoder-only | Encoder-Decoder |
|------|-------------|--------------|-----------------|
| Attention 方向 | 全双向 | 单向（因果） | Enc: 双向；Dec: 单向+跨模态 |
| 适合任务 | 理解、分类 | 生成、对话 | 翻译、摘要 |
| 预训练目标 | MLM（掩码预测） | CLM（下一词预测） | Seq2Seq（多种变体）|
| 代表模型 | BERT / DeBERTa | GPT / LLaMA / Qwen | T5 / BART / Whisper |
| 当前主流地位 | Embedding 模型 | **几乎所有大模型** | 部分专业场景 |

## 五、现代 LLM 的 Decoder 架构细节

原始 Transformer（2017）和现代 LLM（2024）之间有显著的架构演进。以 LLaMA-3 为代表的现代 LLM Decoder 架构如下：

### 5.1 现代 LLM Decoder 层结构

```
Token IDs
    ↓
Embedding（vocab_size × d_model）
    ↓
┌────────────────────────────────────┐
│  Transformer Decoder Block × N     │
│  Input x                           │
│    ↓                               │
│  RMSNorm                           │  ← Pre-Norm（非 Post-Norm）
│    ↓                               │
│  Multi-Head Self-Attention         │  ← GQA（非 MHA）
│  + RoPE 位置编码                   │  ← 相对位置（非绝对位置）
│    ↓                               │
│  x = x + attn_out（残差）          │
│    ↓                               │
│  RMSNorm                           │  ← Pre-Norm
│    ↓                               │
│  SwiGLU FFN                        │  ← 非标准 GELU FFN
│    ↓                               │
│  x = x + ffn_out（残差）           │
└────────────────────────────────────┘
    ↓
Final RMSNorm
    ↓
LM Head（= Embedding 矩阵转置，Weight Tying）
    ↓
Logits（vocab_size）
```

### 5.2 原始 Transformer vs 现代 LLM 的关键差异

| 组件 | 原始 Transformer（2017）| 现代 LLM（2024）| 变化原因 |
|------|------------------------|----------------|---------|
| 归一化 | Post-LayerNorm | Pre-RMSNorm | 更稳定、更快 |
| 位置编码 | 绝对正弦编码 | RoPE（旋转位置编码）| 更好的外推性 |
| 注意力变体 | 标准 MHA | GQA（分组查询注意力）| 节省 KV Cache |
| FFN 激活 | ReLU | SwiGLU | 更强表达力 |
| Bias 项 | 有 | 无（no bias）| 节省参数，效果相当 |
| Dropout | 0.1 | 0.0（预训练）| 数据足够大 |
| 权重共享 | 无 | Embedding ↔ LM Head | 节省参数 |

## 六、参数量与计算量：工程师必会的估算

**会估算参数量和 FLOPs，是大模型工程师的基本功**。面试被问到"训练一个 7B 模型需要多少显存"或"用 256 张 A100 训练 1T token 要多少天"，你不能只会用现成工具，要能口算出数量级。下面给出的公式在实际工程中精确到 10% 以内。

### 6.1 参数量估算

以标准 Decoder-only Transformer 为例（$L$ 层，隐层维度 $d$，FFN 扩展比 $r$，词表大小 $V$）：

| 组件 | 参数量 | 说明 |
|------|--------|------|
| Attention Q/K/V | $3 \times d^2$ | 三个线性投影，无 bias |
| Attention Output | $d^2$ | 输出线性投影 |
| FFN（标准，r=4） | $2 \times 4d^2 = 8d^2$ | 两个线性层 |
| FFN（SwiGLU，r≈8/3）| $3 \times \frac{8}{3}d^2 \approx 8d^2$ | 三个线性层，参数量相近 |
| RMSNorm（×2） | $2d$ | 可忽略 |
| **每层合计** | **≈ $12d^2$** | |

**全模型参数**：

$$N \approx \underbrace{12 \cdot L \cdot d^2}_{\text{Transformer 层}} + \underbrace{V \cdot d}_{\text{Embedding}}$$

注意：Weight Tying 时 LM Head 和 Embedding 共享，只算一次。

```python
def estimate_params(
    n_layers: int,
    d_model: int,
    vocab_size: int,
    ffn_ratio: float = 8/3,     # SwiGLU 约为 8/3
    weight_tying: bool = True,
    verbose: bool = True,
) -> dict:
    """精确估算 Decoder-only Transformer 参数量"""
    # 每层 Attention（Q K V O，各 d×d，无 bias）
    attn_per_layer = 4 * d_model * d_model

    # 每层 FFN（SwiGLU: gate + up + down）
    d_ff = int(d_model * ffn_ratio)
    d_ff = ((d_ff + 63) // 64) * 64   # 对齐 64 的倍数
    ffn_per_layer = 3 * d_model * d_ff  # gate/up/down 各一个

    transformer_params = n_layers * (attn_per_layer + ffn_per_layer)
    embedding_params   = vocab_size * d_model
    # Weight Tying：LM Head 共享 Embedding 权重
    lm_head_params     = 0 if weight_tying else vocab_size * d_model

    total = transformer_params + embedding_params + lm_head_params

    if verbose:
        print(f"模型配置：L={n_layers}, d={d_model}, V={vocab_size:,}, FFN={d_ff}")
        print(f"Transformer 层: {transformer_params/1e9:.3f}B  ({transformer_params/total:.1%})")
        print(f"Embedding:      {embedding_params/1e9:.3f}B  ({embedding_params/total:.1%})")
        print(f"总参数量:        {total/1e9:.2f}B")
        print(f"BF16 显存占用:   {total*2/1e9:.1f} GB（仅参数，不含梯度和优化器状态）")

    return {"total": total, "transformer": transformer_params, "embedding": embedding_params}

# 验证主流模型
configs = {
    "LLaMA-3-8B":   (32, 4096,  32000),
    "LLaMA-3-70B":  (80, 8192,  32000),
    "Qwen-2.5-7B":  (28, 3584, 152064),
    "Mistral-7B":   (32, 4096,  32000),
}

for name, (layers, dim, vocab) in configs.items():
    print(f"\n{'='*10} {name} {'='*10}")
    estimate_params(layers, dim, vocab)
```

### 6.2 FLOPs 估算

**前向传播每参数约 2 次浮点运算**（一次乘法 + 一次加法），所以经验公式（精确到 10% 以内）：

$$\text{FLOPs}_{\text{forward}} \approx 2N$$

**更精确的逐组件公式**（每层每 token）：

| 组件 | FLOPs（每层每 token）|
|------|---------------------|
| QKV 投影 | $6d^2$ |
| Attention 矩阵 QK^T | $2 \times L_{seq} \times d$ |
| Softmax + PV 乘积 | $2 \times L_{seq} \times d$ |
| Output 投影 | $2d^2$ |
| FFN（两个线性层）| $\approx 8d^2$（标准 r=4）|
| **每层每 token 合计** | $\approx 24d^2$（忽略 Attention 的序列项）|

**训练总 FLOPs**（前向 + 反向，反向约是前向的 2 倍）：

$$\text{FLOPs}_{\text{训练}} \approx 6N \cdot D_{\text{tokens}}$$

```python
def estimate_training_cost(
    n_params: float,       # 参数量（非 embedding），如 7e9
    n_tokens: float,       # 训练 token 数，如 1e12
    gpu_flops_bf16: float, # GPU 峰值算力，如 A100=312e12
    n_gpus: int = 1,
    mfu: float = 0.4,      # 模型浮点利用率，大规模训练约 0.3-0.5
):
    """估算训练时间"""
    total_flops   = 6 * n_params * n_tokens    # 前向 + 反向
    gpu_effective = gpu_flops_bf16 * n_gpus * mfu
    hours = total_flops / gpu_effective / 3600
    days  = hours / 24

    print(f"总 FLOPs: {total_flops:.2e}")
    print(f"有效算力: {gpu_effective/1e12:.0f} TFLOPS（{n_gpus}× A100 × MFU{mfu}）")
    print(f"估算训练时间: {hours:.0f} 小时 = {days:.1f} 天")
    print(f"GPU 天数: {days * n_gpus:.0f} GPU-days")

# 示例：LLaMA-3-8B，15T token，2048×A100
print("=== LLaMA-3-8B 训练估算 ===")
estimate_training_cost(
    n_params=7.5e9,       # 非 embedding 参数
    n_tokens=15e12,       # LLaMA-3 训练了 15T token
    gpu_flops_bf16=312e12,
    n_gpus=2048,
    mfu=0.38,
)
```

### 6.3 显存估算速查

**推理时**（BF16，batch_size=1，seq_len=2048）：

| 部分 | 显存 | 说明 |
|------|------|------|
| 模型参数（BF16）| $N \times 2$ bytes | 7B → 14 GB |
| KV Cache（BF16）| $4 \times L \times d \times L_{seq} \times 2$ bytes | 7B → ~4 GB |
| 激活值 | 小，可忽略 | |

**训练时（AdamW，BF16 参数 + FP32 优化器状态）**：

| 部分 | 显存 |
|------|------|
| BF16 参数 | $2N$ bytes |
| BF16 梯度 | $2N$ bytes |
| FP32 Adam m 状态 | $4N$ bytes |
| FP32 Adam v 状态 | $4N$ bytes |
| 激活值（无梯度检查点）| 约 $12 \times d \times L_{seq} \times L \times 2$ bytes |
| **合计（不含激活）** | $\approx 12N$ bytes（7B → ~84 GB）|

## 七、完整的 LLM Decoder 实现

将上述所有内容组合成一个完整、可运行的 Decoder-only Transformer：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math
from dataclasses import dataclass

@dataclass
class LLMConfig:
    vocab_size:  int   = 32000
    d_model:     int   = 512
    n_layers:    int   = 8
    n_heads:     int   = 8
    max_seq_len: int   = 2048
    dropout:     float = 0.0    # 预训练设为 0

class RMSNorm(nn.Module):
    def __init__(self, dim: int, eps: float = 1e-6):
        super().__init__()
        self.scale = nn.Parameter(torch.ones(dim))
        self.eps   = eps

    def forward(self, x):
        rms = x.pow(2).mean(-1, keepdim=True).add(self.eps).sqrt()
        return self.scale * x / rms

class SwiGLUFFN(nn.Module):
    def __init__(self, d_model: int):
        super().__init__()
        hidden = int(d_model * 8 / 3)
        hidden = ((hidden + 63) // 64) * 64
        self.gate_proj = nn.Linear(d_model, hidden, bias=False)
        self.up_proj   = nn.Linear(d_model, hidden, bias=False)
        self.down_proj = nn.Linear(hidden,   d_model, bias=False)

    def forward(self, x):
        return self.down_proj(F.silu(self.gate_proj(x)) * self.up_proj(x))

class CausalSelfAttention(nn.Module):
    def __init__(self, cfg: LLMConfig):
        super().__init__()
        assert cfg.d_model % cfg.n_heads == 0
        self.n_heads = cfg.n_heads
        self.d_head  = cfg.d_model // cfg.n_heads
        self.q_proj  = nn.Linear(cfg.d_model, cfg.d_model, bias=False)
        self.k_proj  = nn.Linear(cfg.d_model, cfg.d_model, bias=False)
        self.v_proj  = nn.Linear(cfg.d_model, cfg.d_model, bias=False)
        self.o_proj  = nn.Linear(cfg.d_model, cfg.d_model, bias=False)

    def forward(self, x):
        B, L, D = x.shape
        def split_heads(t):
            return t.view(B, L, self.n_heads, self.d_head).transpose(1, 2)

        q = split_heads(self.q_proj(x))   # (B, H, L, d_h)
        k = split_heads(self.k_proj(x))
        v = split_heads(self.v_proj(x))

        # Flash Attention 路径（PyTorch 2.0+，自动选择最优 kernel）
        out = F.scaled_dot_product_attention(q, k, v, is_causal=True)
        out = out.transpose(1, 2).contiguous().view(B, L, D)
        return self.o_proj(out)

class DecoderLayer(nn.Module):
    def __init__(self, cfg: LLMConfig):
        super().__init__()
        self.attn_norm = RMSNorm(cfg.d_model)
        self.ffn_norm  = RMSNorm(cfg.d_model)
        self.attn = CausalSelfAttention(cfg)
        self.ffn  = SwiGLUFFN(cfg.d_model)

    def forward(self, x):
        x = x + self.attn(self.attn_norm(x))   # Pre-Norm 残差
        x = x + self.ffn(self.ffn_norm(x))
        return x

class LLM(nn.Module):
    def __init__(self, cfg: LLMConfig):
        super().__init__()
        self.cfg = cfg
        self.embedding   = nn.Embedding(cfg.vocab_size, cfg.d_model)
        self.layers      = nn.ModuleList([DecoderLayer(cfg) for _ in range(cfg.n_layers)])
        self.final_norm  = RMSNorm(cfg.d_model)
        self.lm_head     = nn.Linear(cfg.d_model, cfg.vocab_size, bias=False)
        # Weight Tying
        self.lm_head.weight = self.embedding.weight
        # 初始化
        self.apply(self._init_weights)

    def _init_weights(self, m):
        if isinstance(m, nn.Linear):
            nn.init.normal_(m.weight, std=0.02)
        elif isinstance(m, nn.Embedding):
            nn.init.normal_(m.weight, std=0.02)

    def forward(self, input_ids, labels=None):
        x = self.embedding(input_ids)
        for layer in self.layers:
            x = layer(x)
        x = self.final_norm(x)
        logits = self.lm_head(x)  # (B, L, V)

        loss = None
        if labels is not None:
            # 语言模型损失：每个位置预测下一个 token
            loss = F.cross_entropy(
                logits[:, :-1].contiguous().view(-1, self.cfg.vocab_size),
                labels[:, 1:].contiguous().view(-1),
                ignore_index=-100,
            )
        return logits, loss

    @property
    def n_params(self):
        return sum(p.numel() for p in self.parameters())

# 验证
cfg   = LLMConfig(vocab_size=32000, d_model=512, n_layers=8, n_heads=8)
model = LLM(cfg)
print(f"模型参数量: {model.n_params/1e6:.1f}M")

x      = torch.randint(0, 32000, (2, 64))
labels = x.clone()
logits, loss = model(x, labels)
print(f"输出形状: {logits.shape}")  # (2, 64, 32000)
print(f"Loss:     {loss.item():.4f}")
```

## 八、面试常见问题

**Q1：RNN 有哪些主要问题？Transformer 是如何解决的？**

RNN 有两个核心问题。一是梯度消失导致的长期记忆问题：信息从句子末尾传到开头时信号越来越弱，很难学到长距离依赖。二是串行计算的效率问题：每个时间步必须等待上一步完成，无法利用 GPU 的并行能力。Transformer 用 Self-Attention 替代 RNN：每个词可以直接和任意其他词建立联系，路径长度永远是 1（解决长距离依赖）；整个序列可以并行处理（解决串行效率）。

**Q2：为什么说"Attention is All You Need"这个标题很激进？**

2017 年之前，注意力机制被认为是 RNN 的"增强工具"，用于帮助 RNN 对齐源序列和目标序列，但没人认为可以完全替代 RNN。这篇论文主张完全抛弃 RNN，只用注意力——这在当时非常激进。事实证明这个选择正确，Transformer 在效果和效率上全面超越了 RNN 系列。

**Q3：Transformer 有什么缺点？为什么需要位置编码？**

最主要的缺点是计算复杂度：处理长度为 $L$ 的序列时 Self-Attention 是 $O(L^2)$，序列翻倍计算量变 4 倍，使得处理超长文本非常昂贵（后续有 Flash Attention、稀疏注意力缓解）。需要位置编码是因为注意力机制本身"位置无关"——打乱所有词顺序，注意力结果相同。位置编码给每个位置分配独特"坐标"，和词的语义信息叠加后送进模型。

**Q4：Encoder-only、Decoder-only、Encoder-Decoder 各适合什么任务？为什么现代 LLM 几乎都是 Decoder-only？**

Encoder-only（如 BERT）适合理解类任务（分类、NER），双向注意力能充分利用上下文。Decoder-only 适合生成类任务，自回归生成与预训练目标天然一致。Encoder-Decoder（如 T5）适合翻译、摘要等输入输出差异大的任务。现代 LLM 几乎全是 Decoder-only 有三个原因：预训练目标（下一词预测）和推理行为完全一致；扩展性更好——同等参数下通用能力更强；工程更简单——不需要管理两套参数和 Cross-Attention。

**Q5：原始 Transformer 和现代 LLM（如 LLaMA-3）的架构有哪些关键区别？**

五个主要差异：归一化从 Post-LayerNorm 变为 Pre-RMSNorm（更稳定、更快）；位置编码从绝对正弦编码变为 RoPE（更好的长度外推性）；注意力从标准 MHA 变为 GQA（节省 KV Cache）；FFN 激活从 ReLU 变为 SwiGLU（更强表达力）；几乎所有层去掉了 bias 参数。

**Q6：一个 7B 参数的 LLM，参数量是怎么分配的？训练 FLOPs 经验公式是什么？**

每层约 $12d^2$ 参数（Attention 贡献 $4d^2$，FFN 贡献 $8d^2$）乘以层数，加上 Embedding 的 $V \times d$（Weight Tying 时 LM Head 共享）。7B 模型（$d=4096, L=32, V=32000$）：Transformer 层约 6.4B，Embedding 约 0.13B；FFN 约占 65%，Attention 约 30%，Embedding 约 2-5%。训练 FLOPs $\approx 6ND$：前向约 $2N$，反向约是前向 2 倍，合计 $6N$，乘以训练 token 数 $D$。

**Q7：什么是 Weight Tying？为什么 LLM 几乎都去掉了 Bias？**

Weight Tying 是让输入 Embedding 和输出 LM Head 共享同一组参数（形状互为转置）。好处：减少约 $V \times d$ 参数（7B 模型约省 130M）；语义上合理——把 token 映射到隐空间和把隐空间映射回 token 概率用同一套"词语理解"；实验表明能在不增加参数下提升性能。去掉 bias 同理：实验表明对性能影响很小（有时还有提升），能省约 0.1% 参数，且与 RMSNorm、Weight Tying 等简化趋势一致。
