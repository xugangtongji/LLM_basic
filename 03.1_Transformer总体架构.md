## 零、开篇：一篇论文改变了整个 AI 领域

2017 年，Google Brain 的 Vaswani 等人发表《Attention Is All You Need》。在此之前，序列建模的标准工具是 LSTM 和 GRU——串行计算、难以并行、长距离依赖困难。

这篇论文的核心主张激进到令人怀疑：**把 RNN 完全扔掉，只用注意力机制**。事实证明这个主张完全正确——Transformer 不仅在机器翻译上打败了 RNN，还在接下来的几年里席卷了 NLP、视觉、语音、多模态的所有主要任务。

理解 Transformer 不只是"知道它的结构"，而是理解**它的每一个设计决策解决了什么问题**。本章建立全局框架，后续三章逐一深入各个核心组件。

## 一、从 Seq2Seq 到 Transformer：架构演进的动机

### 1.1 初印象：用费曼视角理解

**RNN 序列模型面临三个致命问题**：

- 串行计算：每个时间步等待上一步，无法并行，训练极慢
- 梯度消失：长序列梯度反传困难，远距离依赖难以学习
- 信息瓶颈：把整句话压成一个固定大小的向量，句子越长信息损失越大

Transformer 一次性解决了全部三个问题：

- **Self-Attention**：每个位置直接和所有其他位置交互，天然并行
- **全局连接**：任意两个位置之间路径长度为 1，远距离依赖和近距离依赖同等容易
- **无信息瓶颈**：编码器输出完整的隐状态序列，解码器通过注意力随机访问任意位置

### 1.2 原始 Transformer 的整体架构

```
输入序列                          目标序列（训练时已知，推理时逐步生成）
    ↓                                    ↓
Token Embedding                   Token Embedding
    +                                    +
Positional Encoding               Positional Encoding
    ↓                                    ↓
┌─────────────────────┐     ┌──────────────────────────────┐
│    Encoder Stack    │     │       Decoder Stack          │
│   ×N 层             │     │       ×N 层                  │
│                     │     │                              │
│  ┌───────────────┐  │     │  ┌────────────────────────┐ │
│  │ Multi-Head    │  │     │  │ Masked Multi-Head       │ │
│  │ Self-Attention│  │     │  │ Self-Attention（Causal）│ │
│  └───────────────┘  │     │  └────────────────────────┘ │
│         ↓           │     │              ↓               │
│  ┌───────────────┐  │     │  ┌────────────────────────┐ │
│  │  Add & Norm   │  │──→  │  │ Cross-Attention         │ │
│  └───────────────┘  │     │  │ (Q←Decoder, K/V←Encoder│ │
│         ↓           │     │  └────────────────────────┘ │
│  ┌───────────────┐  │     │              ↓               │
│  │  Feed-Forward │  │     │  ┌────────────────────────┐ │
│  └───────────────┘  │     │  │  Feed-Forward          │ │
│         ↓           │     │  └────────────────────────┘ │
│  ┌───────────────┐  │     └──────────────────────────────┘
│  │  Add & Norm   │  │                  ↓
│  └───────────────┘  │            Linear + Softmax
└─────────────────────┘                  ↓
                                   输出词概率分布
```

## 二、三种架构变体：Encoder-only、Decoder-only、Encoder-Decoder

### 2.1 初印象：用费曼视角理解

**原始 Transformer 是双人乐队（编码器 + 解码器），但后来人们发现有时候独唱更好**。

- 如果任务是"理解"（分类、NER、问答）→ 只要编码器，让每个 token 都看到全局信息
- 如果任务是"生成"（对话、写作、代码）→ 只要解码器，自左向右逐步生成
- 如果任务是"转换"（翻译、摘要、结构化输出）→ 两者都要，编码器理解输入，解码器生成输出

### 2.2 Encoder-only：双向理解模型

每个 token 可以看到序列中**所有**其他 token（全双向 Self-Attention）。没有因果掩码，模型天然理解上下文。

**典型代表**：BERT、RoBERTa、DeBERTa、ALBERT

**适合任务**：文本分类、命名实体识别、情感分析、句子相似度、RAG 中的 Embedding 模型

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class EncoderBlock(nn.Module):
    """Encoder 块：全双向 Self-Attention + FFN"""
    def __init__(self, d_model: int, n_heads: int, d_ff: int, dropout: float = 0.1):
        super().__init__()
        self.attn  = nn.MultiheadAttention(d_model, n_heads, dropout=dropout, batch_first=True)
        self.ff    = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.drop  = nn.Dropout(dropout)

    def forward(self, x: torch.Tensor, key_padding_mask=None):
        # Pre-Norm
        x = x + self.drop(self.attn(
            self.norm1(x), self.norm1(x), self.norm1(x),
            key_padding_mask=key_padding_mask
        )[0])
        x = x + self.drop(self.ff(self.norm2(x)))
        return x
```

### 2.3 Decoder-only：因果生成模型

每个 token **只能看到自己和它之前**的 token（因果掩码 / Causal Mask）。输出是自回归的：预测下一个 token，把它拼到输入后面，继续预测。

**典型代表**：GPT 系列、LLaMA 系列、Qwen、Mistral、DeepSeek、Claude

**适合任务**：文本生成、对话、代码补全、指令跟随——**几乎所有现代 LLM 都是 Decoder-only**

```python
class DecoderOnlyBlock(nn.Module):
    """Decoder-only 块：Causal Self-Attention + FFN（无 Cross-Attention）"""
    def __init__(self, d_model: int, n_heads: int, d_ff: int):
        super().__init__()
        self.attn  = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ff    = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)

    def forward(self, x: torch.Tensor):
        B, L, D = x.shape
        # 因果掩码：每个位置只能看到 <= 自己位置的 token
        causal_mask = torch.triu(torch.ones(L, L, device=x.device), diagonal=1).bool()

        x_norm = self.norm1(x)
        attn_out, _ = self.attn(x_norm, x_norm, x_norm, attn_mask=causal_mask)
        x = x + attn_out
        x = x + self.ff(self.norm2(x))
        return x
```

### 2.4 Encoder-Decoder：条件生成模型

编码器用全双向 Self-Attention 理解源序列；解码器用 Causal Self-Attention 生成目标序列，同时用 Cross-Attention 查询编码器输出。

**典型代表**：T5、BART、mBART、Whisper（语音识别）、Gemini 的多模态处理部分

**适合任务**：机器翻译、摘要、语音识别、文档问答（需要精确映射输入到输出的任务）

```python
class CrossAttentionDecoderBlock(nn.Module):
    """Encoder-Decoder 架构的 Decoder 块，含 Cross-Attention"""
    def __init__(self, d_model: int, n_heads: int, d_ff: int):
        super().__init__()
        # 1. Causal Self-Attention
        self.self_attn  = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        # 2. Cross-Attention（Q 来自 Decoder，K/V 来自 Encoder 输出）
        self.cross_attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ff    = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model)
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.norm3 = nn.LayerNorm(d_model)

    def forward(self, x, encoder_output, causal_mask=None):
        # Causal Self-Attention
        x_n = self.norm1(x)
        x = x + self.self_attn(x_n, x_n, x_n, attn_mask=causal_mask)[0]
        # Cross-Attention：Q 来自 Decoder（x），K/V 来自 Encoder
        x_n = self.norm2(x)
        x = x + self.cross_attn(x_n, encoder_output, encoder_output)[0]
        x = x + self.ff(self.norm3(x))
        return x
```

### 2.5 三种架构对比速查表

| 维度 | Encoder-only | Decoder-only | Encoder-Decoder |
|------|-------------|--------------|-----------------|
| Attention 方向 | 全双向 | 单向（因果） | Enc: 双向；Dec: 单向+跨模态 |
| 适合任务 | 理解、分类 | 生成、对话 | 翻译、摘要 |
| 预训练目标 | MLM（掩码预测） | CLM（下一词预测） | Seq2Seq（多种变体）|
| 代表模型 | BERT / DeBERTa | GPT / LLaMA / Qwen | T5 / BART / Whisper |
| 当前主流地位 | Embedding 模型 | **几乎所有大模型** | 部分专业场景 |

## 三、现代 LLM 的 Decoder 架构细节

原始 Transformer（2017）和现代 LLM（2024）之间有显著的架构演进。以 LLaMA-3 为代表的现代 LLM Decoder 架构如下：

### 3.1 现代 LLM Decoder 层结构

```
Token IDs
    ↓
Embedding（vocab_size × d_model）
    ↓
┌────────────────────────────────────┐
│  Transformer Decoder Block × N     │
│                                    │
│  Input x                           │
│    ↓                               │
│  RMSNorm                           │  ← Pre-Norm（非 Post-Norm）
│    ↓                               │
│  Multi-Head Self-Attention         │  ← GQA（非 MHA）
│  + RoPE 位置编码                   │  ← 相对位置（非绝对位置）
│    ↓                               │
│  x = x + attn_out（残差）          │
│    ↓                               │
│  RMSNorm                           │  ← Pre-Norm
│    ↓                               │
│  SwiGLU FFN                        │  ← 非标准 GELU FFN
│    ↓                               │
│  x = x + ffn_out（残差）           │
└────────────────────────────────────┘
    ↓
Final RMSNorm
    ↓
LM Head（= Embedding 矩阵转置，Weight Tying）
    ↓
Logits（vocab_size）
```

### 3.2 原始 Transformer vs 现代 LLM 的关键差异

| 组件 | 原始 Transformer（2017）| 现代 LLM（2024）| 变化原因 |
|------|------------------------|----------------|---------|
| 归一化 | Post-LayerNorm | Pre-RMSNorm | 更稳定、更快 |
| 位置编码 | 绝对正弦编码 | RoPE（旋转位置编码）| 更好的外推性 |
| 注意力变体 | 标准 MHA | GQA（分组查询注意力）| 节省 KV Cache |
| FFN 激活 | ReLU | SwiGLU | 更强表达力 |
| Bias 项 | 有 | 无（no bias）| 节省参数，效果相当 |
| Dropout | 0.1 | 0.0（预训练）| 数据足够大 |
| 权重共享 | 无 | Embedding ↔ LM Head | 节省参数 |

## 四、参数量与计算量：工程师必会的估算

### 4.1 初印象：用费曼视角理解

**会估算参数量和 FLOPs，是大模型工程师的基本功**。

面试被问到"训练一个 7B 模型需要多少显存"，或者"用 256 张 A100 训练 1T token 要多少天"——你不能只会用现成工具，要能口算出数量级。

这里给出的公式在实际工程中精确到 10% 以内，完全够用。

### 4.2 参数量估算

以标准 Decoder-only Transformer 为例（$L$ 层，隐层维度 $d$，头数 $h$，FFN 扩展比 $r$，词表大小 $V$）：

**每层的参数**：

| 组件 | 参数量 | 说明 |
|------|--------|------|
| Attention Q/K/V | $3 \times d^2$ | 三个线性投影，无 bias |
| Attention Output | $d^2$ | 输出线性投影 |
| FFN（标准，r=4） | $2 \times 4d^2 = 8d^2$ | 两个线性层 |
| FFN（SwiGLU，r≈8/3）| $3 \times \frac{8}{3}d^2 \approx 8d^2$ | 三个线性层，参数量相近 |
| RMSNorm（×2） | $2d$ | 两个 norm 层，可忽略 |
| **每层合计** | **≈ $12d^2$** | 忽略 norm 的小参数 |

**全模型参数**：

$$N \approx \underbrace{12 \cdot L \cdot d^2}_{\text{Transformer 层}} + \underbrace{V \cdot d}_{\text{Embedding}}$$

注意：Weight Tying 时 LM Head 和 Embedding 共享，只算一次。

```python
def estimate_params(
    n_layers: int,
    d_model: int,
    vocab_size: int,
    ffn_ratio: float = 8/3,     # SwiGLU 约为 8/3
    weight_tying: bool = True,
    verbose: bool = True,
) -> dict:
    """精确估算 Decoder-only Transformer 参数量"""
    # 每层 Attention（Q K V O，各 d×d，无 bias）
    attn_per_layer = 4 * d_model * d_model

    # 每层 FFN（SwiGLU: gate + up + down）
    d_ff = int(d_model * ffn_ratio)
    d_ff = ((d_ff + 63) // 64) * 64   # 对齐 64 的倍数
    ffn_per_layer = 3 * d_model * d_ff  # gate/up/down 各一个

    transformer_params = n_layers * (attn_per_layer + ffn_per_layer)
    embedding_params   = vocab_size * d_model
    # Weight Tying：LM Head 共享 Embedding 权重
    lm_head_params     = 0 if weight_tying else vocab_size * d_model

    total = transformer_params + embedding_params + lm_head_params

    if verbose:
        print(f"模型配置：L={n_layers}, d={d_model}, V={vocab_size:,}, FFN={d_ff}")
        print(f"Transformer 层: {transformer_params/1e9:.3f}B  ({transformer_params/total:.1%})")
        print(f"Embedding:      {embedding_params/1e9:.3f}B  ({embedding_params/total:.1%})")
        print(f"总参数量:        {total/1e9:.2f}B")
        print(f"BF16 显存占用:   {total*2/1e9:.1f} GB（仅参数，不含梯度和优化器状态）")

    return {"total": total, "transformer": transformer_params, "embedding": embedding_params}

# 验证主流模型
configs = {
    "LLaMA-3-8B":   (32, 4096,  32000),
    "LLaMA-3-70B":  (80, 8192,  32000),
    "Qwen-2.5-7B":  (28, 3584, 152064),
    "Mistral-7B":   (32, 4096,  32000),
}

for name, (layers, dim, vocab) in configs.items():
    print(f"\n{'='*10} {name} {'='*10}")
    estimate_params(layers, dim, vocab)
```

### 4.3 FLOPs 估算

**前向传播每 token 的 FLOPs**（关键公式，来自 Chinchilla 论文和 OpenAI scaling book）：

**经验公式**（精确到 10% 以内）：

$$\text{FLOPs}_{\text{forward}} \approx 2N$$

其中 $N$ 是非 Embedding 参数量，前向传播每个参数约有 2 次浮点运算（一次乘法 + 一次加法）。

**更精确的逐组件公式**（每 token）：

| 组件 | FLOPs（每层每 token）|
|------|---------------------|
| QKV 投影 | $2 \times 3 \times d^2 = 6d^2$ |
| Attention 矩阵 QK^T | $2 \times L_{seq} \times d$ |
| Softmax + PV 乘积 | $2 \times L_{seq} \times d$ |
| Output 投影 | $2d^2$ |
| FFN（两个线性层）| $2 \times 2 \times d \times d_{ff} \approx 8d^2$（标准 r=4）|
| **每层每 token 合计** | $\approx 24d^2$（忽略 Attention 的序列项）|

**训练总 FLOPs**（前向 + 反向，反向约是前向的 2 倍）：

$$\text{FLOPs}_{\text{训练}} \approx 6N \cdot D_{\text{tokens}}$$

```python
def estimate_training_cost(
    n_params: float,       # 参数量（非 embedding），如 7e9
    n_tokens: float,       # 训练 token 数，如 1e12
    gpu_flops_bf16: float, # GPU 峰值算力，如 A100=312e12
    n_gpus: int = 1,
    mfu: float = 0.4,      # 模型浮点利用率，大规模训练约 0.3-0.5
):
    """估算训练时间"""
    total_flops   = 6 * n_params * n_tokens    # 前向 + 反向
    gpu_effective = gpu_flops_bf16 * n_gpus * mfu
    hours = total_flops / gpu_effective / 3600
    days  = hours / 24

    print(f"总 FLOPs: {total_flops:.2e}")
    print(f"有效算力: {gpu_effective/1e12:.0f} TFLOPS（{n_gpus}× A100 × MFU{mfu}）")
    print(f"估算训练时间: {hours:.0f} 小时 = {days:.1f} 天")
    print(f"GPU 天数: {days * n_gpus:.0f} GPU-days")

# 示例：LLaMA-3-8B，1T token，256×A100
print("=== LLaMA-3-8B 训练估算 ===")
estimate_training_cost(
    n_params=7.5e9,       # 非 embedding 参数
    n_tokens=15e12,       # LLaMA-3 训练了 15T token
    gpu_flops_bf16=312e12,
    n_gpus=2048,
    mfu=0.38,
)
```

### 4.4 显存估算速查

**推理时**（BF16，batch_size=1，seq_len=2048）：

| 部分 | 显存 | 说明 |
|------|------|------|
| 模型参数（BF16）| $N \times 2$ bytes | 7B → 14 GB |
| KV Cache（BF16）| $4 \times L \times d \times L_{seq} \times 2$ bytes | 7B → ~4 GB |
| 激活值 | 小，可忽略 | |

**训练时（AdamW，BF16 参数 + FP32 优化器状态）**：

| 部分 | 显存 |
|------|------|
| BF16 参数 | $2N$ bytes |
| BF16 梯度 | $2N$ bytes |
| FP32 Adam m 状态 | $4N$ bytes |
| FP32 Adam v 状态 | $4N$ bytes |
| 激活值（无梯度检查点）| 约 $12 \times d \times L_{seq} \times L \times 2$ bytes |
| **合计** | $\approx 12N$ bytes（7B → ~84 GB，只需参数部分就是这样）|

## 五、完整的 LLM Decoder 实现

将上述所有内容组合成一个完整、可运行的 Decoder-only Transformer：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math
from dataclasses import dataclass

@dataclass
class LLMConfig:
    vocab_size:  int   = 32000
    d_model:     int   = 512
    n_layers:    int   = 8
    n_heads:     int   = 8
    max_seq_len: int   = 2048
    dropout:     float = 0.0    # 预训练设为 0

class RMSNorm(nn.Module):
    def __init__(self, dim: int, eps: float = 1e-6):
        super().__init__()
        self.scale = nn.Parameter(torch.ones(dim))
        self.eps   = eps

    def forward(self, x):
        rms = x.pow(2).mean(-1, keepdim=True).add(self.eps).sqrt()
        return self.scale * x / rms

class SwiGLUFFN(nn.Module):
    def __init__(self, d_model: int):
        super().__init__()
        hidden = int(d_model * 8 / 3)
        hidden = ((hidden + 63) // 64) * 64
        self.gate_proj = nn.Linear(d_model, hidden, bias=False)
        self.up_proj   = nn.Linear(d_model, hidden, bias=False)
        self.down_proj = nn.Linear(hidden,   d_model, bias=False)

    def forward(self, x):
        return self.down_proj(F.silu(self.gate_proj(x)) * self.up_proj(x))

class CausalSelfAttention(nn.Module):
    def __init__(self, cfg: LLMConfig):
        super().__init__()
        assert cfg.d_model % cfg.n_heads == 0
        self.n_heads = cfg.n_heads
        self.d_head  = cfg.d_model // cfg.n_heads
        self.q_proj  = nn.Linear(cfg.d_model, cfg.d_model, bias=False)
        self.k_proj  = nn.Linear(cfg.d_model, cfg.d_model, bias=False)
        self.v_proj  = nn.Linear(cfg.d_model, cfg.d_model, bias=False)
        self.o_proj  = nn.Linear(cfg.d_model, cfg.d_model, bias=False)

    def forward(self, x):
        B, L, D = x.shape
        def split_heads(t):
            return t.view(B, L, self.n_heads, self.d_head).transpose(1, 2)

        q = split_heads(self.q_proj(x))   # (B, H, L, d_h)
        k = split_heads(self.k_proj(x))
        v = split_heads(self.v_proj(x))

        # Flash Attention 路径（PyTorch 2.0+，自动选择最优 kernel）
        out = F.scaled_dot_product_attention(q, k, v, is_causal=True)
        out = out.transpose(1, 2).contiguous().view(B, L, D)
        return self.o_proj(out)

class DecoderLayer(nn.Module):
    def __init__(self, cfg: LLMConfig):
        super().__init__()
        self.attn_norm = RMSNorm(cfg.d_model)
        self.ffn_norm  = RMSNorm(cfg.d_model)
        self.attn = CausalSelfAttention(cfg)
        self.ffn  = SwiGLUFFN(cfg.d_model)

    def forward(self, x):
        x = x + self.attn(self.attn_norm(x))   # Pre-Norm 残差
        x = x + self.ffn(self.ffn_norm(x))
        return x

class LLM(nn.Module):
    def __init__(self, cfg: LLMConfig):
        super().__init__()
        self.cfg = cfg
        self.embedding   = nn.Embedding(cfg.vocab_size, cfg.d_model)
        self.layers      = nn.ModuleList([DecoderLayer(cfg) for _ in range(cfg.n_layers)])
        self.final_norm  = RMSNorm(cfg.d_model)
        self.lm_head     = nn.Linear(cfg.d_model, cfg.vocab_size, bias=False)
        # Weight Tying
        self.lm_head.weight = self.embedding.weight
        # 初始化
        self.apply(self._init_weights)

    def _init_weights(self, m):
        if isinstance(m, nn.Linear):
            nn.init.normal_(m.weight, std=0.02)
        elif isinstance(m, nn.Embedding):
            nn.init.normal_(m.weight, std=0.02)

    def forward(self, input_ids, labels=None):
        x = self.embedding(input_ids)
        for layer in self.layers:
            x = layer(x)
        x = self.final_norm(x)
        logits = self.lm_head(x)  # (B, L, V)

        loss = None
        if labels is not None:
            # 语言模型损失：每个位置预测下一个 token
            loss = F.cross_entropy(
                logits[:, :-1].contiguous().view(-1, self.cfg.vocab_size),
                labels[:, 1:].contiguous().view(-1),
                ignore_index=-100,
            )
        return logits, loss

    @property
    def n_params(self):
        return sum(p.numel() for p in self.parameters())

# 验证
cfg   = LLMConfig(vocab_size=32000, d_model=512, n_layers=8, n_heads=8)
model = LLM(cfg)
print(f"模型参数量: {model.n_params/1e6:.1f}M")

x      = torch.randint(0, 32000, (2, 64))
labels = x.clone()
logits, loss = model(x, labels)
print(f"输出形状: {logits.shape}")  # (2, 64, 32000)
print(f"Loss:     {loss.item():.4f}")
```

## 六、面试常见问题

**Q1：Encoder-only、Decoder-only、Encoder-Decoder 各适合什么任务？为什么现代大语言模型几乎都是 Decoder-only？**

Encoder-only（如 BERT）适合理解类任务（分类、NER），因为双向注意力能充分利用上下文。Decoder-only 适合生成类任务（对话、写作），自回归生成与预训练目标天然一致。Encoder-Decoder（如 T5）适合翻译、摘要等输入和输出差异大的任务。

现代 LLM 几乎全是 Decoder-only 有三个原因：预训练目标（下一词预测）和推理行为完全一致；扩展性更好——研究表明同等参数下 Decoder-only 的通用能力更强；工程更简单——不需要管理两套参数和 Cross-Attention。

**Q2：原始 Transformer 和现代 LLM（如 LLaMA-3）的架构有哪些关键区别？**

五个主要差异：归一化位置从 Post-LayerNorm 变为 Pre-RMSNorm（更稳定、更快）；位置编码从绝对正弦编码变为 RoPE（更好的长度外推性）；注意力从标准 MHA 变为 GQA（节省 KV Cache 显存）；FFN 激活从 ReLU 变为 SwiGLU（更强表达力）；几乎所有层都去掉了 bias 参数。

**Q3：一个 7B 参数的 LLM，参数量是怎么分配的？估算方法是什么？**

每层约 $12d^2$ 参数（Attention 贡献 $4d^2$，FFN 贡献 $8d^2$），乘以层数得到 Transformer 部分参数。Embedding 是 $V \times d$，Weight Tying 时 LM Head 共享。7B 模型（$d=4096, L=32, V=32000$）：Transformer 层约 $12 \times 32 \times 4096^2 \approx 6.4$B，Embedding 约 $32000 \times 4096 \approx 0.13$B。FFN 约占总参数的 65%，Attention 约 30%，Embedding 约 2-5%。

**Q4：训练 FLOPs 的经验公式是什么？有什么实际意义？**

$\text{FLOPs}_{\text{训练}} \approx 6ND$，其中 $N$ 是非 Embedding 参数量，$D$ 是训练 token 数。前向传播约 $2N$（每个参数约 2 次浮点运算），反向传播约是前向的 2 倍，合计约 $6N$。

实际意义：根据这个公式可以估算训练时间（结合 GPU 算力和 MFU），验证 Chinchilla 的计算最优训练数据量，以及在多 GPU 配置下估算分布式训练效率。

**Q5：为什么 LLM 几乎都去掉了 Bias？**

三个原因：实验表明去掉 bias 对最终性能影响很小（有时甚至有轻微提升）；节省约 0.1% 的参数（在大模型尺度下绝对值不小，7B 模型约节省 100M 参数）；与 Weight Tying 和 RMSNorm 等简化设计趋势一致，整体模型更干净。从 LayerNorm 已经有 bias 参数这个角度看，额外的线性层 bias 确实显得冗余。

**Q6：什么是 Weight Tying？为什么大多数 LLM 使用它？**

Weight Tying（权重绑定）是让输入 Embedding 矩阵和输出 LM Head 矩阵共享同一组参数（形状互为转置关系：Embedding 是 $V \times d$，LM Head 是 $d \times V$）。好处：减少约 $V \times d$ 的参数（7B 模型约节省 130M 参数）；语义上合理——把 token 映射到隐空间和把隐空间映射回 token 概率，用同一套"词语理解"是一致的；实验表明它在不增加参数的情况下提升性能。
