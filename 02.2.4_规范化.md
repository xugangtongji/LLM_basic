## 零、开篇：为什么深层网络必须有规范化层

随着训练进行，每一层的输入分布在不断变化——因为前一层的参数在更新。这个现象叫 **Internal Covariate Shift（内部协变量偏移）**：下游层需要持续适应上游层分布的变化，相当于每一步训练都在"移动靶子上打靶"，收敛极慢。

规范化层的作用是：**强制每层的输入保持在稳定的数值范围内**，让每层的学习任务不受上游层分布波动的干扰。

这带来了三个直接好处：

1. **训练稳定**：不会因为某层激活值过大而让梯度爆炸
2. **收敛更快**：每层专注学自己的变换，不用适应输入分布的漂移
3. **支持更大学习率**：分布稳定后，步子可以迈大一点

大模型中规范化层的演进：**BatchNorm → LayerNorm → RMSNorm**，每一步都是"去掉多余操作，保留核心效果"。

## 一、BatchNorm：计算机视觉的功臣，NLP 的陌生人

### 1.1 初印象：用费曼视角理解

**BatchNorm 就是"在一个班级里，把所有同学的成绩统一归一化"**。

把这次考试所有同学的成绩减去班级平均分，再除以标准差，得到每个同学的"相对分"。这样，不管这次题难还是题易（分布漂移），"相对分"的分布永远在固定范围内。

"班级"就是一个 batch，"成绩"就是激活值，"归一化"就是 BatchNorm。

### 1.2 数学形式

对 mini-batch $\mathcal{B} = \{x_1, \ldots, x_m\}$（同一特征通道 $c$ 上的 $m$ 个样本）：

$$\mu_\mathcal{B} = \frac{1}{m}\sum_{i=1}^m x_i, \quad \sigma^2_\mathcal{B} = \frac{1}{m}\sum_{i=1}^m (x_i - \mu_\mathcal{B})^2$$

$$\hat x_i = \frac{x_i - \mu_\mathcal{B}}{\sqrt{\sigma^2_\mathcal{B} + \epsilon}}, \quad y_i = \gamma \hat x_i + \beta$$

其中 $\gamma, \beta$ 是可学习的缩放和平移参数（每个通道一对）。

**推理时**：用训练期间的指数移动平均统计量（running_mean / running_var）代替 batch 统计量，确保推理结果确定性。

```python
import torch
import torch.nn as nn

# BatchNorm1d：用于 (N, C) 或 (N, C, L) 的数据
bn = nn.BatchNorm1d(num_features=256)

# 训练时：统计 batch 内的均值和方差
x_train = torch.randn(32, 256)     # (batch=32, features=256)
bn.train()
out_train = bn(x_train)
print(f"训练时输出均值: {out_train.mean().item():.4f}")   # ≈ 0
print(f"训练时输出方差: {out_train.var().item():.4f}")    # ≈ 1

# 推理时：用 running stats，不依赖当前 batch
bn.eval()
x_single = torch.randn(1, 256)     # batch_size=1 也可以
out_eval = bn(x_single)
print(f"推理时输出（单样本）: {out_eval.shape}")   # (1, 256)
```

### 1.3 BatchNorm 为什么不适合 NLP / LLM

BatchNorm 在 NLP 中有三个根本性问题：

**问题一：序列长度可变**

同一个 batch 里，不同句子长度不同。"第 5 个位置"在一些句子里是实词，在另一些里是 padding。把它们混在一起算统计量，没有意义。

**问题二：batch 内统计量不稳定**

NLP 任务的 batch size 通常比 CV 小很多（显存限制）。batch_size=2 时，batch 统计量方差极大，归一化噪声过大。大模型分布式训练时，各卡之间同步 batch 统计量代价极高。

**问题三：自回归生成时 batch_size=1**

推理时用 running_mean/var，但这些统计量是在大 batch 上算的，和单个样本的实际分布差异很大，导致训练和推理行为不一致。

```python
# 演示 BatchNorm 对变长序列的问题
batch = [
    torch.randn(10, 256),   # 长度 10 的句子
    torch.randn(3, 256),    # 长度 3 的句子（padding 到 10）
]
# 统一 padding 到最大长度
padded = torch.zeros(2, 10, 256)
padded[0] = batch[0]
padded[1, :3] = batch[1]
# batch[1] 后 7 个位置是 padding（全零），但 BatchNorm 不区分
# 这些零会污染统计量，导致真实 token 的归一化结果失真
```

## 二、LayerNorm：Transformer 的默认选择

### 2.1 初印象：用费曼视角理解

**LayerNorm 是"每个同学独立归一化自己的各科成绩"**。

不是班级内横向比较，而是每个同学纵向看自己的所有科目：把自己的各科成绩减去自己的科目平均分，再除以自己的标准差，得到"自己内部的相对擅长程度"。

这样，每个样本（每个 token）独立归一化，互不干扰，序列长度、batch 大小完全不影响计算。

### 2.2 数学形式

对单个样本（一个 token 或一个时间步）的特征向量 $\mathbf{x} \in \mathbb{R}^d$：

$$\mu = \frac{1}{d}\sum_{j=1}^d x_j, \quad \sigma^2 = \frac{1}{d}\sum_{j=1}^d (x_j - \mu)^2$$

$$\hat{\mathbf{x}} = \frac{\mathbf{x} - \mu}{\sqrt{\sigma^2 + \epsilon}}, \quad \mathbf{y} = \gamma \odot \hat{\mathbf{x}} + \beta$$

其中 $\gamma, \beta \in \mathbb{R}^d$ 是可学习的缩放和平移向量（每维一个参数），$\epsilon$ 是数值稳定的小常数（如 $10^{-5}$）。

**与 BatchNorm 的关键区别**：归一化维度不同。

| | BatchNorm | LayerNorm |
|--|-----------|-----------|
| 归一化维度 | 跨 batch（同一特征） | 跨特征（同一样本） |
| 统计量 | 依赖 batch | 仅依赖当前样本 |
| 推理 vs 训练 | 有差异（running stats） | 完全一致 |
| 适合场景 | CV，大 batch | NLP，任意 batch size |

### 2.3 手写实现与 PyTorch 验证

```python
import torch
import torch.nn as nn

class LayerNormScratch(nn.Module):
    """从零实现 LayerNorm，便于理解数学原理"""

    def __init__(self, d_model: int, eps: float = 1e-5):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d_model))   # 缩放（初始为 1）
        self.beta  = nn.Parameter(torch.zeros(d_model))  # 平移（初始为 0）
        self.eps   = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # x: (..., d_model)，最后一维是特征维
        mean = x.mean(dim=-1, keepdim=True)               # (..., 1)
        var  = x.var(dim=-1, keepdim=True, unbiased=False) # (..., 1)
        x_hat = (x - mean) / (var + self.eps).sqrt()      # 归一化
        return self.gamma * x_hat + self.beta              # 仿射变换

# 验证与 PyTorch 内置一致
d = 512
x = torch.randn(4, 16, d)   # (batch, seq_len, d_model)

ln_scratch = LayerNormScratch(d)
ln_pytorch = nn.LayerNorm(d)

# 对齐参数
with torch.no_grad():
    ln_pytorch.weight.copy_(ln_scratch.gamma)
    ln_pytorch.bias.copy_(ln_scratch.beta)

out_scratch = ln_scratch(x)
out_pytorch = ln_pytorch(x)
print(f"最大差异: {(out_scratch - out_pytorch).abs().max().item():.8f}")
# 应该非常接近 0

# 验证归一化效果
print(f"归一化后均值: {out_scratch.mean(dim=-1).abs().max().item():.6f}")  # ≈ 0
print(f"归一化后标准差: {out_scratch.std(dim=-1).mean().item():.4f}")      # ≈ 1
```

## 三、RMSNorm：LayerNorm 的精简版，大模型的现任标配

### 3.1 初印象：用费曼视角理解

**RMSNorm 把 LayerNorm 的"中心化"步骤去掉，只保留"缩放"步骤**。

LayerNorm 做了两件事：减去均值（中心化）+ 除以标准差（缩放）。

RMSNorm 的研究者认为：LLM 在推理时，各层的隐状态天然就几乎不含均值偏移（因为模型学会了保持均值接近 0），所以减均值是多余的操作。只除以 RMS（均方根）就够了——效果相当，速度更快。

实验表明 RMSNorm 相比 LayerNorm 在不同模型上速度提升 7%~64%，而效果相当甚至更好，这就是 LLaMA、Qwen、Mistral 等主流 LLM 全面迁移到 RMSNorm 的原因。

### 3.2 数学形式

$$\text{RMS}(\mathbf{x}) = \sqrt{\frac{1}{d}\sum_{j=1}^d x_j^2}$$

$$\text{RMSNorm}(\mathbf{x}) = \frac{\mathbf{x}}{\text{RMS}(\mathbf{x}) + \epsilon} \odot \gamma$$

**与 LayerNorm 的对比**：

| | LayerNorm | RMSNorm |
|--|-----------|---------|
| 减均值 | 是 | 否 |
| 除以 | 标准差 $\sigma$ | 均方根 RMS |
| 可学习参数 | $\gamma$（缩放）+ $\beta$（平移） | 只有 $\gamma$（缩放） |
| 计算量 | 两次累加（均值 + 方差） | 一次累加（平方和） |
| 使用模型 | BERT、GPT-2、早期 Transformer | LLaMA 1/2/3、Qwen、Mistral、Gemma |

### 3.3 手写实现与性能对比

```python
import torch
import torch.nn as nn
import time

class RMSNorm(nn.Module):
    """从零实现 RMSNorm（LLaMA 同款）"""

    def __init__(self, d_model: int, eps: float = 1e-6):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d_model))
        self.eps   = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # 计算均方根
        rms = x.pow(2).mean(dim=-1, keepdim=True).add(self.eps).sqrt()
        return self.gamma * (x / rms)

# PyTorch 2.4+ 内置 RMSNorm
# nn.RMSNorm(d_model, eps=1e-6)

# 验证数学正确性
d = 4096
x = torch.randn(4, 128, d)

rms_scratch = RMSNorm(d)
out = rms_scratch(x)

# 验证：RMS 归一化后，每个向量的均方根应接近 1
rms_after = out.pow(2).mean(dim=-1).sqrt()
print(f"归一化后 RMS 均值: {rms_after.mean().item():.4f}")  # ≈ 1.0

# 性能对比（LayerNorm vs RMSNorm）
rms_norm = RMSNorm(d).cuda()
layer_norm = nn.LayerNorm(d).cuda()
x_gpu = torch.randn(32, 512, d).cuda()

def benchmark(fn, x, n=500):
    torch.cuda.synchronize()
    t = time.perf_counter()
    for _ in range(n):
        _ = fn(x)
    torch.cuda.synchronize()
    return (time.perf_counter() - t) / n * 1000

if torch.cuda.is_available():
    t_rms = benchmark(rms_norm, x_gpu)
    t_ln  = benchmark(layer_norm, x_gpu)
    print(f"RMSNorm:   {t_rms:.3f}ms")
    print(f"LayerNorm: {t_ln:.3f}ms")
    print(f"加速比:    {t_ln/t_rms:.2f}x")
else:
    print("（需要 CUDA 环境测速）")
```

### 3.4 为什么去掉均值计算是安全的

从理论上看，LayerNorm 的均值减法（re-centering）的作用是使归一化后的向量正交于全 1 向量 $(1,1,\ldots,1)^T$。

研究发现，LLM 在推理时其隐状态向量在均值方向上的分量极小——模型在训练过程中自然学到了"不使用均值方向"的特性。因此，LayerNorm 中的均值减法操作在实践中是冗余的。RMSNorm 去掉这一步，在节省计算的同时，性能几乎不受影响。

## 四、GroupNorm 与其他变体

### 4.1 GroupNorm（计算机视觉专用）

**初印象**：GroupNorm 是 BatchNorm 和 LayerNorm 的折中——把通道分成若干组，组内归一化。

$$\mu_{g,n} = \frac{1}{C/G \cdot H \cdot W}\sum_{c \in G_g}\sum_{h,w} x_{n,c,h,w}$$

- $G$ 是组数，$C/G$ 是每组通道数
- $G=1$：等同于 LayerNorm（整个样本内归一化）
- $G=C$：等同于 InstanceNorm（每通道独立归一化）

**在 NLP/LLM 中很少使用**。主要用于小 batch CV 任务（目标检测、分割）。

```python
import torch.nn as nn

# GroupNorm：用于 (N, C, H, W) 的图像
gn = nn.GroupNorm(num_groups=32, num_channels=256)
x_img = torch.randn(2, 256, 28, 28)   # (N, C, H, W)
out_gn = gn(x_img)
print(f"GroupNorm 输出形状: {out_gn.shape}")

# 在 Transformer 里几乎不用 GroupNorm，了解即可
```

### 4.2 各规范化方式的适用场景速查

| 规范化 | 归一化维度 | 适合场景 | 代表模型 |
|--------|-----------|---------|---------|
| BatchNorm | batch 维 | CV 大 batch | ResNet、CNN |
| LayerNorm | 特征维 | NLP、Transformer | BERT、GPT-2 |
| RMSNorm | 特征维（无均值） | LLM（更快） | LLaMA、Qwen |
| GroupNorm | 通道组内 | CV 小 batch | Mask R-CNN |
| InstanceNorm | 单样本单通道 | 风格迁移 | NST |

## 五、Pre-Norm vs Post-Norm：位置决定稳定性

### 5.1 初印象：用费曼视角理解

规范化层放在哪里，对训练稳定性影响巨大。这两种放置方式的名字来自它们相对于子层（Attention/FFN）的位置：

- **Post-Norm**：子层计算 → 和残差相加 → 再归一化
- **Pre-Norm**：先归一化 → 子层计算 → 和（未归一化的）残差相加

**比喻**：Post-Norm 是"先工作，工作完检查整理"；Pre-Norm 是"先整理好材料，再开始工作"。Pre-Norm 的好处是梯度在残差路径上永远不经过归一化层，梯度更稳定。

### 5.2 数学对比

**Post-Norm（原始 Transformer）**：

$$\mathbf{x}' = \text{Norm}(\mathbf{x} + \text{Attention}(\mathbf{x}))$$
$$\mathbf{x}'' = \text{Norm}(\mathbf{x}' + \text{FFN}(\mathbf{x}'))$$

**Pre-Norm（现代 LLM 标配）**：

$$\mathbf{x}' = \mathbf{x} + \text{Attention}(\text{Norm}(\mathbf{x}))$$
$$\mathbf{x}'' = \mathbf{x}' + \text{FFN}(\text{Norm}(\mathbf{x}'))$$

### 5.3 梯度分析：为什么 Pre-Norm 更稳定

**Post-Norm 的梯度路径**：

$$\frac{\partial \mathcal{L}}{\partial \mathbf{x}} = \frac{\partial \mathcal{L}}{\partial \mathbf{x}''} \cdot \frac{\partial \text{Norm}}{\partial \mathbf{x}'} \cdot \left(I + \frac{\partial \text{Attention}}{\partial \mathbf{x}}\right) \cdots$$

梯度必须经过 Norm 层的 Jacobian。在某些情况下，Norm 层可以在梯度路径上引入不稳定性，导致深层 Post-Norm 网络需要精心设计的 warmup。

**Pre-Norm 的梯度路径**：

$$\frac{\partial \mathcal{L}}{\partial \mathbf{x}} = \frac{\partial \mathcal{L}}{\partial \mathbf{x}''} \cdot \left(I + \frac{\partial \text{FFN}(\text{Norm}(\mathbf{x}'))}{\partial \mathbf{x}'}\right)$$

残差路径（$I$ 项）上永远有一个恒等矩阵，梯度不会消失，且不经过 Norm 层。这使得 Pre-Norm 模型即使不用特别精心的 warmup 也能稳定训练。

```python
import torch
import torch.nn as nn

class PostNormBlock(nn.Module):
    """原始 Transformer 风格：Post-Norm"""
    def __init__(self, d_model, n_heads, d_ff):
        super().__init__()
        self.attn  = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ff    = nn.Sequential(nn.Linear(d_model, d_ff), nn.GELU(),
                                    nn.Linear(d_ff, d_model))
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)

    def forward(self, x):
        x = self.norm1(x + self.attn(x, x, x)[0])   # 先残差加，后 norm
        x = self.norm2(x + self.ff(x))
        return x

class PreNormBlock(nn.Module):
    """现代 LLM 风格：Pre-Norm（RMSNorm 版本）"""
    def __init__(self, d_model, n_heads, d_ff):
        super().__init__()
        self.attn  = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ff    = nn.Sequential(nn.Linear(d_model, d_ff), nn.GELU(),
                                    nn.Linear(d_ff, d_model))
        self.norm1 = RMSNorm(d_model)  # 使用之前实现的 RMSNorm
        self.norm2 = RMSNorm(d_model)

    def forward(self, x):
        # 先 norm，后 attn，再加到原始 x 上
        x = x + self.attn(self.norm1(x), self.norm1(x), self.norm1(x))[0]
        x = x + self.ff(self.norm2(x))
        return x
```

### 5.4 大模型的选择：Pre-Norm + RMSNorm

| 模型 | Norm 类型 | Norm 位置 |
|------|----------|---------|
| Transformer（原版）| LayerNorm | Post-Norm |
| BERT | LayerNorm | Post-Norm |
| GPT-2 | LayerNorm | Pre-Norm |
| GPT-3 | LayerNorm | Pre-Norm |
| LLaMA 1/2/3 | **RMSNorm** | **Pre-Norm** |
| Qwen 1/2/2.5 | **RMSNorm** | **Pre-Norm** |
| Mistral | **RMSNorm** | **Pre-Norm** |
| DeepSeek V2/V3 | **RMSNorm** | **Pre-Norm** |
| Gemma | **RMSNorm** | **Pre-Norm** |

几乎所有 2022 年后的大模型都采用了 **Pre-Norm + RMSNorm** 的组合，这成为了事实上的标准。

## 六、实战：在自定义 Transformer 中使用 RMSNorm

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class RMSNorm(nn.Module):
    def __init__(self, dim: int, eps: float = 1e-6):
        super().__init__()
        self.eps   = eps
        self.scale = nn.Parameter(torch.ones(dim))

    def forward(self, x):
        norm_x = x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)
        return self.scale * norm_x


class LlamaStyleBlock(nn.Module):
    """
    LLaMA 风格的 Transformer 块：
    Pre-RMSNorm + Multi-Head Attention（带 KV Cache）+ Pre-RMSNorm + SwiGLU FFN
    """
    def __init__(self, d_model: int, n_heads: int, d_ff: int):
        super().__init__()
        assert d_model % n_heads == 0
        self.d_head = d_model // n_heads
        self.n_heads = n_heads

        # Pre-Norm（RMSNorm）
        self.attn_norm = RMSNorm(d_model)
        self.ffn_norm  = RMSNorm(d_model)

        # Attention 投影
        self.q_proj = nn.Linear(d_model, d_model,    bias=False)
        self.k_proj = nn.Linear(d_model, d_model,    bias=False)
        self.v_proj = nn.Linear(d_model, d_model,    bias=False)
        self.o_proj = nn.Linear(d_model, d_model,    bias=False)

        # SwiGLU FFN
        hidden = int(d_model * 8 / 3)
        hidden = ((hidden + 63) // 64) * 64
        self.gate_proj = nn.Linear(d_model, hidden, bias=False)
        self.up_proj   = nn.Linear(d_model, hidden, bias=False)
        self.down_proj = nn.Linear(hidden,  d_model, bias=False)

    def attention(self, x: torch.Tensor) -> torch.Tensor:
        B, L, D = x.shape
        q = self.q_proj(x).view(B, L, self.n_heads, self.d_head).transpose(1, 2)
        k = self.k_proj(x).view(B, L, self.n_heads, self.d_head).transpose(1, 2)
        v = self.v_proj(x).view(B, L, self.n_heads, self.d_head).transpose(1, 2)

        # Scaled Dot-Product Attention（PyTorch 2.0+ Flash Attention 后端）
        attn_out = F.scaled_dot_product_attention(q, k, v, is_causal=True)
        attn_out = attn_out.transpose(1, 2).contiguous().view(B, L, D)
        return self.o_proj(attn_out)

    def ffn(self, x: torch.Tensor) -> torch.Tensor:
        return self.down_proj(F.silu(self.gate_proj(x)) * self.up_proj(x))

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # Pre-Norm Attention
        x = x + self.attention(self.attn_norm(x))
        # Pre-Norm FFN
        x = x + self.ffn(self.ffn_norm(x))
        return x


# 验证
d, h, ff = 512, 8, 1365
block = LlamaStyleBlock(d, h, ff)
x = torch.randn(2, 16, d)
out = block(x)
print(f"输入形状: {x.shape}  输出形状: {out.shape}")  # (2, 16, 512)

params = sum(p.numel() for p in block.parameters())
print(f"参数量: {params:,}")
```

## 七、面试常见问题

**Q1：BatchNorm 为什么不适合用在 NLP / Transformer 里？**

三个核心原因：序列长度可变，不同位置的 token 语义完全不同，不能混在一起归一化；NLP 的 batch size 通常很小，batch 统计量不稳定噪声大；推理时往往 batch_size=1（自回归生成），用 running stats 和训练行为不一致。这些问题在 CV 里都不存在（固定大小图片、大 batch、批量推理），所以 BatchNorm 在 CV 里大放异彩，在 NLP 里几乎被彻底放弃。

**Q2：LayerNorm 和 RMSNorm 的区别是什么？RMSNorm 为什么更快？**

LayerNorm 对每个样本的特征向量做两步操作：减均值（中心化）+ 除以标准差（缩放），有 $\gamma$ 和 $\beta$ 两组参数。RMSNorm 去掉了减均值这一步，直接除以均方根，只有 $\gamma$ 一组参数。更快的原因：减少了一次均值计算（一次遍历求和），同时减少了一次减法操作和 $\beta$ 参数的更新，在 GPU 上节省了显存带宽。研究表明 RMSNorm 比 LayerNorm 快 7%~64%，且效果相当。

**Q3：Pre-Norm 相比 Post-Norm 有什么优势？**

Pre-Norm 把归一化层放在子层之前，残差路径完全不经过 Norm 层（$x = x + F(\text{Norm}(x))$），梯度路径中总有恒等矩阵保底，梯度不会消失，训练更稳定，不需要精心调整 warmup。Post-Norm 的归一化在残差加法之后，某些情况下 Norm 层的 Jacobian 会影响梯度传播，深层模型需要特别小心的学习率和 warmup 设计。几乎所有现代 LLM 都用 Pre-Norm。

**Q4：RMSNorm 为什么可以去掉均值减法？**

RMSNorm 的作者认为 LayerNorm 起作用的核心是"缩放不变性"（re-scaling invariance），而非"平移不变性"（re-centering invariance）。实验证明，在 LLM 的实际训练中，各层隐状态的均值天然接近 0（模型在训练中自然学到了这个性质）。最新研究也从几何角度证明，LLM 的隐状态向量在推理时自然正交于均匀向量（即均值分量极小），这使得均值减法确实是冗余的步骤。

**Q5：LayerNorm 的 $\gamma$ 和 $\beta$ 参数分别起什么作用？初始化为多少？**

$\gamma$（scale / weight）：归一化后的缩放因子，允许模型恢复被归一化"压缩"的幅度信息，初始化为全 1（不改变幅度）。$\beta$（bias）：归一化后的平移，允许模型将特征中心移动到非零位置，初始化为全 0（不平移）。这样初始状态等同于"什么都不做"，模型从恒等变换出发慢慢学习合适的缩放和平移。RMSNorm 只有 $\gamma$，没有 $\beta$。

**Q6：训练 LLM 时，规范化层通常放在模型的什么位置？**

现代 LLM（LLaMA、Qwen、Mistral 等）的标准做法：每个 Transformer 块内有两个 Pre-RMSNorm（Attention 前一个，FFN 前一个），整个模型最后还有一个 Final RMSNorm（在 LM Head 之前）。代码结构为：`RMSNorm → Attention → 残差 → RMSNorm → FFN → 残差`。这样共有 $2N + 1$ 个 RMSNorm（N 是层数，最后一个在 LM Head 前）。

**Q7：GroupNorm 和 LayerNorm 有什么关系？**

GroupNorm 把通道分成 G 组，在每组内做归一化。当 $G = 1$ 时，整个 C 个通道作为一组归一化，等同于 LayerNorm（对每个样本的所有特征归一化）。当 $G = C$ 时，每个通道独立归一化，等同于 InstanceNorm。LayerNorm 是 GroupNorm 的特例（G=1），主要区别在于 GroupNorm 针对 CV 的 (N,C,H,W) 格式设计，NLP 不常用。

**Q8：规范化层的参数（$\gamma$）会被 weight decay 正则化吗？**

不应该被正则化。LayerNorm 和 RMSNorm 的 $\gamma$ 是缩放参数，不是普通的权重，对它做 L2 正则会阻止模型学到合适的缩放幅度，通常会损害性能。大模型训练的最佳实践是把 Norm 层的参数（以及偏置、Embedding）排除在 weight decay 之外：

```python
# AdamW 分组：norm 参数不做 weight decay
decay_params = [p for n, p in model.named_parameters()
                if p.requires_grad and p.dim() >= 2]  # 权重矩阵
no_decay_params = [p for n, p in model.named_parameters()
                   if p.requires_grad and p.dim() < 2]  # norm, bias, embedding

optimizer = torch.optim.AdamW([
    {"params": decay_params,    "weight_decay": 0.1},
    {"params": no_decay_params, "weight_decay": 0.0},
], lr=3e-4)
```
