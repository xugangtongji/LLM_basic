## 零、开篇：注意力机制到底在做什么

上一章我们搭好了 Transformer 的骨架，但注意力层用了占位符。这一章来填上这个最关键的空白。

先提出一个问题，帮你建立直觉：

> "我喜欢苹果，因为它很甜"

"它"指什么？显然是"苹果"，不是"我"。

当你读到"甜"这个词时，你潜意识里知道要把"甜"和"苹果"联系起来。这种"在读当前词时，回顾并关注相关词"的能力，就是注意力机制要模拟的事情。

## 一、从"在图书馆查资料"理解 Q、K、V

### 1.1 初印象：用费曼视角理解

**Self-Attention 的核心就是三个角色：Query（查询）、Key（索引）、Value（内容）**。

用"在图书馆查资料"来理解：

- 你走进图书馆，手里拿着一张**查询单（Query）**，上面写着"我想找关于苹果的资料"
- 图书馆的书架上，每本书都有一个**索引标签（Key）**，比如"苹果"、"香蕉"、"计算机"……
- 每本书里装的**真正内容（Value）**，是你最终要用的信息

查资料的过程：
1. 拿着你的查询单（Q），挨个对比每本书的标签（K）
2. 哪本书的标签和你的查询单最匹配，你就对它分配更高的注意力
3. 按照注意力高低，加权地把各本书的内容（V）综合起来，得到你最终读到的答案

**这就是 Self-Attention 的完整过程。**

"Self"的意思是：查询单（Q）、书的标签（K）、书的内容（V）**全部来自同一个句子**——每个词既是"读者"（产生 Q），也是"书"（提供 K 和 V）。

### 1.2 用一个句子来演示

句子："猫 喜欢 鱼"

三个词各自生成 Q、K、V：

```
         Q（我在查什么？）  K（我的标签是什么？）  V（我的内容是什么？）
"猫"：       Q_猫              K_猫                V_猫
"喜欢"：     Q_喜欢             K_喜欢              V_喜欢
"鱼"：       Q_鱼              K_鱼                V_鱼
```

当模型处理"喜欢"这个词时：
- 用"喜欢"的 Q，分别和"猫"的 K、"喜欢"的 K、"鱼"的 K 计算匹配度
- 假设匹配度结果是：猫=0.6，喜欢=0.3，鱼=0.1
- 最后用这个权重，把三个词的 V 加权求和：0.6×V_猫 + 0.3×V_喜欢 + 0.1×V_鱼
- 这就是"喜欢"这个词更新后的向量——它包含了"主要关注猫，少量关注自身，轻微关注鱼"的信息

## 二、数学推导：一步一步来

### 2.1 第一步：生成 Q、K、V

Q、K、V 不是直接等于原始词向量，而是通过三个不同的线性变换矩阵生成的：

$$Q = X W_Q, \quad K = X W_K, \quad V = X W_V$$

其中：
- $X$ 是输入矩阵，形状是 $(\text{序列长度} \times d_{\text{model}})$，每一行是一个 token 的向量
- $W_Q, W_K, W_V$ 是三个可学习的权重矩阵，形状是 $(d_{\text{model}} \times d_k)$
- $d_k$ 是 Q 和 K 的维度（通常等于 $d_{\text{model}} / \text{头数}$）

**为什么不直接用 X 作为 Q/K/V？**

因为"查询""索引""内容"是三种不同的角色，需要不同的视角来看待同一段文字。三个可学习的矩阵让模型自己决定如何从原始表示中提取这三种信息。

### 2.2 第二步：计算注意力分数

用 Q 和 K 的内积来衡量"匹配程度"：

$$\text{分数} = Q K^T$$

$Q$ 的形状是 $(L \times d_k)$，$K^T$ 的形状是 $(d_k \times L)$，相乘得到 $(L \times L)$ 的分数矩阵。

分数矩阵的第 $i$ 行第 $j$ 列，就是"第 $i$ 个词在查询时，对第 $j$ 个词的关注程度"。

**为什么用内积衡量相似度？**

两个向量的内积越大，说明它们的"方向"越接近，语义越相似。就像点积越大，两个向量越"对齐"。（1.1.1 线性代数章节讲过。）

### 2.3 第三步：缩放——除以 $\sqrt{d_k}$

$$\text{分数}_{\text{缩放}} = \frac{QK^T}{\sqrt{d_k}}$$

**为什么要除以 $\sqrt{d_k}$？**

这里用一个直觉来解释：

假设 Q 和 K 的每个元素都是标准正态分布（均值 0，方差 1）。两个 $d_k$ 维向量的内积，是 $d_k$ 个乘积项之和，其方差约为 $d_k$，标准差约为 $\sqrt{d_k}$。

当 $d_k$ 很大时（比如 128），内积的数值会很大，送进 softmax 后会接近"赢者通吃"——某个分数远大于其他所有分数，softmax 把 99% 的权重分给它，梯度几乎消失。

除以 $\sqrt{d_k}$ 把方差拉回到 1 左右，让 softmax 的输出分布更合理，梯度也能正常传播。

### 2.4 第四步：Softmax——得到概率分布

$$\text{注意力权重} = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)$$

Softmax 把每一行的分数转换成概率，所有值在 0 到 1 之间，且每行之和等于 1。

这样，每一行就是"这个词对所有其他词的注意力分配方案"——比如"喜欢"这个词，0.6 的注意力给"猫"，0.3 给自己，0.1 给"鱼"。

### 2.5 第五步：加权求和——得到最终输出

$$\text{输出} = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

注意力权重矩阵（$L \times L$）乘以 V（$L \times d_v$），得到 $L \times d_v$ 的输出矩阵。

**每一行的含义**：第 $i$ 个词的输出，是所有词的 V 按注意力权重的加权求和。也就是说，"喜欢"的输出向量 = 0.6 × V_猫 + 0.3 × V_喜欢 + 0.1 × V_鱼。

### 2.6 完整公式

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

**这就是论文中那个著名的 Attention 公式。** 现在你应该能理解每一项的含义了：
- $QK^T$：计算匹配分数
- $/\sqrt{d_k}$：缩放，防止数值过大
- $\text{softmax}$：转换为注意力权重（概率）
- $\times V$：加权求和，得到最终输出

## 三、代码实现：从零手写 Self-Attention

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class SimpleSelfAttention(nn.Module):
    """
    最简单的 Self-Attention 实现，对应上面的数学推导。
    适合理解原理，真实模型会有更多优化。
    """

    def __init__(self, d_model: int, d_k: int):
        """
        d_model: 输入向量的维度（每个 token 用多少维表示）
        d_k:     Q 和 K 的维度（通常 = d_model / 头数）
        """
        super().__init__()
        # 三个线性变换矩阵，把输入 X 分别投影成 Q、K、V
        self.W_Q = nn.Linear(d_model, d_k, bias=False)
        self.W_K = nn.Linear(d_model, d_k, bias=False)
        self.W_V = nn.Linear(d_model, d_k, bias=False)
        self.d_k = d_k

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (batch_size, seq_len, d_model)
        返回: (batch_size, seq_len, d_k)
        """
        # 第一步：生成 Q、K、V
        Q = self.W_Q(x)   # (batch, seq_len, d_k)
        K = self.W_K(x)   # (batch, seq_len, d_k)
        V = self.W_V(x)   # (batch, seq_len, d_k)

        # 第二步：计算注意力分数
        # K.transpose(-2, -1) 把最后两维转置：(batch, d_k, seq_len)
        scores = Q @ K.transpose(-2, -1)   # (batch, seq_len, seq_len)

        # 第三步：缩放
        scores = scores / math.sqrt(self.d_k)

        # 第四步：Softmax，得到注意力权重
        weights = F.softmax(scores, dim=-1)   # (batch, seq_len, seq_len)

        # 第五步：加权求和
        output = weights @ V   # (batch, seq_len, d_k)

        return output, weights


# ── 测试与可视化 ──
torch.manual_seed(42)
d_model, d_k, seq_len = 8, 4, 5
attn = SimpleSelfAttention(d_model, d_k)

# 模拟 "猫 喜欢 鱼 很 甜" 5个token的输入
x = torch.randn(1, seq_len, d_model)   # batch=1
output, weights = attn(x)

print(f"输入形状:     {x.shape}")        # (1, 5, 8)
print(f"输出形状:     {output.shape}")    # (1, 5, 4)
print(f"注意力权重:   {weights.shape}")   # (1, 5, 5)

print("\n注意力权重矩阵（每行和为1）：")
print(weights[0].detach().numpy().round(3))
print(f"\n每行权重之和: {weights[0].sum(dim=-1).detach().numpy().round(4)}")
# 确认每行都是概率分布（和为 1）
```

## 四、因果掩码：生成时不能"偷看未来"

### 4.1 初印象：用费曼视角理解

上面实现的 Self-Attention 有一个问题：**每个词都能看到序列里所有其他词，包括它后面的词**。

这在做"理解"任务（比如分类）时没问题——读一篇文章做题，当然要把全文看完再回答。

但在"生成"任务（比如聊天、写文章）时，就不能"偷看"未来了：

```
句子：今天 天气 很 好

如果模型在预测"天气"时，就已经看到了后面的"很"和"好"，
那它直接"抄答案"就行了，根本没必要真正理解语言。
训练时"偷看"了答案，推理时却没有答案可偷，
模型就什么都不会了。
```

解决方案：**在计算注意力分数后，把"不该看"的位置用一个极大的负数覆盖**，这样经过 softmax 后，这些位置的权重就变成了接近 0，等于"看不到"。

这个操作叫**因果掩码（Causal Mask）**，或者**自回归掩码（Autoregressive Mask）**。

### 4.2 因果掩码长什么样

对一个长度为 4 的序列，因果掩码是一个下三角矩阵：

```
位置:  1    2    3    4
  1 [可见, 屏蔽, 屏蔽, 屏蔽]   ← 处理词1时，只能看词1自己
  2 [可见, 可见, 屏蔽, 屏蔽]   ← 处理词2时，能看词1和词2
  3 [可见, 可见, 可见, 屏蔽]   ← 处理词3时，能看词1、2、3
  4 [可见, 可见, 可见, 可见]   ← 处理词4时，能看所有词
```

"屏蔽"的位置，在注意力分数矩阵里被填入 $-\infty$（实践中用 $-10^9$ 或 `float('-inf')`），经过 softmax 后变成 0。

### 4.3 代码实现

```python
class CausalSelfAttention(nn.Module):
    """
    带因果掩码的 Self-Attention，用于语言生成（GPT / LLaMA 等）。
    """

    def __init__(self, d_model: int, d_k: int):
        super().__init__()
        self.W_Q = nn.Linear(d_model, d_k, bias=False)
        self.W_K = nn.Linear(d_model, d_k, bias=False)
        self.W_V = nn.Linear(d_model, d_k, bias=False)
        self.d_k = d_k

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B, L, _ = x.shape

        Q = self.W_Q(x)   # (B, L, d_k)
        K = self.W_K(x)   # (B, L, d_k)
        V = self.W_V(x)   # (B, L, d_k)

        scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_k)   # (B, L, L)

        # ── 关键：加入因果掩码 ──
        # 生成下三角为 True、上三角为 False 的掩码
        causal_mask = torch.tril(torch.ones(L, L, device=x.device)).bool()
        # 把上三角（不该看的位置）填为 -inf
        scores = scores.masked_fill(~causal_mask, float('-inf'))

        weights = F.softmax(scores, dim=-1)
        output  = weights @ V
        return output, weights


# 演示因果掩码的效果
causal_attn = CausalSelfAttention(d_model=8, d_k=4)
x = torch.randn(1, 4, 8)   # 1个句子，4个token
output, weights = causal_attn(x)

print("因果注意力权重（上三角应全为0）：")
print(weights[0].detach().numpy().round(3))
# 期望输出类似：
# [[1.000, 0.000, 0.000, 0.000],  ← 词1只看自己
#  [0.4xx, 0.6xx, 0.000, 0.000],  ← 词2看词1和词2
#  [0.2xx, 0.3xx, 0.4xx, 0.000],  ← 词3看词1/2/3
#  [0.1xx, 0.2xx, 0.3xx, 0.4xx]]  ← 词4看所有
```

## 五、可视化注意力权重：模型在看什么

import numpy as np

def plot_attention(weights, tokens, title="注意力权重"):
    """
    可视化注意力权重热力图。

    tokens: token 文本列表

    fig, ax = plt.subplots(figsize=(6, 5))
    im = ax.imshow(weights, cmap="Blues", vmin=0, vmax=1)

    ax.set_xticks(range(len(tokens)))
    ax.set_yticks(range(len(tokens)))

## 六、三种注意力的关系

在读论文和文档时，你会遇到三种注意力的名字，这里统一解释清楚：

| 名称 | Q 来自 | K/V 来自 | 用在哪里 |
|------|--------|---------|---------|
| **Self-Attention（自注意力）** | 序列本身 | 序列本身 | Transformer 每一层 |
| **Masked Self-Attention（带掩码的自注意力）** | 序列本身 | 序列本身（只看过去）| Decoder 的第一个注意力层 |
| **Cross-Attention（交叉注意力）** | Decoder 当前状态 | Encoder 的输出 | Encoder-Decoder 架构的 Decoder |

**对于 GPT、LLaMA、Claude 这类 Decoder-only 模型**：
- 只有 Masked Self-Attention（每个词只能看自己和之前的词）
- 没有 Cross-Attention（没有 Encoder）

**对于 BERT 这类 Encoder-only 模型**：
- 只有（全双向）Self-Attention（每个词能看所有词）
- 没有掩码（理解任务不需要因果约束）

## 七、Self-Attention 的计算复杂度

让我们理解一下前面提到的"$O(L^2)$ 问题"：

注意力分数矩阵 $QK^T$ 的形状是 $(L \times L)$，计算它需要 $L^2$ 次内积运算。

| 序列长度 L | 注意力矩阵大小 | 相对计算量 |
|-----------|-------------|-----------|
| 512 token | 512 × 512 = 26万 | 1× |
| 2048 token | 2048 × 2048 = 420万 | 16× |
| 8192 token | 8192 × 8192 = 6700万 | 256× |
| 32768 token | 32768 × 32768 = 10亿 | 4096× |

这就是为什么早期的 GPT-2 只能处理 1024 个 token，而现在的模型处理 128K token 需要专门的优化（Flash Attention，我们在 3.8 章讲）。

```python
# 直观感受计算量
def attention_complexity(seq_len, d_k):
    """计算一次 Self-Attention 的大致计算量"""
    # QK^T: L × d_k 矩阵 × d_k × L 矩阵 = O(L² × d_k) 次乘法
    qk_ops = seq_len * seq_len * d_k
    # softmax(分数) @ V: L × L 矩阵 × L × d_k = O(L² × d_k) 次乘法
    sv_ops = seq_len * seq_len * d_k
    total = qk_ops + sv_ops
    return total

for L in [512, 2048, 8192, 32768]:
    ops = attention_complexity(L, d_k=64)
    print(f"L={L:6d}: 注意力计算量约 {ops/1e9:.2f}B 次乘法")
```

## 八、把 Self-Attention 接入上一章的 Transformer 层

现在把上一章用"占位符"代替注意力的 `TransformerLayer` 换成真正的 `CausalSelfAttention`：

```python
class TransformerLayerWithRealAttention(nn.Module):
    """
    用真正的 Causal Self-Attention 替换占位符，
    形成一个完整的 Decoder 层。
    """
    def __init__(self, d_model: int, d_ff: int):
        super().__init__()
        self.norm1     = nn.LayerNorm(d_model)
        self.norm2     = nn.LayerNorm(d_model)
        # 真正的注意力！d_k = d_model（单头，下一章扩展为多头）
        self.attention = CausalSelfAttention(d_model, d_k=d_model)
        self.ffn       = nn.Sequential(
            nn.Linear(d_model, 4 * d_model),
            nn.GELU(),
            nn.Linear(4 * d_model, d_model),
        )

    def forward(self, x):
        # 注意力 + 残差
        attn_out, _ = self.attention(self.norm1(x))
        x = x + attn_out

        # FFN + 残差
        x = x + self.ffn(self.norm2(x))
        return x


# 测试
layer = TransformerLayerWithRealAttention(d_model=32, d_ff=128)
x     = torch.randn(2, 6, 32)   # batch=2, seq_len=6, d_model=32
out   = layer(x)
print(f"输入形状:  {x.shape}")    # (2, 6, 32)
print(f"输出形状:  {out.shape}")  # (2, 6, 32)  形状不变
```

## 九、Self-Attention 直觉小结

用五句话总结 Self-Attention 在做什么：

1. **把每个词变成三个角色**：Q（在查什么）、K（我的标签）、V（我的内容）
2. **计算所有词对的匹配度**：用 Q 和 K 的内积，再缩放
3. **Softmax 转成概率**：每个词对其他所有词的"注意力分配"
4. **加权求和 V**：按注意力权重，把所有词的内容综合起来
5. **生成时加因果掩码**：每个词只能看自己和之前的词，不能"偷看未来"

这个过程重复 N 层，每层都让每个词的表示变得更"丰富"——融合了越来越多来自其他词的信息，最终能理解复杂的语义关系。

## 十、面试常见问题

**Q1：Self-Attention 的 Q、K、V 分别是什么？为什么需要三个不同的矩阵？**

Q（Query）是"我在查什么"，K（Key）是"我的索引标签"，V（Value）是"我的实际内容"。它们都来自同一个输入 X，但通过三个不同的线性变换矩阵 $W_Q, W_K, W_V$ 生成。需要三个不同矩阵，是因为"查询"、"被查询"和"提供内容"是三种不同的语义角色，同一段文字在不同角色下应该呈现不同的特征，让模型有更大的灵活性。

**Q2：为什么注意力分数要除以 $\sqrt{d_k}$？**

当 $d_k$ 较大时，$QK^T$ 的内积数值会随维度增大（方差约为 $d_k$），导致 softmax 进入"饱和区"——某个词的注意力权重接近 1，其他词接近 0，梯度几乎消失，模型无法有效学习。除以 $\sqrt{d_k}$ 把方差归一化到 1，让 softmax 的输出分布更均匀，梯度也能正常传播。

**Q3：什么是因果掩码？为什么语言生成需要它？**

因果掩码（Causal Mask）是一个下三角矩阵，它把注意力分数矩阵中"未来位置"的值设为负无穷，经过 softmax 后这些位置的权重变成 0。语言生成需要它，是因为自回归生成是逐词进行的：模型在预测第 $t$ 个词时，只能用前 $t-1$ 个词的信息，不能"偷看"第 $t+1, t+2, \ldots$ 个词——否则训练时的目标就失去了意义，模型学不到真正的语言规律。

**Q4：Self-Attention 的计算复杂度是多少？**

空间和时间复杂度都是 $O(L^2 \cdot d_k)$，其中 $L$ 是序列长度，$d_k$ 是 Q/K 的维度。关键是 $L^2$ 这一项——序列长度翻倍，计算量变成 4 倍。这是处理超长序列（如 128K token）时的主要瓶颈，也是 Flash Attention、稀疏注意力等优化技术的出发点。

**Q5：Self-Attention、Masked Self-Attention 和 Cross-Attention 有什么区别？**

Self-Attention：Q、K、V 来自同一序列，每个位置能看所有其他位置（用于 BERT）。Masked Self-Attention：和 Self-Attention 相同，但加了因果掩码，每个位置只能看过去的位置（用于 GPT/LLaMA 等生成模型）。Cross-Attention：Q 来自一个序列（Decoder），K/V 来自另一个序列（Encoder 的输出），用于 Encoder-Decoder 架构（如 T5、Whisper）中让 Decoder 查询 Encoder 的信息。

**Q6：如果去掉 Self-Attention，只用 FFN，会怎样？**

FFN 是"每个词独立处理"的——处理"喜欢"时，完全不知道旁边是"猫"还是"狗"。Self-Attention 让每个词能感知整个句子的上下文，这是理解语义、消解指代、建立长距离关系的关键。没有 Self-Attention，模型就退化成了对每个词独立处理的函数，完全无法捕捉词与词之间的关系。
