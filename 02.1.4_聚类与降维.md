## 零、开篇：无监督结构发现，大模型数据工程的基础设施

聚类和降维是无监督学习里最核心的两类工具。它们在大模型时代不仅没有退场，反而因为 embedding 空间的普及而变得更加重要。

**真实使用场景**：

- **训练数据去重**：LLaMA-3、Qwen2 等模型的预训练数据清洗流水线，都需要在 embedding 空间里做近邻聚类以识别重复或高度相似的文档，MinHash + 聚类是主流方案
- **语义空间可视化**：理解模型学到了什么。把词嵌入或 KV Cache 降维到 2D，可以直观看到"数字聚在一起"、"月份聚在一起"这类语义结构
- **SFT 数据多样性采样**：用聚类保证指令数据在主题上均匀分布，避免某类指令过度重复
- **RAG 知识库去冗余**：对文档 chunk 做 embedding 聚类，每个簇只保留代表性文档，减少冗余检索
- **模型可解释性**：对 Transformer 各层的隐状态做 PCA，分析不同层学到的表示有何差异

## 一、K-means：最经典的聚类算法

### 1.1 初印象：用费曼视角理解

**K-means 就是"选 K 个集合点，让所有人就近集合，然后把集合点移到人群中心，反复迭代"**。

想象广场上散落着 1000 个人，你要把他们分成 3 组。做法是：随机指定 3 个集合点，每个人走向离自己最近的那个；等大家站定了，把每个集合点移到所属人群的中心；再让所有人重新就近走向更新后的集合点……反复几轮，人群自然分成了 3 个聚集的团簇。

### 1.2 算法流程与数学

**输入**：数据 $\{x_1, \ldots, x_n\} \subset \mathbb{R}^d$，目标簇数 $K$

**目标**：最小化簇内总方差（Within-Cluster Sum of Squares，WCSS）：

$$\min_{\{C_k\}, \{\mu_k\}} \sum_{k=1}^K \sum_{x_i \in C_k} \|x_i - \mu_k\|^2$$

**迭代步骤**：

1. 随机初始化 $K$ 个中心 $\mu_1, \ldots, \mu_K$
2. **分配**：每个点归属最近的中心 $c_i = \arg\min_k \|x_i - \mu_k\|^2$
3. **更新**：$\mu_k = \frac{1}{|C_k|}\sum_{x_i \in C_k} x_i$（组内均值）
4. 重复 2-3 直到分配不再变化

**K-means++ 初始化**（sklearn 默认，比随机初始化好很多）：第一个中心随机选，后续中心按"与已有中心距离的平方"概率选，使初始中心尽量分散，降低陷入差解的概率。

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.cluster import KMeans
from sklearn.datasets import make_blobs
from sklearn.metrics import silhouette_score

# 生成模拟 embedding 数据
X, y_true = make_blobs(n_samples=500, centers=5, cluster_std=0.8, random_state=42)

# 用轮廓系数（Silhouette Score）选最优 K
scores = []
K_range = range(2, 10)
for k in K_range:
    km = KMeans(n_clusters=k, init="k-means++", n_init=10, random_state=42)
    labels = km.fit_predict(X)
    scores.append(silhouette_score(X, labels))

best_k = K_range[np.argmax(scores)]
print(f"最优 K={best_k}，轮廓系数={max(scores):.4f}")

# 用最优 K 训练
km_best = KMeans(n_clusters=best_k, init="k-means++", n_init=10, random_state=42)
labels = km_best.fit_predict(X)

# 可视化
plt.figure(figsize=(10, 4))
plt.subplot(1, 2, 1)
plt.scatter(X[:, 0], X[:, 1], c=labels, cmap="tab10", s=15, alpha=0.7)
plt.scatter(km_best.cluster_centers_[:, 0], km_best.cluster_centers_[:, 1],
            c="red", marker="x", s=200, linewidths=3, label="中心")
plt.title(f"K-means 聚类结果（K={best_k}）")
plt.legend()

plt.subplot(1, 2, 2)
plt.plot(list(K_range), scores, "bo-")
plt.axvline(best_k, color="red", linestyle="--", alpha=0.5)
plt.xlabel("K 值"); plt.ylabel("Silhouette Score")
plt.title("选最优 K")
plt.tight_layout()
plt.savefig("kmeans.png", dpi=150, bbox_inches="tight")
```

### 1.3 K-means 在大模型数据工程中的应用

**SFT 数据多样性采样**：确保指令数据覆盖均匀，不让某类型指令过度主导：

```python
from sentence_transformers import SentenceTransformer
from sklearn.cluster import KMeans, MiniBatchKMeans
import numpy as np

def diversity_sample(instructions, n_clusters=500, samples_per_cluster=10):
    """
    用 K-means 对指令做语义聚类，每簇采样固定数量，
    保证 SFT 数据的主题多样性
    """
    # 1. 提取 embedding
    model = SentenceTransformer("all-MiniLM-L6-v2")
    embeddings = model.encode(instructions, batch_size=256, show_progress_bar=True)

    # 2. K-means 聚类（大规模用 MiniBatchKMeans）
    km = MiniBatchKMeans(
        n_clusters=n_clusters,
        batch_size=10000,
        random_state=42,
        n_init=3,
    )
    cluster_ids = km.fit_predict(embeddings)

    # 3. 每簇等量采样
    sampled_indices = []
    for cid in range(n_clusters):
        idx_in_cluster = np.where(cluster_ids == cid)[0]
        n_sample = min(samples_per_cluster, len(idx_in_cluster))
        chosen = np.random.choice(idx_in_cluster, n_sample, replace=False)
        sampled_indices.extend(chosen.tolist())

    return [instructions[i] for i in sampled_indices]
```

## 二、层次聚类与 HDBSCAN

### 2.1 初印象：用费曼视角理解

**层次聚类就是"从下往上合并"或"从上往下分裂"**。

凝聚式层次聚类（最常用）：一开始每个点是一个簇，然后不断找"距离最近的两个簇"把它们合并，一直合并到只剩一个大簇。整个合并过程形成一棵树（树状图 / Dendrogram），你在任意高度"切一刀"就得到对应数量的簇。

优点：不需要预先指定 K。缺点：计算复杂度 $O(n^2)$ 到 $O(n^3)$，大数据不适用。

**HDBSCAN（层次 DBSCAN）**：基于密度的聚类，能发现任意形状的簇，并自动把低密度区域的点标记为"噪声"（-1），不强制每个点都归属某簇。UMAP 与 HDBSCAN 配合使用是一种有效的非线性降维后聚类策略，在 LLM 数据工程中是强大的组合。

```python
import hdbscan
import numpy as np

def semantic_cluster_with_hdbscan(embeddings, min_cluster_size=50):
    """
    HDBSCAN 语义聚类：无需指定 K，自动发现簇数，噪声点标记为 -1
    适合：RAG 知识库去冗余、训练数据质量分析
    """
    clusterer = hdbscan.HDBSCAN(
        min_cluster_size=min_cluster_size,  # 最小簇大小
        min_samples=5,                       # 核心点最小邻居数
        metric="euclidean",
        cluster_selection_method="eom",      # 超出质量法，适合大数据
    )
    labels = clusterer.fit_predict(embeddings)
    n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
    n_noise    = (labels == -1).sum()
    print(f"发现 {n_clusters} 个簇，{n_noise} 个噪声点 ({n_noise/len(labels):.1%})")
    return labels, clusterer

# 典型流水线：先 UMAP 降维，再 HDBSCAN 聚类
# （后面 UMAP 节会给出完整示例）
```

## 三、PCA：线性降维的标准工具

### 3.1 初印象：用费曼视角理解

**PCA 就是"找到数据最'展开'的方向，然后把数据投影到这些方向上"**。

想象你有一张皱巴巴的纸片（高维数据），PCA 找到让纸片"最铺开"的方向（方差最大的方向），把纸片压平到这个方向上——这样损失最小，保留的信息最多。

这些"最铺开的方向"就是主成分，对应数据协方差矩阵的特征向量，按特征值从大到小排列（特征值越大，这个方向的方差越大，即越"重要"）。

### 3.2 数学推导

对数据矩阵 $X \in \mathbb{R}^{n \times d}$（已中心化），PCA 求解：

$$\max_{\mathbf{w}: \|\mathbf{w}\|=1} \text{Var}(\mathbf{w}^T X^T) = \max \mathbf{w}^T \Sigma \mathbf{w}$$

其中 $\Sigma = \frac{1}{n} X^T X$ 是样本协方差矩阵。最优解是 $\Sigma$ 的特征向量（即 $X$ 的奇异向量，联系 1.1.1 章的 SVD）。

**PCA = 截断 SVD**：

$$X = U \Sigma V^T \quad \Rightarrow \quad X_k = U_k \Sigma_k V_k^T$$

保留前 $k$ 个奇异值即是 PCA 到 $k$ 维。

```python
import numpy as np
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

# 模拟 LLM 各层 CLS token 隐状态，分析各层表示差异
np.random.seed(42)
n_samples, d_model = 200, 768

# 模拟浅层（语法信息多，结构性强）和深层（语义信息多，更分散）
shallow = np.random.randn(n_samples, d_model) * 0.5 + \
          np.tile(np.random.randn(1, d_model), (n_samples, 1)) * 2
deep    = np.random.randn(n_samples, d_model)

fig, axes = plt.subplots(1, 2, figsize=(10, 4))
for ax, data, title in zip(axes, [shallow, deep], ["浅层表示（结构化）", "深层表示（分散）"]):
    pca = PCA(n_components=2)
    proj = pca.fit_transform(data)
    ax.scatter(proj[:, 0], proj[:, 1], s=15, alpha=0.6)
    ax.set_title(f"{title}\n前2个PC解释方差: {pca.explained_variance_ratio_[:2].sum():.1%}")
    ax.set_xlabel("PC1"); ax.set_ylabel("PC2")
plt.tight_layout()
plt.savefig("pca_layers.png", dpi=150)

# 选择保留多少维度：解释方差累积曲线
pca_full = PCA().fit(deep)
cumvar = np.cumsum(pca_full.explained_variance_ratio_)
n_95 = np.searchsorted(cumvar, 0.95) + 1
print(f"保留 95% 方差需要 {n_95} 个主成分（共 {d_model} 维）")
```

**PCA 在大模型中的应用**：

- 分析 Transformer 不同层的表示：浅层 PCA 分布通常更结构化（解释方差集中），深层更均匀
- 快速检查词向量质量：近义词应在 PCA 2D 图上靠近
- 作为 UMAP / t-SNE 的预处理步骤（先 PCA 到 50 维再降维，速度快 10 倍）

## 四、t-SNE：局部结构的可视化大师

### 4.1 初印象：用费曼视角理解

**t-SNE 的目标是"邻居还是邻居"**——在高维空间里靠近的点，在 2D 图里也要靠近。

它把高维里每对点的"相似度"转换成概率，然后在 2D 里找一个布局，使得 2D 概率分布和高维概率分布尽量一致（最小化 KL 散度）。用了 t 分布（而非高斯）来表示 2D 相似度，是为了解决"拥挤问题"——让簇之间更分散。

**关键特性**：
- 擅长保留**局部结构**（近邻关系），不擅长保留全局结构（簇间距离没有意义）
- 对超参数 `perplexity` 敏感（通常设 5-50）
- 不可扩展，样本数超过 10 万就很慢

### 4.2 数学原理

**高维相似度**（条件概率，用高斯核）：

$$p_{j|i} = \frac{\exp(-\|x_i - x_j\|^2 / 2\sigma_i^2)}{\sum_{k \neq i}\exp(-\|x_i - x_k\|^2 / 2\sigma_i^2)}, \quad p_{ij} = \frac{p_{j|i} + p_{i|j}}{2n}$$

**低维相似度**（用 t 分布，重尾防拥挤）：

$$q_{ij} = \frac{(1 + \|y_i - y_j\|^2)^{-1}}{\sum_{k \neq l}(1 + \|y_k - y_l\|^2)^{-1}}$$

**损失函数**：最小化两个分布的 KL 散度：

$$\mathcal{L} = \sum_{i,j} p_{ij} \log \frac{p_{ij}}{q_{ij}}$$

```python
from sklearn.manifold import TSNE
from transformers import AutoTokenizer, AutoModel
import torch
import numpy as np
import matplotlib.pyplot as plt

def visualize_token_embeddings_tsne(model_name="bert-base-uncased", n_tokens=2000):
    """可视化 BERT 词嵌入的语义结构"""
    tokenizer = AutoTokenizer.from_pretrained(model_name)
    model = AutoModel.from_pretrained(model_name)

    # 提取 token embedding 矩阵（词表的前 n_tokens 个词）
    with torch.no_grad():
        embeddings = model.embeddings.word_embeddings.weight[:n_tokens].numpy()

    tokens = [tokenizer.convert_ids_to_tokens(i) for i in range(n_tokens)]

    # 先 PCA 预处理（50 维），再 t-SNE（速度提升 10x）
    from sklearn.decomposition import PCA
    embeddings_50d = PCA(n_components=50).fit_transform(embeddings)

    tsne = TSNE(n_components=2, perplexity=30, n_iter=1000, random_state=42)
    embeddings_2d = tsne.fit_transform(embeddings_50d)

    # 给几类词着色
    months = {"january","february","march","april","may","june",
              "july","august","september","october","november","december"}
    digits = {str(i) for i in range(10)}
    colors = []
    for t in tokens:
        t_lower = t.lower().strip("##")
        if t_lower in months:
            colors.append("red")
        elif t_lower in digits:
            colors.append("blue")
        else:
            colors.append("lightgray")

    plt.figure(figsize=(12, 10))
    plt.scatter(embeddings_2d[:, 0], embeddings_2d[:, 1], c=colors, s=5, alpha=0.6)
    plt.title(f"BERT Token Embeddings (t-SNE)\n红=月份 蓝=数字")
    plt.savefig("tsne_embeddings.png", dpi=150, bbox_inches="tight")
    print("可视化完成，保存为 tsne_embeddings.png")

# 调用示例（需要 transformers 库）

```

## 五、UMAP：t-SNE 的全面进化

### 5.1 初印象：用费曼视角理解

**UMAP 和 t-SNE 解决同一个问题，但更快、更好**。

UMAP 速度快，能处理大规模高维数据；能扩展到任意目标维度，不仅仅用于可视化；在保留全局结构上通常优于 t-SNE。

t-SNE 只告诉你"谁和谁近"，不告诉你簇和簇之间的关系（簇间距离没意义）。UMAP 则同时保留局部和全局结构——你能从 UMAP 图里判断哪些大簇本身是近邻关系。

数学上，UMAP 基于黎曼流形的拓扑结构，用模糊拓扑将高维图和低维图之间的差异最小化——本质上是在流形上学习一个保结构的映射。

### 5.2 关键超参数

| 参数 | 含义 | 推荐值 |
|------|------|--------|
| `n_neighbors` | 构建局部图时考虑多少个邻居 | 可视化: 15，聚类: 30-50 |
| `min_dist` | 低维空间中点之间的最小距离 | 可视化: 0.1，聚类: 0.0 |
| `n_components` | 目标维度 | 可视化: 2，聚类预处理: 10-50 |
| `metric` | 距离度量 | `cosine`（embedding 首选），`euclidean` |

### 5.3 完整流水线：LLM Embedding 分析与数据去冗余

```python
import numpy as np
import umap
import hdbscan
import matplotlib.pyplot as plt
from sentence_transformers import SentenceTransformer

def analyze_and_deduplicate_corpus(texts, similarity_threshold=0.95):
    """
    完整的语料分析和去冗余流水线：
    1. 提取 embedding
    2. UMAP 降维
    3. HDBSCAN 聚类
    4. 可视化
    5. 去除高度相似文档
    """
    print("Step 1: 提取 Embedding...")
    encoder = SentenceTransformer("all-MiniLM-L6-v2")
    embeddings = encoder.encode(texts, batch_size=256, normalize_embeddings=True,
                                show_progress_bar=True)

    print("Step 2: UMAP 降维（聚类用 10 维）...")
    reducer_cluster = umap.UMAP(
        n_neighbors=30,
        n_components=10,
        min_dist=0.0,
        metric="cosine",
        random_state=42,
    )
    embeddings_10d = reducer_cluster.fit_transform(embeddings)

    print("Step 3: HDBSCAN 聚类...")
    clusterer = hdbscan.HDBSCAN(
        min_cluster_size=max(5, len(texts) // 100),
        min_samples=3,
        metric="euclidean",
        cluster_selection_method="eom",
    )
    cluster_labels = clusterer.fit_predict(embeddings_10d)
    n_clusters = len(set(cluster_labels)) - (1 if -1 in cluster_labels else 0)
    print(f"  发现 {n_clusters} 个语义簇")

    print("Step 4: UMAP 降维（可视化用 2 维）...")
    reducer_vis = umap.UMAP(
        n_neighbors=15,
        n_components=2,
        min_dist=0.1,
        metric="cosine",
        random_state=42,
    )
    embeddings_2d = reducer_vis.fit_transform(embeddings)

    # 可视化
    plt.figure(figsize=(12, 8))
    unique_labels = sorted(set(cluster_labels))
    cmap = plt.cm.get_cmap("tab20", max(n_clusters, 1))
    for label in unique_labels:
        mask = cluster_labels == label
        color = "lightgray" if label == -1 else cmap(label % 20)
        name  = "噪声" if label == -1 else f"簇 {label}"
        plt.scatter(embeddings_2d[mask, 0], embeddings_2d[mask, 1],
                    c=[color], s=8, alpha=0.6, label=name if label < 10 else "")
    plt.title(f"语料语义空间（UMAP + HDBSCAN）\n{n_clusters} 个语义簇")
    plt.legend(markerscale=2, fontsize=8, ncol=2)
    plt.savefig("corpus_umap.png", dpi=150, bbox_inches="tight")
    print("  可视化保存为 corpus_umap.png")

    print("Step 5: 去除重复文档...")
    # 在每个簇内，找出余弦相似度 > threshold 的重复对，只保留一个
    sim_matrix = embeddings @ embeddings.T
    to_remove = set()
    for i in range(len(texts)):
        if i in to_remove:
            continue
        # 找出和 i 高度相似且下标 > i 的点
        similar = np.where(sim_matrix[i] > similarity_threshold)[0]
        similar = similar[similar > i]
        to_remove.update(similar.tolist())

    kept = [i for i in range(len(texts)) if i not in to_remove]
    print(f"  原始文档: {len(texts)}  去重后: {len(kept)}  "
          f"删除率: {len(to_remove)/len(texts):.1%}")
    return [texts[i] for i in kept], cluster_labels, embeddings_2d
```

### 5.4 t-SNE vs UMAP 实战对比

| 维度 | t-SNE | UMAP |
|------|-------|------|
| 速度 | 慢（10 万样本需数十分钟） | 快（同样数据约 1-2 分钟） |
| 全局结构 | 差（簇间距离无意义） | 好（能反映整体拓扑） |
| 局部结构 | 非常好 | 好 |
| 可扩展性 | 不可扩展到高维目标 | 可降到任意维度 |
| 可复现性 | 随机性大 | 设置 random_state 后可复现 |
| 聚类预处理 | 不推荐（破坏密度） | 推荐（配合 HDBSCAN） |
| 典型使用场景 | 小数据集可视化 | 大规模 embedding 分析 |

## 六、降维方法全景对比

### 6.1 线性 vs 非线性

```
线性方法（保持线性关系）
├── PCA：最大化方差，无监督，速度极快
├── LDA：最大化类间/类内方差比，有监督降维
└── SVD：PCA 的矩阵分解视角，也用于推荐系统、文本 LSA

非线性方法（保持流形结构）
├── t-SNE：保局部，慢，可视化
├── UMAP：保局部+全局，快，可视化+聚类预处理
├── Isomap：测地线距离，保全局流形
└── Autoencoder：神经网络版降维，可端到端训练
```

### 6.2 大模型工程里的工具选择

| 任务 | 推荐工具 | 原因 |
|------|---------|------|
| 快速看 embedding 质量 | PCA 2D | 秒级完成，结果可解释 |
| 小数据集精细可视化（< 5 万） | t-SNE | 局部结构最清晰 |
| 大规模 embedding 可视化（> 5 万） | UMAP | 速度快，全局结构好 |
| 训练数据聚类去重 | UMAP 10D + HDBSCAN | 标准流水线 |
| 下游分类特征预处理 | PCA 到 50-200 维 | 保留方差，加快训练 |
| 可解释性分析（Transformer 层） | PCA + 解释方差比 | 定量分析各层信息量 |

## 七、面试常见问题

**Q1：K-means 的缺点有哪些？怎么解决？**

三个主要缺点：需要预先指定 K（解决：用轮廓系数或肘方法选 K，或改用 HDBSCAN 自动发现簇数）；对初始化敏感（解决：K-means++ 初始化，多次运行取最优）；假设簇是球形且大小相近（解决：改用 DBSCAN 或 GMM，能处理任意形状的簇）。此外 K-means 对异常值敏感，K-medoids 更鲁棒。

**Q2：PCA 和 t-SNE 的根本区别是什么？**

PCA 是线性方法，把数据投影到方差最大的线性子空间，保留的是全局线性结构，速度极快，结果可解释（每个主成分有明确含义），可以对新数据做快速变换。t-SNE 是非线性方法，专注保留局部邻居关系（近邻在低维仍近邻），能展示复杂的非线性结构，但不能对新数据增量变换，速度慢，簇间距离没有意义。实践中先 PCA 到 50 维再 t-SNE，兼顾速度和效果。

**Q3：UMAP 相比 t-SNE 有什么优势？**

UMAP 速度快，可以处理大数据集和高维数据；能扩展到可视化以外的降维任务；在保留全局结构方面通常优于 t-SNE。另外 UMAP 能降到任意目标维度（不仅是 2D），可以配合 HDBSCAN 做聚类预处理；结果更稳定，设置随机种子后可复现。主要代价是数学原理更复杂，超参语义不如 t-SNE 直观。

**Q4：聚类评估没有标准答案时，用什么指标？**

无监督聚类的内部评估指标：轮廓系数（Silhouette Score，-1 到 1，越高越好，衡量簇内紧密度和簇间分离度）、Davies-Bouldin 指数（越低越好）、Calinski-Harabasz 指数（越高越好）。如果有部分标注，用调整兰德指数（ARI）或归一化互信息（NMI）。实际工程中，最重要的是定性检查：随机抽取几个簇内的样本，看它们是否语义上真的相似。

**Q5：在 RAG 系统里，为什么要对知识库做聚类去冗余？**

两个原因：效率——重复内容会占用宝贵的 context window 空间，让 LLM 看到很多意义相同的片段，同时增加了向量检索的计算量；质量——大量相似文档被检索到时，模型容易过度强调这部分内容，产生偏差（好像这个观点有很多"不同来源"支持，实则是同一来源的重复）。去冗余后，知识库更紧凑，检索结果的多样性更高。

**Q6：PCA 为什么要先中心化数据？**

PCA 计算的是协方差矩阵（衡量数据围绕均值的分布），如果不中心化，第一主成分会偏向"数据均值所在的方向"而非"方差最大的方向"，得到的主成分没有意义。中心化就是让数据的均值在原点，让 PCA 专注于发现数据的散布方向。

**Q7：t-SNE 的 perplexity 参数是什么？怎么设置？**

perplexity 控制局部邻域的规模，可以粗略理解为"每个点认为自己有多少个有效邻居"。数学上它决定了高维高斯核的带宽 $\sigma_i$。通常设为 5-50，数据量大时设大一点。perplexity 太小会让图碎片化（每个点只认识很少邻居），太大会让图过于平滑（全局化）。实践中，同一数据在不同 perplexity 下多跑几次，选视觉上簇结构最清晰的那个。

**Q8：UMAP 的 n_neighbors 和 min_dist 分别影响什么？**

`n_neighbors` 控制局部与全局结构的权衡：小值（5-10）更关注局部细节，大值（50-100）更关注全局结构。`min_dist` 控制低维空间中点的"紧凑程度"：接近 0 时点会紧密聚集（有利于聚类），较大值（0.5-1.0）时点分布更均匀（有利于可视化展示数据的连续流形结构）。
