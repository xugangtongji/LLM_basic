## 零、开篇：为什么大模型工程师必须懂信息论

信息论看起来离 AI 很远，实际上**它是大模型评估和训练的度量衣**：

- 训练大模型的 Loss，叫 **Cross-Entropy（交叉熵）**
- 评估语言模型好坏的指标，叫 **Perplexity（困惑度）**，它是交叉熵的指数形式
- PPO、DPO、蒸馏里都有的"别让新分布跑偏"约束，叫 **KL 散度**
- GAN 的稳定版本使用的是 **JS 散度**
- 特征选择、表示学习里的重要量，叫 **互信息（Mutual Information）**
- 大模型"涌现能力"研究里的比特度量，叫 **BPB / BPC**

所有这些名字背后的数学，来自于一个极其朴素的问题：**"信息"是什么？能不能把它量化？** 1948 年 Shannon 在《A Mathematical Theory of Communication》里给出了答案，从此"信息"这个词成了可计算的东西。

## 一、自信息与熵

### 1.1 初印象：用费曼视角理解

**想象一个天气预报员**。

- 在常年艳阳天的沙漠预报"明天晴" — 这条播报几乎没告诉你任何东西（信息量 ≈ 0）
- 在那个沙漠预报"明天下雨" — 这条播报信息量大到爆炸（信息量高）

**越是出人意料的事件，信息量越大**。这就是自信息的直觉：**罕见 = 信息量大，常见 = 信息量小**。

然后把一整天的所有天气可能性都考虑进来，给每个可能性按概率加权平均信息量，得到这一天天气预报"平均给人多少惊讶"——这就是**熵**。熵是一整个概率分布的"不确定性总量"。

### 1.2 自信息

事件 $x$ 发生的自信息（self-information）定义为：

$$
I(x) = -\log P(x)
$$

对数底决定单位：
- $\log_2$ → 单位是 **bit**（比特）
- $\log_e = \ln$ → 单位是 **nat**（奈特，深度学习框架默认用这个）
- $\log_{10}$ → 单位是 hartley（很少见）

三个关键性质：
1. **$I(x) \geq 0$**：信息量非负
2. **$P(x) = 1 \Rightarrow I(x) = 0$**：必然发生的事没有信息量
3. **独立事件可加**：$I(x, y) = I(x) + I(y)$（对应 $P(x,y) = P(x)P(y)$ 取 log）

### 1.3 熵（Entropy）

一个离散随机变量 $X$ 的熵定义为自信息的期望：

$$
H(X) = \mathbb{E}_{x \sim P}[I(x)] = -\sum_{x} P(x) \log P(x)
$$

连续变量用积分代替求和，叫**微分熵**。

**几何直觉**：熵度量分布的"混乱程度"或"不确定度"。

三个极端例子（3 个值的分布）：

| 分布 $P$ | 熵 $H(P)$（以 $\log_2$ 计） | 理解 |
|---------|---------------------------|------|
| $(1, 0, 0)$ | 0 bit | 完全确定 |
| $(0.5, 0.25, 0.25)$ | 1.5 bit | 不均匀 |
| $(1/3, 1/3, 1/3)$ | $\log_2 3 \approx 1.585$ bit | 最大不确定 |

**最大熵原理**：均匀分布熵最大。这是"最大熵模型"（MaxEnt）的理论基础。

### 1.4 熵的编码解释（Shannon Source Coding）

**熵的真正物理含义是：对这个分布的信息进行编码，所需的最短平均编码长度（bit 数）**。

举个例子：4 个符号 A/B/C/D，概率 0.5/0.25/0.125/0.125，熵 = 1.75 bit。
最优编码：A→0、B→10、C→110、D→111，平均码长 = 0.5×1 + 0.25×2 + 0.125×3 + 0.125×3 = 1.75 bit。
完全吻合。

**这个视角贯穿整个信息论**：交叉熵、KL 散度都能从"编码长度"的角度解释。

### 1.5 代码实操

```python
import numpy as np

def entropy(p, base=2):
    p = np.asarray(p, dtype=float)
    p = p[p > 0]  # 避免 log(0)
    return -np.sum(p * np.log(p)) / np.log(base)

print(entropy([1.0, 0, 0]))              # 0.0
print(entropy([0.5, 0.25, 0.25]))        # 1.5
print(entropy([1/3, 1/3, 1/3]))          # 1.585
print(entropy([0.5, 0.25, 0.125, 0.125]))# 1.75
```

## 二、交叉熵（Cross-Entropy）：语言模型损失的真身

### 2.1 初印象：用费曼视角理解

**想象你按照一本过时的"天气出现频率手册"去给每天天气编码**。手册说晴 50%、阴 30%、雨 20%。可今天的真实气候其实是晴 80%、阴 10%、雨 10%。

你用"旧手册"的编码去发电报，**每天平均要多花几个 bit**——因为手册估错了概率，你把最短的编码留给了不该留的事件。

- 如果你的手册和真实分布完全一致 → 平均码长 = 真实熵（最优）
- 如果手册和真实分布不一致 → 平均码长 > 真实熵，多出来的部分就是"代价"

**交叉熵就是"用 Q 分布去编码实际 P 分布数据时的平均码长"**。模型训练，就是在优化这本"手册"（模型 Q），让它越来越贴近真实 P。

### 2.2 定义

给定真实分布 $P$ 和模型分布 $Q$，交叉熵定义为：

$$
H(P, Q) = -\sum_x P(x) \log Q(x) = \mathbb{E}_{x \sim P}[-\log Q(x)]
$$

关键不等式（Gibbs' inequality）：

$$
H(P, Q) \geq H(P)
$$

等号成立当且仅当 $P = Q$。

### 2.3 语言模型里的交叉熵

对一个训练样本（一个 token）：真实标签是 one-hot，只有 "正确词 w" 那一维为 1，其余为 0。

$$
\text{CE}_{\text{token}} = -\sum_i P_i \log Q_i = -1 \cdot \log Q_w - \sum_{i \neq w} 0 \cdot \log Q_i = -\log Q_w
$$

**所以单个 token 的交叉熵 = $-\log$（模型对正确词的预测概率）**。

整段文本的 Loss：

$$
\mathcal{L} = -\frac{1}{N} \sum_{t=1}^{N} \log P_\theta(w_t \mid w_{<t})
$$

你在 PyTorch 里写 `F.cross_entropy(logits, labels)`，内部做的就是：
1. 对 logits 做 log_softmax（比分别做 log 和 softmax 数值稳定）
2. 按 labels 取出对应位置的 log 概率
3. 取负号并平均

### 2.4 CE = MLE（核心顿悟）

这是大模型训练中最核心的数学恒等式之一：

**最小化交叉熵 ≡ 最大化对数似然 ≡ MLE**

推导：
$$
\arg\min_\theta \text{CE}(P_{\text{data}}, Q_\theta) = \arg\min_\theta \left[-\sum P_{\text{data}} \log Q_\theta\right] = \arg\max_\theta \sum P_{\text{data}} \log Q_\theta
$$

最后一项在用采样代替真实分布后，就是对数似然：

$$
\arg\max_\theta \frac{1}{N} \sum_{i=1}^N \log Q_\theta(x_i) = \arg\max_\theta \log \prod_i Q_\theta(x_i)
$$

这解释了为什么几乎所有分类和语言模型都用交叉熵做损失——它就是 MLE 的实现形式。

### 2.5 二分类交叉熵（BCE）

特殊情况：分类只有两个类，$P \in \{y, 1-y\}$，$Q \in \{\hat y, 1-\hat y\}$：

$$
\text{BCE} = -y \log \hat y - (1-y) \log(1-\hat y)
$$

`nn.BCELoss` 和 `nn.BCEWithLogitsLoss` 实现的就是这个。

## 三、KL 散度：两分布差多远

### 3.1 初印象：用费曼视角理解

**KL 散度就是"两本手册之间的距离"**——但它是一个**不公平的距离**：

从 A 手册到 B 手册的"距离"，和从 B 手册到 A 手册的"距离"可能不一样。

所以严格地说，KL 散度**不是一个真正的距离**（不满足对称性，也不满足三角不等式）。它只是"用错误分布编码真实分布要多花的 bit 数"。

**在大模型里你会在三个地方反复遇到它**：
1. PPO / GRPO 的 KL 约束 — 不让新策略跑得离旧策略太远
2. 知识蒸馏 — 让小模型的分布贴近大模型的分布
3. VAE — 让编码后的分布贴近先验高斯

### 3.2 定义与分解

$$
D_{KL}(P \| Q) = \sum_x P(x) \log \frac{P(x)}{Q(x)} = H(P, Q) - H(P)
$$

关键公式：

$$
\boxed{D_{KL}(P \| Q) = H(P, Q) - H(P)}
$$

用人话讲：**KL 散度 = 交叉熵 − 真实熵 = 用 Q 编码 P 多花的 bit 数**。

### 3.3 三大核心性质

1. **非负**：$D_{KL}(P \| Q) \geq 0$，等号 iff $P = Q$
2. **不对称**：$D_{KL}(P \| Q) \neq D_{KL}(Q \| P)$
3. **不满足三角不等式**，所以不是真正意义的距离

**不对称性的工程含义**：

- **Forward KL**：$D_{KL}(P \| Q)$，强迫 Q 在 P 有质量的地方必须有质量 → "zero-avoiding"，产生模糊分布
- **Reverse KL**：$D_{KL}(Q \| P)$，Q 只需要拟合 P 的某个峰 → "mode-seeking"，产生锐利分布

VAE 用 Forward KL，所以生成图像常有点模糊；PPO 用 Reverse KL，所以倾向于"守住"旧策略的主要峰值。

### 3.4 机器学习中 KL 的出现场景

**场景 1：为什么训练时 minimize CE 就等价于 minimize KL？**

因为 $P_{\text{data}}$ 固定，$H(P_{\text{data}})$ 是常数，所以：

$$
\arg\min_\theta D_{KL}(P_{\text{data}} \| Q_\theta) = \arg\min_\theta [H(P_{\text{data}}, Q_\theta) - H(P_{\text{data}})] = \arg\min_\theta H(P_{\text{data}}, Q_\theta)
$$

**场景 2：RLHF / PPO 的 KL 约束**

$$
\mathcal{L}_{\text{PPO}} = \mathbb{E}_\tau[R(\tau)] - \beta \cdot D_{KL}(\pi_\theta \| \pi_{\text{ref}})
$$

这里 $\beta$ 越大，新策略越不敢跑远。当奖励模型被 "reward hacking" 时，KL 惩罚就是最后的护栏。

**场景 3：知识蒸馏（Hinton 2015）**

学生模型的 Loss：

$$
\mathcal{L}_{\text{distill}} = \alpha \cdot \text{CE}(y, \hat y_{\text{student}}) + (1-\alpha) \cdot T^2 \cdot D_{KL}(\hat y_{\text{teacher}}^T \| \hat y_{\text{student}}^T)
$$

（T 是温度系数，用于软化分布）

**场景 4：VAE 的 ELBO**

$$
\mathcal{L}_{\text{VAE}} = \mathbb{E}_{q(z\mid x)}[\log p(x \mid z)] - D_{KL}(q(z \mid x) \| p(z))
$$

### 3.5 代码

```python
import torch
import torch.nn.functional as F

# PyTorch 的 KL：kl_div(input, target)，input 是 log-prob，target 是 prob
P = torch.tensor([0.5, 0.3, 0.2])   # 真实分布
Q = torch.tensor([0.4, 0.4, 0.2])   # 模型分布

kl = F.kl_div(Q.log(), P, reduction='sum')  # D_KL(P || Q)
print(f"D_KL(P||Q) = {kl.item():.4f}")

# 验证 KL = CE - H
def entropy(p):
    p = p[p > 0]
    return -(p * p.log()).sum()

def cross_entropy(p, q):
    return -(p * q.log()).sum()

print(f"CE - H = {(cross_entropy(P, Q) - entropy(P)).item():.4f}")

# 验证不对称
kl_reverse = F.kl_div(P.log(), Q, reduction='sum')
print(f"D_KL(Q||P) = {kl_reverse.item():.4f}  (不等于上面)")
```

## 四、JS 散度、互信息、条件熵

### 4.1 JS 散度：对称版的 KL

**初印象**：KL 散度不对称让人不舒服，所以就把两个方向各算一遍再平均——这就是 JS 散度。

$$
D_{JS}(P \| Q) = \frac{1}{2} D_{KL}(P \| M) + \frac{1}{2} D_{KL}(Q \| M), \quad M = \frac{P + Q}{2}
$$

性质：
- 对称：$D_{JS}(P \| Q) = D_{JS}(Q \| P)$
- 有界：$0 \leq D_{JS} \leq \log 2$
- **$\sqrt{D_{JS}}$ 是真正的距离度量**

**主要应用**：原始 GAN 的目标函数和 JS 散度等价。但 JS 在分布不重叠时梯度为 0，所以 WGAN 改用了 Wasserstein 距离。

### 4.2 条件熵

**初印象**：知道 Y 之后，X 还剩多少不确定性。

$$
H(X \mid Y) = -\sum_{x, y} P(x, y) \log P(x \mid y)
$$

链式法则：

$$
H(X, Y) = H(X) + H(Y \mid X) = H(Y) + H(X \mid Y)
$$

### 4.3 互信息：两变量共享多少信息

**初印象**：知道 Y 能减少多少 X 的不确定性。

$$
I(X; Y) = H(X) - H(X \mid Y) = H(Y) - H(Y \mid X)
$$

等价形式：

$$
I(X; Y) = D_{KL}(P(X, Y) \| P(X)P(Y)) = \sum_{x, y} P(x, y) \log \frac{P(x, y)}{P(x)P(y)}
$$

独立 $\Leftrightarrow I(X; Y) = 0$。

**大模型中的用途**：
- InfoNCE 损失（CLIP、SimCLR 的对比学习核心）本质是 $I(X; Y)$ 的下界
- 特征选择：和标签互信息大的特征更有用
- 信息瓶颈（Information Bottleneck）理论：解释深度网络为什么能泛化

## 五、困惑度（Perplexity）：语言模型评估指标

### 5.1 初印象：用费曼视角理解

**想象一个老师让你猜下一个词**。老师说的是"今天天气真___"，你心里大概在 5 个词之间犹豫（好、冷、差、棒、热）——那你对下一个词的"困惑度"就是 5。

**Perplexity 就是"模型对下一个词，等效在多少个候选词之间犹豫"**。

- 完美模型（100% 猜对）→ PPL = 1（不犹豫）
- 词表里均匀瞎猜（50000 词）→ PPL = 50000
- GPT-2 在 WikiText 上 → PPL ≈ 30（在 30 个词之间犹豫）
- GPT-3 大模型 → PPL 降到 20 以下
- 人类在某些语料上 → PPL ≈ 10-20

### 5.2 定义

给定测试文本 $W = w_1 w_2 \dots w_N$，模型分配概率 $P_\theta(W)$：

$$
\text{PPL}(W) = P_\theta(W)^{-\frac{1}{N}} = \exp\left(-\frac{1}{N} \sum_{t=1}^{N} \log P_\theta(w_t \mid w_{<t})\right)
$$

也就是：

$$
\boxed{\text{PPL} = e^{\text{CE}}}
$$

（以 $e$ 为底时）—— PPL 就是每 token 交叉熵损失的指数形式。

### 5.3 为什么取指数？

直觉：交叉熵是 "平均多花几 nat"，指数化后变成 "等效多少种可能性"。

数学：$H = \log |S|$ 是 $|S|$ 个等概率选择的熵。反过来 $|S| = e^H$ 就是"等效多少种选择"。

### 5.4 代码：用 Hugging Face 模型计算 PPL

```python
import torch
from transformers import GPT2LMHeadModel, GPT2TokenizerFast

device = "cuda" if torch.cuda.is_available() else "cpu"
model_id = "gpt2"
model = GPT2LMHeadModel.from_pretrained(model_id).to(device)
tokenizer = GPT2TokenizerFast.from_pretrained(model_id)

text = "The quick brown fox jumps over the lazy dog."
encodings = tokenizer(text, return_tensors="pt").to(device)

with torch.no_grad():
    outputs = model(**encodings, labels=encodings["input_ids"])
    # 模型已经算了交叉熵损失
    neg_log_likelihood = outputs.loss

ppl = torch.exp(neg_log_likelihood)
print(f"PPL = {ppl.item():.2f}")
```

### 5.5 BPB / BPC：大模型时代的新指标

PPL 受词表大小影响（词表越大 PPL 天然越大），所以跨模型比较时用 **bits per byte（BPB）** 或 **bits per character（BPC）** 更公平：

$$
\text{BPB} = \frac{\text{Total NLL (nats)}}{\text{Total bytes}} / \ln 2
$$

DeepMind 的 Chinchilla、LLaMA 的论文都用 BPB 做主评估指标。

## 六、常见陷阱

1. **Log 底数要一致**：一篇论文里混用 $\log_2$ 和 $\log_e$ 会让熵值差一个 $\ln 2 \approx 0.693$ 的因子。
2. **概率为 0 的项**：$0 \cdot \log 0$ 数学上约定为 0（但 Python 直接算 `np.log(0)` 会报 `-inf`）。实践中加个 $\epsilon$ 或先过滤 $p = 0$ 项。
3. **KL 不对称**：写论文和代码时一定标清楚 $D_{KL}(P \| Q)$ 还是 $D_{KL}(Q \| P)$。
4. **PyTorch `kl_div` 的参数顺序很坑**：`F.kl_div(input, target)` 要求 `input` 是 **log-prob**，`target` 是 **prob**。计算的是 $D_{KL}(\text{target} \| \exp(\text{input}))$。这是新手踩坑高频点。
5. **CrossEntropy vs NLL**：`nn.CrossEntropyLoss` 接收未归一化 logits 自己做 log_softmax；`nn.NLLLoss` 接收 log_softmax 后的结果。**千万不要再对 logits 外面手动 softmax**。
6. **PPL 不能跨 tokenizer 比较**：同一段文本，BPE 切成 100 token 和切成 120 token，PPL 完全不同。比较模型时要统一 tokenizer 或改用 BPB。
7. **温度采样里的熵**：`T > 1` 熵增、`T < 1` 熵减。temperature 的本质就是在调节输出分布的熵。

## 七、面试常见问题

这里把前面那些公式翻成**通俗口水话**，面试时能说清楚就行。

**Q1：什么是熵？一句话解释。**

熵衡量一个概率分布有多"混乱"、多"不确定"。越均匀熵越大，越集中熵越小。完全确定的事（概率 1）熵是 0。

**Q2：交叉熵和 KL 散度有什么区别？**

**KL 散度 = 交叉熵 − 真实分布的熵**。
因为真实分布一般是固定的（训练集摆那儿不变），它的熵是常数，所以**训练时优化交叉熵 ≡ 优化 KL 散度**，结果完全一样。工程上直接写 CE 计算更简单。

**Q3：为什么分类和语言模型都用交叉熵做损失？**

因为**最小化交叉熵 = 最大化似然**。模型在做的事就是让"训练数据在模型下出现的概率"最大。这在统计学里是几百年来最标准的参数估计方法（MLE）。

**Q4：KL 散度为什么不是距离？**

两个原因：
- **不对称**：从 A 到 B 和从 B 到 A 数值不同。
- **不满足三角不等式**：不能保证 $d(A, C) \leq d(A, B) + d(B, C)$。

它只是"分布差异"的一个度量，但数学上不是严格的距离。

**Q5：Perplexity 是什么？2 的 PPL 次方有什么直观含义？**

Perplexity 是交叉熵的指数形式。直观理解就是**模型每预测下一个词时，等效在多少个词之间犹豫**。PPL=1 表示完全确定，PPL=30 表示"心里在 30 个词之间打架"。

**Q6：为什么 GPT 的 PPL 越低模型越好？**

PPL = $e^{\text{平均每 token 的 loss}}$。Loss 小 → PPL 小 → 模型对真实文本分配的概率更高 → 模型更贴近真实语言分布。

**Q7：RLHF 里为什么要加 KL 惩罚？**

防止模型为了讨好奖励模型而"放飞自我"（reward hacking）。KL 散度衡量新策略和旧策略的差异，加个 β 系数把它限制住，模型就不敢乱跑。β 就是"牵绳"的紧度。

**Q8：Forward KL 和 Reverse KL 有什么区别？**

- Forward $D_{KL}(P \| Q)$："Q 必须在 P 有概率的地方也有概率"，结果分布会偏模糊（平均化）
- Reverse $D_{KL}(Q \| P)$："Q 只要拟合 P 的某一个峰就行"，结果分布会偏锐利（取其一）

VAE 用 Forward（所以生成图常模糊），PPO 用 Reverse（所以守旧行为）。

**Q9：互信息在大模型里有什么用？**

衡量"知道 X 能减少多少 Y 的不确定性"。CLIP 和 SimCLR 等对比学习的损失函数（InfoNCE）本质就是最大化图像 embedding 和文本 embedding 的互信息。

**Q10：分类时标签平滑（Label Smoothing）是什么？它和熵什么关系？**

正常交叉熵里真实标签是 one-hot $(1,0,0,\dots)$，对应熵为 0。标签平滑把真实分布换成 $(1-\epsilon, \epsilon/(K-1), \dots)$ —— 真实分布不再是"板上钉钉"，它的熵不再是 0，从而防止模型对自己太自信。这在大模型预训练里很常用。
