## 零、🎯 开篇：用费曼视角理解"概率"本身

**费曼式大白话**：概率就是"这件事我有多确定"的数字化表达。

日常我们说"明天大概会下雨"、"这题我 80% 把握选 B"、"这人八成是骗子"——这些模糊说法背后都有一个 0 到 1 之间的数在隐约翻动。概率论就是把这种"大概"、"八成"掰开揉碎，做成可以计算、可以组合、可以更新的数学语言。

**为什么大模型本质是一个概率机器？** 因为语言本身就是不确定的。给你"今天天气真"这四个字，下一个字可能是"好"、"不错"、"冷"、"差"……GPT 做的事情不是"告诉你下一个字是什么"，而是**给词表里每一个可能的 token 打一个概率分数**，然后从这个分数分布里采样一个出来。

所以你会在大模型里反复看到：概率分布、期望、方差、KL 散度、采样、似然——它们不是装饰品，它们就是大模型本身。

## 一、随机变量与概率分布

### 1.1 🧠 费曼式理解

**想象一台自动贩卖机，你投币后它随机吐出一瓶饮料**：可乐 40% 概率、雪碧 30%、橙汁 20%、矿泉水 10%。这里"吐出的饮料"就是一个**随机变量**，那张"每种饮料各有多少概率"的表就是**概率分布**。

把"饮料"换成"下一个 token"，把"10 种饮料"换成"词表里 15 万个候选词"，你就得到了语言模型。

### 1.2 定义与分类

**随机变量（Random Variable）**：把某个随机现象的每个可能结果映射成一个数值的函数。通常用大写字母 $X$ 表示，取值用小写 $x$ 表示。

- **离散型**：取值可数。例：掷骰子 $X\in\{1,2,3,4,5,6\}$、语言模型下一个 token $X\in \{词表\}$
- **连续型**：取值在实数区间。例：人的身高、神经网络激活值

**概率质量函数（PMF）**（离散）：$P(X = x)$，满足 $\sum_x P(X=x) = 1$。

**概率密度函数（PDF）**（连续）：$p(x)$，满足 $\int p(x)dx = 1$。注意密度 $p(x)$ **不是概率**，只有积分才是概率。

**累积分布函数（CDF）**：$F(x) = P(X \leq x)$，单调递增，从 0 到 1。

### 1.3 大模型中 7 个最常见的分布

| 分布 | 公式 | 大模型里的用途 |
|------|------|---------------|
| 伯努利 Bernoulli | $P(X=1)=p,\ P(X=0)=1-p$ | 二分类（判别模型 yes/no） |
| 类别 Categorical | $P(X=i)=p_i,\ \sum p_i=1$ | **语言模型下一个 token 的分布** |
| 二项 Binomial | $\binom{n}{k}p^k(1-p)^{n-k}$ | A/B 测试、成功次数 |
| 高斯 Normal | $\frac{1}{\sqrt{2\pi}\sigma}e^{-\frac{(x-\mu)^2}{2\sigma^2}}$ | **权重初始化、噪声建模、VAE** |
| 均匀 Uniform | $\frac{1}{b-a}$（$x\in[a,b]$） | Dropout 掩码、初始化 |
| 多元高斯 Multivariate Normal | $\mathcal{N}(\mu,\Sigma)$ | 词向量、扩散模型噪声 |
| 拉普拉斯 Laplace | $\frac{1}{2b}e^{-\frac{|x-\mu|}{b}}$ | L1 正则的贝叶斯解释、差分隐私 |

**语言模型最核心的分布就是 Categorical**——Transformer 最后一层 logits 经过 softmax 就是一个 Categorical 分布。

### 1.4 期望与方差

**期望 $E[X]$**（这个分布"平均会给出什么值"）：
- 离散：$E[X] = \sum_x x \cdot P(X=x)$
- 连续：$E[X] = \int x \cdot p(x) dx$

**方差 $\text{Var}(X)$**（这个分布"有多离散"）：
$$
\text{Var}(X) = E[(X - E[X])^2] = E[X^2] - (E[X])^2
$$

**关键性质**：
- $E[aX+b] = aE[X]+b$
- $E[X+Y] = E[X]+E[Y]$（不需独立）
- $\text{Var}(aX+b) = a^2\text{Var}(X)$
- 若 $X \perp Y$（独立）：$\text{Var}(X+Y)=\text{Var}(X)+\text{Var}(Y)$

**大模型应用**：Transformer 初始化时，权重按 $\mathcal{N}(0, \sigma^2)$ 采样，$\sigma$ 要精心选择以保证前向传播各层输出的方差稳定不爆炸（Xavier / He 初始化的来源）。Attention 里除以 $\sqrt{d_k}$ 也是方差控制：$QK^T$ 的方差随 $d_k$ 线性增长，除以 $\sqrt{d_k}$ 把方差拉回常数。

## 二、联合、边缘、条件概率

### 2.1 🧠 费曼式理解

**把两个骰子并排放着，一个红色一个蓝色**。问：
- 红 3 且蓝 5 的概率？→ **联合概率** $P(R=3, B=5)$
- 不管蓝色多少，红色是 3 的概率？→ **边缘概率** $P(R=3)$（把蓝色的所有可能加起来）
- 已知蓝色是 5 的前提下，红色是 3 的概率？→ **条件概率** $P(R=3 \mid B=5)$

这三个概念看着朴素，但是贝叶斯公式、语言模型、RLHF 全部建立在它们之上。

### 2.2 定义

| 概念 | 公式 |
|------|------|
| 联合概率 | $P(X, Y)$ |
| 边缘概率 | $P(X) = \sum_y P(X, Y=y)$（积分替换求和则为连续情形） |
| 条件概率 | $P(X\mid Y) = \frac{P(X, Y)}{P(Y)}$ |
| 链式法则 | $P(X_1, X_2, \dots, X_n) = \prod_{i=1}^n P(X_i \mid X_1, \dots, X_{i-1})$ |
| 独立 | $X \perp Y \Leftrightarrow P(X, Y) = P(X)P(Y)$ |

### 2.3 语言模型就是链式法则的直接应用

一段文本 $w_1 w_2 \dots w_n$ 出现的概率可以分解为：

$$
P(w_1, w_2, \dots, w_n) = \prod_{i=1}^{n} P(w_i \mid w_1, \dots, w_{i-1})
$$

自回归语言模型（GPT、LLaMA、Claude）的训练目标就是**尽可能准确地估计每一项条件概率** $P(w_i \mid w_{<i})$。Transformer 的每个前向就输出这样一个分布。

**训练：** 把大量文本喂进去，让模型预测每个位置的"下一个词是什么"（给出一个 Categorical 分布），用交叉熵做损失。
**推理：** 从当前分布采样一个词，拼到输入后面，再进模型算下一个分布，循环往复。

## 三、贝叶斯定理

### 3.1 🧠 费曼式理解

**贝叶斯公式是"根据新证据更新信念"的数学规则**。

举个例子：一种病的总人群发病率是 1%（**先验**）。你做了一个准确率 99% 的检测，结果是阳性（**证据**）。你是不是真的得病了（**后验**）？

直觉会让你以为 "99% 肯定得了"，但贝叶斯公式会告诉你真相：**只有 50% 左右**。因为基数少（发病率低），假阳性反而贡献了大量阳性结果。

把"是否得病"换成"哪个参数值最可能产生观察到的数据"，你就得到了机器学习里的参数估计。把"观察到的检测结果"换成"用户看到的回答"，你就得到了对齐问题的贝叶斯视角。

### 3.2 公式

$$
\underbrace{P(\theta \mid D)}_{\text{后验 posterior}} = \frac{\overbrace{P(D \mid \theta)}^{\text{似然 likelihood}} \cdot \overbrace{P(\theta)}^{\text{先验 prior}}}{\underbrace{P(D)}_{\text{证据 evidence}}}
$$

其中 $P(D) = \int P(D\mid \theta)P(\theta) d\theta$ 是归一化常数。实际计算时常常写作：

$$
P(\theta \mid D) \propto P(D \mid \theta) P(\theta)
$$

（"后验 ∝ 似然 × 先验"，七个字打死记住）

### 3.3 Python 代码：癌症检测的贝叶斯计算

```python
# 场景：1% 的人得了某种病；检测敏感性 99%（得病者阳性率），
# 特异性 99%（未得病者阴性率）。某人检测阳性，真的得病概率是多少？

P_disease = 0.01          # 先验：得病率
P_pos_given_disease = 0.99  # 似然：得病时阳性率
P_pos_given_healthy = 0.01  # 假阳性率

# 证据：整体阳性率
P_pos = P_pos_given_disease * P_disease + P_pos_given_healthy * (1 - P_disease)

# 后验：阳性时真得病的概率
P_disease_given_pos = (P_pos_given_disease * P_disease) / P_pos
print(f"阳性检测结果下真得病概率: {P_disease_given_pos:.2%}")
# 输出：阳性检测结果下真得病概率: 50.00%
```

结果反直觉，这就是为什么所有医学检测都要求"阳性后再复检"。

## 四、最大似然估计（MLE）与最大后验估计（MAP）

### 4.1 🧠 费曼式理解

**假设你看着一袋红白球，不知道红球占比**。你随机抓出 10 个，有 7 个红、3 个白。**直觉**会告诉你红球占比是 70%——这个直觉就是 MLE 在做的事：**选那个让"观察到的数据"最可能发生的参数**。

**MAP 是 MLE 的加强版**：你不光看抓到的球，还把你对这袋球的**先验知识**也考虑进来。比如你之前摸过几次这袋球，觉得红球本来就更多——这个"本来就更多"就是先验，MAP 会把它和这次的数据一起算。

**为什么大模型要关心这个？** 因为**最小化交叉熵损失 ≡ 最大似然估计**。你训练模型时每一次梯度更新，本质都是在把模型参数推向 MLE 解。L2 正则（权重衰减）则让它变成 MAP。

### 4.2 MLE 数学形式

给定数据集 $D = \{x_1, \dots, x_n\}$，假设它们独立同分布（i.i.d.），由参数 $\theta$ 的分布产生。**似然函数**：

$$
L(\theta) = P(D \mid \theta) = \prod_{i=1}^{n} P(x_i \mid \theta)
$$

为了避免连乘下溢和方便求导，取对数得到**对数似然**：

$$
\log L(\theta) = \sum_{i=1}^{n} \log P(x_i \mid \theta)
$$

MLE 就是：
$$
\hat\theta_{\text{MLE}} = \arg\max_\theta \sum_{i=1}^n \log P(x_i \mid \theta)
$$

### 4.3 关键顿悟：MLE 就是在最小化负对数似然（NLL）

$$
\hat\theta_{\text{MLE}} = \arg\min_\theta \left[-\sum_{i=1}^n \log P(x_i \mid \theta)\right]
$$

这个 $-\sum \log P$ 就是**负对数似然（NLL, Negative Log-Likelihood）**，也就是 PyTorch 里 `nn.NLLLoss` 和 `nn.CrossEntropyLoss` 实际计算的东西。

**更进一步**：对于语言模型，训练损失就是：

$$
\mathcal{L} = -\sum_{i=1}^{L} \log P(w_i \mid w_{<i}; \theta)
$$

这行公式解释了一切：
- 为什么损失叫"交叉熵" → 因为它等价于真实分布（one-hot）和模型分布的交叉熵
- 为什么说"语言模型困惑度 = $e^{\text{loss}}$" → 因为 perplexity = $\exp(\text{NLL per token})$
- 为什么 scaling law 里损失稳定下降 → 因为模型在越来越接近真实数据分布

### 4.4 MAP：MLE 加上先验

$$
\hat\theta_{\text{MAP}} = \arg\max_\theta \left[\log P(D \mid \theta) + \log P(\theta)\right]
$$

**L2 正则的贝叶斯解释**：如果先验 $P(\theta) = \mathcal{N}(0, \sigma^2 I)$（高斯先验），则 $\log P(\theta) = -\frac{\|\theta\|^2}{2\sigma^2} + \text{const}$，MAP 就等价于 MLE + L2 正则化。所以你加 `weight_decay=0.01` 背后的数学含义是"假设权重本来服从均值为 0 的高斯先验"。

**L1 正则的贝叶斯解释**：拉普拉斯先验 → MAP 等价于 MLE + L1 正则（产生稀疏解）。

### 4.5 PyTorch 代码：从零实现 MLE 训练逻辑回归

```python
import torch
import torch.nn as nn

# 生成数据
torch.manual_seed(0)
n, d = 1000, 10
X = torch.randn(n, d)
true_w = torch.randn(d)
y = (X @ true_w + 0.1 * torch.randn(n) > 0).float()  # 二分类标签

# 模型
w = torch.zeros(d, requires_grad=True)
optimizer = torch.optim.Adam([w], lr=0.1)

# MLE：最大化 sum(log P(y_i | x_i, w))
# 伯努利似然：P(y|x,w) = sigmoid(x·w)^y * (1-sigmoid(x·w))^(1-y)
# 负对数似然（等价于 binary cross-entropy）：
for step in range(200):
    logits = X @ w
    # 标准稳定写法（避免 log(0)）
    nll = nn.functional.binary_cross_entropy_with_logits(logits, y)
    
    optimizer.zero_grad()
    nll.backward()
    optimizer.step()
    
    if step % 50 == 0:
        print(f"Step {step}  NLL={nll.item():.4f}")

# 加 L2 正则 = MAP with 高斯先验
# 只需在 loss 上加：+ 0.01 * (w ** 2).sum()
```

## 五、采样（Sampling）：把概率分布变成具体输出

### 5.1 🧠 费曼式理解

**假设模型算出来"下一个词"的概率分布是：好(40%)、不错(25%)、冷(15%)、差(10%)、……**。

- 你每次都选概率最大的"好"？这叫 **贪心（Greedy）**，说出来的话死板。
- 你按概率随机抽？这叫 **多项采样（Multinomial）**，生动但可能抽到烂尾词。
- 你只从 Top-3 里抽？这叫 **Top-k 采样**。
- 你把分布先"拉平"或"拉尖"再抽？这叫 **温度（Temperature）调节**。
- 你只从"累计概率 90%"的那批词里抽？这叫 **Top-p / Nucleus 采样**。

这几个东西合在一起，决定了同一个 LLM 说出来的话是机器腔还是灵气十足。所有 LLM 服务的 `temperature`、`top_p` 参数就是这些。

### 5.2 采样策略一览

给定分布 $P(x_i)$，常见策略：

**1. 贪心（Greedy / Argmax）**：$x^* = \arg\max_i P(x_i)$。
- 优点：确定性
- 缺点：重复、无聊、容易进入循环

**2. 温度采样（Temperature）**：先把 logits 除以 $T$，再 softmax。

$$
P_T(x_i) = \frac{\exp(\text{logit}_i / T)}{\sum_j \exp(\text{logit}_j / T)}
$$

- $T = 1$：原分布
- $T < 1$：分布变尖（更确定，近贪心）
- $T > 1$：分布变平（更随机，更创意）
- $T \to 0$：等价于贪心

**3. Top-k 采样**：只保留概率最大的 $k$ 个 token，重新归一化后采样。

**4. Top-p / Nucleus 采样**：按概率降序，累加到 $p$（如 0.9），只从这批"核心"token 采样。相比 Top-k，它在分布尖锐时自动保守、平坦时自动多样。

**5. Beam Search**：同时保留多条候选序列，适合翻译这种确定性任务，不适合聊天。

### 5.3 代码：五种采样策略的对比实现

```python
import torch
import torch.nn.functional as F

logits = torch.tensor([3.0, 2.5, 2.0, 1.5, 1.0, 0.5, 0.0, -0.5, -1.0, -1.5])

def greedy(logits):
    return logits.argmax().item()

def temperature_sample(logits, T=1.0):
    probs = F.softmax(logits / T, dim=-1)
    return torch.multinomial(probs, 1).item()

def top_k_sample(logits, k=3, T=1.0):
    values, indices = torch.topk(logits, k)
    probs = F.softmax(values / T, dim=-1)
    idx = torch.multinomial(probs, 1).item()
    return indices[idx].item()

def top_p_sample(logits, p=0.9, T=1.0):
    probs = F.softmax(logits / T, dim=-1)
    sorted_probs, sorted_idx = torch.sort(probs, descending=True)
    cum = torch.cumsum(sorted_probs, dim=-1)
    mask = cum <= p
    mask[0] = True  # 至少保留第一个
    keep_probs = sorted_probs[mask]
    keep_idx = sorted_idx[mask]
    keep_probs = keep_probs / keep_probs.sum()
    choice = torch.multinomial(keep_probs, 1).item()
    return keep_idx[choice].item()

# 对比 10 次
for name, fn in [("greedy", greedy),
                 ("T=1.0",  lambda l: temperature_sample(l, 1.0)),
                 ("T=0.5",  lambda l: temperature_sample(l, 0.5)),
                 ("top-k=3",lambda l: top_k_sample(l, 3)),
                 ("top-p=0.9", lambda l: top_p_sample(l, 0.9))]:
    samples = [fn(logits) for _ in range(20)]
    print(f"{name:<10} → {samples}")
```

## 六、大数定律、中心极限定理与蒙特卡洛

### 6.1 🧠 费曼式理解

**大数定律**：抛硬币抛得足够多，正面比例会无限接近 50%。说白了就是"样本越多，越接近真实"。

**中心极限定理**：**不管原始数据什么分布**，只要你每次抽一组求平均，这些"组平均"的分布会趋向高斯分布。这也是为什么高斯分布在自然界那么普遍——很多现象都是大量独立微小因素的叠加。

**蒙特卡洛**：数学里很多积分算不出来解析解。于是就**多采样、平均**，用"大数定律"来近似。RLHF 里计算策略梯度的期望、扩散模型训练里的去噪期望——全是蒙特卡洛。

### 6.2 公式形式

**大数定律（LLN）**：$\frac{1}{n}\sum_{i=1}^n X_i \xrightarrow{n\to\infty} E[X]$

**中心极限定理（CLT）**：$\sqrt{n}\left(\frac{1}{n}\sum X_i - \mu\right) \xrightarrow{d} \mathcal{N}(0, \sigma^2)$

**蒙特卡洛估计**：
$$
E_{x\sim p}[f(x)] \approx \frac{1}{N}\sum_{i=1}^{N} f(x_i),\ \ x_i \sim p
$$

**RLHF 里的应用**：PPO 的目标函数是 $E_{\tau \sim \pi_\theta}[R(\tau)]$，这个期望无法直接算，就用 Rollout 采样很多轨迹再平均——这就是蒙特卡洛。

## 七、常见陷阱与面试高频点

1. **频率派 vs 贝叶斯派**：前者认为参数 $\theta$ 是固定未知值（用 MLE 估计一个点），后者认为 $\theta$ 本身是随机变量（估计整个分布）。深度学习 99% 是频率派。
2. **概率密度不是概率**：连续分布 $p(x) > 1$ 合法（比如很窄的高斯）。只有 $\int p(x)dx$ 才是概率。
3. **MLE 一定是无偏估计吗？不一定**。经典反例：高斯分布方差的 MLE 估计 $\hat\sigma^2 = \frac{1}{n}\sum(x_i-\bar x)^2$ 是**有偏**的，无偏估计要除以 $n-1$。
4. **NLL 和 Cross-Entropy 的关系**：离散分类场景下，当真实标签是 one-hot 时，二者数值完全相等。PyTorch 里 `NLLLoss` 要手动先 log_softmax，`CrossEntropyLoss` 内部自己做 log_softmax。**不要重复 softmax**。
5. **Logits 溢出**：softmax 直接算 `exp(logit)` 容易溢出。实现要先减最大值：`logits = logits - logits.max()`。所有成熟框架都做了这个技巧。
6. **采样 ≠ argmax**：训练时模型输出概率，推理时需要一个采样策略才能得到具体 token。同一个模型配不同 temperature 可能一个像"机器人"、一个像"疯子"。
