## 零、开篇：一个头够用吗？

上一章我们实现了"单头"Self-Attention。它已经很强——每个词都能关注序列里任意其他词。

但有一个问题：**一次注意力计算，只能学到一种"关注方式"**。

想一想这句话：

> "河岸边的银行职员望着对岸的银行"

"银行"这个词在这里出现了两次，但含义完全不同：第一个是金融机构，第二个是河堤。

单头注意力在处理第一个"银行"时，只能用一套 Q/K/V 来决定关注什么。但理解这句话，同时需要：
- **从语法角度**：这个"银行"是主语还是宾语？
- **从语义角度**：周围的词（"河岸"、"职员"、"对岸"）怎么帮助消歧？
- **从指代角度**："对岸的银行"和前面的"银行"是同一个词吗？

这三个问题，用三种不同的"视角"才能同时回答。

**Multi-Head Attention（多头注意力）就是同时开多个"视角"**——每个"头"学习一种不同的关注方式，最后把所有头的结果汇总。

## 一、多头注意力的直觉

### 1.1 初印象：用费曼视角理解

**想象你要分析一幅画，请了四位评论家同时看**：

- 第一位评论家专注**构图**：这幅画的结构平衡吗，重心在哪里？
- 第二位评论家专注**色彩**：颜色如何搭配，冷暖对比怎样？
- 第三位评论家专注**情绪**：这幅画传递了什么感受？
- 第四位评论家专注**技法**：笔触细节，材质表现？

四位评论家分别给出自己的分析报告，最后汇总成一份完整的评价。

**这就是多头注意力的工作方式**：

- 每个"头"是一位评论家，有自己独立的 $W_Q^i, W_K^i, W_V^i$ 矩阵
- 每个头独立计算注意力，从不同角度"看"同一句话
- 最后把所有头的输出拼接在一起，再做一次线性变换，得到综合结果

### 1.2 单头 vs 多头的对比

```
单头注意力（上一章）：
  输入 X（形状：L × d_model）
      ↓  W_Q、W_K、W_V（各 d_model × d_model）
  Q、K、V（各 L × d_model）
      ↓ Attention
  输出（L × d_model）

多头注意力（本章）：
  输入 X（形状：L × d_model）
      ↓  分别投影到 h 个头
  头1: Q1,K1,V1（各 L × d_k）→ 输出1（L × d_k）
  头2: Q2,K2,V2（各 L × d_k）→ 输出2（L × d_k）
  头3: Q3,K3,V3（各 L × d_k）→ 输出3（L × d_k）
  ...
  头h: Qh,Kh,Vh（各 L × d_k）→ 输出h（L × d_k）
      ↓  拼接所有头的输出
  拼接结果（L × (h × d_k) = L × d_model）
      ↓  输出投影矩阵 W_O（d_model × d_model）
  最终输出（L × d_model）
```

## 二、维度是怎么分配的

### 2.1 头数和维度的关系

多头注意力有一个关键约定：**所有头的维度加起来等于原始模型维度**。

$$d_k = \frac{d_{\text{model}}}{\text{头数 } h}$$

例如：
- LLaMA-3-8B：$d_{\text{model}} = 4096$，头数 $h = 32$，每头 $d_k = 128$
- GPT-2：$d_{\text{model}} = 768$，头数 $h = 12$，每头 $d_k = 64$
- 一个小的演示模型：$d_{\text{model}} = 64$，头数 $h = 4$，每头 $d_k = 16$

这样设计的好处是：多头注意力的**总参数量和单头注意力完全相同**——只是把同样的参数"切分"给了多个头，每个头负责处理更低维度的信息。

```
单头：W_Q 是 (512 × 512)，参数量 = 512 × 512 = 262,144
8头：每头 W_Q 是 (512 × 64)，8个头共 8 × 512 × 64 = 262,144  ← 一样！
```

### 2.2 为什么多头效果更好，但参数量不变？

**关键在于并行性和多样性**：

多头让每个头只关注低维特征空间的一个子集，反而更容易学到专注而清晰的注意力模式——就像每位专家专注自己领域，比一个人什么都想管更有效。

实验表明，多头注意力学到的不同头确实有不同的功能：有的头专注局部依赖（相邻词），有的头专注句法关系，有的头专注语义关系，有的头专注指代消解。

## 三、代码实现：Multi-Head Attention

### 3.1 完整实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    """
    多头注意力机制（Causal，用于语言生成）。
    实现方式：用一个大矩阵同时投影所有头，再 reshape 分头，效率更高。
    """

    def __init__(self, d_model: int, n_heads: int, dropout: float = 0.0):
        super().__init__()
        assert d_model % n_heads == 0, "d_model 必须能被 n_heads 整除"

        self.d_model  = d_model
        self.n_heads  = n_heads
        self.d_k      = d_model // n_heads   # 每个头的维度

        # 用单个大矩阵同时生成所有头的 Q、K、V（效率更高）
        # 等价于 h 个独立的 W_Q、W_K、W_V 拼在一起
        self.W_Q = nn.Linear(d_model, d_model, bias=False)   # 投影到所有头的 Q
        self.W_K = nn.Linear(d_model, d_model, bias=False)   # 投影到所有头的 K
        self.W_V = nn.Linear(d_model, d_model, bias=False)   # 投影到所有头的 V
        self.W_O = nn.Linear(d_model, d_model, bias=False)   # 输出投影

        self.dropout = nn.Dropout(dropout)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B, L, D = x.shape   # batch_size, seq_len, d_model

        # ── 第一步：生成 Q、K、V ──
        Q = self.W_Q(x)   # (B, L, d_model)
        K = self.W_K(x)   # (B, L, d_model)
        V = self.W_V(x)   # (B, L, d_model)

        # ── 第二步：reshape 成多头形式 ──
        # 把 d_model 维度拆分成 (n_heads, d_k)
        # (B, L, d_model) → (B, n_heads, L, d_k)
        def split_heads(tensor):
            return tensor.view(B, L, self.n_heads, self.d_k).transpose(1, 2)

        Q = split_heads(Q)   # (B, n_heads, L, d_k)
        K = split_heads(K)   # (B, n_heads, L, d_k)
        V = split_heads(V)   # (B, n_heads, L, d_k)

        # ── 第三步：计算每个头的注意力分数 ──
        # (B, n_heads, L, d_k) @ (B, n_heads, d_k, L) → (B, n_heads, L, L)
        scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_k)

        # ── 第四步：因果掩码（生成任务必须）──
        causal_mask = torch.tril(torch.ones(L, L, device=x.device)).bool()
        scores = scores.masked_fill(~causal_mask, float('-inf'))

        # ── 第五步：Softmax + 加权求和 ──
        weights = F.softmax(scores, dim=-1)       # (B, n_heads, L, L)
        weights = self.dropout(weights)
        out = weights @ V                          # (B, n_heads, L, d_k)

        # ── 第六步：合并多头输出 ──
        # (B, n_heads, L, d_k) → (B, L, n_heads × d_k) = (B, L, d_model)
        out = out.transpose(1, 2).contiguous().view(B, L, D)

        # ── 第七步：输出投影 ──
        out = self.W_O(out)   # (B, L, d_model)
        return out


# ── 验证 ──
torch.manual_seed(42)
d_model = 64
n_heads = 4      # 4 个头，每头 d_k = 64/4 = 16
seq_len = 8

mha = MultiHeadAttention(d_model, n_heads)
x   = torch.randn(2, seq_len, d_model)   # batch=2, seq=8, d=64
out = mha(x)
print(f"输入:  {x.shape}")     # (2, 8, 64)
print(f"输出:  {out.shape}")   # (2, 8, 64)  形状不变
print(f"参数量: {sum(p.numel() for p in mha.parameters()):,}")
# 4×64×64（W_Q W_K W_V W_O）= 16,384
```

### 3.2 理解 reshape 的过程（重点）

这一步经常让初学者困惑，画图解释一下：

```
输入 X：                      (B=2, L=8, D=64)
    ↓ W_Q 线性变换
Q：                           (B=2, L=8, D=64)
    ↓ .view(B, L, n_heads, d_k)
Q 重塑：                      (B=2, L=8, n_heads=4, d_k=16)
    ↓ .transpose(1, 2)
Q 转置后：                    (B=2, n_heads=4, L=8, d_k=16)
                                        ↑
                              这样每个头的 Q 就独立了
                              第0个头的 Q：(B=2, L=8, d_k=16)
                              第1个头的 Q：(B=2, L=8, d_k=16)
                              ...
```

这种实现方式（一个大矩阵 + reshape）和"$h$ 个独立小矩阵分别计算"在数学上完全等价，但效率高很多——只需要一次矩阵乘法，而不是 $h$ 次。

## 四、把所有零件组装：完整 Transformer 块

### 4.1 完整结构回顾

现在我们拥有了所有零件：

```
输入 x
  ↓
RMSNorm（或 LayerNorm）   ← 稳定数值
  ↓
Multi-Head Attention      ← 词间信息交流
  ↓
x = x + attn_out          ← 残差连接
  ↓
RMSNorm（或 LayerNorm）   ← 再次稳定
  ↓
Feed-Forward Network      ← 每词独立深度加工
  ↓
x = x + ffn_out           ← 残差连接
  ↓
输出（形状与输入相同）
```

### 4.2 完整 Transformer 块代码

```python
class TransformerBlock(nn.Module):
    """
    一个完整的 Transformer Decoder 块。
    包含：Pre-Norm + Multi-Head Attention + 残差 + Pre-Norm + FFN + 残差
    这就是 GPT / LLaMA 每一层的基本结构。
    """

    def __init__(self, d_model: int, n_heads: int, d_ff: int, dropout: float = 0.0):
        super().__init__()
        # 两个归一化层（Pre-Norm 风格，放在子层前面）
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)

        # Multi-Head Attention
        self.attention = MultiHeadAttention(d_model, n_heads, dropout)

        # Feed-Forward Network
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model),
            nn.Dropout(dropout),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # ── 子层一：注意力 ──
        # Pre-Norm：先归一化，再注意力，再残差
        x = x + self.attention(self.norm1(x))

        # ── 子层二：FFN ──
        # Pre-Norm：先归一化，再 FFN，再残差
        x = x + self.ffn(self.norm2(x))

        return x


# ── 验证 ──
block = TransformerBlock(d_model=64, n_heads=4, d_ff=256)
x     = torch.randn(2, 8, 64)
out   = block(x)
print(f"Transformer 块输入:  {x.shape}")    # (2, 8, 64)
print(f"Transformer 块输出:  {out.shape}")  # (2, 8, 64)  形状不变

total_params = sum(p.numel() for p in block.parameters())
print(f"单层参数量: {total_params:,}")
```

### 4.3 堆叠成完整模型

把 Transformer 块重复 $N$ 次，加上 Embedding 和输出层，就是一个完整的 Decoder-only 语言模型：

```python
class GPTStyleModel(nn.Module):
    """
    GPT / LLaMA 风格的完整 Decoder-only 语言模型。
    （简化版，不含位置编码的高级版本 RoPE，用固定位置编码代替）
    """

    def __init__(
        self,
        vocab_size:  int,
        d_model:     int,
        n_heads:     int,
        n_layers:    int,
        d_ff:        int,
        max_seq_len: int = 2048,
        dropout:     float = 0.0,
    ):
        super().__init__()

        # Token Embedding
        self.embedding = nn.Embedding(vocab_size, d_model)

        # 简单的可学习位置 Embedding（下一章会换成 RoPE）
        self.pos_embedding = nn.Embedding(max_seq_len, d_model)

        # N 个 Transformer 块（核心部分）
        self.blocks = nn.ModuleList([
            TransformerBlock(d_model, n_heads, d_ff, dropout)
            for _ in range(n_layers)
        ])

        # 最后的归一化层
        self.final_norm = nn.LayerNorm(d_model)

        # 输出层（把 d_model 映射到 vocab_size）
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)

        # Weight Tying：输出层权重共享 Embedding 权重（节省参数）
        self.lm_head.weight = self.embedding.weight

        # 初始化权重（std=0.02 是 GPT-2 的经验值）
        self.apply(self._init_weights)

    def _init_weights(self, module):
        if isinstance(module, nn.Linear):
            nn.init.normal_(module.weight, std=0.02)
            if module.bias is not None:
                nn.init.zeros_(module.bias)
        elif isinstance(module, nn.Embedding):
            nn.init.normal_(module.weight, std=0.02)

    def forward(self, token_ids: torch.Tensor, labels: torch.Tensor = None):
        B, L = token_ids.shape

        # 1. Token Embedding + 位置 Embedding
        positions = torch.arange(L, device=token_ids.device)
        x = self.embedding(token_ids) + self.pos_embedding(positions)

        # 2. 逐层处理（每层都做注意力 + FFN）
        for block in self.blocks:
            x = block(x)

        # 3. 最终归一化
        x = self.final_norm(x)

        # 4. 输出层：得到每个位置上每个词的"得分"
        logits = self.lm_head(x)   # (B, L, vocab_size)

        # 5. 如果提供了标签，计算训练损失
        loss = None
        if labels is not None:
            # 语言模型训练：预测下一个 token
            # logits 的第 i 个位置预测第 i+1 个 token
            loss = F.cross_entropy(
                logits[:, :-1, :].contiguous().view(-1, logits.size(-1)),
                labels[:, 1:].contiguous().view(-1),
                ignore_index=-100,
            )
        return logits, loss

    @property
    def n_params(self):
        return sum(p.numel() for p in self.parameters())


# ── 搭建一个小 GPT 并测试 ──
model = GPTStyleModel(
    vocab_size  = 1000,
    d_model     = 128,
    n_heads     = 4,
    n_layers    = 6,
    d_ff        = 512,
    max_seq_len = 256,
)

print(f"模型参数量: {model.n_params/1e6:.2f}M")

# 模拟一个训练 batch
token_ids = torch.randint(0, 1000, (2, 32))   # 2个句子，每句32个token
labels    = token_ids.clone()

logits, loss = model(token_ids, labels)
print(f"输出 logits 形状: {logits.shape}")   # (2, 32, 1000)
print(f"训练 Loss: {loss.item():.4f}")       # 随机初始化约 ln(1000) ≈ 6.9

# ── 简单推理：给定开头，生成接下来的词 ──
@torch.no_grad()
def generate(model, start_ids, max_new_tokens=10, temperature=1.0):
    """最简单的贪心/采样生成"""
    x = start_ids.clone()
    for _ in range(max_new_tokens):
        # 只取最后 max_seq_len 个 token（防止超出位置编码范围）
        x_cond = x[:, -256:]
        logits, _ = model(x_cond)
        # 取最后一个位置的预测，除以温度系数
        next_logits = logits[:, -1, :] / temperature
        # 采样下一个 token
        probs     = F.softmax(next_logits, dim=-1)
        next_token = torch.multinomial(probs, num_samples=1)
        x = torch.cat([x, next_token], dim=1)
    return x

start = torch.randint(0, 1000, (1, 5))   # 5个token的开头
generated = generate(model, start, max_new_tokens=10)
print(f"生成结果（token id）: {generated[0].tolist()}")
```

## 五、参数量分析：哪里用了最多参数

以一个典型的配置（$d_{\text{model}}=512$，$n_{\text{heads}}=8$，$d_{ff}=2048$，$n_{\text{layers}}=6$）为例：

```python
def analyze_params(d_model, n_heads, d_ff, n_layers, vocab_size):
    """分析模型各部分参数量"""
    d_k = d_model // n_heads

    # 每层的参数
    attn_params = 4 * d_model * d_model   # W_Q + W_K + W_V + W_O
    ffn_params  = 2 * d_model * d_ff      # 两个线性层
    norm_params = 2 * 2 * d_model         # 两个 LayerNorm（各有 scale 和 bias）
    per_layer   = attn_params + ffn_params + norm_params

    # 全模型
    transformer = per_layer * n_layers
    embedding   = vocab_size * d_model    # Token Embedding（LM Head 共享）
    pos_emb     = 2048 * d_model          # 位置 Embedding（假设 max_len=2048）
    total       = transformer + embedding + pos_emb

    print(f"\n配置: d_model={d_model}, n_heads={n_heads}, d_ff={d_ff}, n_layers={n_layers}")
    print(f"每层参数量:")
    print(f"  注意力 (W_Q+K+V+O): {attn_params/1e6:.2f}M ({attn_params/per_layer:.1%})")
    print(f"  FFN:                {ffn_params/1e6:.2f}M ({ffn_params/per_layer:.1%})")
    print(f"  LayerNorm:          {norm_params/1e3:.1f}K （很小，可忽略）")
    print(f"全模型参数量:")
    print(f"  Transformer 层:    {transformer/1e6:.2f}M ({transformer/total:.1%})")
    print(f"  Embedding:         {embedding/1e6:.2f}M ({embedding/total:.1%})")
    print(f"  总计:              {total/1e6:.2f}M")

analyze_params(d_model=512, n_heads=8, d_ff=2048, n_layers=6, vocab_size=32000)
```

运行后你会发现：
- **注意力层约占 33%，FFN 约占 67%**（FFN 参数量是注意力的 2 倍）
- 层数越多，Embedding 占比越小，Transformer 层占比越大
- 这就是为什么说"FFN 是模型的记忆仓库"——它用了最多的参数

## 六、注意力头的多样性：每个头在看什么

研究者发现，训练好的 Transformer 中，不同注意力头会自然分化出不同的功能：

```
观察到的一些常见模式（来自对 BERT/GPT 的分析研究）：

头1（局部头）：主要关注相邻的 1-2 个词，捕捉局部语境
头2（句法头）：关注语法相关的词对（主语↔谓语，动词↔宾语）
头3（语义头）：关注语义相关的词对（代词↔其指代对象）
头4（位置头）：主要关注固定的相对位置（如前一个词、后一个词）
...
```

**这种自然分化不是人为设计的**，是模型在训练过程中自己学出来的——正因为多头机制给了每个头"各自为政"的空间，才能产生这种专业化。

## 七、本章内容与前三章的关系

读完本章，你已经完整理解了 Transformer 的核心组件。让我们把 3.1-3.4 串起来：

```
3.1 【为什么需要 Transformer？】
    RNN 问题（串行、梯度消失）→ 注意力机制 → Transformer
         ↓
3.2 【基础组件是什么？】
    Embedding + 位置编码 + 残差连接 + LayerNorm + FFN
         ↓
3.3 【核心：注意力怎么工作？】
    Q/K/V 的直觉 → 数学公式 → 因果掩码 → 单头实现
         ↓
3.4 【多头 + 组装】
    多头的动机 → 维度分配 → MHA 实现 → 完整 Transformer 块 → 完整语言模型
```

**现在你手里已经有了一个完整的、可运行的微型语言模型**。它结构上和 GPT-2 完全一样，只是参数量更小。

## 八、面试常见问题

**Q1：为什么需要多头注意力？单头不够用吗？**

单头注意力只能学到一种"关注方式"。语言理解需要同时捕捉多种关系：语法关系（主谓宾）、语义关系（代词指代）、局部关系（相邻词依赖）等。多头注意力让每个头独立学习一种模式，综合多个视角，表达能力远超单头。关键是：多头和单头的总参数量相同，只是把参数分配给了多个专注不同方向的"子注意力"。

**Q2：多头注意力的维度是怎么分配的？**

每个头的维度 $d_k = d_{\text{model}} / h$，其中 $h$ 是头数。比如 $d_{\text{model}}=512$、$h=8$ 时，每头 $d_k=64$。这样所有头的参数量之和等于单头的参数量（$h \times 512 \times 64 = 512 \times 512$），即多头注意力不增加额外参数，只改变了信息处理方式。

**Q3：注意力层和 FFN 层各有什么作用？**

注意力层负责**词间信息交流**：让每个词感知整个序列的上下文，融合来自其他词的信息。FFN 层负责**每词独立加工**：在词已经"看过"上下文之后，对每个词的表示做进一步的非线性变换，提取更复杂的特征。两者分工明确，缺一不可。

**Q4：为什么 FFN 的参数量比注意力多？**

注意力的参数量是 $4d^2$（W_Q + W_K + W_V + W_O），FFN 的参数量是 $2 \times d \times 4d = 8d^2$（两个线性层，扩展比 4）。FFN 是注意力参数量的 2 倍。研究表明 FFN 相当于模型的"知识存储库"——大量的世界知识、语言规律都存储在 FFN 的权重里，所以参数量更多是有意义的。

**Q5：什么是 Weight Tying（权重绑定）？**

语言模型的输入 Embedding（把词 id 映射成向量）和输出 LM Head（把向量映射回词的概率）在形状上是转置关系。Weight Tying 让它们共享同一组参数，减少了 $V \times d_{\text{model}}$ 个参数（对 vocab=32000, d=4096 的模型来说约 1.3 亿参数）。这不仅节省参数，实验也表明共享权重通常能提升模型性能，因为"理解一个词"和"生成一个词"使用相同的语义空间是自然而然的。

**Q6：Transformer 一层的数据流是什么？**

输入 $x$ → Pre-Norm（LayerNorm 或 RMSNorm）→ Multi-Head Attention → 和原始输入 $x$ 残差相加 → Pre-Norm → Feed-Forward Network → 和中间结果残差相加 → 输出。整个过程中，输入和输出的形状完全相同（$B \times L \times d_{\text{model}}$），可以无缝堆叠 $N$ 层。
