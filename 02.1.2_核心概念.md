## 零、开篇：模型学习的两种失败模式

训练一个机器学习模型，最终会遭遇两种失败：

**失败一：模型太简单，训练集上就跑不好** — 欠拟合（Underfitting），专业名词叫高偏差（High Bias）。

**失败二：模型太复杂，训练集上完美，测试集上一塌糊涂** — 过拟合（Overfitting），专业名词叫高方差（High Variance）。

如何在两者之间找到甜蜜点，这是所有机器学习工程的核心问题之一。

有意思的是：**大模型打破了这个经典框架**。一个 7B 参数的模型，训练数据也是万亿量级，居然既不过拟合也不欠拟合，还能泛化到从未见过的任务。这个反常现象叫"双下降"（Double Descent）。理解这些反直觉的现象，需要先把经典理论学扎实。

## 一、偏差与方差：理解误差的来源

### 1.1 初印象：用费曼视角理解

**想象你在打靶**。

- **偏差（Bias）**：你的枪瞄准有系统性偏移，每一枪都偏向左下方。不管你打多少次，子弹都聚集在错误的地方——这是系统性错误，无法通过多打几次来修正。

- **方差（Variance）**：你的枪抖得厉害，每一枪落点分散到处都是。有时打中，有时打飞，结果随机性很大。

理想情况是：**低偏差（准确）+ 低方差（稳定）**，子弹密集落在靶心。

机器学习里：
- "靶心"是真实的数据规律
- "子弹落点"是模型在不同训练数据集上的预测
- 偏差衡量"模型的期望预测"偏离真实规律多远
- 方差衡量"模型预测"随训练数据变化而波动多大

### 1.2 偏差-方差分解（数学推导）

对回归任务，均方误差（MSE）可以精确分解为三项：

$$\text{MSE} = \underbrace{\left(\mathbb{E}[\hat{f}(x)] - f(x)\right)^2}_{\text{Bias}^2} + \underbrace{\mathbb{E}\left[\left(\hat{f}(x) - \mathbb{E}[\hat{f}(x)]\right)^2\right]}_{\text{Variance}} + \underbrace{\sigma^2_\epsilon}_{\text{不可约噪声}}$$

- $f(x)$：真实函数（我们想学的）
- $\hat{f}(x)$：模型预测（随训练集变化而变化）
- $\mathbb{E}[\hat{f}(x)]$：模型在所有可能训练集上的平均预测
- $\sigma^2_\epsilon$：数据本身的噪声，无论模型多好都无法消除

**关键结论**：

$$\text{泛化误差} = \text{Bias}^2 + \text{Variance} + \text{不可约噪声}$$

即使 $\text{Bias}^2 = 0$，方差项也会让泛化误差大于零。这是过拟合的数学本质。

### 1.3 四种情形的诊断方法

**工程上最实用的诊断工具**：训练误差和验证误差的关系。

| 情形 | 训练误差 | 验证误差 | 结论 | 解决方向 |
|------|---------|---------|------|---------|
| 理想 | 低 | 低（接近训练误差） | 泛化好 | 维持 |
| 高偏差（欠拟合） | 高 | 高 | 模型太简单 | 增大模型、更好特征、减少正则 |
| 高方差（过拟合） | 低 | 高（远高于训练误差） | 模型太复杂 | 增加数据、增强正则、减小模型 |
| 两者皆高 | 高 | 极高 | 同时欠拟合+过拟合 | 先解决偏差 |

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import Ridge
from sklearn.pipeline import Pipeline

np.random.seed(42)
n = 60
X = np.sort(np.random.uniform(0, 1, n))
y = np.sin(2 * np.pi * X) + np.random.randn(n) * 0.3   # 真实函数 + 噪声

X_train, X_val = X[:40].reshape(-1, 1), X[40:].reshape(-1, 1)
y_train, y_val = y[:40], y[40:]

fig, axes = plt.subplots(1, 3, figsize=(13, 4))

for ax, degree, title in zip(axes, [1, 4, 15],
                              ["高偏差（欠拟合）\ndegree=1",
                               "恰当\ndegree=4",
                               "高方差（过拟合）\ndegree=15"]):
    model = Pipeline([
        ("poly", PolynomialFeatures(degree=degree)),
        ("ridge", Ridge(alpha=1e-3)),
    ])
    model.fit(X_train, y_train)

    X_plot = np.linspace(0, 1, 200).reshape(-1, 1)
    y_plot = model.predict(X_plot)

    train_err = np.mean((model.predict(X_train) - y_train) ** 2)
    val_err   = np.mean((model.predict(X_val) - y_val) ** 2)

    ax.scatter(X_train, y_train, s=15, alpha=0.6, label="Train")
    ax.scatter(X_val, y_val, s=15, alpha=0.6, color="orange", label="Val")
    ax.plot(X_plot, np.sin(2 * np.pi * X_plot), "k--", alpha=0.4, label="True")
    ax.plot(X_plot, y_plot, "r-", linewidth=2)
    ax.set_title(f"{title}\nTrain MSE={train_err:.3f}  Val MSE={val_err:.3f}")
    ax.legend(fontsize=8)

plt.tight_layout()
plt.savefig("bias_variance.png", dpi=150, bbox_inches="tight")
```

## 二、过拟合与泛化误差

### 2.1 初印象：用费曼视角理解

**过拟合就是"死记硬背而非理解"**。

期末复习时，有两种学生：
- A 同学：把每道历年题的题号和答案都背下来，原题出现时答得完美，题目稍有变化就完全不会。
- B 同学：真正理解了解题思路，看到从未见过的变形题也能应对。

A 同学的学习方式就是过拟合——把"噪声"（偶然的题目特点）也当成了规律。

**过拟合的根本原因**：模型的表达能力（参数量）远超数据量，导致模型有"余力"去记住训练集的噪声。

### 2.2 泛化误差与 VC 维

**泛化界（Generalization Bound）** 是统计学习理论的核心结论，用来回答"为什么训练集上表现好不等于泛化好"：

$$\text{泛化误差} \leq \text{训练误差} + \sqrt{\frac{d \cdot \log(n/d) + \log(1/\delta)}{n}}$$

其中 $d$ 是模型的 VC 维（衡量模型复杂度的理论量），$n$ 是训练样本数，$\delta$ 是置信度。

**通俗含义**：
- 模型越复杂（$d$ 大）→ 泛化差距可能越大
- 数据越多（$n$ 大）→ 泛化差距越小
- 这个界解释了"为什么增大数据总是有帮助的"

**但这个经典理论对大模型失效了**。7B 参数的 LLaMA，VC 维极大，但泛化却很好——因为训练数据也是万亿量级，而且隐含了数据本身的结构先验。

### 2.3 双下降现象：大模型的反常泛化

传统机器学习理论预测：随着模型复杂度增加，泛化误差先降后升，形成经典 U 型曲线。但 Belkin 等人 2019 年发现了**双下降（Double Descent）**：

```
泛化误差
    |
    |   *        第一下降区        第二下降区
    |    *     (欠拟合→过拟合)  (插值→过参数化泛化)
    |      *  *
    |          * *
    |            插值   * * * * *（继续变好）
    |            阈值
    |_______________________________________________→ 模型复杂度
```

当模型参数量超过"插值阈值"（能完美拟合所有训练数据的最小参数量）后，继续增大模型，泛化误差**反而继续下降**，最终比传统"甜蜜点"还好。

**大模型就在第二下降区**。这是 Scaling Law 有效的理论基础之一，也解释了为什么"更大的模型反而不容易过拟合"这个违反直觉的现象。

## 三、正则化：对抗过拟合的工具箱

### 3.1 初印象：用费曼视角理解

**正则化就是"给模型施加约束，让它不要太自由"**。

没有约束的模型像一个过于聪明的学生——它能记住训练集每一个细节，包括噪声。正则化给它加上一条规则："你的答题策略必须保持简洁，不准太花哨。"这样它就被迫学习真正的规律，而不是死记细节。

### 3.2 L2 正则化（权重衰减）

在损失函数上加参数 L2 范数的惩罚：

$$\mathcal{L}_{\text{L2}} = \mathcal{L}_{\text{原始}} + \frac{\lambda}{2} \|\theta\|^2_2 = \mathcal{L}_{\text{原始}} + \frac{\lambda}{2} \sum_i \theta_i^2$$

**效果**：让参数趋向于 0，但不会变成恰好 0（都是小数），防止模型依赖任何单一特征。

**贝叶斯解释**：等价于参数服从高斯先验 $\theta \sim \mathcal{N}(0, 1/\lambda)$ 的 MAP 估计（见 1.1.2 章）。

**大模型里的 weight_decay**：AdamW 的参数 `weight_decay` 就是 L2 正则系数 $\lambda$，通常取 0.01-0.1：

```python
from torch.optim import AdamW

optimizer = AdamW(
    model.parameters(),
    lr=2e-4,
    weight_decay=0.01,   # L2 正则，λ=0.01
    betas=(0.9, 0.999),
)

# AdamW 和 Adam 的区别：
# Adam 把 L2 惩罚混入梯度，和自适应学习率耦合，效果不好
# AdamW 把 weight decay 从梯度解耦，直接作用于参数：
# θ_t = θ_{t-1}(1 - lr * λ) - lr * adam_update
# 这才是"真正的" L2 正则化
```

### 3.3 L1 正则化（Lasso）

$$\mathcal{L}_{\text{L1}} = \mathcal{L}_{\text{原始}} + \lambda \|\theta\|_1 = \mathcal{L}_{\text{原始}} + \lambda \sum_i |\theta_i|$$

**效果**：产生**稀疏解**，许多参数精确变为 0（特征选择）。

**为什么 L1 产生稀疏而 L2 不行**：
- L2 惩罚梯度正比于参数值，参数越小惩罚越轻，永远不会精确到 0
- L1 惩罚梯度是常数（$\pm\lambda$），在 0 附近的参数会被直接推过 0 点归零

**大模型里 L1 用得少**：因为稀疏性对 Tensor Core 不友好（零散的零不能高效利用矩阵乘法），不如结构化剪枝或量化来得实用。

### 3.4 Dropout

**初印象**：每次前向传播，随机把 $p$ 比例的神经元"关掉"（输出设为 0）。

```python
import torch
import torch.nn as nn

# 在 Transformer 里的典型用法
class TransformerLayer(nn.Module):
    def __init__(self, d_model, n_heads, dropout=0.1):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, n_heads, batch_first=True)
        self.ff   = nn.Sequential(
            nn.Linear(d_model, 4 * d_model),
            nn.GELU(),
            nn.Dropout(dropout),       # FFN 中间层 Dropout
            nn.Linear(4 * d_model, d_model),
        )
        self.attn_drop = nn.Dropout(dropout)  # Attention 输出 Dropout
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)

    def forward(self, x):
        attn_out, _ = self.attn(x, x, x)
        x = self.norm1(x + self.attn_drop(attn_out))
        x = self.norm2(x + self.ff(x))
        return x
```

**为什么 Dropout 有效**（三种解释）：

1. **集成效应**：每次用不同的"子网络"训练，相当于训练了 $2^n$ 个共享参数的模型，推理时做集成
2. **防止共适应**：神经元不能依赖固定的邻居，必须学习鲁棒的独立特征
3. **噪声注入**：为训练添加随机噪声，增大模型对扰动的鲁棒性

**大模型中的 Dropout 趋势**：大模型（LLaMA、GPT-4 规模）预训练时 Dropout 率往往设为 0 或极小（0.0-0.05），因为数据量本身已经足够的正则化效果。微调时才会适当开启。

### 3.5 Early Stopping

**初印象**：监控验证集损失，一旦它停止下降甚至上升，就停止训练——就像考试时看到自己开始乱写就停笔。

```python
class EarlyStopping:
    """监控验证集损失，patience 轮不改善则停止训练"""

    def __init__(self, patience: int = 5, min_delta: float = 1e-4, mode: str = "min"):
        self.patience  = patience
        self.min_delta = min_delta
        self.mode      = mode
        self.best      = float("inf") if mode == "min" else float("-inf")
        self.counter   = 0
        self.should_stop = False

    def step(self, metric: float) -> bool:
        improved = (
            metric < self.best - self.min_delta if self.mode == "min"
            else metric > self.best + self.min_delta
        )
        if improved:
            self.best    = metric
            self.counter = 0
        else:
            self.counter += 1
            if self.counter >= self.patience:
                self.should_stop = True
        return self.should_stop

# 训练循环里的用法
early_stop = EarlyStopping(patience=3, min_delta=1e-3)

for epoch in range(100):
    train_one_epoch(model, train_loader, optimizer)
    val_loss = evaluate(model, val_loader)

    if early_stop.step(val_loss):
        print(f"Early stopping at epoch {epoch + 1}，best val_loss={early_stop.best:.4f}")
        break
```

### 3.6 数据增强：从数据端减少方差

**初印象**：如果真实数据不够多，就用变换来"制造"更多数据——但要保证变换不改变数据的本质标签。

| 数据类型 | 常用增强 | 大模型里的应用 |
|---------|---------|--------------|
| 图像 | 随机裁剪、翻转、颜色抖动 | ViT 预训练、多模态模型 |
| 文本 | 同义词替换、回译、EDA | SFT 数据扩充 |
| 文本（对比学习） | Dropout 作为增强（SimCSE） | 训练句子嵌入模型 |
| 代码 | 变量重命名、等价重写 | 代码模型微调 |

**SimCSE 的极简增强**：同一个句子过两次带 Dropout 的编码器，得到两个不同的表示作为正样本对：

```python
# SimCSE 对比学习：用 Dropout 作为数据增强
# 同一 batch 的句子过两次，Dropout 产生随机不同的表示
def simcse_loss(model, input_ids, attention_mask, temperature=0.05):
    # 同一个 input 过两次，Dropout 让两次结果略不同
    z1 = model(input_ids=input_ids, attention_mask=attention_mask).pooler_output
    z2 = model(input_ids=input_ids, attention_mask=attention_mask).pooler_output

    z1 = F.normalize(z1, dim=-1)
    z2 = F.normalize(z2, dim=-1)

    # z1[i] 和 z2[i] 是正样本对，其余是负样本
    sim = (z1 @ z2.T) / temperature   # (N, N)
    labels = torch.arange(len(z1), device=z1.device)
    loss = (F.cross_entropy(sim, labels) + F.cross_entropy(sim.T, labels)) / 2
    return loss
```

### 3.7 Label Smoothing：标签级别的正则化

**初印象**：与其把目标设成硬性的 one-hot（100% 确定是这个类），不如软化一点：正确类给 $(1-\epsilon)$，其余每类给 $\epsilon / (K-1)$。

```python
# 从信息论角度（见 1.1.4）：硬 one-hot 的熵是 0，模型会过于自信
# Label Smoothing 增加了目标分布的熵，防止模型 overconfident

import torch.nn as nn

# PyTorch 内置
loss_fn = nn.CrossEntropyLoss(label_smoothing=0.1)

# SFT 训练时常用，让模型不要对某个 token 过度确信
```

**大模型里的用法**：大多数 LLM 预训练用 label_smoothing=0 或很小的值（0.0-0.1）。过大的 label smoothing 会让模型生成的文本缺乏确定性。

## 四、交叉验证：可靠的泛化估计

### 4.1 初印象：用费曼视角理解

**交叉验证就是"换着法儿考试"**。

假设你只有 100 道题（样本），把其中 80 道用来训练、20 道用来测试，这 20 道考试题的结果可能很随机（运气好或运气坏）。

交叉验证的做法：把 100 道题分成 5 组，每次用 4 组训练、剩下 1 组测试，共做 5 次，取平均。这样每道题都被当过一次测试题，结果更可靠、方差更小。

### 4.2 K-Fold 交叉验证

```python
from sklearn.model_selection import KFold
import numpy as np

def k_fold_cv(X, y, model_fn, k=5):
    """K-Fold 交叉验证，返回每折的验证误差"""
    kf = KFold(n_splits=k, shuffle=True, random_state=42)
    val_errors = []

    for fold, (train_idx, val_idx) in enumerate(kf.split(X)):
        X_train, X_val = X[train_idx], X[val_idx]
        y_train, y_val = y[train_idx], y[val_idx]

        model = model_fn()
        model.fit(X_train, y_train)
        val_pred = model.predict(X_val)
        val_mse  = np.mean((val_pred - y_val) ** 2)
        val_errors.append(val_mse)
        print(f"  Fold {fold+1}: val_MSE={val_mse:.4f}")

    print(f"\nCV Mean={np.mean(val_errors):.4f}  Std={np.std(val_errors):.4f}")
    return val_errors
```

**大模型里的交叉验证**：因为训练成本极高，大模型很少做完整的 K-Fold CV。取而代之的是：
- 固定的 train/val/test 三分（8:1:1 或 9:0.5:0.5）
- 用多个随机种子重复实验取均值
- 用轻量代理任务（小模型）先做 CV 确定超参，再在大模型上跑

## 五、大模型时代的特殊泛化现象

### 5.1 过参数化模型为何不过拟合

经典理论预测：参数量远超样本量时，模型应该严重过拟合。但大语言模型恰恰相反——7B 参数的模型在万亿 token 上训练，泛化性能很好。

几个可能的解释：

**隐式正则化**：SGD/AdamW 的随机性本身有正则化效果。理论表明，梯度下降在多个全局最优解中倾向于收敛到"最平坦"（泛化最好）的那个。

**数据先验**：自然语言文本本身高度结构化、冗余。模型不需要"记住"每个词，只需学会语言的规律——数据内在的压缩性减小了有效问题维度。

**Early Stopping 效果**：大模型训练时 loss 一直在下降但并没有到真正收敛，这本身就是一种隐式的 Early Stopping。

### 5.2 分布偏移（Distribution Shift）：大模型最大的泛化挑战

**初印象**：模型在训练分布上泛化好，不代表在不同分布上也好。

**三类分布偏移**：

| 类型 | 描述 | 大模型场景示例 |
|------|------|--------------|
| 协变量偏移 | 输入 $P(x)$ 变了，$P(y\|x)$ 没变 | 预训练主要是英文，推理时用中文 |
| 标签偏移 | $P(y)$ 变了 | 训练数据正负例均衡，实际应用高度不均衡 |
| 概念偏移 | $P(y\|x)$ 本身变了 | 时事变化（训练时 A 是总统，推理时 B 是） |

**解决策略**：
- RAG（检索增强）：把最新信息动态加入上下文
- 领域微调：用目标领域数据 SFT
- 持续训练（Continual Learning）：增量更新知识

## 六、常见陷阱

1. **用测试集调超参**：测试集只能用一次，用来评估最终模型。反复用测试集调参等于把它变成了验证集，会导致对泛化误差过于乐观。严格的做法：超参用验证集调，最终结果用测试集报告。

2. **训练误差低不等于过拟合**：大模型训练时训练 loss 往往降到很低（接近 0.5-1.0 nats），但验证 loss 也同样低。是否过拟合要看训练 loss 和验证 loss 的**差距**，不是训练 loss 的绝对值。

3. **正则化系数 $\lambda$ 太大**：L2 正则 `weight_decay` 设太大（如 0.1 以上），会让模型所有参数被强力压向 0，相当于强制欠拟合。大模型 SFT 通常 weight_decay=0.01 或更小。

4. **Dropout 在 eval 模式忘记关**：`model.eval()` 会自动关闭 Dropout，但如果手写了 Dropout 而不是用 `nn.Dropout`（会响应 `model.train()/eval()` 的切换），就要手动控制。

5. **学习曲线的纵坐标陷阱**：有时两条曲线（训练/验证）看起来很接近，但纵坐标范围很小，实际上差距显著。看学习曲线时注意 y 轴刻度。

6. **数据泄露（Data Leakage）**：如果预处理（归一化、特征统计）用了验证/测试集的统计量，就是数据泄露，会导致泛化误差被低估。正确做法：所有统计量只在训练集上计算，再应用到验证/测试集。

## 七、面试常见问题

**Q1：什么是偏差？什么是方差？用一句话分别解释。**

偏差衡量模型的"系统性错误"——模型的平均预测偏离真实值的程度，反映模型假设与数据真实规律之间的差距。方差衡量模型的"不稳定性"——模型在不同训练数据上预测结果的波动程度，反映模型对训练数据的敏感程度。

**Q2：过拟合了，有哪些解决方法？**

按效果由强到弱的常用手段：增加训练数据（最根本）、减小模型复杂度（层数/参数量）、增大正则化系数（L2 weight_decay）、加 Dropout、Early Stopping、数据增强、Label Smoothing。大模型 SFT 场景下：减少训练轮数（epoch）、降低学习率、用 LoRA 只训练小部分参数。

**Q3：L1 和 L2 正则化的区别是什么？什么时候用哪个？**

L2 让参数均匀变小但不为零，适合所有特征都有一定价值的场景，是大模型的标配（weight_decay）。L1 产生稀疏解（很多参数精确变为零），适合需要特征选择的场景。L1 产生稀疏的原因：在 0 附近 L1 的梯度是常数 $\pm\lambda$，会把参数推过零点；而 L2 的梯度趋向零，无法精确清零。

**Q4：Dropout 为什么有防过拟合的效果？**

Dropout 每次随机关闭部分神经元，相当于训练了指数级数量的子网络，推理时做集成。此外，它迫使每个神经元不能依赖特定邻居，只能学习鲁棒的、独立的特征，防止特征间的"共适应"。从噪声角度看，Dropout 给训练过程注入随机性，增大了模型对输入扰动的鲁棒性。

**Q5：训练 loss 不断下降，验证 loss 先降后升，该怎么办？**

这是典型的过拟合信号。解决方案按优先级：先用 Early Stopping 在验证 loss 最低点停止；增加正则化（weight_decay、Dropout）；如果数据集较小，考虑数据增强；如果计算资源允许，增加训练数据；也可以减小模型规模或减少训练 epoch。

**Q6：大模型参数量远超训练数据量，为什么没有严重过拟合？**

几个原因：大模型通常有万亿量级的 token 训练数据，单从数量上就已经很大；自然语言高度结构化，模型有效需要学习的"独立信息"远小于参数量；SGD 在过参数化模型中有隐式正则化效果，倾向于找到"最平坦"的最优解；训练通常不到完整收敛（隐式 Early Stopping）。这对应"双下降"现象：超过插值阈值后，更大模型的泛化反而更好。

**Q7：什么是数据泄露？如何避免？**

数据泄露是指训练以外的数据（验证集、测试集、未来信息）不小心影响了模型训练，导致泛化误差被低估。常见形式：用全量数据做归一化统计再划分数据集；Time series 中用了未来数据的特征；用测试集反复调超参。避免方法：严格按时间顺序或随机先划分数据集，所有预处理统计只在训练集上拟合，再变换验证/测试集。

**Q8：Early Stopping 是什么正则化机制？为什么它有效？**

Early Stopping 是一种隐式正则化：在验证误差最低点停止，相当于约束了参数的"更新步数"，限制了模型从训练集中学到多少细节。理论上可以证明，对于线性模型，Early Stopping 等价于 L2 正则化。对于深度学习，虽然没有完全等价的理论，但实践中它非常有效——尤其是大模型 SFT 时，只训练 1-3 个 epoch 往往就能拿到最好的验证集结果。
