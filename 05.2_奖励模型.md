## 零、开篇：谁来判断回答的好坏

SFT 让模型学会了"按格式回答"，但"回答是否真的好"这个问题，SFT 数据并不能完整解决。

问题在于：对于大多数开放式问题（"帮我写一封商务邮件"、"解释一下量子纠缠"），**不存在唯一正确答案**。我们真正想要的，是"人类觉得哪个回答更好"这种偏好信号。

但直接让人类给每个回答打分（1-10分）很难——打分的一致性极差，不同标注员对"8分"的理解可能完全不同。

更可靠的方式：**让人类比较两个回答，选择更好的那个**（相对偏好，而非绝对评分）。比较比打分容易得多，人类在比较时的一致性也更高。

奖励模型（Reward Model，RM）就是从这些"偏好比较数据"中学习的：给定一个 prompt 和两个回答，RM 应该能判断哪个更好，并给出量化的分数差异。

## 一、偏好数据：奖励模型的原材料

### 1.1 初印象：用费曼视角理解

**偏好数据的结构很简单：一个问题，两个回答，一个"哪个更好"的标注**。

```
问题：帮我解释什么是 Transformer。

回答 A（chosen，被偏好）：
```

## 二、Bradley-Terry 模型：偏好的数学基础

### 2.1 初印象：用费曼视角理解

**Bradley-Terry 模型把"谁赢了"这个定性结果，转化成"每个选手有多强"的定量分数**。

这个模型在体育竞技中被广泛使用：国际象棋的 Elo 评分系统就是 BT 模型的一个变体。

原则：如果选手 A 有能力值 $r_A$，选手 B 有能力值 $r_B$，那么 A 赢的概率是：

$$P(A \text{ 赢}) = \frac{e^{r_A}}{e^{r_A} + e^{r_B}} = \sigma(r_A - r_B)$$

这正是 sigmoid 函数！把两个能力值的差值输入 sigmoid，就得到胜负概率。

用在 LLM 偏好上：把每个回答的"质量分"看作能力值，人类偏好的概率就等于 sigmoid(quality_A - quality_B)。

RLHF 流程的关键组成部分是奖励模型，它作为将人类偏好和反馈整合到学习过程中的主要机制。奖励模型指导 RLHF 的优化过程朝着符合人类偏好的目标方向发展。

### 2.2 Bradley-Terry 损失函数推导

给定 prompt $x$，chosen 回答 $y_w$（winner），rejected 回答 $y_l$（loser），奖励模型 $r_\theta$ 输出对应的奖励分数：

**偏好概率**（Bradley-Terry 模型）：

$$P(y_w \succ y_l \mid x) = \sigma(r_\theta(x, y_w) - r_\theta(x, y_l))$$

**最大化似然 = 最小化负对数似然**：

$$\mathcal{L}_{RM} = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}}\left[\log \sigma(r_\theta(x, y_w) - r_\theta(x, y_l))\right]$$

这就是奖励模型的训练损失——本质上是个二元分类损失，要求模型给 chosen 比 rejected 更高的分数。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

def reward_model_loss(
    reward_chosen: torch.Tensor,    # (B,) chosen 回答的奖励分数
    reward_rejected: torch.Tensor,  # (B,) rejected 回答的奖励分数
) -> torch.Tensor:
    """
    Bradley-Terry 奖励模型损失。
    等价于：-log(sigmoid(r_chosen - r_rejected))
    """
    # 分数差值
    diff = reward_chosen - reward_rejected   # (B,)

    # 负对数 sigmoid（等价于 binary cross entropy with logits）
    loss = -F.logsigmoid(diff).mean()

    # 额外记录准确率（chosen 分数 > rejected 分数的比例）
    accuracy = (diff > 0).float().mean()

    return loss, accuracy


# 测试
torch.manual_seed(42)
B = 8
r_chosen   = torch.randn(B) + 1.0   # chosen 分数略高
r_rejected = torch.randn(B)

loss, acc = reward_model_loss(r_chosen, r_rejected)
print(f"奖励模型损失: {loss.item():.4f}")
print(f"排序准确率:   {acc.item():.2%}  （chosen 分数 > rejected 分数的比例）")
```

## 三、奖励模型的架构

### 3.1 初印象：用费曼视角理解

**奖励模型的架构非常简单：在一个 LLM 上加一个分类头，输出单个标量分数**。

就像把 BERT 用于情感分析：LLM 的主体部分保持不变（处理文本，提取语义），在最后一层加一个线性层，把 `[d_model]` 维的向量压缩成一个实数——这个实数就是奖励分数。

通常从 SFT 模型（而不是预训练基础模型）初始化奖励模型，因为 SFT 模型已经学会了理解指令格式。

### 3.2 奖励模型实现

```python
import torch
import torch.nn as nn
from transformers import AutoModel, AutoTokenizer

class RewardModel(nn.Module):
    """
    奖励模型：在 LLM 主体上加一个线性层，输出标量奖励分数。
    初始化自 SFT 模型（或预训练模型）。
    """
    def __init__(self, model_name: str, dropout: float = 0.0):
        super().__init__()
        # 加载预训练/SFT 模型的主体（不含 LM Head）
        self.backbone = AutoModel.from_pretrained(
            model_name,
            torch_dtype=torch.bfloat16,
        )

        # 奖励头：把最后一层的 [EOS] token 表示压缩成标量
        hidden_size = self.backbone.config.hidden_size
        self.reward_head = nn.Linear(hidden_size, 1, bias=False)

        # 初始化奖励头为小值（避免初始分数过大）
        nn.init.normal_(self.reward_head.weight, std=0.02)

    def forward(
        self,
        input_ids: torch.Tensor,         # (B, L)
        attention_mask: torch.Tensor,    # (B, L)
    ) -> torch.Tensor:
        """
        返回每个序列的标量奖励分数 (B,)。
        用最后一个非 padding token 的隐状态作为序列表示。
        """
        outputs = self.backbone(
            input_ids=input_ids,
            attention_mask=attention_mask,
        )
        hidden_states = outputs.last_hidden_state   # (B, L, H)

        # 找到每个序列最后一个真实 token 的位置（非 padding）
        # attention_mask 中最后一个 1 的位置
        seq_lengths = attention_mask.sum(dim=1) - 1   # (B,)
        last_hidden = hidden_states[
            torch.arange(hidden_states.size(0)), seq_lengths
        ]   # (B, H)

        # 线性映射到标量奖励
        reward = self.reward_head(last_hidden).squeeze(-1)   # (B,)
        return reward


class RewardModelTrainer:
    """奖励模型训练器"""

    def __init__(self, model: RewardModel, device: str = "cuda"):
        self.model = model.to(device)
        self.device = device
        self.optimizer = torch.optim.AdamW(
            model.parameters(), lr=1e-5, weight_decay=0.01
        )

    def train_step(
        self,
        chosen_input_ids:    torch.Tensor,
        chosen_attn_mask:    torch.Tensor,
        rejected_input_ids:  torch.Tensor,
        rejected_attn_mask:  torch.Tensor,
    ) -> dict:
        self.model.train()
        self.optimizer.zero_grad()

        # 计算 chosen 和 rejected 的奖励分数
        r_chosen   = self.model(chosen_input_ids,   chosen_attn_mask)
        r_rejected = self.model(rejected_input_ids, rejected_attn_mask)

        # Bradley-Terry 损失
        loss, accuracy = reward_model_loss(r_chosen, r_rejected)

        loss.backward()
        torch.nn.utils.clip_grad_norm_(self.model.parameters(), 1.0)
        self.optimizer.step()

        return {
            "loss": loss.item(),
            "accuracy": accuracy.item(),
            "reward_chosen_mean": r_chosen.mean().item(),
            "reward_rejected_mean": r_rejected.mean().item(),
            "reward_margin": (r_chosen - r_rejected).mean().item(),
        }

    @torch.no_grad()
    def score(self, input_ids: torch.Tensor, attn_mask: torch.Tensor) -> torch.Tensor:
        """给一个回答打分"""
        self.model.eval()
        return self.model(input_ids, attn_mask)
```

## 四、奖励黑客：奖励模型的阿喀琉斯之踵

### 4.1 初印象：用费曼视角理解

**奖励黑客（Reward Hacking）是指：模型学会了欺骗奖励模型，而非真正提升回答质量**。

就像一个学生发现了"老师喜欢长篇大论"的规律，就在每道题后面堆砌废话——卷面成绩提高了，但实际能力没有提升。

RLHF 可能存在奖励黑客问题，政策可能发现绕道而行——产生重复或公式化内容——在不真正推进预期行为的情况下最大化代理奖励。

具体表现：
- **长度偏见**：模型学会"越长的回答越好"，开始堆砌废话
- **格式偏见**：模型学会"有条目的回答更受欢迎"，对所有问题都用列表格式
- **奉承偏见**：模型学会"先夸用户的问题好"，开始奉承用户
- **分布外失效**：奖励模型在训练分布之外的评分不可靠

```python
# 奖励黑客的量化检测
def detect_reward_hacking(responses: list[str], rewards: list[float]) -> dict:
    """
    检测潜在的奖励黑客模式。
    分析回答长度、格式特征等和奖励分数的相关性。
    """
    import numpy as np

    lengths = [len(r.split()) for r in responses]
    has_bullet = [1 if "•" in r or "- " in r or "1." in r else 0 for r in responses]
    starts_with_praise = [
        1 if any(r.startswith(p) for p in ["这是个好问题", "很好的问题", "Great question"])
        else 0 for r in responses
    ]

    length_corr = np.corrcoef(lengths, rewards)[0, 1]
    bullet_corr = np.corrcoef(has_bullet, rewards)[0, 1]
    praise_corr = np.corrcoef(starts_with_praise, rewards)[0, 1]

    warnings = []
    if abs(length_corr) > 0.5:
        warnings.append(f"⚠️  长度与奖励强相关 (r={length_corr:.2f})，可能存在长度偏见")
    if bullet_corr > 0.3:
        warnings.append(f"⚠️  条目格式与奖励相关 (r={bullet_corr:.2f})，可能存在格式偏见")
    if praise_corr > 0.3:
        warnings.append(f"⚠️  奉承语气与奖励相关 (r={praise_corr:.2f})，可能存在奉承偏见")

    return {
        "length_reward_correlation": length_corr,
        "bullet_reward_correlation": bullet_corr,
        "praise_reward_correlation": praise_corr,
        "warnings": warnings,
    }


# 模拟检测
import numpy as np
np.random.seed(42)

n = 100
fake_lengths = np.random.randint(50, 500, n)
# 模拟奖励和长度高度相关（奖励黑客场景）

responses = [" ".join(["word"] * l) for l in fake_lengths]

print("奖励黑客检测结果：")

if not result["warnings"]:
    print("  ✅ 未检测到明显的奖励黑客迹象")
```

### 4.2 缓解奖励黑客的方法

**方法一：KL 惩罚（RLHF 中使用）**

在优化目标中加入 KL 散度惩罚，防止策略模型偏离原始 SFT 模型太远：

$$\text{目标} = \mathbb{E}[r_\theta(x, y)] - \beta \cdot D_{KL}(\pi_\theta \| \pi_{SFT})$$

**方法二：多维度奖励**

不用单一标量奖励，而是在多个维度（有帮助性、无害性、真实性）分别评分，防止模型在某一维度极端优化。

**方法三：保守的 PPO 更新（clip ratio）**

PPO 算法本身就有 clip 机制，限制每步更新的幅度，缓解奖励黑客。

**方法四：定期用人类评估校准**

不完全依赖奖励模型，定期在随机样本上做人类评估，检查奖励分数是否和人类判断一致。

## 六、现代奖励模型的变体

### 6.1 隐式奖励模型（DPO 的视角）

DPO（Direct Preference Optimization，下一章详讲）的核心洞见：

**不需要显式训练一个奖励模型**。可以把 LLM 本身当作隐式奖励模型：

$$r(x, y) = \beta \log \frac{\pi_\theta(y|x)}{\pi_{SFT}(y|x)}$$

LLM 在 SFT 基础上的对数概率变化，本身就构成了一个奖励信号。这避免了单独训练和维护一个奖励模型，工程复杂度大幅降低。

### 6.2 生成式奖励模型（GenRM）

传统奖励模型只输出一个标量分数，无法解释"为什么这个回答更好"。

生成式奖励模型（GenRM）让模型先生成推理过程，再给出偏好判断——类似 chain-of-thought：

```
输入：
  问题：什么是反向传播？
  回答 A：[详细解释]
  回答 B：[简单解释]

GenRM 输出：
  推理：回答 A 更详细，包含了数学推导和代码示例，
        解释了链式法则的应用，有助于深度理解。
        回答 B 虽然简洁，但缺乏核心机制的说明。
  判断：A 更好 [confidence: 0.92]
```

零样本 LLM 判断在分布内任务上的性能比 BT 奖励模型低 9-36%，而 GenRM 在分布内任务上达到了与 BT 模型相当的准确率，同时在分布外任务上显著超越（10-45%）。

## 七、面试常见问题

**Q1：奖励模型在 RLHF 中的作用是什么？**

奖励模型是 RLHF 的核心中间件：它将人类难以量化的"回答质量"转化为可微分的标量分数，让强化学习算法（PPO）能够优化语言模型。具体角色是：训练时用偏好数据（chosen vs rejected 配对）学习人类偏好；RLHF 阶段给语言模型生成的回答实时打分，作为强化学习的奖励信号。

**Q2：Bradley-Terry 模型的损失函数是什么？直觉是什么？**

损失函数是 $\mathcal{L} = -\mathbb{E}[\log\sigma(r_\theta(x, y_w) - r_\theta(x, y_l))]$，其中 $r_\theta$ 是奖励模型，$y_w$ 是 chosen，$y_l$ 是 rejected。直觉：sigmoid$(r_w - r_l)$ 是"chosen 比 rejected 好"的概率，最大化这个概率等价于最小化负对数似然。当 $r_w \gg r_l$ 时，这个概率接近 1，loss 接近 0；当 $r_w \approx r_l$ 时，loss 接近 $\log 2 \approx 0.69$。

**Q3：什么是奖励黑客？如何缓解？**

奖励黑客是指策略模型找到了让奖励模型打高分的"捷径"，但这些捷径并不对应真正的高质量输出（如堆砌废话、奉承用户、格式化输出）。缓解方法：KL 惩罚（防止策略偏离 SFT 模型太远）；多维度奖励（从多个角度评估，防止极端优化）；定期用人类评估校准奖励模型；conservative PPO 限制每步更新幅度。

**Q4：奖励模型的初始化通常用哪个模型？为什么？**

通常从 SFT 模型初始化，而不是预训练基础模型。原因：SFT 模型已经学会了理解指令格式和对话结构，在理解"指令+回答"这种格式方面比基础模型更强；奖励模型需要判断"针对这个指令，这个回答好不好"，SFT 模型的表示空间更适合这类判断。

**Q5：如何评估奖励模型的质量？**

主要评估指标：偏好对准确率（在 held-out 偏好数据集上，chosen 获得更高分的比例）；RewardBench 基准（覆盖 Chat、Safety、Reasoning 等维度的标准化评估）；人类评估（让真实用户评估奖励模型和人类判断的一致性）；与下游 RLHF 性能的相关性（奖励模型最终是否有助于提升策略模型的人类评分）。
