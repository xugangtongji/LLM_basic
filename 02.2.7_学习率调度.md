## 零、开篇：学习率调度往往比优化器选择更重要

一个被广泛认可的工程经验是：**好的学习率调度带来的收益，往往超过切换优化器算法本身**。同样用 AdamW，配好的 Cosine 调度和配差的 Constant 调度，最终 loss 可以差出好几个点。

大模型训练中学习率调度的演进路线也颇有意思：

```
Constant（不调整）
    → Cosine（LLaMA、GPT 系列的经典选择）
    → WSD / Warmup-Stable-Decay（MiniCPM、DeepSeek 引入，
       2024-2025 年快速成为持续训练场景新标准）
    → Schedule-Free（实验性，完全取消衰减，用权重平均代替）
```

理解每种策略的"工作原理"，而不是死记最优配置，才能在新场景里做出正确判断。

## 一、Warmup：所有调度的共同起点

### 1.1 初印象：用费曼视角理解

**Warmup 就是"先慢跑热身，再全速冲刺"**。

大模型训练初期，AdamW 的二阶矩 $v_t$ 从零开始积累，偏差修正 $\hat{v}_t = v_t / (1-\beta_2^t)$ 在 $t$ 很小时分母极小，会让步长爆炸——哪怕梯度正常，参数也会剧烈跳动。Warmup 让学习率从接近 0 线性爬升，给 $v_t$ 时间收敛到稳定估计，再开始正常训练。

在自适应优化器（如 Adam）中，不当的偏差修正和初期较高的更新幅度使得 warmup 对于避免大的不稳定步骤尤为关键。

### 1.2 数学形式与工程配置

线性 Warmup（最常用）：

$$\eta_t = \eta_{\text{peak}} \cdot \frac{t}{T_{\text{warmup}}}, \quad t < T_{\text{warmup}}$$

**工程配置参考（基于主流 LLM 技术报告）**：

| 模型 | 峰值 LR | Warmup 步数 | Warmup 占比 |
|------|--------|------------|------------|
| GPT-3 175B | 6e-5 | 375M tokens | ~0.3% |
| LLaMA-3-8B | 3e-4 | 1000 steps | ~0.1% |
| LLaMA-3-70B | 1.5e-4 | 2000 steps | ~0.1% |
| Qwen-2.5 | 3e-4 | ~2% steps | 2% |
| DeepSeek-V2 | 2.4e-4 | 2000 steps | <1% |

```python
import math
import torch
from torch.optim.lr_scheduler import LambdaLR

def get_warmup_scheduler(optimizer, warmup_steps: int):
    """纯线性 Warmup，不衰减——用于调试或持续训练"""
    def lr_lambda(step):
        if step < warmup_steps:
            return step / max(1, warmup_steps)
        return 1.0   # 保持峰值
    return LambdaLR(optimizer, lr_lambda)
```

## 二、Cosine 衰减：LLM 预训练的经典标准

### 2.1 初印象：用费曼视角理解

**Cosine 调度像一个"平滑刹车"**——从峰值学习率开始，沿余弦曲线缓慢降低，前期慢、后期也慢，中间稍快，整体非常平滑，不会突然"踩刹车"。

相比线性衰减，Cosine 在末期衰减更慢，给模型更多时间在低学习率下精细调整，最终收敛质量通常更好。

### 2.2 数学形式与完整实现

Warmup + Cosine 衰减（最常见组合）：

$$\eta_t = \eta_{\min} + (\eta_{\text{peak}} - \eta_{\min}) \cdot \frac{1 + \cos\left(\pi \cdot \frac{t - T_w}{T - T_w}\right)}{2}, \quad t \geq T_w$$

其中 $T_w$ 是 warmup 步数，$T$ 是总步数，$\eta_{\min}$ 通常是峰值的 1%~10%。

```python
import math
from torch.optim.lr_scheduler import LambdaLR

def get_cosine_warmup_scheduler(
    optimizer,
    warmup_steps: int,
    total_steps: int,
    min_lr_ratio: float = 0.1,
):
    """
    Warmup（线性）+ Cosine 衰减。
    min_lr_ratio：衰减终点 = peak_lr × min_lr_ratio。
    LLaMA / GPT 系列标配。
    """
    def lr_lambda(step):
        if step < warmup_steps:
            return step / max(1, warmup_steps)
        progress = (step - warmup_steps) / max(1, total_steps - warmup_steps)
        cosine = 0.5 * (1.0 + math.cos(math.pi * progress))
        return min_lr_ratio + (1.0 - min_lr_ratio) * cosine

    return LambdaLR(optimizer, lr_lambda)


# 可视化三种衰减终点的差异
import numpy as np
import matplotlib.pyplot as plt

total, warmup = 10000, 500
steps = np.arange(total)

fig, ax = plt.subplots(figsize=(10, 4))
for min_ratio, label, color in [
    (0.0,  "Cosine to 0（decay-to-zero）", "steelblue"),
    (0.1,  "Cosine to 10%（LLaMA 风格）", "darkorange"),
    (0.01, "Cosine to 1%", "green"),
]:
    lrs = []
    for s in steps:
        if s < warmup:
            lrs.append(s / warmup)
        else:
            p = (s - warmup) / (total - warmup)
            lrs.append(min_ratio + (1 - min_ratio) * 0.5 * (1 + math.cos(math.pi * p)))
    ax.plot(steps, lrs, label=label, linewidth=2, color=color)

ax.axvline(warmup, color="red", linestyle="--", alpha=0.5, label="Warmup 结束")
ax.set_xlabel("步数"); ax.set_ylabel("学习率（× 峰值）")
ax.set_title("Warmup + Cosine 衰减：不同终点的对比")
ax.legend(); ax.grid(alpha=0.3)
plt.tight_layout()
plt.savefig("cosine_schedule.png", dpi=150, bbox_inches="tight")
```

### 2.3 Cosine 的核心局限

Cosine 调度有一个**根本性的工程缺陷**：Cosine 方法需要提前固定总训练步数。这导致如果要扩展训练（如加入新数据），就需要从头重新校准整个衰减曲线。这在工业环境中是个大问题——你往往不知道"最优训练步数是多少"，或者中途要追加数据。这正是 WSD 出现的动机。

## 三、线性衰减：简单但有效

### 3.1 数学形式与适用场景

$$\eta_t = \eta_{\text{peak}} \cdot \max\left(\eta_{\min}, 1 - \frac{t - T_w}{T - T_w}\right), \quad t \geq T_w$$

```python
def get_linear_warmup_scheduler(
    optimizer,
    warmup_steps: int,
    total_steps: int,
    min_lr_ratio: float = 0.0,
):
    """Warmup + 线性衰减，SFT / 微调场景常用"""
    def lr_lambda(step):
        if step < warmup_steps:
            return step / max(1, warmup_steps)
        progress = (step - warmup_steps) / max(1, total_steps - warmup_steps)
        return max(min_lr_ratio, 1.0 - progress * (1.0 - min_lr_ratio))
    return LambdaLR(optimizer, lr_lambda)
```

**线性衰减的场景**：
- SFT / RLHF 微调：训练轮数少（1-3 epoch），线性衰减够用且简单
- 研究实验对比：曲线形状简单，方便控制变量
- 近年研究表明，对于足够长的训练，线性衰减的最终 loss 和 Cosine 非常接近

## 四、One-Cycle：快速实验的强力调度

### 4.1 初印象：用费曼视角理解

**One-Cycle 是"先涨后降，但涨的时候比 Warmup 更激进"**。

由 Leslie Smith 提出，学习率从小值快速涨到峰值（比正常 warmup 更激进），再从峰值降到极小值（比 Cosine 末期更低）。动量（momentum）同步反向变化——学习率高时动量低，学习率低时动量高。

```python
from torch.optim.lr_scheduler import OneCycleLR

# 需要提前知道总步数
scheduler = OneCycleLR(
    optimizer,
    max_lr=3e-4,          # 峰值学习率
    total_steps=10000,
    pct_start=0.1,        # 前 10% 步用于涨 LR
    anneal_strategy="cos", # 衰减形状
    div_factor=25.0,       # 初始 lr = max_lr / div_factor
    final_div_factor=1e4,  # 最终 lr = max_lr / (div_factor * final_div_factor)
)
```

**One-Cycle 的适用场景**：
- 训练计算机视觉模型（ResNet 等）的快速实验
- 训练步数固定且较短的场景
- 大模型预训练中很少用（总步数太长，One-Cycle 的激进上升不稳定）

## 五、WSD：持续训练时代的新标准

### 5.1 初印象：用费曼视角理解

**WSD（Warmup-Stable-Decay）是"先热身、再保持油门全开探索、最后踩刹车精调"**。

Cosine 把"探索"和"精调"混在一条曲线里，不能中途停、不能延长训练。WSD 把这两件事分开：

- **Warmup 阶段**：线性升温，稳定优化器状态
- **Stable 阶段**（核心创新）：保持峰值学习率不变，长时间高速探索损失景观
- **Decay 阶段**：在需要收敛时才开始衰减，持续约总步数的 10%

WSD 调度产生非传统的损失曲线：在常数学习率阶段，损失高于 Cosine 调度；但在衰减阶段，损失急剧下降，最终往往优于 Cosine 调度的结果。

### 5.2 WSD 的几何直觉：河谷损失景观

研究者用"河谷"几何来解释 WSD 的有效性：优化轨迹分解为沿平坦"河流"方向的快速推进，以及在陡峭"山坡"方向的振荡。Stable 阶段的高学习率在河流方向快速推进，但会引起山坡方向的振荡（体现为损失偏高）；Decay 阶段则平息振荡，使轨迹坍缩到真正的谷底，表现为损失的急剧下降。

### 5.3 WSD 的核心优势：持续训练友好

```python
def get_wsd_scheduler(
    optimizer,
    warmup_steps: int,
    stable_steps: int,
    decay_steps: int,
    min_lr_ratio: float = 0.0,
):
    """
    三阶段 WSD 调度。
    total_steps = warmup + stable + decay
    适用场景：数据量不确定、需要持续训练、多预算中间检查点。
    """
    total_steps = warmup_steps + stable_steps + decay_steps

    def lr_lambda(step):
        if step < warmup_steps:
            # 线性 warmup
            return step / max(1, warmup_steps)
        elif step < warmup_steps + stable_steps:
            # 稳定期：保持峰值 lr
            return 1.0
        else:
            # 衰减期：线性到 min_lr_ratio（也可以换成 cosine）
            decay_progress = (step - warmup_steps - stable_steps) / max(1, decay_steps)
            return max(min_lr_ratio, 1.0 - decay_progress * (1.0 - min_lr_ratio))

    return LambdaLR(optimizer, lr_lambda)


# 可视化 WSD vs Cosine 的对比
import numpy as np
import matplotlib.pyplot as plt, math

total = 10000
warmup = 200
stable = 8800  # 88% 稳定期
decay  = 1000  # 10% 衰减期

steps = np.arange(total)

def wsd_lr(s):
    if s < warmup: return s / warmup
    elif s < warmup + stable: return 1.0
    else:
        p = (s - warmup - stable) / decay
        return max(0.0, 1.0 - p)

def cosine_lr(s):
    if s < warmup: return s / warmup
    p = (s - warmup) / (total - warmup)
    return 0.1 + 0.9 * 0.5 * (1 + math.cos(math.pi * p))

fig, ax = plt.subplots(figsize=(10, 4))
ax.plot(steps, [cosine_lr(s) for s in steps], label="Cosine（LLaMA 风格）",
        linewidth=2, color="steelblue")
ax.plot(steps, [wsd_lr(s) for s in steps], label="WSD（Warmup-Stable-Decay）",
        linewidth=2, color="darkorange", linestyle="--")
ax.axvline(warmup, color="gray", linestyle=":", alpha=0.7)
ax.axvline(warmup + stable, color="red", linestyle=":", alpha=0.7, label="Decay 开始")
ax.set_xlabel("步数"); ax.set_ylabel("学习率（× 峰值）")
ax.set_title("WSD vs Cosine 调度对比")
ax.legend(); ax.grid(alpha=0.3)
ax.text(warmup + stable/2, 1.05, "Stable（高速探索）", ha="center", fontsize=9)
plt.tight_layout()
plt.savefig("wsd_vs_cosine.png", dpi=150, bbox_inches="tight")
```

### 5.4 WSD 的工程价值：一个 stable checkpoint，多个 decay 实验

WSD 最大的工程优势在于：使用 WSD 调度，可以从主分支（stable 阶段）的同一个 checkpoint 出发，在多个不同计算预算下分叉出不同的 decay 实验，而无需每次重新训练。

```
主分支（stable 阶段，持续高速训练）
    ├── 50B tokens checkpoint ──→ 分叉：decay 10% → 中间版本模型
    ├── 100B tokens checkpoint ──→ 分叉：decay 10% → 更强版本
    └── 200B tokens checkpoint ──→ 分叉：decay 10% → 最终版本
```

这对工业界意义重大：可以在一次训练中高效评估不同数据量下的模型性能，不再需要为每个数据预算重新训练。

**使用 WSD 的主流模型**：MiniCPM（最早大规模应用）、DeepSeek 系列、Falcon-2、部分 Qwen 版本。

## 六、持续训练与学习率重启

### 6.1 初印象：用费曼视角理解

**持续训练就是"给已经训练好的模型，追加新数据继续训练"**。

场景举例：
- 一个 LLM 训练到 1T token，现在要追加 100B token 的领域数据
- 一个已经发布的模型，发现新的高质量数据集，想持续提升

最大的挑战：如果直接在旧模型上用原来的 Cosine 调度继续，学习率已经降到接近 0，模型几乎学不动新数据（"可塑性消失"）。

### 6.2 Re-Warmup：持续训练的标准做法

在持续预训练中，从一个收敛的 checkpoint 出发时"重新 warmup"学习率是必要的，否则对新数据的适应可能会停滞。

```python
def get_continual_training_scheduler(
    optimizer,
    warmup_steps: int,
    total_steps: int,
    restart_lr_ratio: float = 0.3,   # 重启时的初始 lr = peak × ratio
    min_lr_ratio: float = 0.1,
):
    """
    持续训练调度：从已训练模型重新 warmup，再 Cosine 衰减。
    restart_lr_ratio：warmup 起点 lr，不从 0 开始
    （从太低的 lr 开始会浪费 warmup 阶段）
    """
    def lr_lambda(step):
        if step < warmup_steps:
            # 从 restart_lr_ratio 线性涨到 1.0
            return restart_lr_ratio + (1.0 - restart_lr_ratio) * step / max(1, warmup_steps)
        progress = (step - warmup_steps) / max(1, total_steps - warmup_steps)
        cosine = 0.5 * (1.0 + math.cos(math.pi * progress))
        return min_lr_ratio + (1.0 - min_lr_ratio) * cosine
    return LambdaLR(optimizer, lr_lambda)

# WSD 的持续训练优势：只需从 stable checkpoint 重新触发 decay 即可
# 不用 re-warmup，直接接上下一段的 stable 阶段，然后 decay
```

### 6.3 持续训练调度策略对比

| 策略 | Cosine | WSD |
|------|--------|-----|
| 追加数据 | 需要重新设计 decay 曲线 | 从 stable checkpoint 直接续 |
| Re-warmup | 必须 | 可选（可直接接 stable） |
| 中间检查点质量 | 衰减中途的 ckpt 不是最优 | stable ckpt 可以直接发布 |
| 适应新领域 | 高 LR 可能"遗忘"旧知识 | stable 阶段可控制强度 |

## 七、调度策略选型指南

```
你的训练场景是什么？

├── 从头预训练 LLM（预算明确）
│   └── Warmup + Cosine（最成熟，调参经验丰富）
│       - warmup：总步数的 1-2%
│       - min_lr：峰值的 10%
│       - β2=0.95（不要用 0.999）

├── 从头预训练 LLM（预算不确定 / 需持续训练）
│   └── WSD（更灵活，stable ckpt 可复用）
│       - warmup：总步数的 1-2%
│       - stable：总步数的 88-89%
│       - decay：总步数的 10%（线性或 cosine）

├── SFT / RLHF 微调（1-5 epoch）
│   └── Warmup + Linear / Warmup + Cosine 均可
│       - warmup：5-10% 的训练步数
│       - 峰值 lr：比预训练小 10-100 倍

├── 持续训练（从已有模型追加数据）
│   ├── 有 stable ckpt（WSD 训练的模型）→ 直接触发 decay
│   └── 没有 stable ckpt → Re-warmup（从 30% 峰值开始）

├── 快速实验 / 小模型
│   └── One-Cycle 或 Cosine（都行）

└── 显存极度紧张，想省内存
    └── Adafactor + RSqrt 调度（自带的调度，不用另外写）
```

## 八、代码：统一调度工厂

```python
import math
from torch.optim.lr_scheduler import LambdaLR
from typing import Literal

def get_scheduler(
    optimizer,
    strategy: Literal["cosine", "linear", "wsd", "constant"],
    warmup_steps: int,
    total_steps: int,
    min_lr_ratio: float = 0.1,
    wsd_stable_ratio: float = 0.88,
):
    """
    统一调度工厂，支持四种常用策略。
    用法：
        scheduler = get_scheduler(optimizer, "cosine", warmup_steps=1000, total_steps=50000)
        # 训练循环里每步调用：
        scheduler.step()
    """
    if strategy == "constant":
        def lr_fn(step):
            return min(1.0, step / max(1, warmup_steps))

    elif strategy == "linear":
        def lr_fn(step):
            if step < warmup_steps:
                return step / max(1, warmup_steps)
            p = (step - warmup_steps) / max(1, total_steps - warmup_steps)
            return max(min_lr_ratio, 1.0 - p * (1.0 - min_lr_ratio))

    elif strategy == "cosine":
        def lr_fn(step):
            if step < warmup_steps:
                return step / max(1, warmup_steps)
            p = (step - warmup_steps) / max(1, total_steps - warmup_steps)
            return min_lr_ratio + (1.0 - min_lr_ratio) * 0.5 * (1 + math.cos(math.pi * p))

    elif strategy == "wsd":
        stable_steps = int((total_steps - warmup_steps) * wsd_stable_ratio)
        decay_steps  = total_steps - warmup_steps - stable_steps

        def lr_fn(step):
            if step < warmup_steps:
                return step / max(1, warmup_steps)
            elif step < warmup_steps + stable_steps:
                return 1.0
            else:
                p = (step - warmup_steps - stable_steps) / max(1, decay_steps)
                return max(min_lr_ratio, 1.0 - p * (1.0 - min_lr_ratio))
    else:
        raise ValueError(f"未知调度策略: {strategy}")

    return LambdaLR(optimizer, lr_fn)
```

## 九、面试常见问题

**Q1：Cosine 和 Linear 衰减有什么区别？哪个更好？**

形状不同：Cosine 在初期和末期变化慢、中期快，整体更平滑；Linear 匀速下降。效果上，对于足够长的训练两者最终 loss 差距不大，Cosine 口碑稍好，因为末期衰减更慢让模型有更多时间精细收敛。工程上 Linear 更简单，常见于 SFT。大模型预训练几乎都用 Cosine 或 WSD。

**Q2：WSD 调度相比 Cosine 的核心优势是什么？**

两点：第一，不需要提前固定总训练步数，stable 阶段可以随时延长，需要收敛时才触发 decay；第二，从同一个 stable checkpoint 可以分叉出多个不同计算预算的实验，无需为每个预算重新训练。这对数据量不确定或需要持续追加数据的工业训练场景尤其有价值。

**Q3：Warmup 为什么是必须的？不用行不行？**

对于 Transformer 大模型，warmup 几乎是必须的。原因是 AdamW 的二阶矩在初期估计不准，不用 warmup 直接高学习率，参数更新剧烈，容易 loss NaN 或落入差的局部最优。小模型（几十万参数）、SGD、有精心设计的初始化时可以不用，但 LLM 训练不建议省略。

**Q4：持续训练时学习率怎么设？直接从 0 开始 warmup 吗？**

不建议从 0 开始。从 0 warmup 会浪费很多步数在极小的学习率上，并且太低的 lr 更新可能导致模型对新数据的适应性很差。通常做法是从峰值的 20%-50% 开始，线性 warmup 几百步再到峰值，然后衰减。如果原来用 WSD 训练且有 stable checkpoint，可以更简单：直接从 stable checkpoint 的峰值学习率继续，不需要 re-warmup。

**Q5：`min_lr_ratio` 应该设多少？衰减到 0 好还是衰减到峰值的 10% 好？**

近年研究（decay-to-zero，D2Z）表明，在大型 LLM 预训练中，学习率完全衰减到零比衰减到峰值 10% 的策略更优，在计算最优的 tokens-per-parameter 区间可以节省高达 60% 的计算量。但衰减到 0 后无法继续训练（学习率已归零），所以要继续训练的场景通常保留一个小的 `min_lr_ratio`（1%-5%）。固定预算的一次性训练可以考虑衰减到 0。

**Q6：One-Cycle 调度的特点是什么？为什么大模型不用？**

One-Cycle 是学习率先大幅上涨再大幅下降（比普通 warmup 涨得更快更高），动量同步反向变化，适合训练步数有限的中小模型快速实验，能在更少的 epoch 内收敛。大模型不用的原因：激进的 LR 上升阶段对超长序列训练不稳定；大模型对初期 warmup 质量敏感，One-Cycle 的极端峰值容易引起 loss spike；且 One-Cycle 必须提前固定总步数，灵活性差。

**Q7：β2=0.95 和 0.999 在 warmup 的意义上有什么不同？**

$\beta_2$ 决定了二阶矩的有效窗口大小：0.999 约 1000 步，0.95 约 20 步。$\beta_2 = 0.95$ 让 $v_t$ 更快稳定，warmup 所需步数可以更短；$\beta_2 = 0.999$ 收敛更慢，如果 warmup 太短，二阶矩还未稳定就切换到峰值学习率，容易不稳定。这也是大模型训练推荐 $\beta_2 = 0.95$ 的原因之一。
