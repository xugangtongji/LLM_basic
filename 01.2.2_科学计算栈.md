## 一、NumPy：向量化思维是核心

### 1.1 初印象：用费曼视角理解

**NumPy 的核心就一件事：让 Python 不写循环也能快速操作大量数字**。

Python 的 for 循环很慢，因为每次循环都要处理类型检查、动态分发等开销。NumPy 把数据放进一块连续的内存区域（ndarray），然后调用底层 C / Fortran 代码批量处理——速度比纯 Python 快 100 到 1000 倍。

比喻：你要给 100 个瓶子装水。Python 循环是你一个人跑来跑去一瓶一瓶装；NumPy 向量化是一条自动化流水线，100 个瓶子同时装，你只是按了一下开关。

**形状约定（与 PyTorch 一致）**：

| 维度缩写 | 含义 |
|---------|------|
| B | Batch size |
| L / T / S | 序列长度（Length / Time / Seq） |
| H | 注意力头数 |
| D / d_model | 隐层维度 |
| V | 词表大小 |

典型张量形状：`(B, L, D)` 是大模型里最常见的三维张量形状。

### 1.3 向量化：消灭 Python for 循环

**规则只有一条：能用数组操作做到的，就不要写 for 循环**。

```python
# 实例：手算 softmax（大模型里的核心操作）

scores = np.random.randn(4, 10)  # 4个样本，10类

# 错误：用 Python 循环（极慢）
result_slow = np.zeros_like(scores)
for i in range(scores.shape[0]):
    row = scores[i]
    row_max = row.max()
    exp_row = np.exp(row - row_max)
    result_slow[i] = exp_row / exp_row.sum()

# 正确：向量化（快 100x 以上）
def softmax(x):
    # keepdims=True 保持维度，使 broadcasting 正确
    x_max = x.max(axis=-1, keepdims=True)   # (4, 1)
    e_x = np.exp(x - x_max)                  # (4, 10) broadcasting
    return e_x / e_x.sum(axis=-1, keepdims=True)

result_fast = softmax(scores)

# 验证结果一致
assert np.allclose(result_slow, result_fast)
print("两种方法结果相同")

# 速度对比
import time

big = np.random.randn(1000, 10000)

t0 = time.perf_counter()
for i in range(big.shape[0]):
    row = big[i]; e = np.exp(row - row.max()); e / e.sum()
slow_time = time.perf_counter() - t0

t0 = time.perf_counter()
softmax(big)
fast_time = time.perf_counter() - t0

print(f"循环版：{slow_time:.3f}s   向量化：{fast_time:.4f}s   加速比：{slow_time/fast_time:.0f}x")
```

### 1.4 Broadcasting：形状不同也能运算

**Broadcasting 是 NumPy 最精妙的设计，也是最容易出 bug 的地方**。

规则（从右对齐逐维比较）：
1. 维度数不同时，在左边补 1
2. 每一维，两个值要么相等，要么有一个是 1
3. 是 1 的那个维度，运算时自动"重复"到另一个的大小

```python
# 直觉示例：给 batch 里的每个样本减去均值（层归一化的核心操作）
X = np.random.randn(32, 512)    # (B=32, D=512)

# 计算每个样本的均值和标准差
mean = X.mean(axis=-1, keepdims=True)   # (32, 1)  <- keepdims 很关键
std  = X.std(axis=-1, keepdims=True)    # (32, 1)

X_norm = (X - mean) / (std + 1e-8)     # (32,512) - (32,1) → broadcasting 自动扩展
# 等价于把 mean 和 std 各复制 512 份拼成 (32, 512) 再做减法
# 但 NumPy 不会真的复制，只是虚拟扩展，内存零开销

# broadcasting 规则验证
print(X_norm.shape)          # (32, 512)
print(X_norm.mean(axis=-1))  # 每个样本均值约为 0
print(X_norm.std(axis=-1))   # 每个样本标准差约为 1
```

一个更复杂的 Broadcasting 场景：计算两组向量的**余弦相似度矩阵**（RAG 检索的基础）：

```python
def cosine_similarity_matrix(A, B):
    """
    A: (n, d)  查询向量组
    B: (m, d)  文档向量组
    返回: (n, m) 相似度矩阵
    """
    # L2 归一化
    A_norm = A / np.linalg.norm(A, axis=-1, keepdims=True)   # (n, d)
    B_norm = B / np.linalg.norm(B, axis=-1, keepdims=True)   # (m, d)

    # 矩阵乘法得到相似度矩阵
    # (n, d) @ (d, m) = (n, m)
    return A_norm @ B_norm.T

queries = np.random.randn(5, 128)   # 5 个查询
docs    = np.random.randn(100, 128) # 100 个文档
sim     = cosine_similarity_matrix(queries, docs)   # (5, 100)
top3    = np.argsort(sim, axis=-1)[:, -3:]          # 每个查询的 top-3 文档索引
print(top3.shape)  # (5, 3)
```

## 二、Pandas：分析实验结果的瑞士军刀

### 2.1 初印象：用费曼视角理解

**Pandas 就是给 Python 装上了 Excel 超级进化版**。

你有一个训练日志 CSV，里面有 step、loss、lr、val_loss 四列数据，共 10 万行。Pandas 让你用一行代码读进来，用一行代码按 epoch 分组求均值，用一行代码找到验证损失最低的那个 checkpoint——而不需要写任何循环。

在大模型工程里，Pandas 主要用于：
- 读取 / 清洗评测结果（MMLU、GSM8K 等）
- 分析训练日志（loss 曲线、学习率调度）
- 整理人工标注数据、偏好数据集（RLHF 的 preference pairs）
- 对比不同超参实验的结果

### 2.5 实战：清洗训练日志

```python
import pandas as pd

# 模拟从 wandb 导出的训练日志
np.random.seed(0)
steps = range(0, 10001, 10)
log = pd.DataFrame({
    "step":     steps,
    "loss":     2.5 * np.exp(-np.arange(len(steps)) * 0.001) + 0.1 * np.random.randn(len(steps)),
    "val_loss": 2.8 * np.exp(-np.arange(len(steps)) * 0.0008) + 0.15 * np.random.randn(len(steps)),
    "lr":       np.concatenate([
        np.linspace(0, 1e-4, 200),      # warmup
        np.ones(801) * 1e-4,             # 稳定期
    ]),
})

# 常用清洗操作
log = log.dropna()                                 # 删除缺失行
log["loss_smooth"] = log["loss"].rolling(20).mean()  # 平滑曲线（滑动平均）
log["overfit_gap"] = log["val_loss"] - log["loss"]   # 过拟合差距

# 找最佳 checkpoint（val_loss 最低时的 step）
best_step = log.loc[log["val_loss"].idxmin(), "step"]
best_val  = log["val_loss"].min()
print(f"最佳 checkpoint: step={best_step}，val_loss={best_val:.4f}")

# 按区间分析：每 1000 步一段
log["phase"] = (log["step"] // 1000).astype(str) + "k"
phase_stats = log.groupby("phase")[["loss", "val_loss"]].agg(["mean", "min"])
print(phase_stats)
```

## 三、Matplotlib：让实验结果开口说话

### 3.1 初印象：用费曼视角理解

**Matplotlib 就是 Python 版的白板**，你想画什么都行。

在大模型工程里，最常画的三类图：
- **训练曲线**：loss / val_loss / learning rate 随 step 变化

- **评测对比图**：多模型多任务的柱状图或雷达图

这些图既是调试工具（发现训练异常），也是论文和汇报必备。

### 3.3 画注意力热力图

```python
def plot_attention_heatmap(attn_weights, tokens, layer=0, head=0):
    """
    可视化 Transformer 注意力矩阵。
    attn_weights: ndarray，shape=(num_layers, num_heads, seq_len, seq_len)
    tokens: list of str，token 文本
    """
    fig, ax = plt.subplots(figsize=(8, 7))

    attn = attn_weights[layer, head]   # (seq_len, seq_len)
    seq_len = len(tokens)

    im = ax.imshow(attn[:seq_len, :seq_len], cmap="Blues", aspect="auto", vmin=0, vmax=1)
    plt.colorbar(im, ax=ax, fraction=0.046, pad=0.04)

    ax.set_xticks(range(seq_len))
    ax.set_yticks(range(seq_len))
    ax.set_xticklabels(tokens, rotation=45, ha="right", fontsize=9)
    ax.set_yticklabels(tokens, fontsize=9)
    ax.set_title(f"Attention Map — Layer {layer}, Head {head}", fontsize=12, pad=12)
    ax.set_xlabel("Key Tokens", fontsize=10)
    ax.set_ylabel("Query Tokens", fontsize=10)

    plt.tight_layout()
    plt.savefig(f"attn_L{layer}_H{head}.png", dpi=150, bbox_inches="tight")
    plt.show()

# 示例调用
tokens = ["[BOS]", "The", "quick", "brown", "fox", "jumps", "[EOS]"]
fake_attn = np.random.rand(6, 8, len(tokens), len(tokens))
fake_attn = fake_attn / fake_attn.sum(axis=-1, keepdims=True)  # 归一化
plot_attention_heatmap(fake_attn, tokens, layer=2, head=3)
```

### 3.4 画多模型对比柱状图

```python
def plot_benchmark_comparison(results: dict, title: str = "Benchmark Results"):
    """
    results: {model_name: {task: score}}
    示例: {"LLaMA-7B": {"MMLU": 0.53, "GSM8K": 0.35, "HumanEval": 0.14}}
    """
    models = list(results.keys())
    tasks  = list(next(iter(results.values())).keys())
    n_tasks  = len(tasks)
    n_models = len(models)

    x = np.arange(n_tasks)
    width = 0.8 / n_models
    colors = ["#e05c5c", "#4a90d9", "#5cb85c", "#f0a500", "#9b59b6"]

    fig, ax = plt.subplots(figsize=(10, 5))

    for i, (model, scores) in enumerate(results.items()):
        vals = [scores[t] for t in tasks]
        bars = ax.bar(x + i * width - (n_models - 1) * width / 2,
                      vals, width=width * 0.9,
                      color=colors[i % len(colors)], label=model, alpha=0.85)
        # 在柱顶标注数值
        for bar, v in zip(bars, vals):
            ax.text(bar.get_x() + bar.get_width() / 2, bar.get_height() + 0.005,
                    f"{v:.1%}", ha="center", va="bottom", fontsize=8)

    ax.set_xticks(x)
    ax.set_xticklabels(tasks, fontsize=11)
    ax.set_ylabel("Score", fontsize=11)
    ax.set_ylim(0, 1.1)
    ax.set_title(title, fontsize=13, fontweight="bold")
    ax.legend(fontsize=10)
    ax.grid(axis="y", alpha=0.3)
    ax.yaxis.set_major_formatter(plt.FuncFormatter(lambda v, _: f"{v:.0%}"))
```

## 四、常见陷阱

**NumPy 陷阱**

1. **视图 vs 拷贝**：切片产生视图（修改会影响原数组），花式索引产生拷贝。不确定时用 `.copy()` 显式拷贝。
    ```python
    a = np.array([1, 2, 3, 4])
    b = a[1:3]   # 视图
    b[0] = 99    # a 也变了！
    c = a[1:3].copy()  # 安全
    ```

2. **broadcasting 产生意外大数组**：`(1000, 1, 512) + (1, 1000, 512)` 会产生 `(1000, 1000, 512)` 约 2GB，注意检查中间结果大小。

3. **整数除法陷阱**：`np.array([1, 2, 3]) / 2` 在 Python 3 中是浮点除法（结果 `[0.5, 1.0, 1.5]`），但在某些老版本或 dtype=int 场景要注意。

4. **axis 方向搞混**：`axis=0` 沿"行方向"压缩（结果减少一行），不是"对第 0 行操作"。建议配合 `keepdims=True` 和打印 shape 验证。

**Pandas 陷阱**

5. **链式赋值（SettingWithCopyWarning）**：
    ```python
    df[df["loss"] > 1]["loss"] = 0  # 可能无效！
    df.loc[df["loss"] > 1, "loss"] = 0  # 正确写法
    ```

6. **`apply` 很慢**：遇到逐行操作，优先用向量化方法（`str.` 系列、`pd.cut`、`np.where`）替代 `df.apply(lambda row: ...)`。

**Matplotlib 陷阱**

7. **中文显示乱码**：需要指定中文字体或用英文标签。
    ```python
    plt.rcParams["font.sans-serif"] = ["SimHei"]   # Windows
    plt.rcParams["axes.unicode_minus"] = False
    ```

8. **`plt.show()` 在 Jupyter 外阻塞进程**：训练脚本里应只用 `plt.savefig()`，不调用 `plt.show()`，否则会阻塞训练循环。

## 五、面试常见问题

**Q1：NumPy 为什么比 Python 的 for 循环快那么多？**

两个原因：底层用 C/Fortran 实现，省去了 Python 解释器的动态类型开销；以及使用了 SIMD 指令集，CPU 一次能对多个数据做相同操作（Single Instruction Multiple Data）。一个简单的向量加法，NumPy 可以比 Python 快 100 倍以上。

**Q2：什么是 Broadcasting？用一句话解释它的规则。**

两个形状不同的数组运算时，从右往左逐维对齐，只要每一维的大小"相等或其中一个是 1"就兼容——是 1 的那个维度会被虚拟重复扩展到另一个的大小，不实际分配内存。

**Q3：NumPy 的 `view` 和 `copy` 有什么区别？什么时候用哪个？**

`view` 和原数组共享同一块内存，修改 view 会影响原数组，但速度快、不占额外内存（切片操作默认返回 view）。`copy` 是完全独立的副本，修改不影响原数组，但多用内存。数据预处理时如果要修改又不想影响原数据，显式调用 `.copy()`。

**Q4：在 Pandas 里如何高效地对 DataFrame 做条件过滤和赋值？**

用 `.loc[布尔条件, 列名]` 的写法，不要用链式索引（先 `df[条件]` 再 `["列名"] = 值`），因为链式索引可能操作的是临时副本而非原 DataFrame，会触发 `SettingWithCopyWarning` 且赋值可能无效。

**Q5：训练日志的 loss 曲线抖动很剧烈，用 Pandas 怎么平滑？**

用 `rolling` 滑动窗口求均值：`df["loss_smooth"] = df["loss"].rolling(window=50, center=True, min_periods=1).mean()`。`center=True` 让窗口以当前点为中心而非向前看，视觉上更平滑；`min_periods=1` 让开头结尾的数据也能计算而非产生 NaN。

**Q6：NumPy 和 PyTorch Tensor 的互操作需要注意什么？**

两点：首先，CPU Tensor 用 `.numpy()` 转换时和 Tensor 共享内存，修改 numpy 数组会改变 Tensor，反之亦然，要格外小心；其次，GPU Tensor 必须先 `.cpu()` 才能转 numpy。生产代码里推荐用 `tensor.detach().cpu().numpy()` 显式断开梯度跟踪再转换。

**Q7：画训练曲线时，为什么原始曲线要配合平滑曲线一起展示？**

原始曲线保留了真实的噪声信息（方便发现异常跳变），平滑曲线展示了整体趋势（方便判断是否过拟合、收敛情况）。只展示平滑曲线可能掩盖训练中的梯度爆炸或 NaN 问题；只展示原始曲线则太难看出趋势。
