## 零、开篇：经典网络没有死，只是换了外衣

大模型时代，MLP、CNN、RNN 这些架构经常被认为已经"过时"。但如果你仔细拆解现代大模型，会发现它们的影子无处不在：

- **MLP**：Transformer 的 FFN 层就是两层 MLP，占了大模型约 2/3 的参数
- **CNN 的局部特征提取思想**：Mamba 训练时在卷积模式下并行计算，本质上等同于在序列上做因果卷积
- **RNN 的隐状态传递思想**：Mamba 推理时在循环模式下逐步更新，比 Transformer 快且内存固定
- **LSTM 的门控机制**：直接启发了 Transformer 的注意力门控，以及 2024 年新出的 minLSTM / xLSTM 架构

更重要的是：RNN、LSTM 历史上一直是序列建模的主力，直到 Transformer 出现才发生了范式转移。理解它们的局限和突破，是理解 Transformer 为什么要这样设计的最短路径。

本章的核心主线：**每种架构解决了什么问题，又带来了什么新问题**，形成一条清晰的技术演进链。

## 一、MLP：万物的基础构件

### 1.1 初印象：用费曼视角理解

**MLP 是"全连接的信息混合机器"**——输入的每一个维度都会影响输出的每一个维度。

就像一个超级会议室，里面的所有人（输入维度）都能和所有人（输出维度）互相交流，没有任何限制。这带来了极强的表达能力，但代价是参数量巨大（$d_{\text{in}} \times d_{\text{out}}$ 个权重）。

### 1.2 Transformer FFN 就是 MLP

大模型里每个 Transformer 块的 FFN 是一个两层 MLP（+ 激活函数）：

$$\text{FFN}(x) = W_2 \cdot \text{activation}(W_1 x + b_1) + b_2$$

**关键认识**：Transformer 里 **FFN 的参数量约占总参数的 2/3**（Attention 占 1/3）。对于 LLaMA-7B，FFN 约有 4.4B 参数，Attention 层约有 2.3B。

```python
import torch
import torch.nn as nn

def count_transformer_params(d_model=4096, n_heads=32, n_layers=32, vocab=32000):
    """估算 Transformer 各部分参数量"""
    d_ff = int(d_model * 8 / 3)  # SwiGLU
    d_ff = ((d_ff + 63) // 64) * 64

    # 每层
    attn_params = 4 * d_model * d_model   # Q K V O 各一个 d×d
    ffn_params  = 3 * d_model * d_ff      # SwiGLU: gate + up + down

    total_attn = attn_params * n_layers
    total_ffn  = ffn_params * n_layers
    embed_params = vocab * d_model         # Embedding（权重绑定时不算 LM Head）

    total = total_attn + total_ffn + embed_params
    print(f"Attention 层总计: {total_attn/1e9:.2f}B ({total_attn/total:.1%})")
    print(f"FFN 层总计:       {total_ffn/1e9:.2f}B  ({total_ffn/total:.1%})")
    print(f"Embedding:        {embed_params/1e9:.2f}B  ({embed_params/total:.1%})")
    print(f"总参数量:         {total/1e9:.2f}B")

count_transformer_params()
# Attention 层总计: ~2.15B (30%)
# FFN 层总计:       ~4.61B (64%)
# Embedding:        ~0.13B (2%)
# 总参数量:         ~7.15B
```

MLP 作为独立模型的局限：**无法处理顺序信息**。把"我爱你"和"你爱我"送进 MLP，如果把三个词的顺序打乱，输出完全相同——MLP 不知道谁先谁后。这就催生了 CNN 和 RNN。

## 二、CNN：局部感受野与特征层次

### 2.1 初印象：用费曼视角理解

**CNN 是"用一个小窗口滑动扫描，提取局部特征"**。

想象你在审阅一份报纸，不是整张纸同时看（全连接），而是用一个放大镜（卷积核）从左到右逐段扫过，在每个位置提取当前区域的特征。不同的放大镜（不同的卷积核）提取不同的特征——有的找边缘，有的找纹理。

CNN 的三大设计原则：**局部连接**（每个神经元只看邻域）、**权重共享**（同一个卷积核在所有位置用同一组权重）、**层次化**（浅层识别边缘，深层识别轮廓，更深层识别物体）。

### 2.2 一维卷积与 NLP

CNN 在 NLP 里以一维卷积的形式使用，窗口在序列上滑动：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# 一维因果卷积（序列建模的基础）
class CausalConv1d(nn.Module):
    """
    因果卷积：只能看到当前位置及之前的信息，不看未来。
    通过左侧 padding 实现（右侧不 pad）。
    """
    def __init__(self, in_channels, out_channels, kernel_size, dilation=1):
        super().__init__()
        self.padding = (kernel_size - 1) * dilation   # 只在左侧 pad
        self.conv = nn.Conv1d(
            in_channels, out_channels, kernel_size,
            dilation=dilation, padding=0
        )

    def forward(self, x):
        # x: (B, C, L)
        x = F.pad(x, (self.padding, 0))   # 只在左侧补零
        return self.conv(x)


class TextCNN(nn.Module):
    """
    文本 CNN：用不同窗口大小的卷积核捕捉不同 n-gram 特征，
    全局最大池化后分类。经典的文本分类模型。
    """
    def __init__(self, vocab_size, embed_dim, num_filters, kernel_sizes, n_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.convs = nn.ModuleList([
            nn.Conv1d(embed_dim, num_filters, k) for k in kernel_sizes
        ])
        self.classifier = nn.Linear(num_filters * len(kernel_sizes), n_classes)
        self.dropout = nn.Dropout(0.5)

    def forward(self, x):
        # x: (B, L)
        emb = self.embedding(x).transpose(1, 2)   # (B, E, L)
        # 每个卷积核提取特征，再全局最大池化
        features = [F.relu(conv(emb)).max(dim=-1).values for conv in self.convs]
        out = torch.cat(features, dim=-1)          # (B, num_filters * len(kernel_sizes))
        return self.classifier(self.dropout(out))

# 测试
model = TextCNN(vocab_size=10000, embed_dim=128, num_filters=100,
                kernel_sizes=[2, 3, 4], n_classes=5)
x = torch.randint(0, 10000, (32, 50))
print(model(x).shape)   # (32, 5)
```

### 2.3 CNN 的思想在大模型中的延续

S4 的一个关键洞见是：状态空间模型在训练时的行为等价于一个卷积神经网络（CNN），而在推理时的行为等价于一个 RNN。这使得 Mamba 既可以在训练阶段利用卷积的并行性，又可以在推理阶段利用 RNN 的常量内存优势。

CNN 的核心贡献——**局部感受野 + 权重共享 + 层次特征**——启发了 Transformer 的局部注意力（稀疏 Attention）、卷积位置编码，以及 SSM 的卷积训练模式。

## 三、RNN：序列的记忆机器

### 3.1 初印象：用费曼视角理解

**RNN 是"带记忆的序列处理器"**——每一步处理当前输入时，都能参考之前所有步骤积累下来的"记忆"（隐状态）。

就像你在听一段话，脑子里会保留一个"到目前为止的印象"（隐状态），每听到一个新词，就把这个词和之前的印象结合起来，更新出新的印象。

### 3.2 数学形式

在时间步 $t$，RNN 的更新方程：

$$\mathbf{h}_t = \tanh(W_h \mathbf{h}_{t-1} + W_x \mathbf{x}_t + \mathbf{b})$$

$$\mathbf{y}_t = W_y \mathbf{h}_t$$

其中 $\mathbf{h}_t \in \mathbb{R}^d$ 是隐状态，$\mathbf{x}_t$ 是输入，$\mathbf{y}_t$ 是输出。

```python
import torch
import torch.nn as nn

class SimpleRNN(nn.Module):
    """手写简单 RNN，用于理解原理"""
    def __init__(self, input_size, hidden_size, output_size):
        super().__init__()
        self.hidden_size = hidden_size
        self.Wh = nn.Linear(hidden_size, hidden_size)
        self.Wx = nn.Linear(input_size, hidden_size)
        self.Wy = nn.Linear(hidden_size, output_size)

    def forward(self, x, h0=None):
        # x: (B, L, input_size)
        B, L, _ = x.shape
        if h0 is None:
            h = torch.zeros(B, self.hidden_size, device=x.device)
        else:
            h = h0

        outputs = []
        for t in range(L):
            h = torch.tanh(self.Wh(h) + self.Wx(x[:, t, :]))
            outputs.append(self.Wy(h).unsqueeze(1))

        return torch.cat(outputs, dim=1), h  # (B, L, out), (B, hidden)

# PyTorch 内置 RNN（更高效，有 CUDA kernel）
rnn = nn.RNN(input_size=64, hidden_size=256, num_layers=2,
             batch_first=True, bidirectional=False)
x = torch.randn(32, 50, 64)    # (B, L, input)
out, hn = rnn(x)
print(f"输出: {out.shape}   隐状态: {hn.shape}")
# (32, 50, 256)  (2, 32, 256)
```

### 3.3 RNN 的两大缺陷

**缺陷一：梯度消失 / 爆炸**

RNN 通过反向传播时间（BPTT）来训练，梯度需要通过许多时间步反向传播，这导致了梯度消失或爆炸问题，使得 RNN 难以学习长距离依赖。

**缺陷二：无法并行**

RNN 本质上是串行的——每一步必须等待前一步完成，这严重限制了训练效率和可扩展性。Transformer 的 Self-Attention 一次性看所有位置，天然并行，这是它训练效率远超 RNN 的根本原因。

## 四、LSTM：门控机制的经典设计

### 4.1 初印象：用费曼视角理解

**LSTM 是"带选择性记忆和遗忘的 RNN"**。

普通 RNN 的记忆是无差别混合的——新信息直接覆盖旧信息，长距离的信息很快被稀释。LSTM 引入了三个"门"：

- **遗忘门（Forget Gate）**：决定要从长期记忆里丢弃什么（"这件事不重要了，忘掉"）
- **输入门（Input Gate）**：决定要把什么新信息写入长期记忆（"这件事很重要，记住"）
- **输出门（Output Gate）**：决定要从长期记忆里读出什么来影响当前输出（"现在需要用这个记忆"）

这三个门都是 sigmoid 函数（输出 0 到 1），充当"阀门"——接近 0 表示"关闭/丢弃"，接近 1 表示"开放/保留"。

### 4.2 完整的 LSTM 方程

设输入 $\mathbf{x}_t$，前一步隐状态 $\mathbf{h}_{t-1}$，前一步细胞状态 $\mathbf{c}_{t-1}$：

**遗忘门**：决定丢弃多少旧记忆

$$\mathbf{f}_t = \sigma(W_f [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_f)$$

**输入门 + 候选记忆**：决定写入什么新信息

$$\mathbf{i}_t = \sigma(W_i [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_i)$$
$$\tilde{\mathbf{c}}_t = \tanh(W_c [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_c)$$

**细胞状态更新**：旧记忆选择性遗忘 + 新信息选择性写入

$$\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t$$

**输出门**：决定当前输出什么

$$\mathbf{o}_t = \sigma(W_o [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_o)$$
$$\mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{c}_t)$$

```python
import torch
import torch.nn as nn

class LSTMCell(nn.Module):
    """手写 LSTM 单元，展示四个门的计算"""

    def __init__(self, input_size: int, hidden_size: int):
        super().__init__()
        # 把四个门的线性层合并成一个（效率更高）
        # 输出维度 = 4 * hidden_size（对应 f、i、g、o 四个门）
        self.linear = nn.Linear(input_size + hidden_size, 4 * hidden_size)
        self.hidden_size = hidden_size

    def forward(self, x, state):
        h, c = state                          # 隐状态 和 细胞状态

        # 合并输入和隐状态，一次线性变换
        gates = self.linear(torch.cat([h, x], dim=-1))  # (B, 4H)
        f, i, g, o = gates.chunk(4, dim=-1)              # 各 (B, H)

        f = torch.sigmoid(f)    # 遗忘门
        i = torch.sigmoid(i)    # 输入门
        g = torch.tanh(g)       # 候选记忆
        o = torch.sigmoid(o)    # 输出门

        c_new = f * c + i * g                # 细胞状态更新
        h_new = o * torch.tanh(c_new)        # 隐状态输出

        return h_new, (h_new, c_new)


# 对比 PyTorch 内置 LSTM（速度快 10-100 倍）
lstm_builtin = nn.LSTM(input_size=64, hidden_size=256, num_layers=2,
                        batch_first=True)
x = torch.randn(32, 50, 64)
out, (hn, cn) = lstm_builtin(x)
print(f"LSTM 输出: {out.shape}")
print(f"隐状态 h: {hn.shape}")   # (n_layers, B, H)
print(f"细胞状态 c: {cn.shape}") # (n_layers, B, H)
```

### 4.3 LSTM 的思想遗产

LSTM 的门控思想是深度学习史上最重要的发明之一。尽管 Transformer 的出现让 LSTM 退出了主流 NLP，但它的门控机制直接启发了 Transformer 的注意力门控和现代 SSM（Mamba）的选择性机制。

2024 年出现的 **xLSTM**（Extended LSTM）通过引入矩阵值记忆单元和指数门控，使 LSTM 重新具备了与 Transformer 竞争的能力，形成了 LLM 架构的新选项。

## 五、GRU：LSTM 的精简版

### 5.1 初印象：用费曼视角理解

**GRU 是"把 LSTM 的三个门缩减为两个门"的精简设计**。

Cho 等人在 2014 年提出：LSTM 的细胞状态 $\mathbf{c}$ 和隐状态 $\mathbf{h}$ 功能上有一些重叠，可以合并。于是把遗忘门和输入门合并成一个"更新门"，去掉细胞状态，只保留隐状态。

结果：参数量减少约 25%，训练更快，在中等长度序列上效果和 LSTM 相当。

### 5.2 GRU 方程

**重置门**（决定多少旧信息影响候选状态）：

$$\mathbf{r}_t = \sigma(W_r [\mathbf{h}_{t-1}, \mathbf{x}_t])$$

**更新门**（决定保留多少旧状态 vs 新候选）：

$$\mathbf{z}_t = \sigma(W_z [\mathbf{h}_{t-1}, \mathbf{x}_t])$$

**候选隐状态**：

$$\tilde{\mathbf{h}}_t = \tanh(W_h [\mathbf{r}_t \odot \mathbf{h}_{t-1}, \mathbf{x}_t])$$

**最终隐状态**（用更新门在旧状态和候选状态之间插值）：

$$\mathbf{h}_t = (1 - \mathbf{z}_t) \odot \mathbf{h}_{t-1} + \mathbf{z}_t \odot \tilde{\mathbf{h}}_t$$

```python
import torch
import torch.nn as nn

class GRUCell(nn.Module):
    """手写 GRU 单元"""
    def __init__(self, input_size, hidden_size):
        super().__init__()
        # 重置门和更新门的线性层
        self.linear_rz = nn.Linear(input_size + hidden_size, 2 * hidden_size)
        # 候选隐状态的线性层
        self.linear_h  = nn.Linear(input_size + hidden_size, hidden_size)

    def forward(self, x, h):
        # 计算重置门和更新门
        rz = torch.sigmoid(self.linear_rz(torch.cat([h, x], dim=-1)))
        r, z = rz.chunk(2, dim=-1)

        # 候选隐状态（使用重置门过滤旧隐状态）
        h_cand = torch.tanh(self.linear_h(torch.cat([r * h, x], dim=-1)))

        # 更新隐状态（在旧状态和候选状态之间插值）
        h_new = (1 - z) * h + z * h_cand
        return h_new

# PyTorch 内置 GRU
gru = nn.GRU(input_size=64, hidden_size=256, num_layers=2, batch_first=True)
x = torch.randn(32, 50, 64)
out, hn = gru(x)
print(f"GRU 输出: {out.shape}   隐状态: {hn.shape}")

# LSTM vs GRU 参数量对比
lstm = nn.LSTM(64, 256)
gru_model = nn.GRU(64, 256)
lstm_params = sum(p.numel() for p in lstm.parameters())
gru_params  = sum(p.numel() for p in gru_model.parameters())
print(f"LSTM 参数量: {lstm_params:,}   GRU 参数量: {gru_params:,}")
print(f"GRU 比 LSTM 参数少: {(1-gru_params/lstm_params):.1%}")
```

## 六、思想传承：RNN → Transformer → Mamba

### 6.1 各架构的核心权衡

| 架构 | 训练并行 | 推理内存 | 长序列能力 | 参数效率 |
|------|---------|---------|-----------|---------|
| RNN | 串行（慢） | $O(1)$（常量） | 差（梯度消失） | 高 |
| LSTM / GRU | 串行（慢） | $O(1)$ | 中 | 中 |
| Transformer | 并行（快） | $O(L^2)$（随序列长度二次增长） | 强（但贵） | 低 |
| Mamba（SSM） | 训练并行 | $O(L)$ 推理 $O(1)$ | 强（且快） | 中 |

### 6.2 Mamba：CNN + RNN 的统一

Mamba 的核心在于状态空间模型可以有三种等价表示：连续形式、卷积形式和递归形式。训练时使用卷积形式（可并行），推理时使用递归形式（常量内存）。这一设计使得 Mamba 既能像 CNN 一样高效训练，又能像 RNN 一样高效推理。

```python
# Mamba 的核心思想（简化版 SSM）
# 三种等价表示：
#
# 1. 连续形式（微分方程）
#    dh/dt = A·h(t) + B·x(t)
#    y(t) = C·h(t)
#
# 2. 离散递归形式（推理时使用，等价于 RNN）
#    h_t = Ā·h_{t-1} + B̄·x_t       # 常量内存，O(1)/步
#    y_t = C·h_t
#
# 3. 卷积形式（训练时使用，等价于 CNN，可并行）
#    y = x * K                        # K 是由 Ā、B̄、C 导出的卷积核
#    可以用 FFT 高效计算，O(L log L)

class SimplifiedSSM(nn.Module):
    """
    极简 SSM 演示两种推理模式。
    真实 Mamba 的 A、B、C 是依赖输入的（选择性机制），
    这里用固定参数简化展示核心思想。
    """
    def __init__(self, d_model: int, d_state: int = 16):
        super().__init__()
        self.d_model = d_model
        self.d_state = d_state

        # SSM 参数（真实 Mamba 中这些由输入动态生成）
        self.A = nn.Parameter(torch.randn(d_model, d_state))
        self.B = nn.Parameter(torch.randn(d_model, d_state))
        self.C = nn.Parameter(torch.randn(d_model, d_state))

    def recurrent_mode(self, x: torch.Tensor):
        """
        推理模式：逐步递归，O(1) 内存（等价于 RNN）
        x: (B, L, D)
        """
        B_size, L, D = x.shape
        h = torch.zeros(B_size, D, self.d_state, device=x.device)
        outputs = []
        for t in range(L):
            # 状态更新（简化的离散化）
            A_bar = torch.sigmoid(self.A)   # 简化
            h = A_bar.unsqueeze(0) * h + self.B.unsqueeze(0) * x[:, t, :].unsqueeze(-1)
            y = (h * self.C.unsqueeze(0)).sum(-1)  # (B, D)
            outputs.append(y.unsqueeze(1))
        return torch.cat(outputs, dim=1)   # (B, L, D)

# 注意：真实 Mamba 实现需要 CUDA 优化（selective_scan_cuda）
# 参见 state-spaces/mamba 官方仓库
```

### 6.3 minLSTM / minGRU：RNN 的文艺复兴

2024 年，Mila 研究团队提出 minLSTM 和 minGRU：通过移除门控对前一步隐状态的依赖，使模型可以完全并行训练。minGRU 比传统 GRU 减少参数量高达 87%，minLSTM 减少 85%，同时在选择性复制、强化学习等任务上取得与 Mamba 相当的性能。

这说明 RNN 的思想并未过时，只是需要工程上的突破（去掉顺序依赖）才能在大规模场景重焕生机。

```python
class MinGRU(nn.Module):
    """
    minGRU（2024）：去掉隐状态依赖的极简 GRU，可全并行训练
    论文：Were RNNs All We Needed? (Feng et al., 2024)
    """
    def __init__(self, d_model: int):
        super().__init__()
        # 不依赖 h_{t-1}，只依赖 x_t，可以并行计算所有时间步
        self.linear_z = nn.Linear(d_model, d_model)   # 更新门
        self.linear_h = nn.Linear(d_model, d_model)   # 候选隐状态

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (B, L, D)
        由于门不依赖 h_{t-1}，可以对 L 维度完全并行计算
        """
        # 更新门和候选状态都只依赖 x（不依赖 h），可以并行
        z = torch.sigmoid(self.linear_z(x))   # (B, L, D) 完全并行
        h_tilde = torch.relu(self.linear_h(x)) # (B, L, D) 完全并行

        # 串行的部分：h_t = (1-z_t)*h_{t-1} + z_t*h_tilde_t
        # 但可以用 parallel scan 算法在 O(log L) 步内并行完成
        # 这里用简单循环演示逻辑
        B, L, D = x.shape
        h = torch.zeros(B, D, device=x.device)
        outputs = []
        for t in range(L):
            h = (1 - z[:, t]) * h + z[:, t] * h_tilde[:, t]
            outputs.append(h.unsqueeze(1))
        return torch.cat(outputs, dim=1)   # (B, L, D)
```

## 七、面试常见问题

**Q1：RNN 为什么有梯度消失问题？LSTM 是如何缓解的？**

RNN 反向传播时，梯度需要沿时间步从后往前传，每步都要乘以 $W_h^T$ 和激活函数的导数（Tanh 最大为 1）。当序列很长时，矩阵连乘导致梯度指数衰减（若谱范数 < 1）或爆炸（若 > 1），无法学习远距离依赖。LSTM 引入细胞状态 $c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$，梯度可以沿细胞状态路径直接传播（类似残差连接），遗忘门 $f_t$ 接近 1 时梯度几乎无损通过，大幅缓解梯度消失。

**Q2：LSTM 和 GRU 各有几个门？哪个更好？**

LSTM 有三个门（遗忘门、输入门、输出门）和两个状态（隐状态 $h$、细胞状态 $c$），参数量更大。GRU 只有两个门（重置门、更新门）和一个状态，参数少约 25%，训练更快。哪个更好取决于任务：长序列、需要精细记忆控制时 LSTM 略优；资源受限或中等序列长度时 GRU 性价比更高。在大规模 LLM 中两者都已被 Transformer 取代，但 GRU 思想直接影响了 Mamba 的门控设计。

**Q3：CNN 和 RNN 处理序列的方式有什么本质区别？**

CNN 用固定大小的窗口（kernel）在序列上滑动，每个位置只看局部邻域，可以完全并行；能捕捉局部模式（n-gram 特征），但感受野受限于层数 × kernel size，需要堆叠多层才能覆盖全局。RNN 逐步处理序列，每步的隐状态理论上包含所有历史信息，有全局感受野；但必须串行计算，无法并行，且长期记忆受梯度消失限制。

**Q4：Transformer 相比 RNN 的核心优势是什么？有什么代价？**

优势：Self-Attention 一次看所有位置，天然并行，训练速度快；直接建模任意位置对之间的依赖，无梯度消失问题；每个位置的表示直接依赖全局信息，表达能力强。代价：Attention 的计算和内存复杂度都是 $O(L^2)$（序列长度的平方），处理超长序列（如 100K+ tokens）代价极高；推理时 KV Cache 随序列长度线性增长，内存不是常量。

**Q5：Mamba 是如何做到既能并行训练又能高效推理的？**

Mamba 利用了状态空间模型的三种等价表示：连续微分方程形式、离散递归（RNN）形式和卷积形式。训练时用卷积形式（用 FFT 并行计算，避免了 RNN 的串行依赖），推理时用递归形式（每步只维护一个固定大小的隐状态，不随序列长度增长）。

**Q6：MLP 为什么无法直接处理序列？Transformer 是如何引入位置信息的？**

MLP 对输入的所有维度做全连接变换，对输入顺序不敏感——把"猫咬狗"和"狗咬猫"的词向量顺序打乱，MLP 输出相同。Transformer 通过加上位置编码（绝对或相对位置）来告诉模型"谁在哪个位置"，使模型可以区分不同位置的 token。没有位置编码的 Transformer 本质上也是位置无关的。

**Q7：为什么在 LLM 时代 RNN 的推理效率可能优于 Transformer？**

Transformer 推理时需要存储 KV Cache（随序列长度线性增长），自回归生成时每步都要把 Cache 加载到 GPU 上参与 Attention 计算，内存带宽是瓶颈。RNN（或 Mamba）推理时隐状态大小固定（不随序列长度变化），每步只需要更新这个固定大小的隐状态，内存和计算量都是 $O(1)$ per token。因此，在超长序列推理场景，RNN 类模型的推理效率优势显著。

**Q8：LSTM 里细胞状态 $c_t$ 和隐状态 $h_t$ 分别代表什么？**

$c_t$（细胞状态）是 LSTM 的"长期记忆"——通过加法更新（$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$），梯度可以无损流过（遗忘门接近 1 时），适合存储需要保持较长时间的信息。$h_t$（隐状态）是当前步的"工作记忆"——经过 $\tanh$ 和输出门压缩，是实际对外输出和传递给下一步的信号，变化更快，更关注当前输入。这种两级记忆设计让 LSTM 能同时维护长期和短期信息。
