## 零、开篇：把 Transformer 当成一台机器来拆解

上一章我们知道了 Transformer 的大方向——用注意力机制替代 RNN，实现并行计算和长距离依赖。

但"注意力机制"只是 Transformer 的核心零件，不是全部。就像一辆汽车不只有发动机，还需要方向盘、轮胎、变速箱一样，Transformer 也有几个不可缺少的基础组件。

这一章我们先不急着进入最复杂的注意力机制，先把其他几个"脚手架"组件搞清楚：

```
文字输入
   ↓
【组件一】Token Embedding：把文字变成数字向量
   ↓
【组件二】位置编码：告诉模型每个词在第几位
   ↓（这两步叫做"输入表示"）
┌─────────────────────────────────┐
│  Transformer 层（重复 N 次）      │
│                                 │
│  【组件三】注意力机制            │  ← 下一章详讲
│           ↓                    │
│  【组件四】残差连接              │  ← 本章讲
│           ↓                    │
│  【组件五】层归一化              │  ← 本章讲
│           ↓                    │
│  【组件六】前馈网络（FFN）        │  ← 本章讲
│           ↓                    │
│  【组件四】残差连接（再次）       │  ← 本章讲
│           ↓                    │
│  【组件五】层归一化（再次）       │  ← 本章讲
└─────────────────────────────────┘
   ↓
【组件七】输出层：把向量变回词语概率
```

本章依次讲解这七个组件，注意力机制留到下一章单独深讲。

## 一、组件一：Token Embedding——把文字变成数字

### 1.1 初印象：用费曼视角理解

**神经网络只认识数字，不认识文字。所以第一步，要把文字"翻译"成数字**。

但不是简单地给每个字分配一个编号（那样就会让"猫"=1，"狗"=2，"鱼"=3，模型会以为"猫"和"狗"的关系比"猫"和"鱼"更近，仅仅因为 1 和 2 比 1 和 3 更接近——这没有任何语义意义）。

更好的方式是：给每个词（或子词）分配一个**向量**，也就是一组数字。

比如，在一个 4 维的简化空间里（实际模型往往是 4096 维）：

```
"国王" → [0.9,  0.1, 0.8, 0.2]
"王后" → [0.9,  0.9, 0.8, 0.2]
"男人" → [0.1,  0.1, 0.9, 0.1]
"女人" → [0.1,  0.9, 0.9, 0.1]
"苹果" → [0.1,  0.2, 0.1, 0.9]
```

这些向量是**通过训练学出来的**，不是人工定义的。训练之后，语义相似的词，其向量在高维空间里也会靠近——"国王"和"王后"的向量比"国王"和"苹果"更接近。

这种把词语映射成向量的方法，叫做 **词嵌入（Word Embedding）**，存储这些向量的大表格，叫做 **Embedding 矩阵**。

### 1.2 Token 是什么（不是词）

注意：现代模型处理的最小单位不是"词"，而是 **Token**。

Token 可以是：
- 一个完整的词（如 "cat"、"苹果"）
- 一个词的一部分（如 "un-"、"-ing"、"play" 是 "unplaying" 的三个 token）
- 一个标点符号或空格

为什么不直接用词？因为词的数量太多（中文有几十万个词），用子词切分可以把词表大小控制在 3-15 万左右，同时仍能表示任意词语（没见过的词也能用子词拼出来）。

这个切分过程叫做 **分词（Tokenization）**，使用的工具叫 **分词器（Tokenizer）**。

```
例子：
"我喜欢深度学习" → ["我", "喜欢", "深度", "学习"]（可能每个字/词是一个 token）

"unbelievable" → ["un", "believ", "able"]（英文的子词切分）

每个 token 都在 Embedding 矩阵里查找对应的向量
```

### 1.3 一个简单的 Python 演示

```python
import torch
import torch.nn as nn

# 假设词表有 1000 个 token，每个 token 用 16 维向量表示
vocab_size  = 1000
embed_dim   = 16

# Embedding 矩阵：1000 行 × 16 列的可学习参数表
embedding = nn.Embedding(vocab_size, embed_dim)

# 输入：一个句子，被分词成 5 个 token 的编号
sentence = torch.tensor([42, 17, 8, 301, 56])   # 5 个 token id

# 查找每个 token 对应的向量
vectors = embedding(sentence)
print(f"输入：5 个 token")
print(f"输出：{vectors.shape}（5个token，每个用16维向量表示）")
# 输出：torch.Size([5, 16])

# 真实模型中，d_model 通常是 512、1024、4096 等
# LLaMA-3-8B 的 embed_dim = 4096，vocab_size = 32000
```

## 二、组件二：位置编码——告诉模型词的顺序

### 2.1 初印象：用费曼视角理解

**注意力机制有一个盲点：它看词，但不看顺序**。

对注意力机制来说，"猫吃了鱼"和"鱼吃了猫"是一样的——都是三个词，只是顺序不同，但注意力机制看到的信息完全相同（因为它是同时看所有词的）。

显然这不行。语言中词序至关重要。

**解决方案**：在把词送进模型之前，给每个词的向量里额外"加入"一个表示位置的向量——告诉模型"这个词在第 1 位"、"这个词在第 2 位"……

这就是**位置编码（Positional Encoding）**。

### 2.2 两种主要位置编码

**方法一：固定正弦位置编码（原始 Transformer 使用）**

用不同频率的正弦和余弦函数，为每个位置生成一个固定的、独特的"坐标"。

```
位置 1 的编码：[0.00, 1.00, 0.01, 1.00, ...]
位置 2 的编码：[0.84, 0.54, 0.02, 1.00, ...]
位置 3 的编码：[0.91, 0.41, 0.03, 1.00, ...]
...
```

这些编码是固定的，不需要学习。优点是不需要额外参数，并且能推广到训练时没见过的长度。

**方法二：可学习位置编码**

把位置编码也当成模型参数来训练，模型自己学出每个位置的最优编码。GPT-2 使用这种方式。

**方法三：RoPE（旋转位置编码，现代 LLM 主流）**

更先进的方法，不是直接在输入上加位置编码，而是在计算注意力时把位置信息"旋转"进去。好处是对超长序列的外推能力更好。LLaMA、Qwen、Mistral 等都用这种方式。（这是进阶内容，3.7 章会详细讲。）

### 2.3 位置编码的工作方式

```python
import torch
import torch.nn as nn
import math

class SimplePositionalEncoding(nn.Module):
    """
    原始 Transformer 的正弦位置编码。
    不需要学习，直接根据位置算出固定编码。
    """
    def __init__(self, d_model: int, max_len: int = 5000):
        super().__init__()
        # 预先计算好所有位置的编码，存起来备用
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * -(math.log(10000.0) / d_model)
        )
        pe[:, 0::2] = torch.sin(position * div_term)   # 偶数维度用 sin
        pe[:, 1::2] = torch.cos(position * div_term)   # 奇数维度用 cos
        self.register_buffer('pe', pe)

    def forward(self, x):
        # x: (batch_size, seq_len, d_model)
        seq_len = x.size(1)
        # 把位置编码加到词向量上
        return x + self.pe[:seq_len, :]

# 使用示例
d_model = 16
pos_enc = SimplePositionalEncoding(d_model)

# 假设 batch_size=2，序列长度=5，d_model=16
x = torch.zeros(2, 5, d_model)   # 先假设词向量全为 0
output = pos_enc(x)

print(f"输入形状：{x.shape}")
print(f"加入位置编码后：{output.shape}")  # 形状不变
print(f"第1个位置的编码（前8维）：{output[0, 0, :8].detach().numpy().round(3)}")
print(f"第2个位置的编码（前8维）：{output[0, 1, :8].detach().numpy().round(3)}")
# 可以看到两个位置的编码数值不同，模型因此能区分词的位置
```

## 三、组件三：前馈网络（FFN）——每个词独立的"思考"

### 3.1 初印象：用费曼视角理解

在注意力机制让所有词"互相交流"之后，每个词的向量都包含了来自其他词的信息。

但这还不够——模型还需要对每个词的向量做进一步的"深度加工"。

**FFN（前馈网络，Feed-Forward Network）就是这个"深度加工"的步骤**。

它是一个简单的两层神经网络，**独立地**作用在每个词的向量上：

```
词1的向量 → FFN → 加工后的词1向量
词2的向量 → FFN → 加工后的词2向量
词3的向量 → FFN → 加工后的词3向量
...
```

注意"独立地"三个字——不同词之间的 FFN 处理互不影响（而注意力机制恰好相反，它是所有词之间互相交流的）。

### 3.2 FFN 的结构

```python
import torch.nn as nn
import torch.nn.functional as F

class FeedForward(nn.Module):
    """
    Transformer 中的前馈网络。
    两层线性变换，中间加一个激活函数。
    """
    def __init__(self, d_model: int, d_ff: int = None):
        super().__init__()
        if d_ff is None:
            d_ff = 4 * d_model   # 通常扩展到 4 倍
        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)

    def forward(self, x):
        # x: (batch_size, seq_len, d_model)
        # 先扩展维度（增加表达能力），再压缩回去
        x = self.linear1(x)      # d_model → d_ff
        x = F.gelu(x)            # 激活函数（引入非线性）
        x = self.linear2(x)      # d_ff → d_model
        return x

# 示例：d_model=16，d_ff=64
ffn = FeedForward(d_model=16, d_ff=64)
x = torch.randn(2, 5, 16)   # (batch=2, seq_len=5, d_model=16)
output = ffn(x)
print(f"FFN 输入：{x.shape}")
print(f"FFN 输出：{output.shape}")  # 形状不变：(2, 5, 16)
```

**为什么先扩展再压缩？** 扩展到更高维度（比如 4096 → 16384），是为了让模型有更大的"工作空间"来处理复杂的特征变换；处理完再压回原来的维度，保持整体结构的一致性。这个扩展比例（通常是 4 倍）叫做 **FFN 扩展比（FFN ratio）**。

## 四、组件四：残差连接——给梯度留条高速公路

### 4.1 初印象：用费曼视角理解

**残差连接的思想极其简单：把每一层的输入直接加到输出上**。

数学上就是：$\text{输出} = \text{层处理结果} + \text{原始输入}$

为什么这很重要？我们在 2.2.1 和 2.2.8 已经讲过这个问题。用大白话说：

想象你在传话。没有残差连接，话要从最后一个人传回第一个人，每传一次都有点失真；传 20 次之后，原话已经面目全非了——这就是梯度消失。

有了残差连接，就像同时开了一条"直达通道"：不管传了多少层，第一个人的"原话"可以通过这条捷径直接到达任意层，不会失真。

```python
import torch
import torch.nn as nn

# 残差连接的实现极其简单：
class ResidualConnection(nn.Module):
    def __init__(self, sublayer: nn.Module):
        super().__init__()
        self.sublayer = sublayer

    def forward(self, x):
        # 核心就是这一行：输出 = 子层处理结果 + 原始输入
        return x + self.sublayer(x)

# 在 Transformer 中的使用：
# output = x + Attention(x)  ← 注意力层加残差
# output = x + FFN(x)        ← FFN 层加残差
```

这个"$x +$"就是残差连接的全部。别被它简单的外表骗了——正是这个加法，让 Transformer 能训练几十层甚至上百层而不崩溃。

## 五、组件五：层归一化——让数字不要"跑偏"

### 5.1 初印象：用费曼视角理解

**层归一化（Layer Normalization）解决的问题是：数字在经过多层变换后可能变得很大或很小，导致训练不稳定**。

想象你在量体重，每次量完都要把结果"标准化"：减去全班平均体重，除以标准差。这样不管原来是多少斤，标准化后的数字都在一个稳定的范围内（大约在 -3 到 3 之间）。

层归一化做的就是这件事，只不过不是对"全班"（所有样本）统计，而是对"这个词的所有特征维度"统计。

### 5.2 为什么不用 Batch Normalization？

如果你了解 BatchNorm，可能会问：为什么 Transformer 不用 Batch Normalization，而是用 Layer Normalization？

简单说：BatchNorm 是"跨样本"归一化（横向），Layer Normalization 是"跨特征维度"归一化（纵向）。

对文本来说，不同句子长度不同，BatchNorm 的统计量很不稳定；而 LayerNorm 只看当前这一个词/位置的特征维度，不受其他句子的影响，更适合文本。

（我们在 2.2.4 已经详细讲过这个问题。）

```python
import torch
import torch.nn as nn

# LayerNorm 示例
layer_norm = nn.LayerNorm(normalized_shape=16)   # 对最后一个维度（d_model=16）归一化

x = torch.randn(2, 5, 16)   # (batch=2, seq_len=5, d_model=16)
output = layer_norm(x)

# 验证：归一化后每个词向量的均值接近 0，标准差接近 1
print(f"归一化前 - 均值：{x[0,0].mean():.3f}，标准差：{x[0,0].std():.3f}")
print(f"归一化后 - 均值：{output[0,0].mean():.3f}，标准差：{output[0,0].std():.3f}")
```

### 5.3 Pre-Norm vs Post-Norm：放在哪里？

层归一化可以放在注意力/FFN 的前面（Pre-Norm）或后面（Post-Norm）：

```
Post-Norm（原始 Transformer）：
  x → Attention → x + Attention(x) → LayerNorm → 下一步

Pre-Norm（现代 LLM 标配，如 LLaMA）：
  x → LayerNorm → Attention → x + Attention(LayerNorm(x)) → 下一步
```

现代大模型几乎都用 Pre-Norm，因为训练更稳定，不需要太精心设计学习率调度。

## 六、把零件组装起来：一个完整的 Transformer 层

现在把上面所有组件组装成一个完整的 Transformer 层：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class TransformerLayer(nn.Module):
    """
    一个完整的 Transformer 层（Pre-Norm 风格）。
    注意：这里用占位符代替注意力，下一章会换成真正的 Self-Attention。
    """
    def __init__(self, d_model: int, d_ff: int, dropout: float = 0.1):
        super().__init__()
        # 组件：LayerNorm
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        # 组件：前馈网络
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Linear(d_ff, d_model),
        )
        self.dropout = nn.Dropout(dropout)

        # 【占位符】注意力机制，下一章替换为真正的 Self-Attention
        self.attention_placeholder = nn.Linear(d_model, d_model)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # ── 第一个子层：注意力 + 残差 ──
        # Step 1: Pre-Norm
        x_norm = self.norm1(x)
        # Step 2: 注意力（这里暂时用线性层占位）
        attn_output = self.attention_placeholder(x_norm)
        # Step 3: 残差连接
        x = x + self.dropout(attn_output)

        # ── 第二个子层：FFN + 残差 ──
        # Step 4: Pre-Norm
        x_norm = self.norm2(x)
        # Step 5: 前馈网络
        ffn_output = self.ffn(x_norm)
        # Step 6: 残差连接
        x = x + self.dropout(ffn_output)

        return x


# 测试一层的数据流
d_model = 64
d_ff    = 256
layer   = TransformerLayer(d_model, d_ff)

# 模拟一个 batch：2 个句子，每句 10 个 token，每个 token 64 维
x = torch.randn(2, 10, d_model)
output = layer(x)
print(f"输入形状：{x.shape}")    # (2, 10, 64)
print(f"输出形状：{output.shape}")  # (2, 10, 64)  形状不变
```

**数据在一层内的完整流动**：

```
输入 x（形状：batch × seq_len × d_model）
  ↓
LayerNorm（数字归一化）
  ↓
Self-Attention（词与词之间交流，下章讲）
  ↓
+ 原始输入 x（残差连接）
  ↓
LayerNorm（再次归一化）
  ↓
FFN（每个词独立深度加工）
  ↓
+ 上面的中间结果（残差连接）
  ↓
输出（形状不变：batch × seq_len × d_model）
```

这就是 **Transformer 一层的完整数据流**。把这个结构重复 $N$ 次，就是整个 Transformer 的主体（$N$ 通常是 12、24、32、80 等）。

## 七、组件七：输出层——把向量变回词语

最后一层 Transformer 输出的还是一个向量（形状是 `seq_len × d_model`）。要得到"下一个词是什么"的预测，还需要一个输出层：

```python
# 最后一步：把 d_model 维的向量投影到词表大小，再 softmax 得到概率
class LMHead(nn.Module):
    """语言模型的输出头：把最后一层的向量变成每个 token 的概率"""
    def __init__(self, d_model: int, vocab_size: int):
        super().__init__()
        # 线性层：d_model → vocab_size
        self.linear = nn.Linear(d_model, vocab_size, bias=False)

    def forward(self, x):
        # x: (batch, seq_len, d_model)
        logits = self.linear(x)  # (batch, seq_len, vocab_size)
        # logits 就是每个位置预测下一个词的"得分"
        # 训练时配合 CrossEntropy Loss 使用
        # 推理时取 softmax 得到概率，然后采样
        return logits

# 示例
vocab_size = 32000
lm_head = LMHead(d_model=64, vocab_size=vocab_size)
x = torch.randn(2, 10, 64)
logits = lm_head(x)
print(f"输出 logits 形状：{logits.shape}")  # (2, 10, 32000)
# 对最后一个 token 的 logits 做 softmax，得到下一个词的概率分布
probs = F.softmax(logits[:, -1, :], dim=-1)
print(f"预测下一个词的概率分布形状：{probs.shape}")  # (2, 32000)
```

## 八、把全流程串联起来

现在可以把"文字输入"到"词语概率输出"的全流程写出来：

```python
class MiniTransformer(nn.Module):
    """
    一个极简 Transformer，把本章所有组件串联起来。
    注意力层暂用线性层占位，下一章替换。
    """
    def __init__(self, vocab_size, d_model, d_ff, n_layers, max_seq_len):
        super().__init__()
        # 组件一：Token Embedding
        self.embedding = nn.Embedding(vocab_size, d_model)
        # 组件二：位置编码
        self.pos_encoding = SimplePositionalEncoding(d_model, max_seq_len)
        # 组件三-六：N 个 Transformer 层
        self.layers = nn.ModuleList([
            TransformerLayer(d_model, d_ff) for _ in range(n_layers)
        ])
        # 最终的 LayerNorm
        self.final_norm = nn.LayerNorm(d_model)
        # 组件七：输出层
        self.lm_head = nn.Linear(d_model, vocab_size, bias=False)

    def forward(self, token_ids):
        # token_ids: (batch_size, seq_len)

        # 1. Token Embedding
        x = self.embedding(token_ids)   # → (batch, seq_len, d_model)

        # 2. 位置编码
        x = self.pos_encoding(x)        # → 形状不变，但每个位置加了位置信息

        # 3. 逐层处理
        for layer in self.layers:
            x = layer(x)               # → 形状不变

        # 4. 最终归一化
        x = self.final_norm(x)

        # 5. 输出层
        logits = self.lm_head(x)       # → (batch, seq_len, vocab_size)
        return logits


# 搭建一个小模型，看看数据流
model = MiniTransformer(
    vocab_size=1000,
    d_model=64,
    d_ff=256,
    n_layers=4,
    max_seq_len=128,
)

# 模拟输入：2 个句子，每句 8 个 token
token_ids = torch.randint(0, 1000, (2, 8))
logits = model(token_ids)
print(f"输入 token_ids 形状：{token_ids.shape}")     # (2, 8)
print(f"输出 logits 形状：{logits.shape}")            # (2, 8, 1000)
print(f"模型参数量：{sum(p.numel() for p in model.parameters()):,}")
```

## 九、各组件的角色总结

| 组件 | 作用 | 一句话记忆 |
|------|------|-----------|
| Token Embedding | 文字 → 数字向量 | "把词语翻译成模型能理解的语言" |
| 位置编码 | 注入位置信息 | "给每个词打上'我排在第几位'的标签" |
| Self-Attention | 词间信息交流 | "让每个词看清楚它和其他词的关系"（下章详讲）|
| 残差连接 | 保留原始信息 | "每层处理后，把原始输入加回来，防止信息丢失" |
| 层归一化 | 稳定数值范围 | "把数字拉回正常范围，让训练不崩溃" |
| FFN | 每词独立加工 | "每个词在'记住了周围关系'后，自己再深度思考一遍" |
| 输出层 | 向量 → 词概率 | "把处理结果翻译回'下一个词是什么'" |

## 十、面试常见问题

**Q1：Transformer 中 Embedding 和位置编码是怎么结合在一起的？**

两者是直接相加的：`最终输入 = 词语 Embedding + 位置编码`。两个向量维度相同（都是 d_model），逐元素相加。这样每个词的表示既包含了词语的语义信息（来自 Embedding），也包含了位置信息（来自位置编码）。

**Q2：残差连接的公式是什么？为什么需要它？**

公式就是 `output = sublayer(x) + x`。需要它的原因：深层神经网络训练时，梯度从输出层反传到输入层会越来越弱（梯度消失）。残差连接提供了一条"高速公路"——梯度可以直接通过加法操作无损地传递，不必经过每一层的复杂变换。这让 Transformer 能稳定地训练几十甚至上百层。

**Q3：FFN 在 Transformer 里的作用是什么？和注意力机制有什么区别？**

注意力机制负责让不同位置的词"互相交流"，整合全局信息；FFN 负责对每个位置的向量做独立的"深度加工"，进一步变换特征。两者分工明确：注意力管"横向"（位置间），FFN 管"纵向"（每个位置自身）。FFN 的参数量通常占整个模型的 60% 以上，是模型"记忆和推理能力"的重要来源。

**Q4：Pre-Norm 和 Post-Norm 的区别？现代 LLM 用哪个？**

Post-Norm 是先做注意力/FFN，再归一化（原始 Transformer 的做法）；Pre-Norm 是先归一化，再做注意力/FFN（现代 LLM 的做法，如 LLaMA、Qwen）。Pre-Norm 的优点是训练更稳定，因为每个子层的输入都已经被归一化了，梯度更容易传播。几乎所有 2022 年后的主流大模型都用 Pre-Norm。

**Q5：为什么 FFN 要把维度先扩大（到 4d）再压缩回去？**

扩大维度是为了增加模型的"表达能力"。就像解一道复杂数学题，你需要在草稿纸上写很多中间步骤——扩大的维度就是这张"草稿纸"，让模型有足够的"工作空间"来提取和变换复杂的特征。处理完再压缩回原来的维度，是为了保持整个网络维度的一致性，让层与层之间的数据格式匹配。
